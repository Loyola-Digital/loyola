## Story

**As a** Loyola Digital developer,
**I want** database tables for Meta Ads accounts and project associations,
**so that** the platform can store ad account credentials securely and link them to projects.

---

## Acceptance Criteria

1. [ ] Tabela `meta_ads_accounts` criada com campos: `id` (UUID PK), `account_name` (VARCHAR 100 NOT NULL), `meta_account_id` (VARCHAR 50 NOT NULL — formato `act_XXXXXXX`), `encrypted_access_token` (TEXT NOT NULL), `access_token_iv` (TEXT NOT NULL), `is_active` (BOOLEAN DEFAULT true), `created_by` (FK → users.id ON DELETE RESTRICT), `created_at` (TIMESTAMPTZ DEFAULT NOW), `updated_at` (TIMESTAMPTZ DEFAULT NOW)
2. [ ] Indice `uq_meta_ads_meta_account_id` UNIQUE em `meta_ads_accounts(meta_account_id)`
3. [ ] Indice `idx_meta_ads_created_by` em `meta_ads_accounts(created_by)`
4. [ ] Tabela `meta_ads_account_projects` criada com: `id` (UUID PK), `account_id` (FK → meta_ads_accounts.id ON DELETE CASCADE), `project_id` (FK → projects.id ON DELETE CASCADE), `created_at` (TIMESTAMPTZ DEFAULT NOW)
5. [ ] Constraint UNIQUE `uq_meta_ads_account_project` em `(account_id, project_id)`
6. [ ] Indices em `account_id` e `project_id` na junction table
7. [ ] Tabelas exportadas no `packages/api/src/db/schema.ts`
8. [ ] Migration gerada via `npx drizzle-kit generate` e aplicavel via `npx drizzle-kit push`
9. [ ] `pnpm typecheck` e `pnpm lint` passam sem erros

## Dev Notes

### Architecture Context

- **ORM:** Drizzle ORM — schema em `packages/api/src/db/schema.ts` [Source: EPIC-3 pattern]
- **DB:** PostgreSQL 16 (Railway managed)
- **Padrao de referencia:** `instagramAccounts` + `instagramAccountProjects` — seguir exatamente mesmo pattern de columns, indices e constraints
- **Encriptacao:** Campos `encrypted_access_token` + `access_token_iv` (mesma estrategia AES-256-CBC do Instagram)
- **UUID PKs:** `uuid("id").defaultRandom().primaryKey()` em todas as tabelas
- **Timestamps:** `timestamp("...", { withTimezone: true }).defaultNow().notNull()`

### Schema Drizzle Reference

```typescript
// ============================================================
// META ADS TABLES (EPIC-6)
// ============================================================

export const metaAdsAccounts = pgTable(
  "meta_ads_accounts",
  {
    id: uuid("id").defaultRandom().primaryKey(),
    accountName: varchar("account_name", { length: 100 }).notNull(),
    metaAccountId: varchar("meta_account_id", { length: 50 }).notNull(),
    accessTokenEncrypted: text("access_token_encrypted").notNull(),
    accessTokenIv: text("access_token_iv").notNull(),
    isActive: boolean("is_active").notNull().default(true),
    createdBy: uuid("created_by")
      .notNull()
      .references(() => users.id, { onDelete: "restrict" }),
    createdAt: timestamp("created_at", { withTimezone: true })
      .defaultNow()
      .notNull(),
    updatedAt: timestamp("updated_at", { withTimezone: true })
      .defaultNow()
      .notNull(),
  },
  (table) => [
    uniqueIndex("uq_meta_ads_meta_account_id").on(table.metaAccountId),
    index("idx_meta_ads_created_by").on(table.createdBy),
  ]
);

export const metaAdsAccountProjects = pgTable(
  "meta_ads_account_projects",
  {
    id: uuid("id").defaultRandom().primaryKey(),
    accountId: uuid("account_id")
      .notNull()
      .references(() => metaAdsAccounts.id, { onDelete: "cascade" }),
    projectId: uuid("project_id")
      .notNull()
      .references(() => projects.id, { onDelete: "cascade" }),
    createdAt: timestamp("created_at", { withTimezone: true })
      .defaultNow()
      .notNull(),
  },
  (table) => [
    unique("uq_meta_ads_account_project").on(table.accountId, table.projectId),
    index("idx_meta_ads_account_projects_account").on(table.accountId),
    index("idx_meta_ads_account_projects_project").on(table.projectId),
  ]
);
```
