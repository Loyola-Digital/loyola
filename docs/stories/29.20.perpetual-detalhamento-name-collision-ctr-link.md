# Story 29.20 — Detalhamento: subcontagem por nome repetido (M1) + CTR/CPC link em linha mesclada (M2)

**Epic:** 29
**Status:** Draft
**Points:** 5
**Tipo:** Bug / Data Correctness
**Prioridade:** P2
**Origem:** QA Gate `docs/qa/29.10-29.19-perpetual-gate.md` (findings M1, M2)

## Contexto

Durante o QA gate do batch 29.10–29.19, o review adversarial encontrou 2 problemas reais de contabilização na tabela Detalhamento, diferidos pra não travar o ship das melhorias já validadas.

## Problema M1 — Subcontagem quando adset/ad têm nome igual (Média)

O backend (`packages/api/src/services/traffic-analytics.ts:719-756` adsets, ~789 ads) agrega **por nome** (`key = adset_name.trim()` / `ad_name.trim()`) e fixa `id: a.adset_id`/`a.ad_id` **do primeiro visto**. No frontend, `overlaySpreadsheetMetrics` (`perpetual-dashboard.tsx:151`) casa `byId.get(r.campaignId)` — só o id do primeiro. Vendas da planilha cujo `utm_medium`/`utm_content` apontam pra **outro id de mesmo nome** somem da linha do Detalhamento.

**Sintoma:** a mesma entidade mostra Receita/Vendas menor no Detalhamento do que no gráfico "Receita por Público/Criativo" ao lado (que re-agrupa **todos** os `porUtmMedium`/`porUtmContent` por nome). Dois números diferentes na mesma tela.

## Problema M2 — CTR/CPC "(link)" errado em linha mesclada (Baixa)

O merge do 29.19 (`perpetual-dashboard.tsx` ~585) recalcula `ctr = clicks/impressions` e `cpc = spend/clicks` de **cliques totais**; linhas não-mescladas mantêm o valor do backend (que usa **link clicks**). O header diz "(link)". Então linha com "— Cópia" mesclada mostra CTR/CPC de base diferente sob rótulo que promete link.

## Acceptance Criteria

- [ ] **AC1 (M1):** No filtro Público/Criativo, vendas da planilha de **todos** os ids que compartilham o mesmo nome são somadas na linha correspondente. Opções: (a) backend expõe lista de ids por nome; ou (b) overlay soma por nome cruzando o resolve de nomes.
- [ ] **AC2 (M1):** A linha do Detalhamento passa a bater com o gráfico "Receita por Público/Criativo" para a mesma entidade.
- [ ] **AC3 (M2):** Linha mesclada re-deriva CTR/CPC de **link clicks** somados (não cliques totais), OU o header deixa de prometer "(link)" quando há merge. Definir com consistência.
- [ ] **AC4:** Sem regressão nas linhas de id único; typecheck + build OK.
