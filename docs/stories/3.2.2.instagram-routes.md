## Story

**As a** Loyola Digital employee,
**I want** REST API endpoints to manage Instagram accounts and fetch metrics,
**so that** the frontend can perform CRUD operations and display analytics data.

---

## Acceptance Criteria

1. [ ] `POST /api/instagram/accounts` — cadastrar conta: valida token via Instagram service, encripta token, salva no banco, retorna dados da conta (sem token)
2. [ ] `GET /api/instagram/accounts` — listar contas do usuario autenticado (sem expor tokens)
3. [ ] `GET /api/instagram/accounts/:id` — detalhes de uma conta (sem token)
4. [ ] `PUT /api/instagram/accounts/:id` — atualizar nome e/ou token (se token novo: valida, encripta, salva)
5. [ ] `DELETE /api/instagram/accounts/:id` — remover conta (cascade deleta metricas cache)
6. [ ] `GET /api/instagram/accounts/:id/profile` — perfil atualizado via Instagram service
7. [ ] `GET /api/instagram/accounts/:id/insights?period=30d&since=&until=` — metricas da conta
8. [ ] `GET /api/instagram/accounts/:id/media?limit=25&after=` — posts com metricas
9. [ ] `GET /api/instagram/accounts/:id/demographics` — dados de audiencia
10. [ ] `GET /api/instagram/accounts/:id/stories` — stories ativos com metricas
11. [ ] `GET /api/instagram/accounts/:id/reels` — reels com metricas
12. [ ] `POST /api/instagram/accounts/:id/refresh` — invalida cache e forca re-fetch
13. [ ] Todos endpoints protegidos por auth (Clerk JWT)
14. [ ] Scoped por `user_id` — usuario so acessa suas proprias contas (retorna 404 para contas de outros)
15. [ ] Validacao de input com JSON Schema (Fastify built-in)
16. [ ] Error responses padronizadas: `{ error: string, code?: string }`
17. [ ] Testes de integracao para cada endpoint

- **Instagram Service:** Disponivel via Fastify decorate (Story 3.2.1)

### API Contract

**POST /api/instagram/accounts**
```json
// Request
{ "accountName": "Cliente XYZ", "instagramUserId": "17841400...", "accessToken": "EAABx..." }

// Response 201
{ "id": "uuid", "accountName": "Cliente XYZ", "instagramUsername": "@clientexyz", "instagramUserId": "17841400...", "profilePictureUrl": "https://...", "followersCount": 12500, "isActive": true, "tokenExpiresAt": "2026-05-15T00:00:00Z", "createdAt": "2026-03-16T..." }
```

**GET /api/instagram/accounts**
```json
// Response 200
[{ "id": "uuid", "accountName": "...", "instagramUsername": "...", "isActive": true, "lastSyncedAt": "...", "tokenExpiresAt": "..." }]
```

**GET /api/instagram/accounts/:id/insights?period=30d**
```json
// Response 200
{ "period": "30d", "since": "2026-02-14", "until": "2026-03-16", "data": { "impressions": [...], "reach": [...], "followerCount": [...], "profileViews": [...] } }
```

### Test Scenarios

1. POST /accounts com token valido → 201, conta criada
2. POST /accounts com token invalido → 400, erro claro
3. POST /accounts duplicado (mesmo instagram_user_id) → 409 conflict
4. GET /accounts → lista apenas contas do usuario autenticado
5. GET /accounts/:id de outro usuario → 404
6. PUT /accounts/:id com novo token → valida e atualiza
7. DELETE /accounts/:id → remove conta + cascade cache
8. GET /accounts/:id/profile → retorna dados atualizados
9. GET /accounts/:id/insights?period=30d → retorna metricas
10. POST /accounts/:id/refresh → limpa cache
11. Request sem auth → 401
