# Story 25.1 — Match Code + Detecção + Banner de Campanhas Órfãs

## Status

InReview

## Story

**As an** analista do Loyola Digital X,
**I want** cadastrar um `matchCode` no funil (ex: `dg-pg02`) e ver um banner amarelo no dashboard sempre que houver campanhas Meta Ads com esse código no nome **não selecionadas** em nenhuma etapa,
**so that** eu não rode meses com métricas incompletas porque esqueci de adicionar uma nova campanha ao funil.

## Acceptance Criteria

### Database Schema

1. Adicionar coluna `match_code` (varchar(50), nullable) na tabela `funnels`:
   - Drizzle: `matchCode: varchar("match_code", { length: 50 })`
   - Migration `0025_funnel_match_code.sql` aplicada em PROD via `node -e` + `pg`

2. Shared type `Funnel` em `@loyola-x/shared` ganha `matchCode: string | null`.

### Settings UI

3. No popup/sheet de configuração do funil (`funnels/[funnelId]/page.tsx` — Settings popover existente), adicionar campo:
   - Label: "Código de match (Meta Ads)"
   - Sub-label: "Texto a buscar no nome das campanhas Meta. Ex: `dg-pg02`. Deixe vazio para desativar o alerta."
   - Input text, max 50 chars, normaliza pra lowercase ao salvar
   - Botão Salvar dispara `useUpdateFunnel` (já existe) com `{ matchCode: "..." }`
   - Sub-texto dinâmico: "X campanhas matcham este código" (atualiza ao digitar — debounce 300ms)

4. API `PUT /api/projects/:projectId/funnels/:funnelId` (já existe) aceita `matchCode?: string | null` no body.

### API — Detecção de Órfãs

5. Novo endpoint:
   - `GET /api/projects/:projectId/funnels/:funnelId/orphan-campaigns`
   - Verifica `getProjectAccess`
   - Lê `funnel.matchCode`. Se `null` ou vazio → retorna `{ orphans: [], byStage: {}, hasMatchCode: false }`
   - Lê todas as campanhas Meta Ads da conta vinculada ao projeto (reusa lógica de `useCampaignPicker` no backend, ou seja, lê das tabelas/cache existentes)
   - Filtra: campanhas onde `name.toLowerCase().includes(matchCode.toLowerCase())`
   - Coleta IDs selecionados:
     - `funnel.campaigns[].id` (nível funil — legacy)
     - `stages[].campaigns[].id` (todas as etapas do funil)
   - **Órfã** = bate matchCode E **não está em** nenhum dos sets acima
   - Retorna:
     ```typescript
     {
       hasMatchCode: boolean;
       matchCode: string | null;
       totalMatching: number;       // matcham o code, mesmo que selecionadas
       orphans: Array<{
         id: string;
         name: string;
         status: string;             // ACTIVE | PAUSED | DELETED | ARCHIVED
         objective?: string;
       }>;
       byStage: Record<string, {     // chave = stageId
         stageName: string;
         orphans: Array<{ id: string; name: string; status: string }>;
       }>;
     }
     ```
   - `byStage` lista órfãs DAQUELA etapa específica = campanhas que matcham + estão em outras etapas/funil mas não nesta. Útil pro banner da etapa.

### Banner Visual

6. Componente novo `<OrphanCampaignsBanner>` em `packages/web/components/funnels/`:
   - Props: `{ projectId, funnelId, stageId?: string }`
   - Hook `useOrphanCampaigns(projectId, funnelId)` — query simples, staleTime 1min
   - Quando `stageId` é passado → lê `byStage[stageId]?.orphans` (granular da etapa)
   - Quando `stageId` não é passado → lê `orphans` (agregado do funil — todas as não-selecionadas em lugar nenhum)
   - Se `hasMatchCode === false` ou `orphans.length === 0` → não renderiza nada
   - Visual: banner amarelo `bg-amber-500/10 border-amber-500/30 text-amber-900 dark:text-amber-200` com ícone `AlertTriangle`
   - Texto curto:
     - Funil: "⚠️ {N} campanhas Meta com **`{matchCode}`** não estão em nenhuma etapa"
     - Etapa: "⚠️ {N} campanhas Meta com **`{matchCode}`** não estão selecionadas nesta etapa"
   - Lista até 5 nomes com badge de status (ACTIVE verde, PAUSED amber, DELETED/ARCHIVED cinza)
   - Se houver mais que 5: "+N outras"
   - Botão "Configurar etapa" (etapa) ou "Ver etapas" (funil) com link

### Integração

7. Banner integrado em:
   - `app/(app)/projects/[id]/funnels/[funnelId]/page.tsx` (página do funil) — sem `stageId`, mostra agregado
   - `app/(app)/projects/[id]/funnels/[funnelId]/stages/[stageId]/page.tsx` — passa `stageId`, mostra granular da etapa
   - Posicionado logo abaixo do header, antes das tabs/abas

### Validação

## Dev Notes

### Reuso

- `useCampaignPicker(projectId)` já lista campanhas Meta da conta vinculada ao projeto. Backend pode reusar a query/cache subjacente (verificar onde `getCampaignsForProject` está implementado — provavelmente em `routes/funnels.ts` ou `meta-ads.ts`).
- `useUpdateFunnel(projectId)` já existe — patch acrescenta `matchCode`.

### Match logic

```ts
function isMatch(campaignName: string, matchCode: string): boolean {
  if (!matchCode || !matchCode.trim()) return false;
  return campaignName.toLowerCase().includes(matchCode.trim().toLowerCase());
}
```

### Sets de campanhas selecionadas

- Coletar de `funnel.campaigns` (legacy nível funil)
- Coletar de TODAS as `stages[].campaigns` do funil
- União dos dois sets → "selectedIds"
- Órfãs = campaign.id NOT IN selectedIds AND campaignName matches

### byStage (banner da etapa)

Pra cada etapa:
- selectedInThisStage = `stage.campaigns.map((c) => c.id)`
- orphansInStage = matching AND NOT IN selectedInThisStage
- Inclui campanhas selecionadas em OUTRAS etapas — porque do ponto de vista DESTA etapa, elas continuam órfãs aqui
