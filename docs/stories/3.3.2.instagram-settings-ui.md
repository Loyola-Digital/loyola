## Story

**As a** Loyola Digital employee,
**I want** a settings interface to add, edit, and remove Instagram accounts with their access tokens,
**so that** I can manage which client accounts are connected to the platform.

---

## Acceptance Criteria

1. [ ] Tab "Meta / Instagram" em `/settings/instagram` funcional (substituir placeholder de 3.1.1)
2. [ ] Lista de contas cadastradas em cards com: avatar do perfil, @username, nome amigavel, status do token (valido/expirado), ultima sincronizacao, data de expiracao do token
3. [ ] Acoes por conta: Editar, Remover (com confirmacao), Refresh
4. [ ] Botao "Adicionar Conta" abre dialog/modal com form:
   - Campo: Nome da conta (texto livre)
   - Campo: Instagram Business Account ID (com helper text)
   - Campo: Access Token (textarea, masked por default, toggle mostrar/esconder)
5. [ ] Validacao em tempo real: ao colar token, chama `POST /api/instagram/accounts` para validar
6. [ ] Feedback visual: spinner enquanto valida → checkmark verde (com dados do perfil) ou erro vermelho
7. [ ] Ao salvar com sucesso: mostra perfil encontrado (@username, followers) como confirmacao
8. [ ] Edicao: dialog pre-preenchido (token mascarado, so substitui se digitar novo)
9. [ ] Remocao: modal de confirmacao com nome da conta
10. [ ] Guia colapsavel: "Como obter seu Access Token no Business Manager" com instrucoes textuais
11. [ ] Toast notifications: sucesso ao adicionar/editar/remover
12. [ ] Empty state atrativo quando sem contas
13. [ ] Indicador visual de token proximo de expirar (< 7 dias: badge amarelo "Expira em X dias")
14. [ ] Responsivo: cards em stack no mobile

### Token Guide Content

Guia colapsavel com passos:
1. Acesse developers.facebook.com e selecione seu App
2. Va para "Instagram Graph API" no painel de ferramentas
3. Gere um User Token com as permissoes: `instagram_basic`, `instagram_manage_insights`, `pages_show_list`, `pages_read_engagement`
4. Copie o token (expira em ~60 dias)
5. Para Long-Lived Token: use o endpoint de exchange no Graph API Explorer

### UI Patterns

- Account card: usar `Card` com `Avatar`, badges para status, `DropdownMenu` para acoes
- Dialog: usar `Dialog` + `DialogContent` + `DialogHeader` + `DialogFooter`
- Token input: `Textarea` com `type="password"` toggle via `Eye`/`EyeOff` icons
- Validacao: `useMutation` do TanStack Query com onSuccess/onError handlers
- Toast: `toast.success("Conta adicionada!")`, `toast.error("Token invalido")`
