## User Story

Como gestor de tráfego do Loyola, quando abro o **Launch dashboard** de um funil e olho a seção **"Top Criativos"**, eu preciso ver (1) **o preview real da imagem do criativo** — não um quadrado preto — pra reconhecer o anúncio na hora, e (2) o **bloco de qualificação por pesquisa** (Faturamento top / Profissão top / etc) **dentro de cada card**, pra saber qual ad trouxe os leads mais qualificados. Hoje os dois estão quebrados, o que apaga o valor principal do dashboard de captação paga.

---

## Bug A — Preview de imagem preto

### Cenário atual

- No Launch dashboard, abre seção "Top Criativos"
- Lugar onde deveria aparecer o thumbnail do ad → **fundo escuro/preto, sem imagem** (não é o placeholder `ImageOff` esperado pra URL expirada)
- Acontece pra a maioria dos criativos (não casos isolados)

### Hipóteses a investigar (ordenadas por probabilidade)

| # | Hipótese | Como validar |
|---|----------|--------------|
| H1 | `imageUrl` E `thumbnailUrl` vindo **vazios** do hook `useTopPerformers` → `creativeImgSrc()` retorna `""` → cai no branch `!src` mas o container herda `bg-*` escuro do parent e o ícone `ImageOff` está com cor quase invisível (`text-muted-foreground/40`) sobre dark | Abrir DevTools → inspecionar card → ver se `<div>` placeholder está renderizado mas só não dá pra ver o ícone |
| H2 | URLs do Meta CDN **expirando** em massa (TTL ~ horas/dias) → `<img onError>` dispara → fallback renderiza, mesmo cenário do H1 visualmente | Pegar `imageUrl` no console e abrir direto no browser — se 404, é TTL |
| H3 | Mudança na API Meta deixou de devolver `image_url` em `object_story_spec` / `creative` → backend manda vazio, frontend cai em fallback | Logar payload do `/insights` ou `/ads` no backend e ver se o campo veio |
| H4 | Backend está populando os campos mas com chave errada (ex: `image_url` em snake_case quando o type espera `imageUrl` camelCase) | Inspecionar response da rota que alimenta `useTopPerformers` |
| H5 | `next/image` config sem domínio do Meta whitelisted causando bloqueio silencioso | Verificar `next.config.js` — mas o componente usa `<img>` nativo (linha 130), então provavelmente não é isso |

### Comportamento desejado

1. Quando `imageUrl` existe e carrega → mostra a imagem HD do criativo
2. Quando só `thumbnailUrl` existe → mostra low-res com blur sutil (já implementado, manter)
3. Quando ambos faltam OU URL falhou → placeholder visível e legível (não preto chapado): fundo `bg-muted/50` + ícone `ImageOff` em cor com contraste mínimo de 4.5:1 contra o fundo
4. Bonus: se TTL do Meta expirou, ter um caminho de re-buscar a URL via backend (chamada Meta Ads Marketing API) — **fora de escopo desta story se for muito complexo, registrar como follow-up**

## Bug B — Galeria de criativos só lê 4 perguntas hardcoded, mas pesquisas agora são mapeadas dinamicamente

### Causa raiz (confirmada pelo Lucas)

O `top-creatives-gallery.tsx` (linhas 770-804) ainda consome o contrato **legacy** `SurveyAdData`:

```ts
interface SurveyAdData {
  faturamento: Array<{ label: string; count: number }>;
  profissao:   Array<{ label: string; count: number }>;
  funcionarios: Array<{ label: string; count: number }>;
  voce_e:      Array<{ label: string; count: number }>;
}
```

E renderiza 4 linhas com emojis hardcoded (`💰 Faturamento`, `👤 Profissão`, `👥 Funcionários`, `📋 Você é`).

Acontece que após o commit **`117b8b2`** ("feat: mapping configurável substitui hardcoded survey questions"), as pesquisas atuais são configuradas via **column mapping dinâmico** — o usuário escolhe quais colunas da planilha de pesquisa quer trackear, com **labels custom** e flag `showInDashboard`.

Resultado: o hook `useSurveyAggregation` agora popula `byQuestion` (dinâmico, indexado por `columnName`), mas o `byAdId` legacy fica vazio pra pesquisas que usam mapping novo — porque as keys hardcoded (`faturamento`, etc) simplesmente não existem mais.

Comprovação no código: linhas 41-48 do hook deixam isso explícito:
```ts
/**
 * Legacy: top-creatives-gallery ainda usa 4 perguntas hardcoded.
 * Migração pra dinâmico fica pra story futura.
 */
```

**Essa é a story futura.**

### Comportamento desejado

1. Galeria passa a renderizar as perguntas do **mapping** (filtradas por `showInDashboard: true`), com os **labels custom** definidos pelo gestor — não os 4 hardcoded
2. Cada card mostra **N linhas** dinâmicas (em vez de 4 fixas), preservando o formato `<emoji> <label top> · X/Y (Z%)`
   - Limite máximo sugerido: 4-5 linhas por card pra não estourar layout (truncar excedente ou mostrar só top-N por relevância)
   - Emoji por pergunta: vir do mapping (campo novo `emoji` no `columnMapping`?) OU heurística por label OU emoji genérico (`📊`) — **decisão do dev em consulta com @ux-design-expert se necessário**
3. **Pesquisas sem mapping (legacy)** continuam funcionando via fallback dos 4 matchers hardcoded — sem regressão pros funis antigos
4. Quando não há dados (`byAdId` ou equivalente dinâmico vazio), mostrar "— Sem dados de pesquisa" (estado atual, linha 781)

### Trabalho previsto

| Camada | Mudança |
|--------|---------|
| `use-survey-aggregation.ts` | Expor um novo retorno `byAdIdDynamic: Record<adId, Record<columnName, Array<{label, count}>>>` — ou estender `byAdId` pra ser dinâmico mantendo as keys legacy quando estão presentes |
| `top-creatives.ts` (`mergeSurveyForGroup`) | Receber lista dinâmica de `questions` (do `questions: SurveyQuestionMeta[]`) em vez de hardcoded; retornar mapa por `columnName` |
| `top-creatives-gallery.tsx` (770-804) | Iterar sobre `questions` (do hook) e renderizar uma linha por pergunta com `top` resolvido em runtime |
| Schema `funnel_surveys.column_mapping` | **Opcional:** adicionar campo `emoji?: string` em cada coluna mapeada — se decidir suportar emoji custom |

### Riscos / decisões abertas

- **Emoji**: hardcoded por label / vem do mapping / genérico. Default: genérico `📊` no MVP, evolui depois
- **Ordenação das perguntas**: usar `orderindex` do mapping ou ordem de criação
- **Backward compat**: como detectar "pesquisa usa mapping novo" vs "pesquisa legacy" no runtime — usar presença/ausência de `columnMapping` no record do `funnel_surveys`

## Acceptance Criteria

### AC1 — Preview de imagem (Bug A)

- **AC1.1:** Em pelo menos **80% dos cards** do Launch dashboard de um funil de captação ativo, a imagem HD do criativo aparece renderizada (não placeholder)
- **AC1.2:** Quando a URL falha (TTL expirado ou indisponível), o placeholder mostra `ImageOff` visível com contraste mínimo de 4.5:1 (WCAG AA) — não pode ficar quadrado preto sem indicação visual

- **AC2.3:** Em funil com pesquisa **legacy (sem `columnMapping`)**, fallback dos 4 matchers hardcoded continua funcionando — sem regressão pros funis antigos
- **AC2.4:** Quando não há respostas matched para o ad, card mostra "— Sem dados de pesquisa" (estado claro)
- **AC2.5:** Console do browser fica limpo (sem warnings/errors de tipo) ao renderizar a galeria
- **AC2.6:** Hook `useSurveyAggregation` mantém retorno legacy `byAdId` populado quando pesquisa for legacy (não quebrar outros consumers)
