## Story

Como gestor de tráfego do Loyola X, quero uma nova aba **"Meta Ads 2"** dentro do funil que cruze os dados diários da API do Meta Ads (investimento, cliques, impressões) com os dados das planilhas vinculadas ao funil (leads por categoria), exibindo tudo em uma **tabela diária unificada**, para que eu consiga analisar a performance completa do funil em uma única visão — do investimento até o lead.

## Contexto Técnico

- Project ID: `e25369be-1d04-4153-8178-14a3b617e70e`
- Funnel ID: `5492a226-4378-44fa-8775-dec28140117e`
- Planilha de leads já vinculada via aba "Planilhas" (EPIC-17)

### Fontes de dados

| Coluna | Fonte | API/Hook existente |
|--------|-------|--------------------|
| Dia | Meta Ads API | `useMetaAdsDailyInsights` ou `useCampaignDailyInsights` |
| Investimento | Meta Ads API | campo `spend` |
| Cliques | Meta Ads API | `actions[]` → `action_type: "link_click"` (**NÃO** o campo `clicks` que inclui todos os cliques) |
| Impressões | Meta Ads API | campo `impressions` |
| CPM | **Calculado** | `Investimento / Impressões * 1000` |
| CPC | **Calculado** | `Investimento / Link Clicks` |
| CTR | **Calculado** | `Link Clicks / Impressões * 100` |
| LP View | Meta Ads API | `actions[]` → `action_type: "landing_page_view"` |
| Connect Rate | **Calculado** | `LP View / Link Clicks * 100` |
| Tx Conv. | **Calculado** | `Total Leads / LP View * 100` (onde Total Leads = Leads pagos + Leads org + Leads s/ track) |
| Leads pagos | **Planilha** | Leads por dia onde `utm_source` ∈ {meta, meta-ads, google-ads} (case-insensitive) |
| Leads org | **Planilha** | Leads por dia onde `utm_source` tem valor mas ∉ {meta, meta-ads, google-ads} |
| Leads s/ track | **Planilha** | Leads por dia onde `utm_source` está vazio/ausente |
| CPL Pg | **Calculado** | `Investimento / Leads pagos` |
| CPL G | **Calculado** | `Investimento / (Leads pagos + Leads org + Leads s/ track)` |

### Lógica de cruzamento

1. **Meta Ads**: Buscar insights diários das campanhas do funil (campo `campaigns[]` do funil) — agregar por dia se houver múltiplas campanhas
2. **Planilha**: Buscar dados da planilha vinculada ao funil via `useFunnelSpreadsheetData` → usar a coluna mapeada como `date` para agrupar por dia → contar linhas que correspondem a cada categoria (pagos, orgânicos, sem tracking)
3. **Join por dia**: Cruzar ambos os datasets pela data (chave: `YYYY-MM-DD`)
4. **Colunas calculadas**: Derivar Connect Rate, Tx Conv., CPL Pg, CPL G a partir dos dados já cruzados

### Estrutura da coluna `actions[]` do Meta Ads

```typescript
interface CampaignDailyInsight {
  date_start: string;
  date_stop: string;
  impressions: string;
  clicks: string;      // ⚠️ NÃO USAR — são todos os cliques (social + link)
  spend: string;
  ctr: string;          // ⚠️ NÃO USAR — calculado pelo Meta com total clicks
  cpc: string;          // ⚠️ NÃO USAR — calculado pelo Meta com total clicks
  cpm: string;          // ⚠️ NÃO USAR — recalcular no frontend
  actions?: { action_type: string; value: string }[];
}
// Link Clicks = actions.find(a => a.action_type === "link_click")?.value ?? "0"
// LP View = actions.find(a => a.action_type === "landing_page_view")?.value ?? "0"
// IMPORTANTE: CPM, CPC, CTR devem ser recalculados no frontend usando Link Clicks
```

### Identificação de categorias na planilha

A lógica de categorização depende do **mapeamento de colunas** (`columnMapping`) definido pelo usuário na aba Planilhas. O dev deve:

1. Usar a coluna `date` do mapping para agrupar por dia
2. Usar a coluna `utm_source` para categorizar em 3 grupos
3. Se `utm_source` não estiver mapeada, todos os leads caem em "Leads s/ track"

**Regra de categorização (confirmada pelo usuário):**
- Coluna mapeada: `utm_source` do `columnMapping`
- **Leads pagos:** `utm_source` é `meta`, `meta-ads` ou `google-ads` (comparação case-insensitive)
- **Leads org:** `utm_source` tem valor preenchido mas NÃO é nenhum dos acima
- **Leads s/ track:** `utm_source` está vazio, nulo ou a coluna não está mapeada

## Dev Notes

- **Frontend-only:** Esta story é 100% frontend. Nenhum endpoint novo de API é necessário — reusa hooks existentes (`useFunnelSpreadsheets`, `useFunnelSpreadsheetData`, `useMetaAdsDailyInsights` ou `useCampaignDailyInsights`).
- **Planilha vinculada:** Usar `useFunnelSpreadsheets` para listar planilhas do funil → encontrar a de tipo "leads" ou a primeira disponível → buscar dados com `useFunnelSpreadsheetData`.
- **Múltiplas campanhas:** O funil tem campo `campaigns: { id, name }[]`. Se houver várias, somar os insights diários de todas.
- **Regra de categorização:** Usar `utm_source` do mapping. Valores `meta`, `meta-ads`, `google-ads` (case-insensitive) = Leads pagos. Outro valor preenchido = Leads org. Vazio/ausente = Leads s/ track.
