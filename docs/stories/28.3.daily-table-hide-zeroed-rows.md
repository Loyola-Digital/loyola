# Story 28.3 — Tabela de dados diários: ocultar linhas com tudo zerado

**Epic:** 28 — Match Inteligente de Campanhas (follow-up de qualidade dos dashboards)
**Status:** InReview
**Priority:** Medium-High
**Estimate:** XS (mudança cirúrgica em 1 função pura)
**Type:** Bug fix
**Surface:** `packages/web/lib/utils/funnel-metrics.ts` → `buildDailyRows()`

---

## User Story

Como gestor de tráfego, quando olho a **tabela de dados diários** do Launch dashboard, quero que **dias sem nenhum dado relevante não apareçam** — investir, ver impressão, ter lead ou faturamento. Hoje aparecem linhas como `12/05/2026 | R$ 0,00 | R$ 0,00 | 0 | 0 | 0 | — | — | …` que ocupam espaço sem trazer informação e poluem a leitura.

---

## Bug atual

Exemplo das linhas zeradas (reportado pelo Lucas):

```
12/05/2026  R$ 0,00  R$ 0,00  0  0  0  —  —  R$ 0,00  R$ 0,00  0.00%  0  —  —  0  0  0
13/05/2026  R$ 0,00  R$ 0,00  0  0  0  —  —  R$ 0,00  R$ 0,00  0.00%  0  —  —  0
```

### Causa raiz (confirmada)

`buildDailyRows()` em `packages/web/lib/utils/funnel-metrics.ts:271-306` faz **union de todas as datas** de 3 fontes:

```ts
const allDates = new Set([...metaMap.keys(), ...sheetMap.keys()]);
if (extraDates) for (const d of extraDates) allDates.add(d);
```

E pra cada data, preenche com **zeros como fallback** quando uma fonte não tem dados (linhas 280-281):

```ts
const meta = metaMap.get(date) ?? { spend: 0, impressions: 0, ... };
const sheet = sheetMap.get(date) ?? { leadsPagos: 0, leadsOrg: 0, ... };
```

Resultado: se uma data aparece em **uma** das fontes mesmo sem dados úteis (ex: planilha de vendas tem a data mas com zero vendas, e Meta API não tem essa data), ela vira uma linha 100% zerada na tabela.

---

## Comportamento desejado

Filtrar **dentro do `buildDailyRows()`** linhas onde **todas** as métricas significativas são zero/null. Critério explícito:

```
hasData = (
  spend > 0 ||
  impressions > 0 ||
  linkClicks > 0 ||
  lpView > 0 ||
  totalLeads > 0 ||  // leadsPagos + leadsOrg + leadsSemTrack
  faturamento > 0
)
```

Se todas falsas → linha não entra no array de retorno.

A linha "Total" (montada por `computeTotals()` separadamente) **não é afetada** — agrega as linhas restantes.

---

## Acceptance Criteria

### AC1 — Filtragem no utility
- **AC1.1:** Linhas onde `spend`, `impressions`, `linkClicks`, `lpView`, `totalLeads (leadsPagos+leadsOrg+leadsSemTrack)` e `faturamento` são **todos** 0 são removidas do array retornado por `buildDailyRows()`
- **AC1.2:** Linhas com **qualquer** métrica > 0 continuam aparecendo (ex: dia só com leads orgânicos sem spend, ou dia com spend sem leads)
- **AC1.3:** Ordenação ascendente por data é preservada

### AC2 — Consumers herdam o fix
- **AC2.1:** Tabela em `crossed-funnel-daily-table.tsx` (Launch dashboard) deixa de mostrar linhas zeradas
- **AC2.2:** Tabela em `meta-ads-spreadsheet-tab.tsx` (Meta Ads Spreadsheet) também — mesmo hook
- **AC2.3:** Linha "Total" no footer continua aparecendo com os agregados corretos das linhas restantes
- **AC2.4:** `computeTotals` recebe array já filtrado — totais batem com a soma visível das linhas

### AC3 — Sem regressão
- **AC3.1:** Se TODOS os dias do range têm dados, nenhuma linha é removida (no-op)
- **AC3.2:** Tipos `DailyRow` inalterados — sem breaking change pra consumers
- **AC3.3:** Marcadores de virada de lote (Story 27.1) continuam funcionando — eles operam sobre o array já filtrado e a referência por data ainda é única

## Implementation guidance

Mudança proposta cirúrgica em `funnel-metrics.ts:271-306`:

```ts
for (const date of allDates) {
  const meta = metaMap.get(date) ?? { spend: 0, ... };
  const sheet = sheetMap.get(date) ?? { leadsPagos: 0, ... };
  const totalLeads = sheet.leadsPagos + sheet.leadsOrg + sheet.leadsSemTrack;

  // Story 28.3: pula linhas 100% zeradas (todas as métricas significativas em 0)
  const hasData =
    meta.spend > 0 ||
    meta.impressions > 0 ||
    meta.linkClicks > 0 ||
    meta.lpView > 0 ||
    totalLeads > 0 ||
    sheet.faturamento > 0;
  if (!hasData) continue;

  rows.push({ ... });
}
```

Localizado dentro do loop, antes do `rows.push`. ~7 linhas adicionadas.
