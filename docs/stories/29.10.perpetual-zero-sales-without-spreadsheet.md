# Story 29.10 — Funil Perpétuo: zerar Vendas/Receita quando não há planilha de vendas conectada

**Epic:** 29
**Status:** InReview
**Points:** 3
**Depende de:** 29.4 (reverte parcialmente a AC2 dela — ver Contexto)
**Tipo:** Bug Fix / Mudança de regra de negócio

## Contexto (por que existe)

Bug reportado por Danilo: no **Funil Perpétuo**, etapa/painel de vendas, ao **desvincular a planilha** de vendas (ex: `[FZ-A1] KPIs Aquisição`), os KPIs de topo **continuam contando vendas** (528) e **Receita** (R$ 56.2K). Não deveria — sem fonte de vendas conectada, não pode haver dado de venda no painel.

**Causa raiz (confirmada no código):** A story 29.4 definiu, por design, um fallback: quando `usingSpreadsheet === false`, o painel usa `overview` (dados do **Meta Ads / pixel purchases**) como fonte de vendas. Isso está em `perpetual-dashboard.tsx:297-299`:

```ts
if (!usingSpreadsheet || !salesData) {
  return { ...overview, totalSpend: effectiveSpend }; // ← herda totalSales/totalRevenue do Meta
}
```

Quando a planilha é desvinculada, `usingSpreadsheet` vira `false` (`perpetual-dashboard.tsx:175`), e `overview.totalSales` / `overview.totalRevenue` vêm de `traffic-analytics.ts:407-408` (compras do pixel Meta) — independentes da planilha. Por isso os números persistem.

**Decisão de negócio (Danilo, 2026-07-06):** Reverter esse fallback para a etapa de vendas do Funil Perpétuo. **Sem planilha de vendas conectada = zero venda**, com aviso visual "Conectar fonte de vendas". A planilha passa a ser a **única** fonte de Vendas/Receita neste painel — o pixel Meta deixa de ser fallback silencioso.

> ⚠️ Isto contradiz intencionalmente a **AC2 da Story 29.4** (que preservava o Meta como fallback). É mudança de regra deliberada, não regressão. @po deve validar ciente disso.

## User Story

Como usuário de funil perpétuo, quando **não tenho planilha de vendas conectada** (nunca conectei ou desvinculei), quero que os KPIs de **Vendas** e **Receita** mostrem **zero** com um aviso claro **"Conectar fonte de vendas"**, para que eu não seja induzido a acreditar em números de venda que na verdade vêm do pixel Meta.

## Dev Notes

### Ponto único de correção
A raiz do bug é **um único ramo** do `effectiveMetrics` (`perpetual-dashboard.tsx:297-299`). Não é preciso mexer no backend (`traffic-analytics.ts` continua entregando o `overview` do Meta — só não usamos os campos de venda dele aqui). Fix é front-only.

### Validação manual obrigatória (o teste do bug)
Reproduzir o cenário do Danilo:
1. Abrir funil perpétuo com planilha conectada → Vendas/Receita preenchidos com badge 📄.
2. **Desvincular** a planilha (`[FZ-A1] KPIs Aquisição`).
3. Confirmar: Vendas = 0, Receita = R$ 0, aviso "Conectar fonte de vendas", ROAS/CAC/Margem = "—", Investimento **inalterado**.
4. Reconectar a planilha → tudo volta ao normal (sem regressão).

Funil de exemplo do report:
`project 4d7f55ea-ff1b-4fa8-b3cc-caed182878b3 / funnel b67b738a-7480-4939-acd1-e37bb08696a8 / stage fd2d424f-d800-470d-ac96-62e89078185e`

### Não tocar
- `StageSalesSection` / `stage-sales-data.ts` — a seção por-etapa já trata o caso corretamente (retorna `semDados` quando `spreadsheets.length === 0`).
- Backend `perpetual-sales-data.ts` / `traffic-analytics.ts` — mudança é só de apresentação no dashboard.

### Consideração de UX (para @ux/@dev decidir na hora)
"Conectar fonte de vendas" pode virar link/CTA que abre o wizard de conexão de planilha (`usePerpetualSpreadsheet` connect flow). Não é obrigatório para o fix, mas é natural. **Não implementado** (aviso é texto estático) — fica como nice-to-have.

### Decisões de implementação (@dev)

1. **Receita mostra "—", não "R$ 0,00" literal.** O formatador `fmtCurrency(val)` do arquivo retorna `"—"` para `val == null || val === 0` (convenção usada em todo o dashboard). Não criei exceção pra esse card — o aviso "⚠️ Conectar fonte de vendas" já comunica a ausência de dados, e manter a convenção evita inconsistência com os outros cards. **Vendas mostra "0"** (fmtNumber(0)). → Se o Danilo preferir "R$ 0,00" literal, é ajuste trivial de 1 linha.

2. **`noSalesSource` derivado no render**, não no `effectiveMetrics`. Usei `const noSalesSource = !usingSpreadsheet` junto do `fromSheet` existente — evita estender o tipo do `effectiveMetrics` com flag e mantém a fonte da verdade única (`usingSpreadsheet`).

3. **Gráfico no tempo (AC7): margem = 0 quando sem planilha**, não `receitaLíquida − spend` (que daria margem negativa = −spend). Sem fonte de vendas não sabemos que houve zero vendas — mostrar −spend afirmaria isso falsamente. Linha de margem fica em 0 (flat), receita em 0. Investimento continua real.

4. **`KpiCard` é componente local** (definido no próprio `perpetual-dashboard.tsx:967`), não o `kpi-card.tsx` de subscriptions. Adicionei a prop `warning?: string` só nele. Nenhum outro arquivo tocado.
