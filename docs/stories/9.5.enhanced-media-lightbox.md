## Story

**As a** gestor de tráfego da Loyola Digital,
**I want** to preview ad creatives in high quality — watching videos inline and seeing images in full resolution — and navigate between creatives without closing the lightbox,
**so that** I can quickly evaluate creative quality and compare ads visually without leaving the dashboard.

---

## Acceptance Criteria

### Video Player

1. [ ] When clicking a VIDEO creative (in gallery, top performers, or drill-down), the lightbox opens with an embedded video player instead of a static thumbnail
2. [ ] Use the Meta Graph API to get the video source URL: `GET /{videoId}?fields=source` (returns direct MP4 URL). The `videoId` is already available in `MetaAdCreative.videoId`
3. [ ] Video player must have: play/pause, progress bar, mute toggle, fullscreen button. Use native HTML5 `<video>` element with `controls` attribute
4. [ ] Fallback: if video source fetch fails, show the thumbnail with a "Ver no Meta" link (current behavior)
5. [ ] Video source URLs should be cached alongside creative data (same 1h TTL)

### HD Images

6. [ ] For IMAGE creatives, the lightbox must show the full-resolution image from `creative.imageUrl` instead of the compressed `thumbnailUrl`
7. [ ] If `imageUrl` is null, fallback to `thumbnailUrl` (current behavior)
8. [ ] Image should render with `object-contain` to preserve aspect ratio, max-height 70vh
9. [ ] Add a subtle loading skeleton while the HD image loads

### Prev/Next Navigation

10. [ ] Lightbox includes left/right arrow buttons to navigate between creatives in the current list context
11. [ ] Arrow key navigation (Left/Right) also works when lightbox is open
12. [ ] Navigation context: if opened from Gallery, navigate through gallery items. If opened from drill-down, navigate through ads in that adset
13. [ ] Show current position indicator: "3 / 12"
14. [ ] Wrap-around: after last item, go to first (and vice versa)

### Lightbox UI Improvements

15. [ ] Close on Escape key (already works) and click outside (already works)
16. [ ] Show ad name, campaign name, adset name in lightbox header
17. [ ] Show key metrics below the media: Spend, Impressions, Clicks, CTR, CPC
18. [ ] For video ads, also show: Video Views (thruplay), Retention curve (p25→p100)

## Dev Notes

- `MetaAdCreative.videoId` is already fetched and available — just need to call `GET /{videoId}?fields=source` to get the MP4 URL
- The Meta video source URL is temporary (expires after a few hours) — cache it with shorter TTL or fetch on-demand when lightbox opens
- For the video player, use native `<video>` with `controls` — no need for a library. The Meta source URL returns a direct MP4
- The lightbox currently lives inside `DrillDownAds` component as inline JSX. It needs to be extracted into a standalone `<CreativeLightbox>` component that can be used from Gallery, Top Performers, and drill-down
- `image_url` from the Meta API is the full resolution version. For IMAGE objects this is the original upload. For VIDEO objects, `image_url` is a full-res frame capture
- Navigation requires passing the full list of creatives + current index to the lightbox

### API Addition Needed

Add a new endpoint or extend existing creative fetch to get video source:

```typescript
// In meta-ads.ts
export async function fetchVideoSource(videoId: string, accessToken: string): Promise<string | null> {
  const data = await fetchMeta<{ source?: string }>(`/${videoId}?fields=source`, accessToken);
  return data.source ?? null;
}
```

Expose via route: `GET /api/traffic/analytics/:projectId/video-source?videoId=XXX`
