## Story

Como usuário do Loyola X, quero uma nova aba **"Planilhas"** dentro de cada funil onde eu possa vincular planilhas do Google Sheets (leads, vendas, custom), selecionar a aba e **mapear os campos** (nome, email, telefone, UTMs, data, status, valor) através de um wizard, para ter dados estruturados do funil disponíveis sem depender da aba "Pesquisa" existente.

## Contexto Técnico

### Padrões reusados

- **Wizard de referência:** `MappingDialog` em `packages/web/app/(app)/settings/sales/page.tsx:1-461` — 3-step: planilha → aba → colunas. Copiar UX, adaptar para novos campos e salvar em `funnelSpreadsheets`.
- **Tab pattern:** `packages/web/components/funnels/survey-funnel-tab.tsx:1-324` — aba "Pesquisa" já existente serve de modelo estrutural.
- **Picker dialog:** `SheetsPickerDialog` do survey — componente de seleção planilha→aba já resolvido.
- **Hooks:** `useFunnelSpreadsheets*` (criados em 17.1), `useSpreadsheets` + `useSpreadsheetSheets` + `useSheetData` (já existem).

### Campos do mapeamento (UI)

O wizard deve oferecer os seguintes campos opcionais (usuário arrasta/seleciona coluna da planilha para cada um):

| Campo | Label | Obrigatório? |
|-------|-------|--------------|
| `name` | Nome | Opcional |
| `email` | Email | Opcional |
| `phone` | Telefone | Opcional |
| `date` | Data | Opcional |
| `status` | Status | Opcional |
| `value` | Valor | Opcional |
| `utm_source` | UTM Source | Opcional |
| `utm_medium` | UTM Medium | Opcional |
| `utm_campaign` | UTM Campaign | Opcional |
| `utm_content` | UTM Content | Opcional |
| `utm_term` | UTM Term | Opcional |

**Validação:** pelo menos UM campo preenchido (enforçado no backend por 17.1, refletido no botão "Salvar" desabilitado).

### Tipo da planilha (UI)

Select com 3 opções: **Leads**, **Vendas**, **Custom**. Afeta apenas ícone/badge — não altera comportamento.

## Acceptance Criteria

### AC1: Nova aba "Planilhas" no funil
- [ ] Aba "Planilhas" adicionada no funnel page (ao lado de "Pesquisa", Meta Ads, YouTube Ads)
- [ ] Ícone distintivo (ex: `Table` ou `FileSpreadsheet` do lucide-react)
- [ ] Aba "Pesquisa" permanece **inalterada** e totalmente funcional

### AC2: Listagem de planilhas vinculadas
- [ ] Tab renderiza lista de `funnelSpreadsheets` via `useFunnelSpreadsheets`
- [ ] Cada item mostra: label, badge de tipo (Leads/Vendas/Custom), nome da planilha, aba, contagem de campos mapeados
- [ ] Estado vazio com CTA "Vincular primeira planilha"
- [ ] Botão "Vincular planilha" no topo sempre visível

### AC3: Wizard — Step 1 (Selecionar planilha)
- [ ] Lista de planilhas do Drive via `useSpreadsheets`
- [ ] Busca por nome
- [ ] Ao selecionar, avança para Step 2

### AC4: Wizard — Step 2 (Selecionar aba + tipo + label)
- [ ] Dropdown de abas via `useSpreadsheetSheets`
- [ ] Input de `label` (ex: "Leads captados abril")
- [ ] Select de `type` (Leads/Vendas/Custom)
- [ ] Ao confirmar, avança para Step 3

### AC5: Wizard — Step 3 (Mapeamento de colunas)
- [ ] Carrega headers da aba via `useSheetData`
- [ ] Para cada campo mapeável (11 opções listadas acima), oferece **select de coluna** com headers da planilha + opção "Não mapear"
- [ ] Pré-visualização: tabela com 5 primeiras linhas mostrando como fica após mapping
- [ ] Botão "Salvar" desabilitado até pelo menos 1 campo estar mapeado
- [ ] Ao salvar, chama `useCreateFunnelSpreadsheet` e fecha dialog

### AC6: Edição de mapeamento
- [ ] Botão "Editar" em cada item da lista abre dialog no Step 3 (pula 1 e 2, planilha já selecionada)
- [ ] Permite alterar label, type e mapeamento de colunas (não planilha/aba — forçar recriação para mudar essas)
- [ ] Salva via `useUpdateFunnelSpreadsheet`

### AC7: Remoção
- [ ] Botão "Remover" (ícone lixeira) em cada item com confirmação (dialog)
- [ ] Ao confirmar, chama `useDeleteFunnelSpreadsheet`

### AC8: Estados de carregamento e erro
- [ ] Skeleton loader durante fetch
- [ ] Toast de erro se API falhar
- [ ] Toast de sucesso após criar/atualizar/deletar

### AC9: Sem regressão
- [ ] Aba "Pesquisa" (`survey-funnel-tab.tsx`) renderiza e funciona idêntico ao antes
- [ ] Dashboards Launch/Perpetual continuam funcionando (não dependem de funnelSpreadsheets)

### AC10: Validação em browser
- [ ] Testado manualmente no dev server: criar vínculo de leads + vínculo de vendas + editar + deletar funciona end-to-end
- [ ] Recarregar página mantém estado (dados persistidos corretamente)
