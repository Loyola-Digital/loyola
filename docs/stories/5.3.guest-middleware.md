## Story

**As a** backend service,
**I want** middleware that restricts guest access by project membership and module permissions,
**so that** guests can only access data they were explicitly granted.

---

## Acceptance Criteria

1. [ ] Middleware `guestGuard` criado — detecta `user.role === 'guest'` via Clerk metadata
2. [ ] Para rotas `/api/projects/:id/*`: valida que guest está em `project_members` para o projeto `:id`
3. [ ] Rotas de módulo validam `project_members.permissions`:
   - `/api/projects/:id/instagram/*` → requer `permissions.instagram = true`
   - `/api/projects/:id/conversations` → requer `permissions.conversations = true`
   - `POST /api/chat` com `project_id` → requer `permissions.mind = true`
4. [ ] Módulo bloqueado retorna `403 { error: "module_not_allowed", module: "instagram" }`
5. [ ] Projeto não autorizado retorna `403 { error: "project_access_denied" }`
6. [ ] Rotas globais bloqueadas para guest:
   - `GET /api/projects` → filtra apenas projetos onde guest é membro (não 403, mas filtrado)
   - `POST/PUT/DELETE /api/projects/*` → 403
   - `GET /api/instagram/accounts` (global, sem project_id) → 403
   - `GET /api/conversations` (global) → 403
   - `GET /api/tasks` (global) → 403
7. [ ] `getChatTools(fastify, userRole)` aceita `userRole` como parâmetro
   - `role === 'guest'`: retorna apenas `instagram_metrics` (sem `clickup_*` tools)
   - `get_past_conversations` para guest: filtra apenas conversas do próprio guest
8. [ ] `routes/chat.ts` passa `request.userRole` para `getChatTools`
9. [ ] `middleware/auth.ts` expõe `request.userRole` no tipo do request Fastify
10. [ ] Testes: guest bloqueado em módulo sem permissão, guest bloqueado em projeto sem membership, mind sem ClickUp tools para guest

### guestGuard Pattern

```typescript
// middleware/guest-guard.ts
export async function guestGuard(request: FastifyRequest, reply: FastifyReply) {
  if (request.userRole !== 'guest') return; // admin passa direto

  const projectId = (request.params as { id?: string }).id;
  if (!projectId) return; // rota global — verificar bloqueio em handler

  // Verifica membership
  const member = await getMember(projectId, request.userId);
  if (!member) return reply.code(403).send({ error: "project_access_denied" });

  // Verifica permission do módulo (detectado pela URL)
  const module = detectModule(request.url);
  if (module && !member.permissions[module]) {
    return reply.code(403).send({ error: "module_not_allowed", module });
  }
}
```
