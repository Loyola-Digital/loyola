# Story 22.1 — Lead Scoring Tab (v2, schema enriquecido)

## Status

Ready for Review

## Story

**As an** analista de tráfego do Loyola Digital X,
**I want** uma aba "Lead Scoring" dentro de cada etapa do funil onde eu posso colar um JSON de schema de pontuação e vinculá-lo a uma planilha de pesquisa já conectada à etapa,
**so that** o app pontue cada lead da planilha conforme as regras do schema, classifique em bandas A/B/C/D e me devolva um relatório com distribuição por banda + CPL ideal/breakeven por banda.

## Acceptance Criteria

1. **Aba "Lead Scoring"** aparece na página da etapa (`projects/[id]/funnels/[funnelId]/stages/[stageId]`), depois da aba "Links", com ícone `Brain` da lucide-react.
2. **Seletor de planilha de pesquisa**: dropdown lista apenas as planilhas vinculadas à etapa atual (via `funnelSurveys` filtrada por `stageId`). Estado vazio mostra link/dica para vincular planilha primeiro na aba "Pesquisas".
3. **Editor JSON**: textarea grande (font monospace, ~16 linhas), aceita o schema completo no formato fornecido (ver "Schema Reference" em Dev Notes). Validação no salvar:
   - JSON deve ser parseable (toast de erro caso contrário)
   - `schema_version === "1.0"` (warning se diferente, mas salva)
   - Estrutura mínima exigida: `scoring_model.questions[]`, `bands[]`. Demais campos são opcionais e exibidos como metadados.
4. **Persistência**: schema é salvo em uma tabela `stage_lead_scoring_schemas` (1 schema por `stageId`, upsert). Dados salvos: `surveyId` selecionada + `schemaJson` completo (como JSONB).
5. **Cálculo de scoring** — backend lê a planilha vinculada e aplica:
   - Para cada linha, para cada `question` em `scoring_model.questions[]`:
     - Achar a coluna no header pelo `new_survey_column` (case-insensitive, trim)
     - Achar a resposta em `answers[].value` (case-insensitive, trim, comparação exata após normalização)
     - **Regra especial Q3 (Empresário↔Q4)**: se a resposta atual é "Empresário" e tem `points_conditional`, usar `if_q4_filled` quando a coluna do Q4 (resolvida via `new_survey_column` do Q4) estiver não-vazia, senão `if_q4_empty`
     - Se a resposta não bate com nenhuma `value`, usar `unmapped_default` da questão (default `0`)
   - Somar pontos das 8 questões → `totalScore`
   - Classificar em banda: `band.range.min <= totalScore < band.range.max` (note: ranges são half-open conforme o JSON; banda A vai 80-101, então score 100 cai em A)
   - Casos não-classificados (score fora de qualquer range) contam separadamente como `unclassified`.
6. **Tabela de resultados** mostra por banda (ordem A → D):
   - Banda (badge colorido: A=verde, B=azul, C=amarelo, D=vermelho)
   - `description` da banda (ex: "Contatar agora — lead quente")
   - Leads classificados na banda (count + %)
   - CPL Ideal (de `cpl_ideal.per_band[bandId].cpl`)
   - CPL Breakeven (de `cpl_ideal.per_band[bandId].breakeven`)
   - Ação recomendada (mapeada: CONTACT_NOW→"Contatar agora", NURTURE→"Nutrir", QUALIFY_MORE→"Qualificar mais", DISQUALIFY→"Desqualificar")
   - Footer: total de leads, total %, CPL global (`cpl_ideal.global`)
7. **Botão "Recalcular"** com ícone `RefreshCw` recompõe os resultados (re-fetch da planilha, sem precisar salvar schema de novo).
8. **Project info card** (acima da tabela) mostra `project.name`, `ticket`, `roas`, `cpa_ceiling` quando o schema os define.
9. **Empty states** claros:
   - Sem schema salvo: "Cole o JSON do modelo de Lead Scoring acima para começar."
   - Sem planilha selecionada: "Selecione uma planilha de pesquisa para calcular os resultados."
   - Sem dados ou matching falhou: "Sem dados suficientes para calcular. Verifique se a planilha tem respostas e se os `new_survey_column` batem com os headers."
10. **Typecheck** passa em web e api sem erros.

### Scoring Algorithm

Pseudocode da função `computeBands(schema, { headers, rows })`:

```
questions = schema.scoring_model.questions
bands = schema.bands

# Mapa: questionId -> coluna index na planilha
questionColMap = {}
for q in questions:
  idx = headers.findIndex(h => h.trim().toLowerCase() === q.new_survey_column.trim().toLowerCase())
  questionColMap[q.id] = idx  # -1 se não achou

bandCounts = {bandId: 0 for band in bands}
unclassified = 0

for row in rows:
  totalScore = 0
  q4Idx = questionColMap.get("Q4")
  q4Value = q4Idx !== -1 ? row[q4Idx]?.trim() : ""
  q4Filled = q4Value !== "" && q4Value !== "(sem resposta)"

  for q in questions:
    colIdx = questionColMap[q.id]
    if colIdx === -1:
      totalScore += q.unmapped_default ?? 0
      continue
    answer = row[colIdx]?.trim() ?? ""
    match = q.answers.find(a => a.value.trim().toLowerCase() === answer.toLowerCase())
    if match:
      if match.points_conditional:  # Q3 Empresário
        rule = match.points_conditional
        totalScore += q4Filled ? rule.if_q4_filled : rule.if_q4_empty
      else:
        totalScore += match.points
    else:
      totalScore += q.unmapped_default ?? 0

  # Classificar (range half-open: min <= score < max, exceto última banda que inclui max)
  band = bands.find(b => totalScore >= b.range.min && totalScore < b.range.max)
  if !band:
    # Tenta a última banda inclusiva (score == max permitido)
    band = bands.find(b => totalScore === b.range.max)
  if band:
    bandCounts[band.id] += 1
  else:
    unclassified += 1
```

### Padrões de codebase a seguir

- **API routes**: dentro de `fp()` closure, usar `fastify.db` injetado pelo `dbPlugin`. Ver `packages/api/src/routes/funnel-spreadsheets.ts` como referência.
- **Drizzle FK auto-ref**: já não se aplica aqui (sem auto-referência). Para FKs cross-table, usar `references(() => funnelStages.id, { onDelete: "cascade" })`.
- **Zod v4**: `z.record(keySchema, valueSchema)` exige 2 args. NÃO use `z.record(z.unknown())` — vai dar erro `_zod` em runtime.
- **Migration manual**: `cd packages/api && node -e "const { Pool } = require('pg'); require('dotenv').config(); const p = new Pool({ connectionString: process.env.DATABASE_URL }); p.query('CREATE TABLE IF NOT EXISTS stage_lead_scoring_schemas (...)').then(() => p.end())"`
- **Web hooks**: padrão `useApiClient()` + `useQuery`/`useMutation`. Ver `packages/web/lib/hooks/use-funnel-spreadsheets.ts` como referência.
- **Components UI**: shadcn (Card, Table, Badge, Select, Textarea, Button, Skeleton). Cores das bandas (manter consistente):
  - A: `bg-green-100 text-green-800 border-green-200`
  - B: `bg-blue-100 text-blue-800 border-blue-200`  ← (atualização: era amarelo na v1; usar azul agora pra distinguir de C)
  - C: `bg-yellow-100 text-yellow-800 border-yellow-200`
  - D: `bg-red-100 text-red-800 border-red-200`

### Diferenças da implementação anterior (v1, removida)

| Item | v1 (removida) | v2 (esta) |
|------|---------------|-----------|
| Matching coluna | `question.label` | `question.new_survey_column` |
| Pontuação Q3 | Linear | Condicional via `points_conditional` (Q4-aware) |

| Banda B color | Amarelo | Azul (C agora é amarelo) |
| Bands `description` | Não exibia | Mostra ao lado do badge |
