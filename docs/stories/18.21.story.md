# Story 18.21 — Dashboard: Ticket Médio por Origem (Pago/Orgânico/Sem Track)

## 📖 User Story

Como gestor de tráfego, quero ver o Ticket Médio das vendas separado por origem (Pago, Orgânico, Sem Track) abaixo do card "Venda Ingressos", para que eu possa entender o valor médio de cada tipo de cliente e otimizar investimento baseado em ticket, não apenas em volume.

---

## 🎯 Acceptance Criteria

### AC1: Cálculo de Ticket Médio Correto
- [ ] Ticket Médio = Faturamento Total / Número de Vendas (por origem)
- [ ] Faturamento Total = soma de `valor_bruto` ou `valor_liquido` (conforme stage de vendas)
- [ ] Número de Vendas = contagem de linhas deduplicadas (1 email = 1 venda)
- [ ] Fórmula: `ticket_medio = faturamento_total / total_vendas`

### AC2: Separa por Origem (3 Breakdowns)
- [ ] **Pago:** `utm_source = 'meta'` → "Pago"
- [ ] **Orgânico:** `utm_source` != 'meta' E utm_source não é vazio → "Orgânico"
- [ ] **Sem Track:** `utm_source` vazio ou null → "Sem Track"
- [ ] Mesmo matching logic usada em Story 18.7 (Faturamento por Canal)

### AC3: Renderizar Abaixo do Card Venda Ingressos
- [ ] 3 cards/boxes exibindo:
  - Card 1: "Ticket Pago" → R$ XXXX,XX
  - Card 2: "Ticket Orgânico" → R$ XXXX,XX
  - Card 3: "Ticket Sem Track" → R$ XXXX,XX
- [ ] Layout alinhado com padrão do dashboard (similar aos cards de Venda)
- [ ] Fonte, tamanho, espaçamento mantêm consistência

### AC4: Filtro de Data Funciona
- [ ] Quando DayRangePicker muda, Ticket Médio recalcula automaticamente
- [ ] Dados refetch via API com novo intervalo de datas
- [ ] Se intervalo não contém vendas, mostrar "—" ou "0" (conforme padrão)

### AC5: Dados da Planilha de Vendas
- [ ] Lê `valor_bruto` (ou `valor_liquido`) do stage Sales (subtype capture/main_product)
- [ ] Lê `utm_source` da coluna mapeada
- [ ] Deduplica por email (mesma lógica de Story 18.7)
- [ ] Se coluna não está mapeada, feature é desabilitada gracefully

### AC6: Edge Cases
- [ ] Se total vendas = 0 → mostrar "—"
- [ ] Se faturamento = 0 → mostrar "R$ 0,00"
- [ ] Se uma origem não tem vendas → "R$ 0,00" ou "—"
- [ ] Arredonda para 2 casas decimais (padrão monetário)

### AC7: Sem Regressão
- [ ] Card "Venda Ingressos" continua funcionando
- [ ] Outros elementos do dashboard não são afetados
- [ ] Performance não degrada

## 🎬 Implementation Notes

### Reuse Matching Logic from Story 18.7
Story 18.7 já implementou matching de `utm_source`:
```javascript
const fonte = utmSource === "meta" ? "Pago" : utmSource ? "Orgânico" : "Sem Track";
```

**Reutilize exatamente essa lógica para garantir consistência.**

### Calculation Example
```
Pago:
  - Total Faturamento Bruto: R$ 500.000
  - Total de Vendas: 125
  - Ticket Médio = 500.000 / 125 = R$ 4.000

Orgânico:
  - Total Faturamento Bruto: R$ 200.000
  - Total de Vendas: 80
  - Ticket Médio = 200.000 / 80 = R$ 2.500

Sem Track:
  - Total Faturamento Bruto: R$ 50.000
  - Total de Vendas: 25
  - Ticket Médio = 50.000 / 25 = R$ 2.000
```
