## 📖 User Story

Como gestor de tráfego, quero ver o Ticket Médio das vendas separado por origem (Pago, Orgânico, Sem Track) abaixo do card "Venda Ingressos", para que eu possa entender o valor médio de cada tipo de cliente e otimizar investimento baseado em ticket, não apenas em volume.

---

## 🎯 Acceptance Criteria

### AC1: Cálculo de Ticket Médio Correto
- [x] Ticket Médio = Faturamento Total / Número de Vendas (por origem)
- [x] Faturamento Total = soma de `valor_bruto` ou `valor_liquido` (conforme stage de vendas)
- [x] Número de Vendas = contagem de linhas deduplicadas (1 email = 1 venda)
- [x] Fórmula: `ticket_medio = faturamento_total / total_vendas`

- [x] **Pago:** `utm_source = 'meta'` → "Pago"
- [x] **Orgânico:** `utm_source` != 'meta' E utm_source não é vazio → "Orgânico"
- [x] **Sem Track:** `utm_source` vazio ou null → "Sem Track"
- [x] Mesmo matching logic usada em Story 18.7 (Faturamento por Canal)

  - Card 1: "Ticket Pago" → R$ XXXX,XX

  - Card 3: "Ticket Sem Track" → R$ XXXX,XX
- [x] Layout alinhado com padrão do dashboard (similar aos cards de Venda)
- [x] Fonte, tamanho, espaçamento mantêm consistência

### AC4: Filtro de Data Funciona
- [x] Quando DayRangePicker muda, Ticket Médio recalcula automaticamente
- [x] Dados refetch via API com novo intervalo de datas
- [x] Se intervalo não contém vendas, mostrar "—" ou "0" (conforme padrão)

### AC5: Dados da Planilha de Vendas
- [x] Lê `valor_bruto` (ou `valor_liquido`) do stage Sales (subtype capture/main_product)
- [x] Lê `utm_source` da coluna mapeada
- [x] Deduplica por email (mesma lógica de Story 18.7)
- [x] Se coluna não está mapeada, feature é desabilitada gracefully

### AC6: Edge Cases
- [x] Se total vendas = 0 → mostrar "—"
- [x] Se faturamento = 0 → mostrar "R$ 0,00"

### AC7: Sem Regressão

## 🎬 Implementation Notes

### Reuse Matching Logic from Story 18.7
Story 18.7 já implementou matching de `utm_source`:
```javascript
const fonte = utmSource === "meta" ? "Pago" : utmSource ? "Orgânico" : "Sem Track";
```

**Reutilize exatamente essa lógica para garantir consistência.**

### Calculation Example
```
Pago:
  - Total Faturamento Bruto: R$ 500.000
  - Total de Vendas: 125
  - Ticket Médio = 500.000 / 125 = R$ 4.000

Orgânico:
  - Total Faturamento Bruto: R$ 200.000
  - Total de Vendas: 80
  - Ticket Médio = 200.000 / 80 = R$ 2.500

Sem Track:
  - Total Faturamento Bruto: R$ 50.000
  - Total de Vendas: 25
  - Ticket Médio = 50.000 / 25 = R$ 2.000
```

### Rendering Position
Cards devem aparecer **imediatamente abaixo** do card "Venda Ingressos":

```
[Card] Venda Ingressos
  Total: 230 vendas
  Faturamento Bruto: R$ 750.000
  Faturamento Líquido: R$ 700.000

[Card] Ticket Pago        [Card] Ticket Orgânico        [Card] Ticket Sem Track
    R$ 4.000                  R$ 2.500                          R$ 2.000
```
