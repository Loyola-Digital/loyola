## Story

**As a** gestor de trafego da Loyola Digital,
**I want** the dashboard to cross-reference Meta Ads campaign data with real leads from the Google Sheets CRM via UTM parameters,
**so that** I can see real conversions, CPL (cost per lead), and identify which campaigns, ad sets, and ads are actually generating leads — not just clicks.

---

## Acceptance Criteria

1. [ ] Service `traffic-analytics.ts`: cruza dados Meta Ads + Google Sheets leads via UTM matching
2. [ ] Match logic: UTM Campaign ↔ campaign name/id, UTM Medium ↔ adset name/id, UTM Content ↔ ad name/id
3. [ ] Match flexivel: case-insensitive, trim whitespace, ignore leading/trailing spaces
4. [ ] `GET /api/traffic/analytics/:projectId/overview?days=30` — resumo: total spend, total leads, CPL medio, total campanhas
5. [ ] `GET /api/traffic/analytics/:projectId/campaigns?days=30` — campanhas com metricas de midia + leads + CPL
6. [ ] `GET /api/traffic/analytics/:projectId/adsets?campaignId=X&days=30` — ad sets com leads + CPL
7. [ ] `GET /api/traffic/analytics/:projectId/ads?adsetId=X&days=30` — ads com leads + CPL
8. [ ] Resposta de cada rota inclui: nome | spend | impressions | clicks | CTR | CPC | CPM | leads | CPL
9. [ ] CPL calculado: spend / leads (retorna null se leads = 0)
10. [ ] Dashboard `/traffic` atualizado: tabela de campanhas agora mostra colunas Leads e CPL quando projeto tem Sheets conectado
11. [ ] Indicador visual no dashboard: badge "CRM Conectado" (verde) ou "Apenas Midia" (cinza) por projeto
12. [ ] Leads sem UTM atribuivel mostrados como "Sem atribuicao" em linha separada na tabela
13. [ ] Cache de dados cruzados: TTL 15min (invalidar ao re-mapear abas no settings)
14. [ ] Drill-down de campanhas agora inclui leads/CPL em cada nivel (campanha → ad set → ad)
15. [ ] `pnpm typecheck` e `pnpm lint` passam sem erros

### UTM Matching Logic

```typescript
// O match entre Meta Ads e Google Sheets e feito assim:
//
// 1. Buscar campanhas do Meta Ads (com metricas)
// 2. Buscar dados da aba "leads" do Sheets (com column mapping)
// 3. Para cada lead:
//    - Extrair UTM Campaign, UTM Medium, UTM Content do lead
//    - Normalizar: lowercase, trim, remove whitespace extra
```

### Normalizacao de UTMs

```typescript
function normalizeUtm(value: string | undefined | null): string {
  if (!value) return "";
  return value.trim().toLowerCase().replace(/\s+/g, " ");
}

function matchUtmToCampaign(
  utmValue: string,
  campaignName: string,
  campaignId: string
): boolean {
  const normalized = normalizeUtm(utmValue);
  return (
    normalized === normalizeUtm(campaignName) ||
    normalized === normalizeUtm(campaignId)
  );
}
```
