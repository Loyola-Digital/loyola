# Story 18.31: Tabela Dados Diários - Condicionais por Etapa (Captação Paga vs Gratuita)

**Epic:** 18  
**Status:** Ready for Review  
**Assignee:** @dev  
**Branch:** feature/18.31-dados-diarios-stage-conditionals

## Story

Tabela Dados Diários deve exibir nomenclaturas diferentes conforme a etapa (stage) selecionada:
- **Captação Gratuita:** Manter nomenclaturas atuais (Total Leads, Leads Pg, Leads Org, Leads s/ track)
- **Captação Paga:** Alterar nomenclaturas (Total Ingressos, Ingressos Pg, Ingressos Org, Ingressos s/ track)

Atualizar tooltips condicionalmente também.

## Acceptance Criteria

- [x] AC1: Detectar stage atual (Captação Paga ou Captação Gratuita)
- [x] AC2: Em Captação Gratuita, manter nomenclatura:
  - Total Leads
  - Leads Pg
  - Leads Org
  - Leads s/ track
  - Tooltip original (Total Leads = Leads Pg + Leads Org + Leads s/ track)
- [x] AC3: Em Captação Paga, alterar nomenclatura:
  - Total Ingressos (em vez de Total Leads)
  - Ingressos Pg (em vez de Leads Pg)
  - Ingressos Org (em vez de Leads Org)
  - Ingressos s/ track (em vez de Leads s/ track)
- [x] AC4: Tooltip em Captação Paga atualizado:
  - Total Ingressos = Ingressos Pg + Ingressos Org + Ingressos s/ track
  - Ingressos Pg = Ingressos que vieram de mídia paga
  - Ingressos Org = Ingressos com origem orgânica
- [x] AC5: Transição suave entre etapas (sem flicker)
- [x] AC6: Dados numéricos não são afetados, apenas labels
