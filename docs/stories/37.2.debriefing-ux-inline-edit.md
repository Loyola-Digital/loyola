## 📖 Story

**Como** membro do time usando a aba Debriefing (37.1),
**Quero** ver o doc HTML em altura total (sem caixinha com scroll interno), editar o conteúdo clicando direto no documento renderizado, e ter os comentários sempre visíveis ao lado,
**Para** que o debriefing seja legível "igual ele é", correções sejam naturais (sem mexer em código) e a discussão do time não fique escondida abaixo da dobra.

_Feedback direto do Lucas sobre a 37.1: "melhorar a visibilidade do html", "quando clica em editar quero editar direto na tela", "faltou os comentários, não vi isso lá". Causa raiz dos itens 1 e 3: iframe fixo em 75vh com scroll interno esconde o fim da página._

---

## ✅ Acceptance Criteria

1. **Auto-altura do doc:** helper `lib/debriefing-frame.ts` injeta um script-agente no `srcDoc` que reporta a altura real do documento via `postMessage` (load + ResizeObserver + retries); a página ajusta a altura do iframe (mín. 400px) — o doc aparece inteiro, sem scrollbar interna. O parent valida `event.source === iframe.contentWindow`.
2. **Edição inline (WYSIWYG):** botão "Editar" liga o modo de edição — o iframe recarrega com `document.designMode = "on"`; o usuário clica no texto renderizado e altera direto. Banner de modo de edição com "Salvar" e "Cancelar". Salvar → parent pede o HTML via `postMessage`, o agente devolve o documento serializado (SEM o script injetado, preservando DOCTYPE) → `PUT /api/debriefings/:id { html }` → toast + volta ao modo leitura. Cancelar descarta.
3. **Sandbox preservado:** iframe continua `sandbox="allow-scripts"` sem `allow-same-origin` em ambos os modos — sem `dangerouslySetInnerHTML`.
4. **Comentários em destaque:** layout de 2 colunas no desktop (`lg:`) — doc à esquerda, comentários em coluna lateral direita sempre visível (sticky); empilhado no mobile. Contagem de comentários visível no header do detalhe.
5. **Dialog de edição simplificado:** o dialog perde a textarea de código-fonte (substituída pela edição inline) e vira "Renomear / substituir arquivo", acessível por menu (⋮) junto com Excluir.
6. **Qualidade:** `pnpm --filter @loyola-x/web build` compila com types ok; lint sem erros novos; API intocada.

| Risk | Severity | Mitigation |
|------|----------|-----------|
| Script-agente vazar no HTML salvo | MEDIUM | Serialização remove o nó por id antes de responder; id único `__loyola_debriefing_agent__`. |
| HTML normalizado pelo browser ao salvar (atributos reordenados) | LOW | Aceitável — conteúdo/estilos preservados; documentado. |
| Doc gigante → página muito longa | LOW | Altura clampada em 20000px; comentários seguem visíveis na lateral sticky. |
