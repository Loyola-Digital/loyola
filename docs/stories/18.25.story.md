## Overview

**Objetivo:** Corrigir bug em produção (PR #168 — Story 18.24) onde tabela de desempenho de criativos não agrupa anúncios com mesmo `ad_name` e exibe totalizadores incorretos.

**Problema:** Implementação agrupa por `ad_id` mas deveria agrupar por `ad_name`. Anúncios duplicados aparecem como linhas separadas, causando somatório errado dos totalizadores (R$ 121K vs R$ 5.1K esperado).

**Impacto:** Valores totais mostrados no dashboard estão errados, quebrando confiabilidade de análise de ROI/CPA por criativo.

## Technical Notes

### Problema Raiz
Endpoint retorna array de ads agrupado por `ad_id`:
```typescript
for (const ad of allAds) {
  const adId = ad.ad_id; // ← agrupa por ad_id
  creatives.push({
    adId,
    adName: ad.ad_name,
    spend: parseFloat(ad.spend || "0"),
    // ... outras métricas
  });
}
```

Deveria agrupar por `ad_name` e agregar múltiplos `ad_id`:
```typescript
const groupedByName = new Map<string, {spend: number; leads: number; ...}>();
// agrupa todos ads por ad_name
// soma spend, leads, revenue, etc
```

### Validação de Temperatura
Após agrupamento, se houver múltiplos `ad_id` com mesmo `ad_name`:
- `utm_term` pode ter múltiplos valores
- Usar **moda** (valor mais frequente) para classificar como hot/cold
- Se split entre hot/cold, considerar como "misto" (ambos filtros devem mostrar)
