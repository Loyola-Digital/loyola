## Dev Notes

### Dockerfile Existente

O Dockerfile ja existe em `packages/api/Dockerfile` (Story 1.1.1):
```dockerfile
FROM node:20-alpine AS base
RUN corepack enable && corepack prepare pnpm@9 --activate

FROM base AS build
WORKDIR /app
COPY pnpm-lock.yaml pnpm-workspace.yaml package.json ./
COPY packages/api/package.json packages/api/
COPY packages/shared/package.json packages/shared/
RUN pnpm install --frozen-lockfile
COPY packages/api packages/api
COPY packages/shared packages/shared
RUN pnpm --filter @loyola-x/api build

FROM base AS runtime
WORKDIR /app
COPY --from=build /app/packages/api/dist ./dist
COPY --from=build /app/node_modules ./node_modules
COPY squads ./squads
EXPOSE 3001
CMD ["node", "dist/server.js"]
```

**Nota:** O `EXPOSE 3001` e apenas documentacao — Railway injeta `$PORT` e o server.ts ja le `process.env.PORT`.

[Source: packages/api/Dockerfile]

### Server Startup

O `server.ts` faz `app.listen({ port: config.PORT, host: config.HOST })`. As env vars PORT e HOST sao lidas via Zod validated config.

[Source: packages/api/src/server.ts, packages/api/src/config/env.ts]

### Env Vars Schema

Todas as env vars sao validadas pelo Zod schema em `packages/api/src/config/env.ts`:
- `PORT` — number, default 3001
- `HOST` — string, default "0.0.0.0"
- `NODE_ENV` — enum development/production/test
- `DATABASE_URL` — string required
- `CLERK_SECRET_KEY` — string required
- `CLERK_PUBLISHABLE_KEY` — string required
- `CORS_ORIGIN` — string, default "http://localhost:3000"
- `MINDS_BASE_PATH` — string, default "./squads"
- `ANTHROPIC_API_KEY` — string required
- `CLICKUP_API_TOKEN` — string optional
- `CLICKUP_LIST_ID` — string optional

[Source: packages/api/src/config/env.ts]

### Health Check Endpoint

`GET /api/health` retorna `{ status: "ok", db: "connected" }` com verificacao de conexao ao DB.

[Source: packages/api/src/routes/health.ts]
