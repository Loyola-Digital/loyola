## Story

**As a** Loyola Digital platform,
**I want** to automatically sync user data from Clerk to the local PostgreSQL database via webhooks,
**so that** the system has a local user record for FK relationships, display data, and business logic without depending on Clerk API calls at runtime.

---

## Acceptance Criteria

1. [ ] `POST /api/webhooks/clerk` endpoint no backend Fastify
2. [ ] Valida webhook signature usando Clerk Webhook Secret (via `svix` library)
3. [ ] Handles evento `user.created` â€” INSERT na tabela `users` (clerk_id, email, name, avatar_url)
4. [ ] Handles evento `user.updated` â€” UPDATE email, name, avatar_url na tabela `users`
5. [ ] Handles evento `user.deleted` â€” DELETE user (cascade limpa conversations e delegated_tasks)
6. [ ] Endpoint NAO requer JWT auth (webhook vem do Clerk, nao do user)
7. [ ] Idempotente: re-processar mesmo evento nao causa erro (upsert on conflict)
8. [ ] Retorna 200 para o Clerk em todos os casos (evita retry infinito)
9. [ ] Env var `CLERK_WEBHOOK_SECRET` adicionada ao schema Zod de env config
10. [ ] `pnpm typecheck` passa sem erros
11. [ ] `pnpm lint` passa sem erros

### Clerk Webhook â€” Signature Verification

Clerk usa `svix` para assinar webhooks. Os headers sao:
- `svix-id` â€” ID unico do evento
- `svix-timestamp` â€” timestamp do evento
- `svix-signature` â€” assinatura HMAC

A library `svix` fornece `Webhook.verify(body, headers)` que valida automaticamente.

**IMPORTANTE:** O `svix.Webhook.verify()` precisa do body como **string raw**, nao como JSON parseado. O Fastify parseia JSON automaticamente, entao precisamos capturar o raw body antes do parsing via `addContentTypeParser` ou `preParsing` hook.

[Source: Clerk docs â€” Webhooks]

### Cascade Behavior

A tabela `conversations` tem `userId` com `onDelete: 'cascade'`, e `delegated_tasks` referencia `conversations`. Ao deletar um user, todas as conversations e tasks sao removidas automaticamente pelo PostgreSQL.

[Source: packages/api/src/db/schema.ts]

- Nao ha necessidade de queue/background job para este volume
