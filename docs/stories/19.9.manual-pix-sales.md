CREATE TABLE IF NOT EXISTS manual_sales (

## Risks

- **Vendedor deletado da plataforma:** FK `ON DELETE SET NULL` preserva a venda. `seller_name` snapshot garante que UI continua exibindo nome. Risco baixo.
- **ConfusГЈo com vendas da planilha:** UsuГЎrio pode achar que vendas manuais somam com planilha. Resolver com **tГ­tulo claro** "Vendas PIX Direto" + separaГ§ГЈo visual forte (border / spacing). Tooltip opcional.
- **LanГ§amento duplicado:** NГЈo hГЎ dedupe automГЎtico. UsuГЎrio responsГЎvel por nГЈo lanГ§ar 2x. AceitГЎvel pra v1.
- **Performance:** Tabela cresce no tempo. Index em `(stage_id, sale_date)` + filtro de `days` mantГ©m queries leves. Sem paginaГ§ГЈo na v1 (atГ© 1k vendas/etapa Г© OK).
- **`value` em numeric vs centavos int:** Optei por `numeric(12,2)` por consistГЄncia com `parseNumber` da planilha (que retorna float). Trade-off: pequena perda de precisГЈo em ops aritmГ©ticas вЂ” aceitГЎvel pra valores monetГЎrios atГ© R$ 10 bilhГµes.
