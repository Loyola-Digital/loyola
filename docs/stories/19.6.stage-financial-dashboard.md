# Story 19.6 — Dashboard Financeiro no Meta Ads (Etapas Pagas)

**Epic:** 19 — Funnel Stages  
**Status:** Ready  
**Points:** 5  
**Depende de:** Stories 19.4, 19.5

---

## User Story

Como usuário de uma etapa paga com planilhas de vendas conectadas, quero ver uma seção financeira **dentro da aba Meta Ads** com dados de faturamento, canal de origem e forma de pagamento, para cruzar investimento em tráfego com resultado de vendas.

---

## Acceptance Criteria

- [ ] AC1: API: endpoint `GET /api/projects/:projectId/funnels/:funnelId/stages/:stageId/sales-data?subtype=capture|main_product&days=N` que lê a planilha conectada, agrupa por email (1 venda por email, soma faturamento da mesma pessoa), retorna métricas agregadas
- [ ] AC2: Agrupamento por email: linhas com mesmo email = 1 venda, `valorBruto` e `valorLiquido` somados, `formaPagamento` e `canalOrigem` do registro mais recente (ou mais frequente)
- [ ] AC3: Resposta inclui: `totalVendas`, `faturamentoBruto`, `faturamentoLiquido`, `ticketMedioBruto`, `ticketMedioLiquido`, `porCanal: [{canal, vendas, bruto, liquido}]`, `porFormaPagamento: [{forma, vendas, bruto, liquido}]`
- [ ] AC4: No `LaunchDashboard` e `PerpetualDashboard`, quando `stage.stageType === 'paid'` e há planilha de captação conectada, exibir seção "Vendas de Captação" com:
  - Cards: Total Vendas, Faturamento Bruto, Faturamento Líquido, Ticket Médio Líquido
  - Tabela: por Canal de Origem
  - Tabela: por Forma de Pagamento
- [ ] AC5: Se planilha de produto principal também estiver conectada, exibir seção "Produto Principal" com os mesmos cards/tabelas
- [ ] AC6: DayRangePicker existente controla o período das duas seções (usar campo de data da planilha — mapeável como `dataVenda` opcional nas colunas)
- [ ] AC7: Empty state quando `stageType !== 'paid'` ou planilha não conectada
- [ ] AC8: Hook `useStageSalesData(projectId, funnelId, stageId, subtype, days)` no web

## API Response Shape

```ts
interface StageSalesData {
  totalVendas: number;
  faturamentoBruto: number;
  faturamentoLiquido: number;
  ticketMedioBruto: number;
  ticketMedioLiquido: number;
  porCanal: { canal: string; vendas: number; bruto: number; liquido: number }[];
  porFormaPagamento: { forma: string; vendas: number; bruto: number; liquido: number }[];
  semDados: boolean; // true quando planilha não conectada ou sem linhas no período
}
```
