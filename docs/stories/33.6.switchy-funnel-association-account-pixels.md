## 📖 Story

**Como** gestor de tráfego do projeto,
**Quero** que os shortlinks gerados pelo Switchy fiquem atrelados ao **funil** (e que o gerador + a lista de links vivam dentro da página do funil), e poder **selecionar** os pixels já cadastrados na conta Switchy ao invés de digitar pixel manualmente,
**Para** que cada funil tenha seu próprio conjunto de links rastreados e que os pixels anexados sejam exatamente os do expert correto, sem erro de digitação.

---

## ✅ Acceptance Criteria

1. **Coluna `funnel_id` em `switchy_shortened_links`**
   - Migration `0049_switchy_funnel_link.sql` adiciona `funnel_id uuid` (nullable) na tabela `switchy_shortened_links`.
   - FK `funnel_id` → `funnels(id)` com `ON DELETE SET NULL` (apagar o funil NÃO apaga o histórico de links — só desvincula).
   - Índice `idx_switchy_links_funnel` em `funnel_id`.
   - Migration idempotente: `ADD COLUMN IF NOT EXISTS`, guard de constraint via `DO $$`, `CREATE INDEX IF NOT EXISTS`.
   - Script `apply-migration-0049.mjs` (node + pg) no padrão do `apply-migration-0048.mjs` — **NÃO executado** nesta story.

2. **Schema Drizzle**
   - `switchyShortenedLinks` ganha `funnelId: uuid("funnel_id").references(() => funnels.id, { onDelete: "set null" })`.
   - Índice `idx_switchy_links_funnel` declarado no array de índices da tabela.
   - `funnels` já está definido no mesmo módulo (`schema.ts`) — sem novo import.

3. **Service — `fetchSwitchyPixels`** (`services/switchy.ts`)
   - Nova função `fetchSwitchyPixels(token): Promise<SwitchyAccountPixel[]>` que usa o `graphql<T>()` existente com a query `{ pixels { id platform value title workspaceId } }`.
   - Novo tipo exportado `SwitchyAccountPixel = { id, platform, value, title, workspaceId }`.
   - Mesma auth (`Api-Authorization`) do restante do GraphQL. Conta GLOBAL (1 token) — a query retorna pixels de vários experts.

4. **Rota — `GET /switchy/pixels`**
   - `GET /api/projects/:projectId/switchy/pixels` → `{ pixels }` via `fetchSwitchyPixels(getSwitchyToken())`.
   - Reusa `getProjectAccess` (404 sem acesso, 400 id inválido). Acessível por guest (leitura).
   - Erro do Switchy → `502 { error: "Falha ao buscar pixels do Switchy" }`, logado via `fastify.log.error`.

5. **`POST /switchy/generate` aceita `funnelId`**
   - `generateBodySchema` aceita `funnelId: z.string().uuid().optional()`.
   - Ao persistir cada linha em `switchy_shortened_links`, grava `funnelId: body.funnelId ?? null`.
   - Não quebra chamadas existentes (campo opcional).

6. **`GET /switchy/links/history` filtra por `funnelId`**
   - `historyQuerySchema` aceita `funnelId: z.string().uuid().optional()`.
   - Quando presente, filtra `eq(switchyShortenedLinks.funnelId, funnelId)` combinado com o `projectId` via `and(...)`.
   - Quando ausente, comportamento atual preservado (todos os links do projeto).

7. **Não-regressão**
   - Os 13 testes do builder de URL (`switchy.test.ts`) continuam passando.
   - `typecheck`, `lint` e `test` do pacote `@loyola-x/api` verdes.
   - Nenhuma rota/assinatura existente quebrada (incremento puro).

## ⚠️ Risks & Mitigations

| Risk | Severity | Mitigation |
|------|----------|-----------|
| FK `funnel_id` com `ON DELETE` errado apagaria histórico ao remover funil | HIGH | `ON DELETE SET NULL` — funil apagado só desvincula, link permanece. |
| Migration re-rodada quebrar (coluna/constraint já existe) | MEDIUM | `ADD COLUMN IF NOT EXISTS`, guard `DO $$` por `information_schema`, `CREATE INDEX IF NOT EXISTS`. |
| Quebrar chamadas existentes do generate/history | HIGH | `funnelId` é `.optional()` em ambos os schemas; persistência usa `?? null`; filtro só aplica quando presente. |

### Decisões do Lucas
1. Links atrelados ao **funil** (`funnel_id`).
2. Gerador + lista de links vivem **dentro da página do funil** (funil implícito, sem picker no frontend).
3. Pixels: **não digitar manual** — selecionar da lista de pixels já cadastrados no Switchy. Switchy **não** herda pixel de folder/workspace (campo `pixels[]` é por-link), então buscamos via GraphQL e anexamos no create.

### GraphQL `pixels` (confirmado por introspection autenticada)
- Endpoint: `https://graphql.switchy.io/v1/graphql` (já usado em `services/switchy.ts`).
- Query: `{ pixels { id platform value title workspaceId } }`.
- Conta GLOBAL (1 token): retorna pixels de vários experts (ex.: `PX | Dr Pacheco` facebook, `GTM - Pacheco` gtm).
- Header de auth: `Api-Authorization` (igual ao GraphQL já usado).

### Migration (idempotente)
```sql
ALTER TABLE "switchy_shortened_links" ADD COLUMN IF NOT EXISTS "funnel_id" uuid;
-- FK via DO $$ guard (information_schema.table_constraints) → ON DELETE set null
CREATE INDEX IF NOT EXISTS "idx_switchy_links_funnel" ON "switchy_shortened_links" USING btree ("funnel_id");
```

### Schema Drizzle
```ts
funnelId: uuid("funnel_id").references(() => funnels.id, { onDelete: "set null" }),
// ...
index("idx_switchy_links_funnel").on(table.funnelId),
```

### Service
```ts
export interface SwitchyAccountPixel {
  id: string; platform: string; value: string; title: string; workspaceId: string | null;
}
export async function fetchSwitchyPixels(token: string): Promise<SwitchyAccountPixel[]> {
  const data = await graphql<{ pixels: SwitchyAccountPixel[] }>(
    token, `{ pixels { id platform value title workspaceId } }`,
  );
  return data.pixels;
}
```

### Rota / schemas
- `GET /switchy/pixels` adicionada após `GET /switchy/links` (mesmo padrão de access/erro 502).
- `generateBodySchema`: `+ funnelId: z.string().uuid().optional()`; insert grava `funnelId: body.funnelId ?? null`.
- `historyQuerySchema`: `+ funnelId: z.string().uuid().optional()`; `where` vira `and(eq(projectId), eq(funnelId))` quando presente.
