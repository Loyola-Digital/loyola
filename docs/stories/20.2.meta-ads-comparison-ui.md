# Story 20.2 — Meta Ads com Métricas Comparativas

**Epic:** 20 — Funil Comparativo  
**Status:** Ready for Review  
**Points:** 8  
**Depende de:** Story 20.1

---

## User Story

Como usuário com um funil de comparação vinculado, quero ver no Meta Ads as métricas do funil atual lado a lado com as métricas equivalentes do funil de referência, para saber se estou melhor ou pior nos primeiros N dias em comparação com a campanha anterior.

---

## Acceptance Criteria

- [x] AC1: Novo endpoint `GET /api/projects/:projectId/funnels/:funnelId/stages/:stageId/meta-ads-comparison?days=N`:
  - Busca `funnel.compareFunnelId`; se nulo → retorna `{ semDados: true }`
  - Busca o funil de comparação e o nome dele
  - Encontra stage correspondente: `funnel_stages WHERE funnel_id = compareFunnelId AND stage_type = currentStage.stageType LIMIT 1`
  - Se nenhuma stage correspondente → retorna `{ semDados: true, reason: "no_matching_stage" }`
  - Busca os dados Meta Ads da stage correspondente (usa `metaAccountId` + `campaigns` do funil de comparação + campanhas da stage de comparação)
  - Agrega todos os dias do período (N dias), ordena por data ASC, indexa como Dia 1, Dia 2… Dia N
  - Retorna `MetaAdsComparisonData` (shape abaixo)

- [x] AC2: Response shape `MetaAdsComparisonData` adicionado ao `@loyola-x/shared`:
  ```ts
  interface ComparisonDayMetrics {
    dayIndex: number;           // 1..N
    impressions: number;
    clicks: number;
    spend: number;
    reach: number;
    ctr: number;
    cpc: number;
  }
  interface MetaAdsComparisonData {

    totals: {
      impressions: number;
      clicks: number;
      spend: number;
      reach: number;
    };
    semDados: boolean;
    reason?: string;
  }
  ```

  - Se comparação ausente ou `semDados: true` → cards sem mudança visual (comportamento atual)

## Dev Notes

### Endpoint — busca do meta account e campanhas do stage de comparação
O `metaAccountId` e o token ficam no **funil** de comparação (não na stage). As campanhas podem estar tanto no funil de comparação quanto na stage de comparação (`stage.campaigns`). Prioridade:
- Se a stage de comparação tiver `campaigns` preenchido → usar campanhas da stage (igual ao fluxo da story 19)
- Senão → usar campanhas do funil de comparação

Seguir o mesmo padrão de `stageAsFunnel` da Story 19: construir um objeto funnel-like a partir do funil de comparação + stage de comparação para aproveitar a lógica existente de fetch.

### Normalização de dias
```ts
// Ordenar por date_start ASC
const sorted = rawInsights.sort((a, b) => a.date_start.localeCompare(b.date_start));
// Indexar a partir de 1
const days = sorted.map((item, idx) => ({
  dayIndex: idx + 1,
  impressions: Number(item.impressions),
  clicks: Number(item.clicks),
  spend: Number(item.spend),
  reach: Number(item.reach),
  ctr: Number(item.ctr),
  cpc: Number(item.cpc),
}));
```

### Delta percentual
```ts
// delta > 0 = melhor (verde), delta < 0 = pior (vermelho)
const delta = current === 0 ? 0 : ((current - comparison) / Math.abs(comparison)) * 100;
```
Para métricas de custo (spend, cpc, cpm): delta negativo = melhor (gastou menos). Para impressões, cliques: delta positivo = melhor. A UI deve tratar isso corretamente — levantar com o desenvolvedor se necessário.

### Localização dos cards de KPI
Verificar nos componentes `LaunchDashboard` e `PerpetualDashboard` onde os cards de métricas Meta Ads são renderizados. Provavelmente em um componente separado (ex: `MetaAdsInsightsCards` ou similar). Identificar e modificar para aceitar props opcionais de comparação.
