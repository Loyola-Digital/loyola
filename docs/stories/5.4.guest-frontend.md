## Story

**As a** guest user (client),
**I want** to see only the projects and modules I was invited to,
**so that** I have a focused experience without access to internal Loyola data.

---

## Acceptance Criteria

**Fluxo de convite (admin):**
1. [ ] Dialog "Convidar pessoa" dentro da overview do projeto (`/projects/[id]`)
   - Campo: e-mail do convidado
   - Toggles por módulo: Instagram, Conversas, Mind (todos on por default)
   - Botão "Enviar convite" → `POST /api/projects/:id/invitations`
   - Exibe URL do convite para admin copiar
2. [ ] Tela de membros do projeto exibe cada guest com suas permissões atuais
3. [ ] Admin pode editar permissões inline (toggles) via `PATCH .../permissions`
4. [ ] Admin pode remover guest ("Remover acesso") via `DELETE .../members/:userId`

**Fluxo de aceite (guest):**
5. [ ] Página `/invite/[token]` exibe: nome do projeto, quem convidou, e-mail, módulos liberados
6. [ ] Botão "Aceitar convite" → `POST /api/invitations/:token/accept` → redirect para `/projects/[id]`
7. [ ] Token expirado → mensagem "Este convite expirou. Peça um novo ao seu gestor."
8. [ ] Token inválido → mensagem de erro

**Experiência do guest:**
9. [ ] Sidebar do guest (`GuestSidebar`) renderiza apenas projetos onde é membro
10. [ ] Dentro de cada projeto, exibe apenas módulos com `permissions[module] = true`
11. [ ] Sem Settings, sem Tasks, sem navegação global, sem outros projetos
12. [ ] Guest tentando acessar módulo bloqueado via URL → redirect com toast "Você não tem acesso a este módulo"
13. [ ] Layout não exibe botões de admin (ex: "+ Novo Projeto", "+ Convidar", "+ Adicionar conta")
14. [ ] `useUserRole()` hook retorna o role do usuário autenticado
15. [ ] `app-sidebar.tsx` renderiza `GuestSidebar` se `role === 'guest'`, `AppSidebar` se admin
16. [ ] `middleware.ts` do Next.js: guest que tenta acessar rota global → redirect para `/projects` (primeiro projeto)

### useUserRole Hook

```typescript

import { useUser } from "@clerk/nextjs";
export function useUserRole() {
  const { user } = useUser();
  return user?.publicMetadata?.role as string ?? "admin";
}
```
