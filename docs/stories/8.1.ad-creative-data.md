## Story

**As a** gestor de trafego da Loyola Digital,
**I want** the system to fetch creative data (thumbnail, image, title, body, type) from the Meta Ads API for each ad,
**so that** I can see the actual creative assets alongside performance metrics instead of just ad names.

---

## Acceptance Criteria

1. [ ] Nova funcao `fetchAdCreatives(metaAccountId, accessToken, adIds[])` em `meta-ads.ts` — busca `/{adId}/adcreatives?fields=thumbnail_url,image_url,title,body,link_url,call_to_action_type,object_type,video_id` para cada ad
2. [ ] Novo tipo `MetaAdCreative`: `{ adId, thumbnailUrl, imageUrl, title, body, linkUrl, ctaType, objectType, videoId }`
3. [ ] Batching: agrupar ate 50 ad IDs por request usando batch API do Meta (ou requests paralelos com rate limit)
4. [ ] `getTopPerformers()` atualizado — retorna campo `creative: MetaAdCreative | null` junto com cada TopPerformerAd
5. [ ] `getProjectAdAnalytics()` atualizado — retorna `creative` junto com cada ad no drill-down
6. [ ] Nova rota `GET /api/traffic/analytics/:projectId/ad-creatives?adIds=id1,id2,...` — fallback para buscar criativos sob demanda
7. [ ] Cache de criativos separado com TTL 1h (criativos mudam raramente)
8. [ ] Tratamento graceful: se criativo nao encontrado, retorna `creative: null` (nunca quebra o fluxo)
9. [ ] Rate limit respeitado: criativos sao puxados APENAS quando necessario (top performers, drill-down ads), nunca em batch de todas campanhas
10. [ ] Tipos exportados no hook `use-traffic-analytics.ts`: `TopPerformerAd` ganha campo `creative`
11. [ ] `pnpm typecheck` e `pnpm lint` passam

## Dev Notes

### Architecture Context

- **Meta Ads service:** `packages/api/src/services/meta-ads.ts` — `fetchAds()` ja retorna `creative?: { id: string }` mas nao busca detalhes do criativo
- **Traffic analytics:** `packages/api/src/services/traffic-analytics.ts` — `getTopPerformers()` e `getProjectAdAnalytics()` ja retornam ads com metricas cruzadas
- **Hooks:** `packages/web/lib/hooks/use-traffic-analytics.ts` — tipos `TopPerformerAd` e `AdAnalyticsResponse`
- **Rate limit:** 200 req/h atual, criativos adicionam ~5-20 requests por sessao

### Meta API — Creative Endpoint

```
GET /{ad-id}?fields=creative{thumbnail_url,image_url,title,body,link_url,call_to_action_type,object_type,video_id}
```

Alternativa mais eficiente (field expansion — 1 request por ad, inclui creative inline):
```
GET /act_{id}/ads?fields=id,name,status,creative{thumbnail_url,image_url,title,body,link_url,call_to_action_type,object_type,video_id}
```

### Rate Limit Impact

| Cenario | Requests extras | Mitigacao |
|---------|----------------|-----------|
| Top Performers (5 ads) | +5 | Cache 1h |
| Drill-down ads (10-20 ads) | +10-20 | Lazy load, cache 1h |
| Ad creatives fallback route | +N (on-demand) | Batch max 50, cache 1h |

### Completion Notes

- Used field expansion approach (`/{adId}?fields=creative{...}`) — 1 request per ad, most efficient
- Creative cache is separate from analytics cache (1h TTL vs 15min)
- Batching: parallel requests in groups of 50, each with try/catch for graceful fallback
- Top performers fetch creatives only for final top N ads (not all ads)
- Drill-down ads fetch creatives for all visible ads in the adset
- Fallback route allows frontend to fetch creatives on-demand (max 50 per request)
