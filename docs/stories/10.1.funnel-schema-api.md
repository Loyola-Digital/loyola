## Story

**As a** gestor da Loyola Digital,
**I want** a funnel entity linked to my project that references a specific Meta Ads campaign,
**so that** I can organize my launches and evergreen funnels separately with dedicated analytics.

---

## Acceptance Criteria

### Database Schema

1. [x] Create `funnels` table with columns:
   - `id` (uuid, PK)
   - `project_id` (uuid, FK → projects)
   - `name` (varchar, not null) — e.g. "Lançamento Black Friday"
   - `type` (enum: 'launch' | 'perpetual', not null)
   - `meta_account_id` (uuid, FK → meta_ads_accounts, nullable)
   - `campaign_id` (varchar, nullable) — Meta Ads campaign ID
   - `campaign_name` (varchar, nullable) — denormalized for display
   - `created_at` (timestamptz)
   - `updated_at` (timestamptz)
2. [x] Add index on `project_id` for fast lookups
3. [x] Add unique constraint on `(project_id, campaign_id)` — one campaign per funnel per project
4. [x] Run migration via `drizzle-kit generate` + `drizzle-kit push`

### API Endpoints

5. [x] `GET /api/projects/:projectId/funnels` — List all funnels for a project
6. [x] `POST /api/projects/:projectId/funnels` — Create funnel (name, type, campaign_id, meta_account_id)
7. [x] `PUT /api/projects/:projectId/funnels/:funnelId` — Update funnel (name, type, campaign link)
8. [x] `DELETE /api/projects/:projectId/funnels/:funnelId` — Delete funnel
9. [x] `GET /api/projects/:projectId/funnels/:funnelId` — Get single funnel with details
10. [x] All endpoints require authentication and project membership

### Shared Types

11. [x] Add `Funnel` type to `@loyola-x/shared` package:
    ```typescript
    interface Funnel {
      id: string;
      projectId: string;
      name: string;
      type: 'launch' | 'perpetual';
      metaAccountId: string | null;
      campaignId: string | null;
      campaignName: string | null;
      createdAt: string;
      updatedAt: string;
    }
    ```
