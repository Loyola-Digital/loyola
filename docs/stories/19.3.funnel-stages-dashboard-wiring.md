# Story 19.3 — Web: Dashboard Wired à Etapa

**Epic:** EPIC-19 Funnel Stages
**Status:** Draft
**Priority:** High
**Estimate:** L
**Depends on:** Story 19.2 (navegação + CRUD de etapas)

---

## Story

Como gestor de tráfego, quero que cada etapa exiba o dashboard completo (Meta Ads, Google Ads, Pesquisas, Planilhas, Links) usando a configuração de campanhas daquela etapa específica, e que eu consiga configurar as campanhas de cada etapa diretamente pelo dashboard, para que cada frente do meu funil tenha seus próprios dados isolados.

## Contexto

Após a Story 19.2, `stages/[stageId]/page.tsx` é um placeholder. Esta story:
1. Move o conteúdo de `funnels/[funnelId]/page.tsx` (tabs + dashboards) para `stages/[stageId]/page.tsx`
2. Substitui a prop `funnel` nos dashboards pela prop `stage` (que tem os mesmos campos de campanhas)
3. Adiciona "Configurações da Etapa" — drawer/dialog para gerenciar campanhas Meta/Google/Switchy da etapa
4. Transforma `funnels/[funnelId]/page.tsx` em puro redirect/stage-list (já feito na 19.2 para list, agora também remove o antigo dashboard)

## Escopo

### IN
- Mover tabs do dashboard (Meta Ads, Meta Ads 2, YouTube, Pesquisas, Planilhas, Links) para `stages/[stageId]/page.tsx`
- Criar hook `useFunnelStage(projectId, funnelId, stageId)` para buscar etapa individual
- Adaptar `LaunchDashboard` e `PerpetualDashboard` para aceitar `FunnelStage` como config de campanhas (em vez de `Funnel`)
- `StageSettingsPanel` — botão "⚙ Configurar Etapa" no header da stage page, abre drawer com:
  - Renomear etapa
  - Gerenciar campanhas Meta (picker igual ao atual do funnel settings)
  - Gerenciar campanhas Google Ads
  - Gerenciar Switchy (folder picker + link picker)
- Breadcrumb na stage page: `[Funnel name] › [Stage name]`
- Funil com 1 etapa: o redirect da 19.2 já lida com isso — a UX é transparente
- `funnels/[funnelId]/page.tsx`: remover o antigo dashboard (agora é só stage-list + redirect)

### OUT
- Per-stage planilhas (Story 19.4) — planilhas continuam vinculadas ao funil, não à etapa
- Per-stage pesquisa Tally (Story 19.4)
- Consolidação entre etapas (sem totals cross-stage)

### Compatibilidade retroativa
- Funnels com 1 etapa "Principal" (todos os existentes após migration 19.1): transparente para o usuário, experiência idêntica à atual
- O `funnels/[funnelId]/page.tsx` com redirect automático garante que links existentes continuam funcionando

```tsx

const { data: stage, isLoading } = useFunnelStage(params.id, params.funnelId, params.stageId);

```

### Adaptação dos Dashboards

**Antes:**
```typescript
interface LaunchDashboardProps {
  funnel: Funnel;  // usa funnel.campaigns, funnel.metaAccountId, etc.
  projectId: string;
}
```

**Depois:**
```typescript
interface LaunchDashboardProps {
  funnel: Funnel;           // mantido para: funnel.id (planilhas), funnel.type (para display)
  stage: FunnelStage;       // usa stage.campaigns, stage.metaAccountId, etc.
  projectId: string;
}
```

`LaunchDashboard` e `PerpetualDashboard` passam a ler campanhas/metaAccountId/googleAds/switchy de `stage` em vez de `funnel`. O `funnel.id` continua sendo passado para os hooks de planilha (que ficam por funil até Story 19.4).

**Hooks afetados:** `useCrossedFunnelMetrics`, `useTrafficOverview`, `useTrafficCampaigns`, `useCampaignDailyInsights`, `useTopPerformers`, `useGoogleAdsCampaignPicker`, `useSwitchyFolders` — todos recebem campanhas de `stage` em vez de `funnel`.

### `StageSettingsPanel`

Botão discreto no header da stage page: `⚙ Configurar Etapa`

Abre `Sheet` (drawer lateral) com 4 seções colapsáveis:

**1. Nome da Etapa**
- Input de texto, salva com `useUpdateStage` on blur ou botão Salvar

**2. Campanhas Meta Ads**
- Mesmo componente/picker atual do funnel settings (já existe em `funnel-settings` ou inline no funil)
- Lista campanhas disponíveis, checkboxes, salva via `useUpdateStage`

**3. Campanhas Google Ads**
- Mesmo picker do Google Ads atual

**4. Links Switchy**
- Folder picker + link picker (já existe em `SwitchyLinksTab`)
- Reutilizar lógica existente adaptada para stage

> **Nota:** Este drawer substitui a necessidade de editar campanhas via página de configuração do funil. A config do funil-pai (funnels table) pode ser mantida como fallback histórico mas não é mais a fonte de verdade para campanhas.

## Technical Notes

- `LaunchDashboard` e `PerpetualDashboard` recebem `stage` como nova prop. Internamente, toda referência a `funnel.campaigns`, `funnel.metaAccountId`, `funnel.googleAdsCampaigns`, `funnel.googleAdsAccountId`, `funnel.switchyFolderIds`, `funnel.switchyLinkedLinks` muda para `stage.*`
- `funnel.id` e `funnel.type` ainda vêm de `funnel` (não mudam por etapa)
- O drawer de configuração pode reutilizar os pickers já existentes no projeto (ex: `CampaignSelector`, lógica do `FolderPickerDialog` do `SwitchyLinksTab`)
- Invalidar `["funnel-stage", projectId, funnelId, stageId]` após qualquer update no drawer para forçar refetch do dashboard
