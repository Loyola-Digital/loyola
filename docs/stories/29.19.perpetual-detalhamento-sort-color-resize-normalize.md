## Dev Notes

### Merge por nome base (AC5) — confirmar comportamento
Interpretei "atribuindo ao respectivo nome" como **agrupar** a cópia no nome base (ex.: "Campanha A" + "Campanha A — Cópia" viram uma linha "Campanha A" com métricas somadas). Se o Danilo quiser só **renomear** sem juntar (manter linhas separadas), é ajuste simples. Merge só acontece quando 2+ entidades caem no mesmo nome normalizado.

### Preserva CTR link em linha única
Só re-derivo CTR/CPC/CPM quando há merge real (2+ membros). Linha única mantém o valor original do Meta (que pode ser CTR de link, diferente de clicks/impressions).

### Ordenação client-side
Ordena o array já mergeado no front. Nulls tratados como −∞ (caem no fim em desc).
