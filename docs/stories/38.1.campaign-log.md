## 📖 Story

**Como** membro do time Loyola operando uma campanha (funil),
**Quero** uma seção fixa "Log de Campanha" dentro de cada funil onde registro rapidamente cada ação executada — disparo de e-mail, disparo de WhatsApp (X1/grupo/canal), SMS/ligação, publicação em rede social, ações no gerenciador de anúncios, automações etc. — com data/hora, ferramenta, categoria e observação,
**Para** substituir a planilha manual de log (hoje: colunas Data, Hora, Evento, Aplicativo, Categoria, Responsável, Observações) por um registro centralizado, rápido de lançar, filtrável e com autoria automática.

---

## ✅ Acceptance Criteria

1. **Schema — tabela nova `campaign_log_entries` em `packages/api/src/db/schema.ts`:**
   - `id` uuid PK; `funnelId` uuid NOT NULL → `funnels.id` (`onDelete: "cascade"`);
   - `occurredAt` timestamptz NOT NULL (data+hora da ação — a planilha separa Data e Hora; aqui é um campo só, default "agora" na UI);
   - `evento` varchar(80) NOT NULL; `aplicativo` varchar(80); `categoria` varchar(80); (valores dos vocabulários do AC 2, ou texto livre quando "Outro")
   - `notes` text (Observações / título da mensagem);
   - `responsavel` varchar(255) (opcional — default visual é o usuário logado; preenche só quando a ação foi de outra pessoa);
   - `createdBy` uuid NOT NULL → `users.id` (`onDelete: "restrict"`); `createdAt`/`updatedAt` timestamptz default now.
   - Index `(funnelId, occurredAt desc)`. Migration aditiva e idempotente (padrão 0071/0072). ⚠️ DB dev = prod.

2. **Vocabulários (espelham os dropdowns da planilha atual)** — constantes em `packages/web/lib/campaign-log-options.ts` (⚠️ NÃO exportar valor de `@loyola-x/shared` — memória `web-shared-type-only`; a API valida só tamanho/não-vazio, não o conjunto):
   - **Evento:** Disparo de e-mail; Disparo de mensagem de WhatsApp; Disparo de SMS / ligação; Publicação em rede-social; Transmissão ao-vivo; Atualização de perfil / conta; Ação no gerenciador de anúncios; Ação em automação; Atualização de página; Outro.
   - **Aplicativo:** ActiveCampaign, Letalk, Z-API, Devzapp, Manychat, Ligueleads, Instagram, YouTube, Facebook, TikTok, Threads, X, LinkedIn, Pinterest, Make, N8N, Meta Ads, Google Ads, Tally, Google Forms, Calendly, Zoom, Google Meet, StreamYard, Hotmart, Kiwify, Wordpress, Hospedagem, Domínio, Webflow, Mautic, Chatwoot, SendFlow, Outro.
   - **Categoria:** Campanha, Privado (X1), Grupo, Canal (WPP), SMS, Ligação, Feed, Carrossel, Story, Sequência, Vídeo, Reel, Short, Thread, Live, Edição de Bio, Edição de Link da Bio, Edição de Descrição, Edição de Foto de Perfil, Edição de Legenda, Edição de Capa, Edição de Thumbnail, Campanha Ligada, Campanha Desligada, Ajuste de Budget, Ajuste de Público, Publicação de Criativos, Automação Ligada, Automação Desligada, Automação Editada, Edição de Web-Design, Edição de Texto, Edição de Infra, Outro.
   - Selecionar "Outro" habilita input de texto livre (gravado no mesmo campo).

3. **API — `packages/api/src/routes/campaign-log.ts`** (registrado em `app.ts`), path base `/api/projects/:projectId/funnels/:funnelId/campaign-log`:
   - `GET /` → lista paginada (query: `limit`/`offset` ou `days`, + filtros opcionais `evento`, `aplicativo`, `categoria`, `q` busca em notes). Ordena `occurredAt` desc. Retorna autor (nome/avatar via join users).
   - `POST /` → cria entrada (zod: `occurredAt` ISO ou YYYY-MM-DDTHH:mm, `evento` obrigatório, demais opcionais). `createdBy = request.userId`.
   - `PATCH /:entryId` → edita campos (parcial).
   - `DELETE /:entryId` → remove (qualquer membro; sem soft-delete).
   - Acesso: mesmas regras das rotas de funil (guest com membership no projeto pode ler E escrever — o time de tráfego lança ações).

4. **UI — entrada FIXA no funil (não é `funnel_stage`):** no grid de etapas do funil (`app/(app)/projects/[id]/funnels/[funnelId]/page.tsx`), renderizar um **card fixo "Log de Campanha"** (ícone `ScrollText`, badge própria) sempre presente, após os cards de etapas — sem drag, sem delete, sem contar pro mínimo de 1 etapa. Click navega pra `/projects/:id/funnels/:funnelId/campaign-log`. Como funis de 1 etapa fazem auto-redirect pra etapa, a página da etapa também ganha acesso (botão/atalho "Log de Campanha" no header).

5. **Página do log — `app/(app)/projects/[id]/funnels/[funnelId]/campaign-log/page.tsx`:**
   - Header: nome do funil + "Log de Campanha" + botão primário **"Registrar ação"**.
   - **Dialog de lançamento rápido** (meta: < 10 segundos): data/hora default = agora (editável); Evento (select); Aplicativo (select com busca — 30+ opções); Categoria (select); Observações (input); Responsável = usuário logado por padrão (chip), com opção de digitar outro nome. Ao salvar: toast + lista atualiza; dialog reabre limpo com "Registrar outra" (checkbox "continuar lançando").
   - **Timeline agrupada por dia** (mais recente primeiro): cada entrada mostra hora, badge do Evento (cor por tipo), Aplicativo, Categoria, observação, avatar/nome do responsável. Menu por entrada: editar / excluir (com confirmação).
   - **Filtros** no topo: Evento, Aplicativo, Categoria (selects), busca por texto; contador "N ações no período"; `DayRangePicker` de período (default 90 dias).
   - Empty state com CTA "Registrar primeira ação".

6. **Hooks — `lib/hooks/use-campaign-log.ts`** (React Query + `useApiClient`): `useCampaignLog(projectId, funnelId, filters)`, `useCreateLogEntry`, `useUpdateLogEntry`, `useDeleteLogEntry` — invalidação por prefixo `["campaign-log", projectId, funnelId]`.

7. **Qualidade:** `tsc --noEmit` limpo em api e web; migration aplicada e verificada via information_schema; lint ok.

## ⚠️ Risks & Mitigations

- **DB dev = prod:** migration é aditiva (`CREATE TABLE IF NOT EXISTS`) e aplicada com verificação — mesmo protocolo das 0071/0072.
- **Vocabulário divergir do uso real:** valores gravados como texto (não enum no banco) — adicionar/renomear opção é só editar a constante no web, sem migration.
- **"Etapa fixa" vs `funnel_stage` real:** implementar como card fixo evita mexer em sort/mínimo-1-etapa/cascades; se depois o Lucas quiser como etapa de verdade (tipo "log"), a tabela já liga por `funnelId` e a migração de UI é pequena.
- **Web-shared type-only:** vocabulários e tipos inline no web (memória `web-shared-type-only`).
