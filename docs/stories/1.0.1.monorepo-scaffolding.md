### Fastify Entry Point Pattern

Architecture define dois arquivos para o backend:
- `app.ts` — factory function `buildServer()` que registra plugins e retorna a instancia Fastify
- `server.ts` — entry point que chama `buildServer()` e faz `.listen()`

Nesta story, ambos devem ser **minimos** (apenas Fastify instanciado, sem plugins). Os plugins serao adicionados na Story 1.1.1.

[Source: architecture/fullstack-architecture#5.3]

### Next.js App Router Structure

Architecture define que o frontend usa App Router com route groups:
- `(auth)/` — rotas publicas (Clerk)
- `(app)/` — rotas autenticadas

Nesta story, criar apenas o root layout e page placeholder. Route groups serao criados na Story 1.3.1.

[Source: architecture/fullstack-architecture#4.2]

### Cross-Package Import Pattern

Packages referenciam `@loyola-x/shared` via workspace protocol (`"workspace:*"`):
```json
{
  "dependencies": {
    "@loyola-x/shared": "workspace:*"
  }
}
```

O `transpilePackages` no `next.config.ts` e necessario para Next.js processar o shared package:
```typescript
const nextConfig = {
  transpilePackages: ["@loyola-x/shared"],
};
```
