## 📖 Story

**Como** gestor de tráfego / copywriter da Loyola,
**Quero** colar um link de checkout cru, escolher uma folder do Switchy, digitar o `utm_campaign` e disparar a geração em lote de shortlinks rastreados para todos os canais marcados,
**Para** que eu obtenha de uma só vez N links curtos (um por canal) já com UTMs, `sck`, `vk_source` e pixels do projeto corretos — e veja o histórico do que já foi gerado.

---

## ✅ Acceptance Criteria

1. **Generator (formulário de lote) dentro da aba "Switch" (nível projeto):**
   - Input de **checkout URL** (texto livre). Abaixo dele, exibir a **URL crua** colada (preview do que entra como base).
   - **Select de folder** alimentado pelo hook `useSwitchyFolders(projectId)` existente (label = `folder.name`, value = `folder.id`).
   - Input **campaign** (`utm_campaign`) — **OBRIGATÓRIO**. Botão Gerar desabilitado enquanto vazio.
   - Inputs **term** (`utm_term`) e **content** (`utm_content`) — **OPCIONAIS**, pré-preenchidos com `defaultUtmTerm` / `defaultUtmContent` vindos de `useSwitchySettings` (mas editáveis e podendo ser apagados).
   - **Checkboxes de canais** montados a partir dos presets com `enabled === true` (via `useSwitchyPresets`). Por padrão **todos marcados**; usuário pode desmarcar. Cada checkbox mostra `label` (e idealmente `utm_medium`/`utm_source` como subtítulo).
   - Botão **Gerar** chama `useGenerateSwitchyLinks` enviando apenas os canais marcados.

2. **Validações do formulário (antes de chamar a API):**
   - `checkoutUrl` não vazio.
   - `folderId` selecionado.
   - `campaign` não vazio.
   - Ao menos **1 canal** marcado.
   - Se alguma falhar, exibir feedback (toast `sonner` + estado desabilitado do botão) e **não** disparar a request.

3. **Geração em lote (batch):**
   - Ao clicar Gerar, montar o body **exatamente** no contrato de `POST /api/projects/:projectId/switchy/generate`:
     `{ checkoutUrl, folderId, folderName, campaign, term?, content?, channels: [{ label, medium, source }] }`.
   - `folderName` resolvido a partir da folder selecionada (nome exibido no Select).
   - Durante a chamada, mostrar estado de loading (botão com spinner + tabela com Skeleton).

4. **Results (tabela de resultados):**
   - Após sucesso, renderizar tabela com colunas **Canal | medium | source | short url | copiar**.
   - Cada linha vem de `results[]` retornado: `{ label, medium, source, fullUrl, shortUrl, switchyLinkId, error? }`.
   - **Falha parcial por-canal:** linhas com `error` aparecem com indicador de erro (badge/ícone vermelho + mensagem curta) **sem** derrubar as demais. Linhas com sucesso mostram o `shortUrl` clicável.
   - Botão **Copiar** por linha copia o `shortUrl` (fallback para `fullUrl` se `shortUrl` ausente) para o clipboard com toast de confirmação.
   - Resumo curto acima da tabela: "X de N gerados com sucesso" quando houver ao menos 1 erro.

5. **Histórico:**
   - Lista/tabela dos últimos links gerados, vinda de `useSwitchyHistory(projectId)` (`GET /api/projects/:projectId/switchy/links/history`).
   - Colunas mínimas: canal (`channelLabel`), `utm_campaign`, `shortUrl` (copiável), data (`createdAt`).
   - Atualiza automaticamente após uma geração bem-sucedida (invalidação da query de histórico).
   - Estados de loading (Skeleton) e vazio ("Nenhum link gerado ainda").

6. **Hooks novos em `use-switchy.ts`:**
   - `useGenerateSwitchyLinks(projectId)` — mutation (React Query) que faz `POST .../switchy/generate` e, no sucesso, invalida `["switchy-history", projectId]`.
   - `useSwitchyHistory(projectId)` — query do histórico, seguindo o padrão `useApiClient` + `useQuery` já presente no arquivo.

7. **Permissão / guest:**
   - Geração é uma ação de escrita. Para usuário `guest` (`role === "guest"`), os controles de geração ficam desabilitados / ocultos (consistente com o gating binário do projeto: `role !== null && role !== "guest"` = pode editar). Histórico e leitura permanecem visíveis.

## ⚠️ Risks & Mitigations

| Risk | Severity | Mitigation |
|------|----------|-----------|
| **Shape da response de `generate` divergir do contrato** | MEDIUM | Tipar `results[]` exatamente como no contrato (`{label,medium,source,fullUrl,shortUrl,switchyLinkId,error?}`). Renderizar defensivamente: `shortUrl` pode ser `null` (cair para `fullUrl`); `error` opcional. |
| **Falha parcial silenciosa (batch "verde" com canais quebrados)** | HIGH | Linhas com `error` recebem indicador visual explícito + resumo "X de N gerados". Nunca esconder erro por-canal. |
| **Dependência de 33.3/33.4 ainda não mergeadas** | MEDIUM | Hooks e tipos são auto-contidos no front; se o endpoint não existir em dev, a UI mostra estado de erro/empty sem quebrar. Marcar dependências explicitamente. |
| **Pré-preenchimento de term/content sobrescrever edição do usuário** | MEDIUM | Inicializar inputs com defaults **uma vez** quando settings carregam (e o campo ainda está "pristine"); não reaplicar default após o usuário digitar/apagar. |
| **Copiar para clipboard falhar (contexto não seguro / permissão)** | LOW | `navigator.clipboard.writeText` com `try/catch` e toast de erro; sem fallback obrigatório. |
| **`campaign` com espaços/caracteres especiais** | LOW | Encoding é responsabilidade do backend (`buildTrackedCheckoutUrl` faz `encodeURIComponent`). Front apenas faz `trim()` antes de enviar. |

### Contrato de hooks (em `packages/web/lib/hooks/use-switchy.ts` — ADICIONAR, manter o GraphQL/read-only existente)

Padrão do arquivo: `"use client"` + `useApiClient()` + `useQuery`/`useMutation` do `@tanstack/react-query`. Já existem `useSwitchyFolders` e `useSwitchyLinks` — **reusar `useSwitchyFolders` no Select de folder**.

```ts
// Tipos do batch (nomes EXATOS do contrato)
export interface SwitchyGenerateChannel {
  label: string;
  medium: string;
  source: string;
}

export interface SwitchyGeneratePayload {
  checkoutUrl: string;
  folderId: string;       // folderId do Switchy (string no body; o Select trabalha com number — converter ao montar)
  folderName: string;
  campaign: string;
  term?: string;
  content?: string;
  channels: SwitchyGenerateChannel[];
}

export interface SwitchyGenerateResult {
  label: string;
  medium: string;
  source: string;
  fullUrl: string;
  shortUrl: string | null;
  switchyLinkId: string | null;
  error?: string;
}

export interface SwitchyHistoryItem {
  id: string;
  channelLabel: string | null;
  utmCampaign: string | null;
  utmMedium: string | null;
  utmSource: string | null;
  shortUrl: string | null;
  fullUrl: string;
  createdAt: string;
}

// useGenerateSwitchyLinks: POST .../switchy/generate -> { results: SwitchyGenerateResult[] }
// no onSuccess: queryClient.invalidateQueries({ queryKey: ["switchy-history", projectId] })

// useSwitchyHistory: GET .../switchy/links/history -> { links: SwitchyHistoryItem[] }
// queryKey: ["switchy-history", projectId]; enabled: !!projectId
```

> **Nota de tipo:** o body do contrato envia `folderId` como string; o `useSwitchyFolders` devolve `folder.id` numérico. Ao montar o payload, usar `String(folderId)` e `folderName` = nome da folder selecionada. Confirmar com a 33.3 se o backend aceita number ou string e alinhar (preferir string conforme contrato).

### Contrato do endpoint consumido (referência — implementado na 33.3)

```
POST /api/projects/:projectId/switchy/generate
body { checkoutUrl, folderId, folderName, campaign, term?, content?, channels:[{label,medium,source}] }
-> { results: [{ label, medium, source, fullUrl, shortUrl, switchyLinkId, error? }] }
Falha por-canal NÃO derruba o batch (parcial OK).

GET /api/projects/:projectId/switchy/links/history
-> últimos links gerados (do nosso banco) — colunas mínimas: channelLabel, utmCampaign, shortUrl, createdAt
```

Ícones `lucide-react` (ex.: `Copy`, `ExternalLink`, `Loader2`, `AlertCircle`, `Sparkles`/`Link2`). Toasts via `sonner` (`import { toast } from "sonner"`).
- Componentes sugeridos:
  - `switchy-generator.tsx` — o formulário (checkout URL + preview cru, Select folder via `useSwitchyFolders`, input campaign obrigatório, term/content opcionais, checkboxes de canais a partir de `useSwitchyPresets` enabled).
  - `switchy-results-table.tsx` — tabela `Canal | medium | source | short url | copiar` com tratamento de erro por-canal.

- **Folder Select:** value = `String(folder.id)`, label = `folder.name`. Guardar tanto `folderId` quanto `folderName` no estado para montar o payload.
- **Checkboxes de canais:** estado tipo `Set<string>` (por `label` ou `id` do preset). Inicializar com todos os `enabled` marcados. Enviar só os marcados como `channels: [{ label, medium, source }]`.
- **Copiar:** `await navigator.clipboard.writeText(value)` dentro de `try/catch`; `toast.success("Link copiado")` / `toast.error(...)`.
- **Gating guest:** `const role = useUserRole();` (hook já existente, ver `packages/web/app/(app)/projects/[id]/page.tsx`); `const canEdit = role !== null && role !== "guest";` Desabilitar/ocultar o botão Gerar e inputs de escrita quando `!canEdit`.

### Pré-preenchimento de defaults (term/content)
- Ao carregar `useSwitchySettings`, semear os inputs term/content **uma única vez** com `defaultUtmTerm`/`defaultUtmContent`. Usar um flag de "já inicializado" ou inicializar via estado derivado controlado — **não** sobrescrever depois que o usuário tocou no campo. Campo pode ficar vazio (term/content são opcionais e omitidos da URL pelo backend quando vazios).

### Estados a tratar
- Loading: Select de folders (`useSwitchyFolders.isLoading`), botão Gerar (mutation `isPending`), Results (Skeleton enquanto `isPending`), Histórico (`useSwitchyHistory.isLoading`).
- Vazio: nenhum result ainda; histórico vazio ("Nenhum link gerado ainda").
- Erro: mutation `isError` → toast; histórico `isError` → mensagem inline. Erro por-canal → indicador na linha.
