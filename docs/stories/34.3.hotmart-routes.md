## 📖 Story

**Como** estrategista/gestor de um projeto na Loyola Digital X,
**Quero** endpoints REST que conectem minha conta Hotmart ao projeto, listem os produtos disponíveis e devolvam o dashboard de assinaturas agregado,
**Para** que o frontend da aba "Assinaturas" possa configurar a credencial Hotmart e renderizar métricas de recorrência (Total, Vigentes, Canceladas, Reembolsadas, MRR, LTV, LT, Retenção, Renovações) de um produto escolhido num dropdown.

---

## ✅ Acceptance Criteria

1. **Arquivo de rotas criado:** `packages/api/src/routes/hotmart.ts` exporta um plugin `fastify-plugin` (`fp`) — registrado em `packages/api/src/app.ts` ao lado de `mauticRoutes` —, reutilizando o helper interno `getProjectAccess(projectId, userId, userRole)` no padrão de `mautic.ts` (guest sem vínculo → 404; projeto inexistente → 404).

2. **GET `/api/projects/:projectId/hotmart/connection`** retorna `{ connected: boolean }`. Não retorna nem `client_id`, nem `client_secret`, nem o Basic. Sem credencial salva → `{ connected: false }`.

3. **PUT `/api/projects/:projectId/hotmart/connection`** (bloqueia guest com 403):
   - Body validado por zod: `{ clientId: string (min 1), clientSecret: string (min 1) }`.
   - **VALIDA as credenciais antes de persistir**, batendo no token endpoint da Hotmart (`getHotmartToken(clientId, clientSecret)` do service). Se a Hotmart recusar (token inválido) → `502` com mensagem amigável (`"Falha ao conectar na Hotmart. Verifique client_id/client_secret."`) e `details` do erro.
   - Só após sucesso: criptografa `clientId` e `clientSecret` via `services/encryption.ts` e faz **upsert** em `hotmart_connections` (`onConflictDoUpdate` no `project_id`), gravando `updated_at`.
   - Invalida o cache LRU do dashboard/products desse projeto.
   - Retorna `{ connected: true }`.

4. **DELETE `/api/projects/:projectId/hotmart/connection`** (bloqueia guest com 403): remove a linha de `hotmart_connections` do projeto, invalida o cache LRU do projeto e retorna `{ connected: false }`.

5. **GET `/api/projects/:projectId/hotmart/products?months=12`** retorna `{ products: [{ id, name }] }`:
   - `months` validado por zod (`coerce.number().int().min(1).max(36).default(12)`).
   - Deriva os produtos distintos varrendo `/subscriptions/summary` via `listHotmartProducts(token, accessionFrom)` do service (accessionFrom = hoje − `months`).
   - Projeto sem conexão Hotmart → `409` (`"Hotmart não conectado neste projeto"`).
   - Resultado cacheado em LRU (ttl ~30min), chave inclui `projectId` + `months`.

6. **GET `/api/projects/:projectId/hotmart/dashboard?productId=&months=12`** retorna o objeto de métricas agregadas de `computeHotmartDashboard(token, { productId, months })`:
   - `productId` obrigatório (zod `string().min(1)`); `months` igual ao de products.
   - Projeto sem conexão → `409`.
   - Resultado cacheado em LRU (ttl ~30min), chave inclui `projectId` + `productId` + `months`.

7. **Guard de escrita:** todas as rotas de escrita (`PUT`, `DELETE`) retornam `403` quando `request.userRole === "guest"`. As rotas de leitura (`GET`) seguem o `getProjectAccess` (guest com vínculo lê; sem vínculo → 404).

8. **Cache LRU:** uma única instância `LRUCache` module-level no arquivo de rotas (`lru-cache`, ttl ≈ 30min), compartilhada por products e dashboard. Função helper `invalidateHotmartCache(projectId)` remove todas as entradas cujo prefixo da chave é `projectId` e é chamada no `PUT` e no `DELETE` de connection.

9. **Tratamento de erro:** todo `catch` de chamada externa (token/products/dashboard) loga via `fastify.log.error(err, "...")` / `request.log.error(...)` e responde `502` com `{ error, details }`. **NUNCA** logar `client_secret`, o Basic, nem PII do assinante.

10. **Validação estática:** `pnpm --filter @loyola/api typecheck` e `pnpm --filter @loyola/api lint` passam sem erros/warnings novos.

## ⚠️ Risks & Mitigations

| Risk | Severity | Mitigation |
|------|----------|-----------|
| **Vazamento de `client_secret` / Basic em logs** | HIGH | `getCreds` nunca loga; catches logam só `err.message`. Revisão CodeRabbit foca em ausência de `client_secret`/`Authorization` em qualquer `log.*`. Credenciais criptografadas em repouso (`encryption.ts`). |
| **Service 34.2 com assinatura diferente do contrato** | MEDIUM | Story documenta o contrato exato esperado (`getHotmartToken`, `listHotmartProducts`, `computeHotmartDashboard`). Se 34.2 divergir, ajustar imports — divergência detectada em typecheck. |
| **Cache LRU servindo dado de outro projeto/produto** | MEDIUM | Chave composta `${projectId}:${productId}:${months}` (dashboard) e `${projectId}:products:${months}` (products). Teste manual com 2 projetos distintos. |
| **Cache stale após troca de credencial** | MEDIUM | `invalidateHotmartCache(projectId)` no PUT e DELETE de connection limpa todas as entradas do projeto. |
| **PUT persistindo credencial inválida** | HIGH | Validação obrigatória no token endpoint antes do upsert (espelha `testMauticConnection`). Falha → 502, nada gravado. |
| **`/subscriptions` default 30 dias mascarando produtos antigos** | MEDIUM | `accessionFrom` derivado de `months` (default 12) sempre passado a `listHotmartProducts` — pegadinha já documentada no Epic. |


### Contrato de Service consumido (Story 34.2 — `packages/api/src/services/hotmart.ts`)

```ts
getHotmartToken(clientId, clientSecret): Promise<string>          // cache em memória, refresh 5min antes do expires_in
listHotmartProducts(token, accessionFrom): Promise<{ id, name }[]> // distinct product das assinaturas
computeHotmartDashboard(token, { productId, months }): Promise<HotmartDashboard> // todas as métricas

```

> O service ainda pode expor `encrypt`/`decrypt` diretamente de `services/encryption.ts` (assinatura `encrypt(plaintext) -> { encrypted, iv }`, `decrypt(encrypted, iv) -> plaintext`). Reusar o padrão de `mautic.ts` (`encryptMauticPassword`/`decryptMauticPassword`). Se 34.2 expor helpers próprios (`encryptHotmartSecret`/`decryptHotmartSecret`), preferi-los.

### Contrato de Dados (Story 34.1 — tabela `hotmart_connections`)

```
id uuid pk default gen_random_uuid()
project_id uuid NOT NULL UNIQUE FK->projects(id) ON DELETE cascade
client_id_encrypted text NOT NULL
client_id_iv text NOT NULL
client_secret_encrypted text NOT NULL
client_secret_iv text NOT NULL
created_at timestamptz default now()
updated_at timestamptz default now()
index idx_hotmart_connections_project on (project_id)
```

> O Basic (`base64(client_id:client_secret)`) é **derivado em runtime no service**, nunca armazenado.

### Padrões a reusar (de `packages/api/src/routes/mautic.ts`)

- **Plugin:** `export default fp(async function hotmartRoutes(fastify) { ... })`.
- **`getProjectAccess(projectId, userId, userRole)`**: guest sem `projectMembers` → `null` (404); projeto inexistente → `null` (404). Copiar literalmente de `mautic.ts` (linhas 58-73).
- **`getConnectionRow(projectId)`** e **`getCreds(projectId)`**: ler `hotmartConnections` por `project_id`; `getCreds` decifra `client_id`/`client_secret` num `try/catch` (retorna `null` se decifragem falhar) e devolve `{ clientId, clientSecret }`.
- **Guard guest:** `if (request.userRole === "guest") return reply.code(403).send({ error: "Acesso negado" });` no topo de PUT e DELETE.
- **Validação de cred no PUT:** espelhar o bloco `testMauticConnection` (linhas 137-145) — chamar `getHotmartToken(clientId, clientSecret)` dentro de `try/catch`; falha → `502` com `{ error, details }`.
- **Upsert:** `fastify.db.insert(hotmartConnections).values({...}).onConflictDoUpdate({ target: hotmartConnections.projectId, set: {...} })` (linhas 149-168).
- **Catches:** `502` com `{ error, details: err instanceof Error ? err.message : String(err) }`, precedido de `request.log.error(err, "...")`.

### Cache LRU (padrão `lru-cache`, ver `services/artifact-cache.ts`)

```ts
import { LRUCache } from "lru-cache";

// module-level, compartilhado por products e dashboard
const hotmartCache = new LRUCache<string, unknown>({
  max: 500,
  ttl: 30 * 60 * 1000, // ~30min
});

function invalidateHotmartCache(projectId: string) {
  for (const key of hotmartCache.keys()) {
    if (key.startsWith(`${projectId}:`)) hotmartCache.delete(key);
  }
}
```

- Chave products: `` `${projectId}:products:${months}` ``
- Chave dashboard: `` `${projectId}:dashboard:${productId}:${months}` ``
- `get` → se hit, retorna direto; miss → chama service, `set`, retorna.
- `invalidateHotmartCache(projectId)` no PUT e DELETE de connection.
