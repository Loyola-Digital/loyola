# Story 19.2 — Web: Navegação de Etapas + CRUD

**Epic:** EPIC-19 Funnel Stages
**Status:** Draft
**Priority:** High
**Estimate:** L
**Depends on:** Story 19.1 (API de etapas)

---

## Story

Como gestor de tráfego, quero ver uma lista de etapas ao entrar em um funil e poder criar, renomear e remover etapas, para que eu consiga segmentar meu funil em frentes independentes (ex: "Captação Paga", "Captação Gratuita", "Modelado").

## Contexto

Hoje `funnels/[funnelId]/page.tsx` renderiza o dashboard diretamente. Com stages, o fluxo muda:

```
/funnels/:funnelId              → StageListPage (lista de etapas)
/funnels/:funnelId/stages/:stageId  → [StageDetailPage — Story 19.3]
```

A `StageListPage` mostra cards clicáveis, um botão "Nova Etapa" e opções de renomear/deletar por etapa.

## Escopo

### IN
- Hook `useFunnelStages` e `useCreateStage` / `useUpdateStage` / `useDeleteStage` em `packages/web/lib/hooks/use-funnel-stages.ts`
- Transformar `funnels/[funnelId]/page.tsx` em **Stage List Page** (cards de etapas + "Nova Etapa")
- Criar rota `funnels/[funnelId]/stages/[stageId]/page.tsx` como placeholder (renderiza loading/empty — wiring do dashboard é Story 19.3)
- Componente `StageCard` — card clicável com nome da etapa, quantidade de campanhas Meta/Google, badge de campanhas
- Dialog "Nova Etapa" — input de nome + submit
- Dialog "Renomear Etapa" — inline ao clicar em `⋯` → "Renomear"
- Botão "Deletar" no menu `⋯` com confirm dialog. Disabled + tooltip se for a última etapa
- Funil com apenas 1 etapa: redireciona automaticamente para `stages/:stageId` (bypass da list page) para não quebrar o fluxo atual
- Breadcrumb na stage page: `[Funnel name]` → `[Stage name]`

### OUT
- Dashboard wired à etapa (Story 19.3)
- Config de campanhas dentro da etapa via UI (Story 19.3 ou Settings dentro do dashboard)
- Reordenação de etapas
- Per-stage planilhas (Story 19.4)

## Technical Notes

- Stale time do `useFunnelStages`: 2 min (dados mudam pouco)
- `useCreateStage` deve resetar o form e fechar o dialog no `onSuccess`
- O redirect automático (funil com 1 etapa) usa `useRouter().replace` para não poluir o histórico de navegação
- Não remover a lógica de dashboard do `funnels/[funnelId]/page.tsx` ainda — a Story 19.3 vai fazer essa remoção dentro da nova `stages/[stageId]/page.tsx`
- O `funnels/[funnelId]/page.tsx` atual tem tabs (Meta Ads, YouTube, Pesquisas, etc.). Essas tabs MIGRAM para `stages/[stageId]/page.tsx` na Story 19.3. Nesta story, apenas a rota de stages/[stageId] existe como placeholder.
