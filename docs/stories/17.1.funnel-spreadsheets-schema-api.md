## Story

Como sistema, preciso de uma tabela `funnel_spreadsheets` que armazene vínculos de planilha ↔ funil com **tipo** e **mapeamento rico de colunas** (nome, email, telefone, UTMs, data, status, valor), expostos via endpoints REST e hooks React, para suportar a nova aba "Planilhas" no funil.

### Schema proposto

```typescript
// packages/api/src/db/schema.ts

export const funnelSpreadsheetTypeEnum = pgEnum("funnel_spreadsheet_type", [
  "leads",
  "sales",
  "custom",
]);

export const funnelSpreadsheets = pgTable(
  "funnel_spreadsheets",
  {
    id: uuid("id").defaultRandom().primaryKey(),
    funnelId: uuid("funnel_id")
      .notNull()
      .references(() => funnels.id, { onDelete: "cascade" }),
    label: varchar("label", { length: 255 }).notNull(),
    type: funnelSpreadsheetTypeEnum("type").notNull(),
    spreadsheetId: varchar("spreadsheet_id", { length: 255 }).notNull(),
    spreadsheetName: varchar("spreadsheet_name", { length: 255 }).notNull(),
    sheetName: varchar("sheet_name", { length: 255 }).notNull(),
    columnMapping: jsonb("column_mapping")
      .notNull()
      .$type<{
        name?: string;
        email?: string;
        phone?: string;
        date?: string;
        status?: string;
        value?: string;
        utm_source?: string;
        utm_medium?: string;
        utm_campaign?: string;
        utm_content?: string;
        utm_term?: string;
      }>(),
    createdBy: uuid("created_by")
      .notNull()
      .references(() => users.id, { onDelete: "restrict" }),
    createdAt: timestamp("created_at", { withTimezone: true })
      .defaultNow()
      .notNull(),
    updatedAt: timestamp("updated_at", { withTimezone: true })
      .defaultNow()
      .notNull(),
  },
  (table) => [
    index("idx_funnel_spreadsheets_funnel").on(table.funnelId),
  ]
);
```

### Endpoints propostos

Arquivo novo: `packages/api/src/routes/funnel-spreadsheets.ts`

| Método | Rota | Descrição |
|--------|------|-----------|
| GET | `/api/projects/:projectId/funnels/:funnelId/spreadsheets` | Lista vínculos do funil |
| POST | `/api/projects/:projectId/funnels/:funnelId/spreadsheets` | Cria vínculo + mapeamento |
| PUT | `/api/projects/:projectId/funnels/:funnelId/spreadsheets/:id` | Atualiza label/tipo/mapeamento |
| DELETE | `/api/projects/:projectId/funnels/:funnelId/spreadsheets/:id` | Remove vínculo |
| GET | `/api/projects/:projectId/funnels/:funnelId/spreadsheets/:id/data` | Retorna dados lidos (headers + rows) — reusa `readSheetData()` |

### Validação de payload (POST/PUT)

- `label`: string não-vazia, max 255
- `type`: enum (leads|sales|custom)
- `spreadsheetId`, `spreadsheetName`, `sheetName`: strings não-vazias
- `columnMapping`: objeto com pelo menos UM campo preenchido; todas as chaves opcionais
- `funnelId` deve pertencer ao `projectId` (validação cruzada)

## Dev Notes

- **Reuso é obrigatório:** serviço `google-sheets.ts` é a única fonte de verdade para Drive/Sheets. Não reimplementar.
- **Não deduplicar com sales:** `salesSpreadsheetMappings` e `funnelSpreadsheets` são tabelas **separadas por design** (escopo diferente: produto vs funil). Consolidação é fora de escopo.
- **Validação cruzada projectId/funnelId:** evita vazamento entre projetos.
- **`columnMapping` todos opcionais:** permite flexibilidade (ex: planilha de leads pode ter só email+name+utms, sem valor/status).
- **Enum `type`:** UI usa para ícones/labels, mas sistema não processa diferente — a granularidade do comportamento vem do `columnMapping`.
