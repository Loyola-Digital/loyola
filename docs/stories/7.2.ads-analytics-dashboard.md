## Story

**As a** gestor de trafego da Loyola Digital,
**I want** a dashboard with charts showing campaign performance (spend, CTR, CPC, CPM over time) and a drill-down table by campaign > ad set > ad,
**so that** I can analyze media metrics and identify top-performing campaigns and creatives.

---

## Acceptance Criteria

1. [ ] Dashboard `/traffic` refatorado: seletor de conta Meta Ads no topo e filtro de periodo (7d, 14d, 30d, 90d)
2. [ ] Cards de resumo: Total Spend, Total Impressions, Total Clicks, CTR medio, CPC medio, CPM medio
3. [ ] Grafico de linha: spend diario + clicks diarios no periodo selecionado (eixo duplo Y)
4. [ ] Tabela de campanhas com colunas: Nome | Status | Spend | Impressions | Clicks | CTR | CPC | CPM
5. [ ] Metricas calculadas: CTR = clicks/impressions × 100, CPC = spend/clicks, CPM = (spend×1000)/impressions
6. [ ] Drill-down: clicar em campanha expande e mostra ad sets da campanha; clicar em ad set expande e mostra ads
7. [ ] `GET /api/meta-ads/accounts/:id/adsets?campaignId=X` — nova rota: ad sets de uma campanha com metricas
8. [ ] `GET /api/meta-ads/accounts/:id/ads?adsetId=X` — nova rota: ads de um ad set com metricas
9. [ ] `GET /api/meta-ads/accounts/:id/insights/daily?days=30` — nova rota: insights com breakdown por dia
10. [ ] Hooks: `useMetaAdsAdSets(accountId, campaignId)`, `useMetaAdsAds(accountId, adsetId)`, `useMetaAdsDailyInsights(accountId, days)`
11. [ ] Loading skeletons para graficos e tabelas
12. [ ] Empty state quando conta nao tem campanhas ativas
13. [ ] Formatacao de valores: spend em R$ (ou currency da conta), percentuais com 2 casas, numeros grandes com K/M
14. [ ] `pnpm typecheck` e `pnpm lint` passam sem erros

### Meta Marketing API — Novas chamadas necessarias

```typescript
// Ad Sets de uma campanha:
// GET /act_{ID}/adsets?fields=id,name,status,targeting,bid_amount,daily_budget&filtering=[{"field":"campaign_id","operator":"EQUAL","value":"{CAMPAIGN_ID}"}]

// Ads de um ad set:
// GET /act_{ID}/ads?fields=id,name,status,creative&filtering=[{"field":"adset_id","operator":"EQUAL","value":"{ADSET_ID}"}]

// Insights diarios (breakdown por dia):
// GET /act_{ID}/insights?fields=impressions,reach,clicks,spend,ctr,cpc,cpm&time_increment=1&date_preset=last_30d&level=account

// Insights por campanha:
// GET /act_{ID}/insights?fields=impressions,reach,clicks,spend,ctr,cpc,cpm&date_preset=last_30d&level=campaign

// Insights por ad set:
// GET /act_{ID}/insights?fields=impressions,reach,clicks,spend,ctr,cpc,cpm&date_preset=last_30d&level=adset&filtering=[{"field":"campaign_id","operator":"EQUAL","value":"{CAMPAIGN_ID}"}]

// Insights por ad:
// GET /act_{ID}/insights?fields=impressions,reach,clicks,spend,ctr,cpc,cpm&date_preset=last_30d&level=ad&filtering=[{"field":"adset_id","operator":"EQUAL","value":"{ADSET_ID}"}]
```

### Novas funcoes no service meta-ads.ts

```typescript
// Adicionar ao packages/api/src/services/meta-ads.ts:

export async function fetchAdSets(
  metaAccountId: string,
  accessToken: string,
  campaignId: string
): Promise<MetaAdSet[]> { ... }

export async function fetchAds(
  metaAccountId: string,
  accessToken: string,
  adsetId: string
): Promise<MetaAd[]> { ... }

export async function fetchDailyInsights(
  metaAccountId: string,
  accessToken: string,
  days: number = 30
): Promise<MetaDailyInsight[]> { ... }

export async function fetchCampaignInsights(
  metaAccountId: string,
  accessToken: string,
  days: number = 30
): Promise<MetaCampaignInsight[]> { ... }

// Tipos adicionais:
interface MetaAdSet {
  id: string; name: string; status: string;
  targeting?: object; bid_amount?: string; daily_budget?: string;
}

interface MetaAd {
  id: string; name: string; status: string;
  creative?: { id: string };
}

interface MetaDailyInsight {
  date_start: string; date_stop: string;
  impressions: string; reach: string; clicks: string;
  spend: string; ctr: string; cpc: string; cpm: string;
}

interface MetaCampaignInsight extends MetaDailyInsight {
  campaign_id: string; campaign_name: string;
}
```
