# Story 13.1 ŌĆö Settings: Conectar Canal YouTube

**Epic:** EPIC-13 YouTube Organic Analytics
**Status:** Draft
**Priority:** High
**Estimate:** M

## Story

Como usuario do Loyola X, quero conectar meu canal do YouTube nas configuracoes, para que o sistema possa puxar dados organicos de performance do canal e dos videos.

## Contexto Tecnico

Reaproveitar o OAuth flow do Google Ads (Epic 12) adicionando scopes do YouTube. O usuario pode conectar via o mesmo botao "Conectar com Google" ŌĆö so precisa autorizar scopes adicionais.

### APIs necessarias

- **YouTube Data API v3** ŌĆö `https://www.googleapis.com/youtube/v3/`
  - `channels.list` ŌĆö info do canal (nome, thumbnail, inscritos)
  - `search.list` / `videos.list` ŌĆö listar videos do canal
- **YouTube Analytics API v2** ŌĆö `https://youtubeanalytics.googleapis.com/v2/`
  - `reports.query` ŌĆö metricas de performance

### OAuth Scopes

Adicionar aos scopes existentes:
- `https://www.googleapis.com/auth/youtube.readonly`
- `https://www.googleapis.com/auth/yt-analytics.readonly`

## Acceptance Criteria

### AC1: Schema DB
- [ ] Tabela `youtube_channels`: id, channel_id (varchar 50), channel_name, thumbnail_url, subscriber_count, refresh_token_encrypted, refresh_token_iv, is_active, created_by, created_at, updated_at
- [ ] Tabela `youtube_channel_projects` (many-to-many): id, channel_id (FK), project_id (FK), created_at
- [ ] Unique constraint em channel_id
- [ ] Cascade delete em channel_projects
- [ ] Migration aplicavel

### AC2: OAuth ŌĆö Adicionar scopes YouTube
- [ ] Atualizar `getGoogleOAuthUrl()` em `services/google-ads.ts` para incluir scopes youtube.readonly e yt-analytics.readonly
- [ ] OU criar novo endpoint OAuth dedicado para YouTube com scopes proprios
- [ ] Callback retorna refresh token que funciona para YouTube APIs

### AC3: API ŌĆö Listar canais apos auth
- [ ] Apos OAuth, chamar YouTube Data API `channels.list?mine=true` para listar canais do usuario
- [ ] Retornar: channelId, channelName, thumbnailUrl, subscriberCount

### AC4: API Routes
- [ ] `POST /api/youtube-channels` ŌĆö salvar canal (channel_id + refresh_token)
- [ ] `GET /api/youtube-channels` ŌĆö listar canais com projetos vinculados
- [ ] `DELETE /api/youtube-channels/:id` ŌĆö remover canal
- [ ] `POST /api/youtube-channels/:id/projects/:projectId` ŌĆö vincular projeto
- [ ] `DELETE /api/youtube-channels/:id/projects/:projectId` ŌĆö desvincular
- [ ] Guests bloqueados (403)

### AC5: Settings UI
- [ ] Nova pagina `/settings/youtube` (ou tab dentro do grupo Google)
- [ ] Botao "Conectar com Google" (reaproveitando OAuth flow)
- [ ] Apos auth, mostrar canais disponiveis para selecionar
- [ ] Lista de canais conectados com thumbnail, nome, inscritos
- [ ] Vincular/desvincular projetos
- [ ] Delete com confirmacao

### AC6: Navegacao
- [ ] Adicionar "YouTube" como sub-item no grupo Google em settings/layout.tsx
- [ ] Ou criar sub-item separado no grupo Google: "YouTube Ads" + "YouTube Canal"

## Dev Notes

- YouTube Data API requer habilitar no Google Cloud Console (mesmo projeto do Google Ads)
- YouTube Analytics API tambem precisa ser habilitada
- `channels.list?mine=true` retorna canais do usuario autenticado
- channel_id formato: `UC` + 22 chars (ex: `UCxxxxxxxxxxxxxxxxxxxxxxxx`)
- Thumbnail do canal: `snippet.thumbnails.default.url`
- Subscriber count: `statistics.subscriberCount`
- Se o usuario ja conectou Google Ads com OAuth, pode pedir re-auth com scopes adicionais ou criar novo consent
