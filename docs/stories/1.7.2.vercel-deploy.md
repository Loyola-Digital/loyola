## Dev Notes

### Frontend Existente

O frontend Next.js 15 esta completo com todas as features implementadas:
- Auth: Clerk sign-in/sign-up + middleware (`(auth)` e `(app)` route groups)
- Layout: AppSidebar + Topbar + main content area
- Minds: Catalog grid + Profile page
- Chat: Full chat interface with SSE streaming
- Tasks: Task page + task detection (Story 1.6.2)
- Conversations: History page (Story 1.5.3)

[Source: packages/web/app/]

### Root Layout Metadata

O root layout em `packages/web/app/layout.tsx` ja tem `ClerkProvider` e providers. Metadata pode ser adicionada via `export const metadata: Metadata = {...}`.

```typescript
// Next.js 15 metadata pattern
export const metadata: Metadata = {
  title: { default: "Loyola Digital X", template: "%s | Loyola Digital X" },
  description: "Central de Mentes — Plataforma de AI para Loyola Digital",
  openGraph: { title: "Loyola Digital X", description: "..." },
};
```

[Source: packages/web/app/layout.tsx]

### Vercel Monorepo Configuration

Para monorepos com Turborepo, Vercel precisa:
- Root Directory: `packages/web`
- Build Command: pode usar o default do Next.js ou custom com turbo
- Install Command: `pnpm install` (na raiz)

Se necessario, criar `vercel.json` na raiz:
```json
{
  "buildCommand": "pnpm turbo build --filter=@loyola-x/web",
  "installCommand": "pnpm install",
  "framework": "nextjs",
  "outputDirectory": "packages/web/.next"
}
```

### Clerk Webhook Backend

O endpoint `POST /api/webhooks/clerk` ja esta implementado no backend (Story 1.3.2). Trata eventos `user.created`, `user.updated`, `user.deleted`.

[Source: packages/api/src/routes/webhooks.ts]

### Backend URL

O backend sera acessivel via Railway URL (configurado em Story 1.7.1). A env var `NEXT_PUBLIC_API_URL` no Vercel deve apontar para essa URL.
