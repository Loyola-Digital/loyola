# Story 24.2 — Página HTML do Relatório Mensal Instagram

## Status

Done

## Story

**As a** membro do projeto,
**I want** uma página HTML compartilhável que renderiza o snapshot do relatório mensal de Instagram com todas as seções (top/bottom posts, saldo de seguidores, totais, distribuição, gráfico diário, demografia, comparativo),
**so that** eu possa visualizar a performance consolidada do mês de forma clara e enviar o link pra clientes/stakeholders sem precisar gerar slides manualmente.

## Acceptance Criteria

### Route

1. Nova route Next.js: `/projects/:id/reports/instagram/:reportId`
   - Server component ou client (dev decide) — fetch do `GET /reports/instagram/:reportId`
   - Loading skeleton enquanto carrega
   - Erro 404 amigável se reportId não existe ou usuário não tem acesso
   - Header com breadcrumb: Projetos / {projectName} / Relatórios Instagram / {monthLabel}

### Header do Relatório

2. Seção topo:
   - Título: `Relatório Instagram — {monthLabel}` (ex: "Relatório Instagram — Abril 2026")
   - Sub: `Período {periodStart} a {periodEnd}` + `Gerado em {generatedAt}` por `{generatedBy}`
   - Botão "Compartilhar" → copia URL atual pra clipboard com toast de confirmação
   - Quando há múltiplas contas, dropdown ou tabs no topo pra navegar entre elas (anchors)

### Por conta IG (1 seção por conta)

Cada conta renderiza as seções abaixo, separadas por linha divisória/espaçamento.

3. **Card identificação da conta:**
   - Avatar (`profilePictureUrl`)
   - `@instagramUsername`
   - `accountName` (se diferente do username)

4. **Cards de totais (grid 4 cols, 2 em mobile):**
   - Posts publicados, Alcance, Visualizações, Interações
   - Cada card: número grande + label + (se `comparison`) delta % vs mês anterior com seta ↑ verde / ↓ vermelho / → cinza
   - Cores: green-600/red-600/muted

5. **Saldo de seguidores (card destaque, full-width):**
   - Net: `+N` ou `-N` em fonte maior, cor por sinal
   - Sub: `+{gained} novos / -{lost} unfollows`
   - Comparativo: "vs mês anterior: +X% / -X%" ou "—" se sem comparison

6. **Distribuição por tipo de mídia (donut/barras horizontal):**
   - Reels vs FEED — count + reach
   - Tooltip mostra split: count(N), reach(M), shares X% do total

7. **Tendência diária (gráfico):**
   - Recharts LineChart 2 séries: seguidores ganhos por dia (eixo Y esquerdo) + alcance por dia (eixo Y direito)
   - Cores: emerald (seguidores) e cyan (alcance)
   - Tooltip mostra os dois valores ao hover na data
   - Vazio se `dailyTrend` é array vazio

8. **Top 5 melhores por Engagement (lista ranqueada):**
   - Posição (1-5), thumbnail, caption truncada, métricas (engagement, reach, likes, comments, follows se FEED)
   - Click no card abre o post no IG (target=_blank)
   - Visual similar ao `<TopPostsByFollowersCard>` (PR #83)

9. **Top 5 melhores por Alcance (mesma estrutura visual de #8):**
   - Métrica destacada: alcance absoluto

10. **Top 5 piores por Engagement:**
    - Mesma estrutura, ordenado ascendente
    - Header com tom amarelo/laranja sutil pra diferenciar de "melhores"

11. **Top 5 piores por Alcance:**
    - Mesma estrutura

12. **Demografia (3 colunas — idade/gênero, cidades, países):**
    - Idade × gênero: barras ou tabela (top 5 buckets por count)
    - Top 5 cidades + count
    - Top 5 países + count
    - Empty state quando `demographics === null`: "Demografia indisponível (conta com menos de 100 seguidores ou Meta não retornou)"

### Empty / Loading / Error states

13. Loading: skeleton com proporção correta de cada card/seção
14. Erro 404: mensagem "Relatório não encontrado ou você não tem acesso a este projeto" com link de volta pro dashboard
15. Erro 500: mensagem genérica + botão "Tentar novamente"
16. Conta sem nenhum post no mês: seção mostra "Nenhuma publicação em {monthLabel}" e oculta cards de top/bottom

### Compartilhamento

17. Link compartilhável com auth: usuários logados que têm acesso ao projeto podem abrir; outros recebem 404. Não há link público sem auth na v1.

### Print-friendly

18. CSS `@media print`: oculta header de navegação do app, mostra só o conteúdo do relatório, quebra de página entre contas. Permite ao usuário "Imprimir → Salvar como PDF" do navegador como workaround antes da v2 ter export.

### Validação

19. `pnpm --filter @loyola-x/web typecheck` → 0 erros
20. `pnpm --filter @loyola-x/web lint` → 0 erros
21. Smoke manual: abrir relatório gerado (24.1) → todas as seções populadas com dados reais.

## Dev Notes

### Componentes a referenciar

- `<TopPostsByFollowersCard>` (PR #83) — padrão de ranking de posts
- `<OverviewCards>` (instagram) — padrão de KpiCard com gradient/border por categoria
- `<ReachChart>` — pattern de Recharts LineChart
- `<HotColdSpendDonut>` (funnel) — pattern de donut Recharts

### Layout responsivo

- Mobile: 2 cols nos cards de totais, ranking em lista vertical, gráficos full-width
- Tablet+: 4 cols nos totais, rankings em grid 2x2

### Performance

- A página é fetch-once + render. Sem polling. Sem refetch.
- Imagens dos posts vêm de CDN do IG — usar `<img>` simples (não Next Image, IG não permite optimização por proxy)
