## 📖 Story

**Como** gestor de tráfego/copywriter do projeto,
**Quero** uma aba "Switch" a nível de projeto onde eu configuro os pixels (Meta + GTM) que serão injetados nos shortlinks e gerencio os presets de canal (cada par utm_medium + utm_source),
**Para** que toda geração de link em lote use as configurações certas do projeto e os canais corretos, sem depender do time de engenharia.

---

## ✅ Acceptance Criteria

1. **Aba "Switch" registrada a nível de PROJETO:**
   - Novo item de navegação "Switch" aparece na sidebar do projeto (junto com Ads, Vendas, Conversas, etc.), apontando para `/projects/:id/switch`.
   - A aba existente `switchy-links-tab.tsx` (nível etapa, read-only) permanece **intacta** — não é movida nem alterada.
   - A nova rota `/projects/:id/switch` renderiza um host de componentes da aba (config panel + presets manager).

2. **Painel de configuração (pixels + GDPR + defaults):**
   - Carrega settings via `GET /api/projects/:projectId/switchy/settings` → `{ pixels, showGdpr, defaultUtmTerm, defaultUtmContent }`.
   - Pixel **Meta**: input para Meta pixel ID → gera `{ platform: "facebook", value, title }`.
   - Pixel **GTM**: input para GTM container (`GTM-XXXX`) → gera `{ platform: "gtm", value, title }`.
   - Toggle **show_gdpr** (boolean).
   - Inputs **defaultUtmTerm** e **defaultUtmContent** (opcionais, pré-preenchem o gerador na Story 33.5).
   - Botão "Salvar" persiste via `PUT /api/projects/:projectId/switchy/settings` (upsert). Toast de sucesso/erro com `sonner`.

3. **Presets manager (CRUD dos canais):**
   - Carrega presets via `GET /api/projects/:projectId/switchy/presets` (semeia os 7 defaults se vazio — comportamento do backend da Story 33.3).
   - Tabela lista cada preset com colunas: `label`, `utm_medium`, `utm_source`, toggle `enabled`, ações (editar / remover).
   - "Adicionar canal" cria preset via `POST .../presets` (campos: label, utm_medium, utm_source).
   - Editar atualiza via `PUT .../presets/:presetId`.
   - Remover via `DELETE .../presets/:presetId` (com confirmação).
   - Toggle `enabled` atualiza via `PUT .../presets/:presetId`.

4. **Permissão (guest):**
   - Edição (salvar settings, criar/editar/remover/toggle presets) é **bloqueada para guest** — `role !== null && role !== "guest"` libera; guest vê leitura mas controles desabilitados/ocultos (alinha com o gating do backend da Story 33.3).
   - O backend já bloqueia guest; o frontend não deve mostrar/permitir os controles de escrita para guest.

5. **Hooks novos em `use-switchy.ts`:**
   - `useSwitchySettings(projectId)` (GET settings).
   - `useSetSwitchySettings(projectId)` (PUT settings, invalida `["switchy-settings", projectId]`).
   - `useSwitchyPresets(projectId)` (GET presets).
   - `useCreateSwitchyPreset(projectId)` / `useUpdateSwitchyPreset(projectId)` / `useDeleteSwitchyPreset(projectId)` (mutations, invalidam `["switchy-presets", projectId]`).
   - Hooks existentes `useSwitchyFolders` / `useSwitchyLinks` permanecem **intactos**.

6. **Estados de UI:**
   - Loading: `Skeleton` enquanto settings/presets carregam.
   - Erro: mensagem clara + botão "Tentar de novo" (padrão `mautic-stage-tab.tsx`).
   - Empty: presets nunca ficam vazios (backend semeia 7), mas tratar defensivamente.

## 🔧 Dev Notes

### Ponto de integração da aba (DESCOBERTO)

O projeto **não usa `TabsList`** a nível de projeto. As "abas" de projeto são **itens de navegação na sidebar**, registrados em:

- **Arquivo:** `packages/web/components/layout/project-folder.tsx`
- **Array:** `PROJECT_SUBITEMS` (linhas 94–100), cada item = `{ label, href, icon }`.
- Cada `href` mapeia para um **segmento de rota** em `app/(app)/projects/[id]/<href>/page.tsx`.

Exemplo atual:
```ts
const PROJECT_SUBITEMS = [
  { label: "Minds", href: "minds", icon: Brain },
  { label: "Ads", href: "traffic", icon: TrendingUp },
  { label: "YouTube Ads", href: "youtube", icon: Youtube },
  { label: "Vendas", href: "sales", icon: ArrowUpDown },
  { label: "Conversas", href: "conversations", icon: MessageSquare },
] as const;
```

**Plug da aba "Switch":**
1. Adicionar `{ label: "Switch", href: "switch", icon: Link2 }` (importar `Link2` de `lucide-react`) ao `PROJECT_SUBITEMS`.
2. Criar `app/(app)/projects/[id]/switch/page.tsx` (segmento de rota = `switch`, igual ao `href`).
3. (Opcional/cosmético) Os checks de active-state em `ProjectFolder` (linha 460 `isProjectActive` e linha 507 que exclui `/instagram`, `/traffic`, `/conversations`) — verificar se a nova aba precisa de ajuste no highlight; o `Button variant` por item (linhas 675–692) já usa `pathname.startsWith(href)`, então a aba nova já fica destacada sem mudança extra. Não é necessário alterar a linha 507 (ela só controla o destaque do nó-pai "projeto").

### Contrato de dados (settings)

```
GET  /api/projects/:projectId/switchy/settings
  -> { pixels, showGdpr, defaultUtmTerm, defaultUtmContent }
PUT  /api/projects/:projectId/switchy/settings   (upsert, bloquear guest)
```
- `pixels`: array de `{ platform: "facebook" | "gtm", value: string, title: string }`.
- `showGdpr`: boolean.
- `defaultUtmTerm` / `defaultUtmContent`: string opcional (varchar 120).

Tabela `project_switchy_settings` (já definida na Story 33.1): `pixels jsonb default '[]'`, `show_gdpr boolean default false`, `default_utm_term varchar(120)`, `default_utm_content varchar(120)`, UNIQUE em `project_id`.

### Contrato de dados (presets)

```
GET    /api/projects/:projectId/switchy/presets             (semeia os 7 se vazio)
POST   /api/projects/:projectId/switchy/presets             (cria — bloquear guest)
PUT    /api/projects/:projectId/switchy/presets/:presetId   (atualiza — bloquear guest)
DELETE /api/projects/:projectId/switchy/presets/:presetId   (remove — bloquear guest)
```
- Preset = `{ id, label, utmMedium, utmSource, sortOrder, enabled }` (tabela `switchy_channel_presets`).
- 7 defaults semeados pelo backend:

| label | utm_medium | utm_source |
|-------|-----------|-----------|
| bio | bio | ig |
| direct | direct | ig |
| stories | stories | ig |
| manychat | automacao | manychat |
| chatwoot | disparo | chatwoot |
| email | email | mautic |
| grupo | grupo | whatsapp |

### Hooks novos (mirror de `use-mautic.ts` — query + mutation + invalidate)

Adicionar em `packages/web/lib/hooks/use-switchy.ts` (manter `useSwitchyFolders`/`useSwitchyLinks` intactos):

```ts
export interface SwitchyPixel { platform: "facebook" | "gtm"; value: string; title: string; }
export interface SwitchySettings {
  pixels: SwitchyPixel[];
  showGdpr: boolean;
  defaultUtmTerm: string | null;
  defaultUtmContent: string | null;
}
export interface SwitchyPreset {
  id: string; label: string; utmMedium: string; utmSource: string;
  sortOrder: number; enabled: boolean;
}

// GET settings
export function useSwitchySettings(projectId: string | null) { /* useQuery ["switchy-settings", projectId] */ }
// PUT settings (upsert) — invalida ["switchy-settings", projectId]
export function useSetSwitchySettings(projectId: string) { /* useMutation */ }
// GET presets
export function useSwitchyPresets(projectId: string | null) { /* useQuery ["switchy-presets", projectId] */ }
// POST / PUT / DELETE presets — invalidam ["switchy-presets", projectId]
export function useCreateSwitchyPreset(projectId: string) { /* useMutation */ }
export function useUpdateSwitchyPreset(projectId: string) { /* useMutation (presetId no body/path) */ }
export function useDeleteSwitchyPreset(projectId: string) { /* useMutation */ }
```
Padrão exato: `useApiClient()` + `useQuery`/`useMutation` + `useQueryClient().invalidateQueries(...)` no `onSuccess`. `staleTime = 5 * 60 * 1000` (igual `SWITCHY_STALE_TIME`).
