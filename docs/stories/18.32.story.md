## Dev Notes

**CRITICAL Pattern Reference:**
See `/packages/api/src/routes/stage-creative-performance.ts` (lines 321-385) for the grouping pattern:
1. Fetch from Meta Ads API (e.g., ad_id, ad_name, spend, etc.)
2. Read sales sheet (extract utm_medium / utm_content IDs)
3. Create index: ID → metrics
4. **GROUP BY NAME** (not ID) — because one name can have multiple IDs
5. Consolidate metrics across all IDs for each name

**For "Por Medium (Adset)":**
```
fetched from Meta Ads → { adset_id, adset_name, spend, impressions, clicks }
sales data → { utm_medium, leads, revenue }
match: utm_medium (from sales) crosses with adset_id (from Meta API)
group by: adset_name
aggregate: spend, impressions, clicks, leads, revenue by adset_name
```

**For "Por Content (Ad)" — Exact Same Pattern:**
```
fetched from Meta Ads → { ad_id, ad_name, spend, impressions, clicks }
sales data → { utm_content, leads, revenue }
match: utm_content (from sales) crosses with ad_id (from Meta API)
group by: ad_name
aggregate: spend, impressions, clicks, leads, revenue by ad_name
```
