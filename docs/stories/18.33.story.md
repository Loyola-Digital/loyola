## Story

Na seção "Por Content (Ad)" do dashboard de vendas, vários IDs numéricos de ad (utm_content) não estão sendo resolvidos para nomes de anúncios. Eles aparecem com a badge "⚠️ id sem nome", impedindo que o usuário identifique qual anúncio gerou aquelas vendas.

**Problema Root:** A chamada `resolveEntityNames()` para Ad IDs está falhando ou retornando resultados incompletos.

**Objetivo:** Garantir que todos os Ad IDs tenham labels legíveis, seja via Meta API ou fallback inteligente.

## Acceptance Criteria

### AC1: Investigar Falha de Resolução
- [ ] Verificar logs/debug do `resolveEntityNames` para Ad IDs
- [ ] Confirmar se Meta API está retornando dados para esses IDs específicos
- [ ] Identificar se é timeout, rate limit, ou ID inválido

### AC2: Implementar Retry Logic
- [ ] Se `resolveEntityNames` falhar para um Ad ID, tentar novamente com backoff exponencial
- [ ] Max 2 tentativas (não sobrecarregar Meta API)
- [ ] Log de falhas para análise posterior

### AC3: Fallback Inteligente
- [ ] Se ID não resolver após retries, gerar label legível:
  - Formato: `"Ad #120251580121030273"` (em vez de só o número)
  - Ou: `"Ad (ID desconhecido)"` se houver muitos IDs falhados
- [ ] Badge "⚠️ id sem nome" SÓ aparece para Meta IDs numéricos que falharam legitimamente

### AC4: Validação de Dados
- [ ] Nenhum Ad ID deve aparecer como "só número nu" na UI
- [ ] Todos os Ad IDs devem ter uma label (resolvida ou fallback)
- [ ] Sem duplicatas de IDs falhados na tabela

### AC5: Testes
- [ ] Verificar que Ad IDs resolvidos aparecem com nomes corretos
- [ ] Verificar que Ad IDs falhados usam fallback
- [ ] Validar sorting por vendas funciona corretamente
