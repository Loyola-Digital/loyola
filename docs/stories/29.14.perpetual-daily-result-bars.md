# Story 29.14 — Funil Perpétuo: gráficos "no tempo" viram barra de resultado/dia

**Epic:** 29
**Status:** InReview
**Points:** 3
**Tipo:** UX / Dataviz
**Prioridade:** P2

## Contexto

Os gráficos "Investimento no Tempo" e "Margem no Tempo" (linhas com rótulos numéricos em cada ponto) estavam **poluídos** — difícil interpretar. Danilo pediu redesenho e escolheu (2026-07-06) a direção **barra de resultado/dia**: uma barra por dia, verde se lucro / vermelho se prejuízo, valores no tooltip, com linhas-guia no fundo.

## User Story

Como usuário de funil perpétuo, quero ver de relance se cada dia foi **lucro ou prejuízo** (barra verde/vermelha) sem poluição de números, e ver Investimento/Receita detalhados só ao passar o mouse.

## Acceptance Criteria

- [x] **AC1:** "Investimento no Tempo" → renomeado **"Resultado no Tempo"**: barra única por dia = **Receita − Investimento** (resultado bruto). Verde se ≥0, vermelho se <0.
- [x] **AC2:** "Margem no Tempo": barra única por dia = **Margem líquida** (com fees). Verde se ≥0, vermelho se <0. *(Diferencia do gráfico 1: bruto vs líquido — não redundante.)*
- [x] **AC3:** **Sem rótulos numéricos** nas barras (era a poluição). Valores aparecem só no **tooltip**.
- [x] **AC4:** Tooltip mostra: Data, **Investimento**, **Receita** e o **Resultado/Margem** (colorido verde/vermelho).
- [x] **AC5:** Linhas-guia de fundo (`CartesianGrid`) + linha do zero (`ReferenceLine y=0`) pra facilitar leitura dos valores.
- [x] **AC6:** Herda o fix da 29.10: sem planilha conectada, receita/margem/resultado = 0 (não vaza Meta).
- [x] **AC7:** Typecheck + build OK.

## Dev Notes

### Bruto vs líquido — por que os dois gráficos não são redundantes
- **Resultado no Tempo** = Receita **bruta** − Investimento (com imposto). Visão de caixa/topo.
- **Margem no Tempo** = Receita **líquida** (descontados fees da plataforma) − Investimento. Visão de lucro real.
Assim os dois entregam leituras diferentes do mesmo dia.

### Cor por barra
Recharts `<Bar>` com `<Cell>` por ponto — `fill` verde (`hsl(150 60% 45%)`) ou vermelho (`hsl(0 72% 55%)`) conforme o sinal.

### Aberto pra ajuste visual
Cores, altura (260px), raio das barras e textos dos subtítulos são fáceis de ajustar — Danilo valida no Local e a gente afina.
