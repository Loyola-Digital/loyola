## Story

Como desenvolvedor, quero uma tabela `funnel_stages` com CRUD completo via API, para que o frontend possa criar e gerenciar etapas independentes dentro de um funil, cada uma com sua própria configuração de campanhas.

## Contexto

O funil hoje tem configuração única: `campaigns`, `metaAccountId`, `googleAdsCampaigns`, `googleAdsAccountId`, `switchyFolderIds`, `switchyLinkedLinks`. A Story 19.1 extrai esses campos para uma tabela `funnel_stages`, mantendo retrocompatibilidade via migration que cria uma etapa "Principal" para cada funil existente.

## Escopo

### IN
- Criar tabela `funnel_stages` no schema Drizzle (`packages/api/src/db/schema.ts`)
- Criar migration SQL (`0012_funnel_stages.sql`)
- Criar migration de dados SQL (`0013_funnel_stages_seed.sql`) — copia config de cada funil existente para uma etapa "Principal"
- Criar arquivo de rotas `packages/api/src/routes/funnel-stages.ts` com CRUD completo
- Registrar rotas em `packages/api/src/app.ts`
- Adicionar tipo `FunnelStage` em `packages/shared/src/types/funnel.ts`
- Exportar `FunnelStage` de `packages/shared/src/index.ts`

### OUT
- Frontend / hooks React
- Mudanças no dashboard
- Per-stage planilhas e pesquisas (Story 19.4)
- Remoção dos campos de campanha do `funnels` table (retrocompat — ficam lá)

### Schema — `funnel_stages`

// packages/api/src/db/schema.ts
export const funnelStages = pgTable(
  "funnel_stages",

      { onDelete: "set null" }

    googleAdsCampaigns: jsonb("google_ads_campaigns")
      .notNull()
      .default([])
      .$type<{ id: string; name: string }[]>(),

      .notNull()
      .default([])

### Migration SQL — `0012_funnel_stages.sql`

```sql
CREATE TABLE "funnel_stages" (
  "id" uuid PRIMARY KEY DEFAULT gen_random_uuid() NOT NULL,
  "funnel_id" uuid NOT NULL REFERENCES "funnels"("id") ON DELETE CASCADE,
  "name" varchar(255) NOT NULL,
  "meta_account_id" uuid REFERENCES "meta_ads_accounts"("id") ON DELETE SET NULL,
  "campaigns" jsonb DEFAULT '[]'::jsonb NOT NULL,
  "google_ads_account_id" uuid REFERENCES "google_ads_accounts"("id") ON DELETE SET NULL,
  "google_ads_campaigns" jsonb DEFAULT '[]'::jsonb NOT NULL,
  "switchy_folder_ids" jsonb DEFAULT '[]'::jsonb NOT NULL,
  "switchy_linked_links" jsonb DEFAULT '[]'::jsonb NOT NULL,
  "sort_order" integer DEFAULT 0 NOT NULL,
  "created_at" timestamptz DEFAULT now() NOT NULL,
  "updated_at" timestamptz DEFAULT now() NOT NULL
);
CREATE INDEX "idx_funnel_stages_funnel" ON "funnel_stages" ("funnel_id");
```

### API Routes — `packages/api/src/routes/funnel-stages.ts`

Prefixo de todos os endpoints: `/api/projects/:projectId/funnels/:funnelId/stages`

| Método | Path | Descrição |
|--------|------|-----------|
| GET | `/` | Lista etapas do funil |
| POST | `/` | Cria nova etapa |
| GET | `/:stageId` | Busca etapa por ID |
| PUT | `/:stageId` | Atualiza etapa (name, campaigns, etc.) |
| DELETE | `/:stageId` | Remove etapa (mínimo de 1 etapa por funil — retorna 409 se for a última) |

**Schemas Zod:**
```typescript
const createStageSchema = z.object({
  name: z.string().min(1).max(255),
  metaAccountId: z.string().uuid().nullable().optional(),
  campaigns: z.array(campaignSchema).default([]),
  googleAdsAccountId: z.string().uuid().nullable().optional(),
  googleAdsCampaigns: z.array(campaignSchema).default([]),
  switchyFolderIds: z.array(switchyFolderSchema).default([]),
  switchyLinkedLinks: z.array(switchyLinkRefSchema).default([]),
});

const updateStageSchema = createStageSchema.partial();
```

**Regra de negócio:** DELETE retorna 409 `{ error: "Não é possível remover a única etapa do funil" }` se `count(stages where funnel_id = X) === 1`.

**Helper `stageShape`:** Mesmo padrão do `funnelShape` — mapeia campos do DB para o tipo `FunnelStage`.
