## Valor de Negócio

É a entrega final que fecha o objetivo do Lucas: um **MCP server** que embrulha a API pública (36.3/36.5) como **tools** que a IA dele chama diretamente — "liste campanhas", "performance dos criativos da SCA", "série temporal desse ad", "ROAS real desse criativo". Mais o **`llms.txt`**, um documento que ensina qualquer LLM a usar a API/MCP sem adivinhação. Sem esta story, a API existe mas a IA não sabe usá-la.

## Dependências

- **Bloqueante:** Story 36.3 (endpoints públicos). 36.5 é desejável (ROAS) mas o MCP pode lançar com os dados básicos e ganhar ROAS quando 36.5 mergear.
- **Auth:** usa uma API Key gerada na 36.1 (configurada no MCP via env).
- **Publicação/registro:** @devops (config de MCP é exclusiva dele — ver `mcp-usage.md`).

---

## Contexto

- Stack já é Node/TypeScript monorepo (pnpm + turbo). O MCP server entra como **pacote novo** (`packages/mcp`), seguindo o layout dos outros packages.
- O MCP é um cliente HTTP fino: chama `https://<api-host>/api/public/meta/v1/*` com header `X-API-Key`, expõe as respostas como tools MCP.
- Construir AI apps → usar SDK Anthropic e modelos Claude mais recentes quando aplicável (mas aqui o MCP é só transporte; a IA consumidora é externa).

- [ ] Erros da API (401/403/429) traduzidos em mensagens úteis para a IA (ex.: "API key inválida ou revogada").

- **llms.txt:** seguir a convenção do formato (https://llmstxt.org) — markdown estruturado, conciso, navegável por LLM.
