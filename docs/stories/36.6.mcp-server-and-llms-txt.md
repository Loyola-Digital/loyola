# Story 36.6: MCP Server + `llms.txt` — a IA consome a Meta Ads Creative Intelligence API

**Epic:** 36 - Meta Ads Creative Intelligence API + MCP
**Story:** 36.6
**Status:** Draft
**Priority:** High
**Complexity:** Medium (~1,5 dia)
**Points:** 8
**Assignee:** @dev

---

## Valor de Negócio

É a entrega final que fecha o objetivo do Lucas: um **MCP server** que embrulha a API pública (36.3/36.5) como **tools** que a IA dele chama diretamente — "liste campanhas", "performance dos criativos da SCA", "série temporal desse ad", "ROAS real desse criativo". Mais o **`llms.txt`**, um documento que ensina qualquer LLM a usar a API/MCP sem adivinhação. Sem esta story, a API existe mas a IA não sabe usá-la.

## Dependências

- **Bloqueante:** Story 36.3 (endpoints públicos). 36.5 é desejável (ROAS) mas o MCP pode lançar com os dados básicos e ganhar ROAS quando 36.5 mergear.
- **Auth:** usa uma API Key gerada na 36.1 (configurada no MCP via env).
- **Publicação/registro:** @devops (config de MCP é exclusiva dele — ver `mcp-usage.md`).

---

## Contexto

- Stack já é Node/TypeScript monorepo (pnpm + turbo). O MCP server entra como **pacote novo** (`packages/mcp`), seguindo o layout dos outros packages.
- O MCP é um cliente HTTP fino: chama `https://<api-host>/api/public/meta/v1/*` com header `X-API-Key`, expõe as respostas como tools MCP.
- Construir AI apps → usar SDK Anthropic e modelos Claude mais recentes quando aplicável (mas aqui o MCP é só transporte; a IA consumidora é externa).

## Aceitação

### AC1: Pacote MCP server
- [ ] Novo pacote `packages/mcp` (TypeScript), usando o SDK oficial de MCP (`@modelcontextprotocol/sdk`).
- [ ] Config via env: `LOYOLA_API_BASE_URL`, `LOYOLA_API_KEY`.
- [ ] Transporte stdio (padrão para MCP local) — confirmar com @devops se precisa também HTTP/SSE.

### AC2: Tools expostas
- [ ] `list_campaigns(from?, to?, accountId?)` → `GET /campaigns`
- [ ] `get_creative_performance(campaignId?, from?, to?, orderBy?, limit?)` → `GET /creatives`
- [ ] `get_creative_timeseries(adId, from?, to?)` → `GET /creatives/:adId/timeseries`
- [ ] `get_real_roas(...)` → campos de ROAS (quando 36.5 disponível; caso contrário, tool documenta que ROAS pode vir `null`).
- [ ] Cada tool com schema de input claro (JSON Schema) e descrição que ensina a IA quando usá-la.
- [ ] Erros da API (401/403/429) traduzidos em mensagens úteis para a IA (ex.: "API key inválida ou revogada").

### AC3: `llms.txt`
- [ ] Gerar `docs/llms.txt` (e/ou na raiz, conforme convenção llms.txt) documentando:
  - O que é a API, base URL, autenticação (`X-API-Key`), scopes.
  - Cada endpoint: método, path, params, shape de response, exemplo.
  - As tools MCP equivalentes e quando usar cada uma.
  - Limites (rate limit, read-only, `partial`/`attributionCoverage`/`null` honestos).
- [ ] Conteúdo derivado do contrato real das stories 36.3/36.5 (não inventar campos).

### AC4: Documentação de setup
- [ ] README do pacote `packages/mcp`: como configurar a env, gerar a API key (link p/ tela da 36.1), registrar o MCP no cliente da IA do Lucas.

## Out of Scope
- Hospedar o MCP como serviço remoto multi-tenant (MVP é local/single-key).
- Tools de escrita (a API é read-only).
- Publicar o pacote em registry público.

---

## Riscos

| Risco | Prob. | Impacto | Mitigação |
|-------|-------|---------|-----------|
| Contrato da API mudar e quebrar as tools | Média | Médio | Versão `/v1/`; tools e `llms.txt` derivam do contrato; mudanças aditivas |
| API key vazar na config do MCP | Média | Alto | Documentar uso de env/secret; chave revogável (36.1) |
| `llms.txt` desatualizar vs. API | Média | Médio | Derivar do contrato; checklist de atualização ao mudar endpoint |

## Notas Técnicas
- **MCP é transporte fino:** zero lógica de negócio no MCP — só mapeia tool → request HTTP → response. Toda regra vive na API.
- **Config de MCP é @devops** (`mcp-usage.md`): o registro/ativação do MCP no ambiente da IA do Lucas é feito/coordenado pelo @devops.
- **llms.txt:** seguir a convenção do formato (https://llmstxt.org) — markdown estruturado, conciso, navegável por LLM.
