# Story 23.1 — Schema & API: Stage Organic Posts

## Status

InReview

## Story

**As an** analista do Loyola Digital X,
**I want** uma tabela `stage_organic_posts` que vincule posts orgânicos (YouTube + Instagram) a etapas de funil, com endpoints REST para criar, listar e remover vínculos,
**so that** as histórias 23.2 (UI no Dash Orgânico) e 23.3 (tab no Stage Page) tenham uma base de dados estável e API consistente para consumir.

## Acceptance Criteria

### Database Schema

1. Criada tabela `stage_organic_posts` em `packages/api/src/db/schema.ts` com colunas:
   - `id` (uuid, PK, defaultRandom)
   - `stage_id` (uuid, FK → `funnel_stages.id`, ON DELETE CASCADE, NOT NULL)
   - `project_id` (uuid, FK → `projects.id`, ON DELETE CASCADE, NOT NULL) — denormalizado para queries por projeto e checagem de permissão
   - `source` (enum `organic_post_source`: `'youtube' | 'instagram'`, NOT NULL)
   - `external_id` (varchar(100), NOT NULL) — `videoId` para YouTube, `mediaId` para Instagram
   - `created_by` (uuid, FK → `users.id`, NOT NULL)
   - `created_at` (timestamptz, default NOW(), NOT NULL)

2. Constraints e índices:
   - `UNIQUE (stage_id, source, external_id)` — impede vincular o mesmo post à mesma etapa duas vezes
   - `INDEX (stage_id)` — para listagem por etapa
   - `INDEX (project_id, source)` — para consulta de "quais posts já estão vinculados" no Dash Orgânico
   - Enum `organic_post_source` criado via `pgEnum`

3. Migration aplicada em PROD via `node -e` com `pg` Pool (padrão do projeto, ver Epic 19/20). Migration file commitada em `packages/api/src/db/migrations/`.

### API Endpoints

4. `POST /api/projects/:projectId/funnels/:funnelId/stages/:stageId/organic-posts`
   - Body: `{ source: 'youtube' | 'instagram', externalId: string }`
   - Valida com Zod: `source` enum, `externalId` non-empty string max 100 chars
   - Verifica `getProjectAccess(userId, projectId)` (espelhar padrão de `lead-scoring.ts` e demais routes da etapa)
   - Verifica que `stageId` pertence a `funnelId` e que `funnelId` pertence a `projectId` (404 caso contrário)
   - Insere vínculo. Em conflict (UNIQUE), retorna 409 com `{ error: 'already_linked' }`
   - Retorna 201 com o registro criado

5. `DELETE /api/projects/:projectId/funnels/:funnelId/stages/:stageId/organic-posts/:linkId`
   - Verifica `getProjectAccess` e ownership da chain (project → funnel → stage → link)
   - Hard delete
   - Retorna 204 No Content
   - 404 se link não existe ou não pertence à stage

6. `GET /api/projects/:projectId/funnels/:funnelId/stages/:stageId/organic-posts`
   - Verifica `getProjectAccess` e ownership da chain
   - Retorna array de vínculos com hidratação de métricas:
     ```typescript
     Array<{
       id: string;            // link id
       source: 'youtube' | 'instagram';
       externalId: string;
       createdAt: string;
       createdBy: string;
       hydration: {
         isStale: boolean;    // true se post não foi encontrado na fonte
         title: string | null;
         thumbnailUrl: string | null;
         externalUrl: string;          // youtube.com/watch?v=... ou instagram.com/p/...
         metrics: Record<string, number | null>; // shape varia por source — ver Dev Notes
       } | null; // null em caso de erro de fetch (não-stale, mas indisponível agora)
     }>
     ```

7. `GET /api/projects/:projectId/organic-posts/links?source={youtube|instagram}` (auxiliar para o Dash Orgânico)
   - Retorna `Array<{ externalId: string, stageIds: string[] }>` — todos os vínculos do projeto agrupados por `externalId` para o frontend pintar "este post já está vinculado às etapas X, Y"
   - Verifica `getProjectAccess`

8. Todos os endpoints registrados em `packages/api/src/app.ts` e seguem o padrão de `fp()` closure (per `project_loyola_digital_x.md` memory).

### Hidratação (server-side)

9. Para `source = 'youtube'`: usar o serviço/query existente que alimenta `useYouTubeVideos` (canal vinculado ao projeto via `youtubeChannelProjects`). Buscar pelos `videoIds` solicitados e retornar `title`, `thumbnailUrl`, `viewCount`, `likeCount`, `commentCount`, `watchTimeMinutes` (se disponível), `avgRetention` (se disponível). Se vídeo não retornar da API → `isStale: true`.

10. Para `source = 'instagram'`: usar serviço Instagram Graph existente (conta vinculada via `instagramAccountProjects`). Buscar `media_id` e retornar `title` (caption truncada a 80 chars), `thumbnailUrl` (`media_url` ou `thumbnail_url`), `reach`, `impressions`, `likeCount` (`like_count`), `commentCount` (`comments_count`), `saved` (se `media_type === 'VIDEO'`). Se mídia não retornar → `isStale: true`.

11. Cache server-side de 5 min em memória (Map com TTL) para evitar bater na API externa a cada GET. Chave: `${source}:${externalId}`. Compartilhado entre os dois GETs (4 e 7).

### Shared Types

12. Adicionar tipos em `@loyola-x/shared`:
    ```typescript
    type OrganicPostSource = 'youtube' | 'instagram';

    interface StageOrganicPost {
      id: string;
      stageId: string;
      source: OrganicPostSource;
      externalId: string;
      createdAt: string;
      createdBy: string;
    }

    interface StageOrganicPostHydrated extends StageOrganicPost {
      hydration: OrganicPostHydration | null;
    }

    interface OrganicPostHydration {
      isStale: boolean;
      title: string | null;
      thumbnailUrl: string | null;
      externalUrl: string;
      metrics: YouTubeOrganicMetrics | InstagramOrganicMetrics;
    }

    interface YouTubeOrganicMetrics {
      viewCount: number | null;
      likeCount: number | null;
      commentCount: number | null;
      watchTimeMinutes: number | null;
      avgRetention: number | null;
    }

    interface InstagramOrganicMetrics {
      reach: number | null;
      impressions: number | null;
      likeCount: number | null;
      commentCount: number | null;
      saved: number | null;
    }
    ```

## Dev Notes

### Padrões a seguir (do projeto)

- **FK auto-referência (Drizzle):** se precisar, usar `(): AnyPgColumn => table.id` (per memory)
- **Rotas dentro de `fp()` closure** (per memory)
- **Migrations manuais via `node -e` + `pg`** — drizzle-kit push **não** é o padrão deste projeto (per memory + Epic 19/20)
- **`getProjectAccess`** já existe em vários routes (ver `lead-scoring.ts`, `funnels.ts`) — copiar shape

### Referências

- Schema relacionado: `funnels`, `funnelStages` (linhas 538-643 em `packages/api/src/db/schema.ts`)
- Conta Instagram: `instagramAccounts` + `instagramAccountProjects` (linhas 221-270)
- Canal YouTube: `youtubeChannels` + `youtubeChannelProjects` (linhas 484-528)
- Padrão de hidratação multi-source: ver `meta-ads-comparison` endpoint do Epic 20 (referência conceitual)

### Riscos técnicos

- **Cache em memória não escala em multi-instância** — ok para MVP (Railway 1 instância). Se escalar, migrar para Redis. Documentar como tech debt.
- **Rate limit do Instagram Graph** — se hidratar 50 posts numa stage, pode estourar. Mitigação: cache 5min + fallback `null` em erro 429.
- **YouTube API quota** — mesmo raciocínio.
