## Story

**As a** Loyola Digital admin,
**I want** to invite guests to projects with granular module permissions,
**so that** clients can access only the data I choose to share.

---

## Acceptance Criteria

1. [ ] `POST /api/projects/:id/invitations` — Criar convite
   - Body: `{ email, permissions: { instagram: bool, conversations: bool, mind: bool } }`
   - Gera token com `crypto.randomBytes(32).toString('hex')`
   - Salva em `project_invitations` com `expires_at = now + 7 days`
   - Retorna: `{ inviteUrl: "{FRONTEND_URL}/invite/{token}", expiresAt }`
2. [ ] `GET /api/invitations/:token` — Verificar token
   - Token válido: retorna `{ projectName, invitedByName, email, permissions, expiresAt }`
   - Token expirado ou já aceito: retorna 404 com mensagem clara
3. [ ] `POST /api/invitations/:token/accept` — Aceitar convite
   - Cria user em `users` com `role = 'guest'` se não existir (busca pelo email no Clerk)
   - Insere em `project_members` com `permissions` copiadas do convite
   - Marca `accepted_at = now()`
   - Retorna: `{ projectId, userId, permissions }`
4. [ ] `DELETE /api/projects/:id/members/:userId` — Remover guest do projeto
   - Remove de `project_members` (não deleta user do Clerk)
5. [ ] `PATCH /api/projects/:id/members/:userId/permissions` — Editar permissões
   - Body: `{ permissions: { instagram: bool, conversations: bool, mind: bool } }`
   - Atualiza `project_members.permissions`
6. [ ] `GET /api/projects/:id/members` — Listar membros com permissões
7. [ ] Token inválido retorna 404 com `{ error: "Convite não encontrado ou expirado" }`
8. [ ] Token já aceito retorna 409 com `{ error: "Convite já foi aceito" }`
9. [ ] Endpoint `POST /invitations/:token/accept` não requer auth (é o fluxo de onboarding)
10. [ ] Todos os outros endpoints de invitações/membros requerem auth de admin
11. [ ] Testes de integração para cada endpoint

## Dev Notes

### Architecture Context

- **Routes:** Plugin Fastify — seguir padrão de `routes/projects.ts` (Story 4.2)
- **Auth:** `POST /api/invitations/:token/accept` é rota pública (guest ainda não tem auth)
- **Clerk:** MVP não envia email; URL retornada no response para admin copiar manualmente
- **FRONTEND_URL:** disponível via `app.config.FRONTEND_URL` (variável de ambiente)

### Token Generation

```typescript
import { randomBytes } from "crypto";
const token = randomBytes(32).toString("hex"); // 64 chars
```

### Accept Flow (Simplified MVP)

O accept flow do MVP não cria usuário no Clerk automaticamente — apenas registra na tabela `users` com `role = 'guest'`. O guest fará login separadamente via Clerk usando o email do convite. A associação é feita pelo email no momento do aceite.
