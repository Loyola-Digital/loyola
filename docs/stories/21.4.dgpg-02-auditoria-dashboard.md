## Story

### Problema
Atualmente, não há forma de rastrear quando um usuário validou/auditou os dados do dashboard Meta Ads. Isso dificulta:
- Validar a integridade dos dados exibidos
- Rastrear quem foi responsável pela validação
- Saber quando foi a última auditoria realizada
- Auditoria compliance (rastreabilidade)

### Solução Proposta
Implementar um componente de auditoria que:
1. Exibe um **badge/status visual** no dashboard indicando estado de auditoria (verde = auditado, cinza = pendente)
2. **Registra** data, hora e usuário quando auditoria é realizada
3. **Exibe** essas informações em um widget visível no dashboard
4. **Permite** o usuário marcar como "Auditado" com um botão/ação

### User Story
**Como** usuário responsável pelo dashboard Meta Ads,  
**Eu quero** marcar o dashboard como "Auditado" e visualizar quem fez e quando,  
**Para que** eu tenha rastreabilidade completa das validações de dados.

## Acceptance Criteria

### AC-1: Badge de Status de Auditoria Visível
**Given** dashboard Meta Ads está carregado  
**When** página renderiza  
**Then** um badge/widget de status de auditoria é exibido (posição: próximo aos KPIs topo-direita ou na seção de tools)

### AC-2: Status "Não Auditado" (Padrão)
**Given** primeira carga ou nenhuma auditoria registrada  
**When** dashboard renderiza  
**Then** badge exibe "Não auditado" com cor cinza e ícone de alerta

### AC-3: Ação de Auditoria
**Given** usuário clica no badge/widget de auditoria  
**When** confirma intenção  
**Then** sistema registra:
- Data/hora ISO (UTC)
- ID do usuário autenticado
- Nome do usuário (full_name ou clerk profile)

### AC-4: Status "Auditado" (Após Validação)
**Given** auditoria foi registrada com sucesso  
**When** página recarrega ou estado é atualizado  
**Then** badge exibe:
- "Auditado em [data]" ou "Auditado 2 min atrás"
- Nome do usuário: "por [Nome do Usuário]"
- Cor verde com ícone de checkmark
- Data/hora completa em tooltip ao hover

### AC-5: Persistência no Banco de Dados
**Given** auditoria é marcada  
**When** sessão é encerrada  
**Then** ao retornar, auditoria anterior ainda é exibida (dados persistem)

### AC-6: Reset de Auditoria (Optional)
**Given** dados do dashboard são atualizados/recarregados  
**When** data da última sincronização é mais recente que auditoria  
**Then** status volta para "Não auditado" ou exibe aviso (TBD com UX)

- Análises/dashboards de auditoria
