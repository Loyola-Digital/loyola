## Story

Story 18.32 AC2-3 deveria agrupar dados por **name resolvido** (vários IDs com mesmo name consolidam em uma linha). Atualmente, quando um Ad ID não é resolvido pela Meta API, aparece como número (`120251580121030273`) em vez de:
1. Um name (se resolvido)
2. Um identificador inteligente (se não resolvido)

**Problema:** Backend retorna IDs em vez de names na chave da resposta (`content` field).

**Objetivo:** Garantir agrupamento correto por name — vários IDs da mesma campanha/criativos consolidam em uma única linha.

```typescript
for (const [contentId, metrics] of utmContentMap.entries()) {
  const adName = (contentNames.get(contentId) ?? contentId).trim() || "(sem nome)";
  // ... grouping ...
}
const porUtmContentGrouped = Array.from(contentByName.values())
  .map(({ name, vendas, bruto, liquido }) => ({ content: name, name, vendas, bruto, liquido }))
```

**Issue:** Quando `contentNames.get(contentId)` retorna `undefined`, usa `contentId` como fallback. Depois agrupa por isso.

**Fix:**
1. Se `contentNames.get(contentId)` é undefined E contentId é numérico, usar label: `"[Unresolved] ID"`
2. Isso garante que todos os IDs falhados com padrão similar agrupem juntos
