## Story

**As a** Loyola Digital employee,
**I want** a file attachment button in the chat input that lets me pick a document,
**so that** I can easily upload .txt, .docx, or .pdf files as context for my conversation with a mind.

---

## Acceptance Criteria

1. [ ] Paperclip/attachment button to the left of the textarea in `chat-input.tsx`
2. [ ] Clicking opens native file picker filtered to `.txt,.docx,.pdf`
3. [ ] After selecting a file, a preview chip appears above the textarea showing: filename, file size, and a remove (X) button
4. [ ] While uploading/extracting, the chip shows a loading spinner
5. [ ] On file select: immediately POST to `/api/upload`, receive extracted text
6. [ ] If upload fails, show error toast/inline message and remove chip
7. [ ] When user sends message with an attached file: include `attachmentContext` and `attachmentMeta` in the chat request
8. [ ] After sending, the chip disappears and the attachment is part of the message
9. [ ] In message history, messages with attachments show a small document badge/icon with filename
10. [ ] Disabled state: attachment button disabled while streaming or while another file is being processed
11. [ ] File size client-side check: reject files > 10MB before upload with user-friendly message
12. [ ] Keyboard accessible: attachment button focusable and activatable via Enter/Space

## Dev Notes

### UX Flow

```
1. User clicks 📎 button
2. Native file picker opens (accept=".txt,.docx,.pdf")
3. User selects file
4. Chip appears: "📄 briefing.docx (45KB)" with X button + spinner
5. POST /api/upload → extracts text
6. Spinner stops, chip shows ✓
7. User types message and hits Enter
8. Message sent with attachmentContext + attachmentMeta
9. Chip disappears, message shows in bubble with 📎 badge
```

### Design Tokens (Loyola Brand)

- Chip background: `bg-card border border-border/50`
- Chip text: `text-sm text-foreground`
- Attachment badge in message: `text-xs text-muted-foreground` with `Paperclip` icon from lucide
- Loading state: `Loader2` spinner from lucide (same as thinking-indicator)
- Error state: `text-destructive` inline message

### API Integration

```typescript
// Upload file
const formData = new FormData();
formData.append("file", selectedFile);

const uploadRes = await fetch(`${API_URL}/api/upload`, {
  method: "POST",
  headers: { Authorization: `Bearer ${token}` },
  body: formData,
});

const { filename, mimeType, textLength, extractedText } = await uploadRes.json();

// Then send chat with attachment
sendMessage(mindId, message, {
  attachmentContext: extractedText,
  attachmentMeta: { filename, mimeType, textLength },
});
```
