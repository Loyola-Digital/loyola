# Story 18.41: Tabela Desempenho de Criativos em Captação Gratuita

**Epic:** 18 — Dashboard Improvements  
**Status:** Draft  
**Created:** 2026-06-10  
**Priority:** Medium  

---

## Summary

Extend Creative Performance table to Free Lead stage (Captação Gratuita) with data-driven insights. Currently the table only appears in Paid Lead stage (Captação Paga). Users need the same performance metrics for organic/free traffic with revenue-independent columns.

---

## Acceptance Criteria

### AC1: Table Visibility in Free Stage
- [x] Table "Desempenho de Criativos" appears in Captação Gratuita stage
- [x] Same table behavior as Captação Paga (sorting, filtering, pagination)
- [x] Data loads correctly for free stage creative performance

### AC2: Data Crossings Maintained (Detailed)
- [x] **Meta Ads integration:** Resolve creative names from Meta Ads API (same as paid)
- [x] **Daily data aggregation:** Sum impressions + clicks + leads by creative across period
- [x] **CPL calculation:** Cost-per-Lead = Total Spend ÷ Total Leads (same formula)
- [x] **Filtering logic:** Apply date range, funnelId, stageId filters consistently
- [x] **Sorting:** Maintain sort by Impressions, Leads, CPL, CTR, etc.
- [x] **Pagination:** Keep current page state across stage switches

### AC3: Column Suppression for Free Stage
- [x] **SUPPRESS in Captação Gratuita:**
  - Faturamento (Revenue) — no monetary value in free stage
  - ROAS (Return on Ad Spend) — cannot calculate without revenue
- [x] **MAINTAIN in Captação Gratuita:**
  - Impressões (Impressions)
  - Cliques (Clicks)
  - CTR (Click-Through Rate)
  - Leads (Lead count)
  - CPL (Cost-Per-Lead)
  - % Meta (attribution %)
  - Any other non-revenue columns
- [x] **MAINTAIN in Captação Paga:**
  - All columns unchanged (full feature parity with current)
- **Implementation:** Added `stageType` prop to StageSalesSectionProps + StageCreativePerformanceTable interface; created `visibleColumns` useMemo in table component that filters out 'revenue' and 'roas' when stageType === 'free'; propagated stageType through component tree (launch-dashboard + perpetual-dashboard)

### AC4: UX Consistency
- [x] Table header layout responsive (no column shift when switching stages)
- [x] Empty state handling (if no data for free stage)
- [x] No visual regressions in paid stage table
