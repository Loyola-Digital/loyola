## 📖 Story

**Como** desenvolvedor da plataforma Loyola,
**Quero** estender o service `switchy.ts` com um cliente REST de escrita (`createSwitchyLink`) e um construtor determinístico de URL rastreada (`buildTrackedCheckoutUrl`),
**Para** que a rota de geração em lote (Story 33.x) possa montar a URL completa de cada canal e encurtá-la na folder escolhida do Switchy, reusando o token global e injetando os pixels do projeto.

## ✅ Acceptance Criteria

1. **Service estendido, não recriado:**
   - O código GraphQL existente (`fetchSwitchyFolders`, `fetchSwitchyLinks`, helper `graphql`, tipos `SwitchyFolder`/`SwitchyLink`/`SwitchyPixel`) permanece **intacto** em `packages/api/src/services/switchy.ts`.
   - As novas funções são **adicionadas** ao mesmo arquivo (exports nomeados), sem quebrar consumidores existentes.

2. **`buildTrackedCheckoutUrl({ baseUrl, campaign, medium, source, term, content })` → string:**
   - Ordem **exata** dos parâmetros na URL final: `utm_campaign`, `utm_medium`, `utm_source`, `utm_term`, `utm_content`, `sck`, `vk_source`.
   - `sck` = `[campaign, medium, source, term, content].filter(não-vazio).join("_")`.
   - `vk_source` **sempre presente e VAZIO** (`vk_source=`).
   - Se `term` vazio/ausente → **omitir** `utm_term` da URL **e** excluí-lo do `sck`.
   - Se `content` vazio/ausente → **omitir** `utm_content` da URL **e** excluí-lo do `sck`.
   - Preservar querystring pré-existente da `baseUrl`: se a base já tem `?`, usar `&` como separador; caso contrário, usar `?`.
   - Aplicar `encodeURIComponent` nos **valores** (não nas chaves).

3. **`createSwitchyLink(token, payload)` → link criado:**
   - `POST https://api.switchy.io/v1/links/create` com header `Api-Authorization: <token>` e `Content-Type: application/json`.
   - Body: `{ link: { url, domain (default "hi.switchy.io"), folderId (int), pixels: [{platform,value,title}], showGDPR (bool), tags?, note? }, autofill: false }`.
   - **Tratamento defensivo da response:** aceitar tanto `{ link: {...} }` quanto shape flat; em caso de shape inesperado, logar o raw e retornar um objeto normalizado (sem lançar por campo ausente, exceto erro de rede/HTTP).
   - Em erro HTTP (`!res.ok`) → lançar `Error` com status/statusText (mesmo padrão do helper GraphQL existente).

4. **Testes unitários OBRIGATÓRIOS (`buildTrackedCheckoutUrl`):**
   - 7 exemplos-ouro validados (bio, direct, stories, manychat, chatwoot, email, grupo) com `campaign=fzl1`, `term=cpl`, `content=org` produzindo exatamente:
     `https://pay.tmbeducacao.com.br/FernandaZapp/Y5B1006856G?utm_campaign=fzl1&utm_medium=<m>&utm_source=<s>&utm_term=cpl&utm_content=org&sck=fzl1_<m>_<s>_cpl_org&vk_source=`
   - Caso `term`/`content` vazios → omitidos da URL e do `sck` (ex.: `...&sck=fzl1_bio_ig&vk_source=`).
   - Caso base com querystring pré-existente → separador `&` correto.
   - Caso com caractere que precisa encode → valor encodado.

5. **Qualidade:**
   - `npm run lint` e `npm run typecheck` passam sem erros no pacote `api`.
   - Sem dependências novas (usar `fetch` nativo, como o GraphQL existente).

## ⚠️ Risks & Mitigations

| Risk | Severity | Mitigation |
|------|----------|-----------|
| **Shape da response do `/links/create` não 100% documentado** | HIGH | Tratamento defensivo: aceitar `{link:{...}}` ou flat; logar raw em caso inesperado; normalizar para `{ shortUrl, switchyLinkId, switchyUniq, raw }` sem lançar por campo ausente. |
| **Ordem de params divergente do esperado pelo checkout** | HIGH | Ordem fixa e hardcoded (`utm_campaign, utm_medium, utm_source, utm_term, utm_content, sck, vk_source`) + 7 testes-ouro byte-a-byte travando a saída. |
| **`vk_source` removido por engano (parece vazio/inútil)** | MEDIUM | AC explícito + teste verificando que a URL termina com `&vk_source=` (presente e vazio). |
| **Omissão de term/content quebrar o `sck`** | MEDIUM | `sck` derivado do **mesmo filtro** dos params (single source: `[campaign,medium,source,term,content].filter(Boolean)`), testado nos dois cenários. |
| **Double-encoding ou encode de chave** | LOW | `encodeURIComponent` aplicado **só nos valores**; teste com caractere especial. |
| **Quebra de consumidores GraphQL existentes** | LOW | Apenas adição de exports; nenhum símbolo existente alterado/removido. Lint+typecheck do pacote inteiro. |

## 🔧 Dev Notes

### Arquivo alvo
`packages/api/src/services/switchy.ts` — **manter** o bloco GraphQL existente (linhas atuais 1-107). **Adicionar** a seção REST abaixo dele.

### Constantes REST

```ts
const REST_ENDPOINT = "https://api.switchy.io/v1";
const DEFAULT_DOMAIN = "hi.switchy.io";
```

Header de auth é o **mesmo** do GraphQL: `Api-Authorization: <token>` + `Content-Type: application/json`. Token vem de `fastify.config.SWITCHY_API_TOKEN` (global, resolvido na camada de rota — o service só recebe `token: string`).

### Contrato `buildTrackedCheckoutUrl`

Assinatura:
```ts
export interface BuildTrackedUrlParams {
  baseUrl: string;
  campaign: string;       // OBRIGATÓRIO (ex.: "fzl1")
  medium: string;         // do preset (ex.: "bio")
  source: string;         // do preset (ex.: "ig")
  term?: string;          // OPCIONAL (ex.: "cpl")
  content?: string;       // OPCIONAL (ex.: "org")
}
export function buildTrackedCheckoutUrl(params: BuildTrackedUrlParams): string;
```

Regras determinísticas:
1. Normalizar `term`/`content`: tratar `undefined`, `null`, `""` e strings só-espaço como **vazio** (recomendado `(term ?? "").trim()`).
2. `sck = [campaign, medium, source, term, content].filter((v) => v && v.length > 0).join("_")`.
3. Montar pares **nesta ordem exata**, pulando os vazios de `utm_term`/`utm_content`:
   - `utm_campaign=<enc(campaign)>`
   - `utm_medium=<enc(medium)>`
   - `utm_source=<enc(source)>`
   - `utm_term=<enc(term)>` (somente se term não-vazio)
   - `utm_content=<enc(content)>` (somente se content não-vazio)
   - `sck=<enc(sck)>`
   - `vk_source=` (SEMPRE, valor vazio — não encodar string vazia muda nada, mas a chave deve aparecer)
4. Separador inicial: `baseUrl.includes("?") ? "&" : "?"`. Demais pares unidos por `&`.
5. `enc = encodeURIComponent` aplicado **apenas aos valores**.

> Atenção: `vk_source` é sempre incluído mesmo vazio — a URL final termina literalmente em `&vk_source=`.

### Contrato `createSwitchyLink`

Assinatura:
```ts
export interface SwitchyCreateLinkPixel {
  platform: string;       // "facebook" | "gtm" (Meta / GTM)
  value: string;          // pixel id / container id
  title: string;
}
export interface SwitchyCreateLinkPayload {
  url: string;
  folderId: number;
  pixels?: SwitchyCreateLinkPixel[];
  showGDPR?: boolean;
  domain?: string;        // default DEFAULT_DOMAIN
  tags?: string[];
  note?: string;
}
export interface CreatedSwitchyLink {
  shortUrl: string | null;     // url curta retornada (pode vir como url/shortUrl/short)
  switchyLinkId: string | null;
  switchyUniq: number | null;
  raw: unknown;                // resposta crua para depuração/persistência
}
export async function createSwitchyLink(
  token: string,
  payload: SwitchyCreateLinkPayload,
): Promise<CreatedSwitchyLink>;
```

Implementação esperada:
1. Montar request body:
```ts
{
  link: {
    url: payload.url,
    domain: payload.domain ?? DEFAULT_DOMAIN,
    folderId: payload.folderId,
    pixels: payload.pixels ?? [],
    showGDPR: payload.showGDPR ?? false,
    ...(payload.tags ? { tags: payload.tags } : {}),
    ...(payload.note ? { note: payload.note } : {}),
  },
  autofill: false,
}
```
2. `fetch(`${REST_ENDPOINT}/links/create`, { method: "POST", headers: { "Content-Type": "application/json", "Api-Authorization": token }, body: JSON.stringify(body) })`.
3. Se `!res.ok` → `throw new Error(`Switchy REST error: ${res.status} ${res.statusText}`)`.
4. Parse defensivo do JSON:
   - `const json = await res.json()`.
   - `const link = json?.link ?? json` (aceita `{link:{...}}` ou flat).
   - `shortUrl = link?.shortUrl ?? link?.short ?? link?.url ?? null`.
   - `switchyLinkId = link?.id ?? null` (string).
   - `switchyUniq = typeof link?.uniq === "number" ? link.uniq : null`.
   - Se nada disso resolver (`shortUrl == null && switchyLinkId == null`) → logar o raw (sem token) para diagnóstico e ainda retornar o objeto normalizado com `raw: json`.
5. Retornar `{ shortUrl, switchyLinkId, switchyUniq, raw: json }`.

> Observação: o service não tem acesso ao `fastify.log`. Para "logar raw" sem acoplar o logger, usar `console.warn("[switchy] resposta inesperada de /links/create", json)` (a rota chamadora já loga em nível mais alto). Não logar o token.

### Padrão de testes (vitest)

Seguir o padrão de `packages/api/src/__tests__/encryption.test.ts`: import relativo `../services/switchy`, `describe`/`it`/`expect`. O builder é função pura — **não precisa mockar fetch**. Os testes do `createSwitchyLink` ficam fora do escopo obrigatório desta story (response não 100% documentada); se incluídos, mockar `global.fetch`. - **REUSE:** padrão de header `Api-Authorization`, padrão de erro `!res.ok → throw`, e `fetch` nativo já presentes no service GraphQL.
- **CREATE justificado:** funções REST de escrita não existem (o service atual é só leitura GraphQL); novo endpoint REST (`api.switchy.io/v1`) e nova lógica de URL builder não têm equivalente reutilizável.
