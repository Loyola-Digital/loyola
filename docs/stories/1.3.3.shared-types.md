## Dev Notes

### Tipos Existentes em @loyola-x/shared

Ja existem tipos em `packages/shared/src/types/`:
- `mind.ts` — MindArtifactPaths, MindMetadata, MindSummary, MindDetail, Squad
- `chat.ts` — ChatRequest, SSEEvent

Exportados via `packages/shared/src/index.ts`.

[Source: packages/shared/src/index.ts, packages/shared/src/types/]

### Schema do Banco (referencia para tipos)

Os tipos devem alinhar com o schema Drizzle em `packages/api/src/db/schema.ts`:
- `delegated_tasks` — conversationId (FK), messageId (FK nullable), userId (FK), mindId, clickupTaskId (unique), clickupUrl (LIKE 'https://%'), title, description, status (enum), priority (enum), tags (text array)
- `users` — clerkId (unique), email (unique), name, avatarUrl, role (enum)
- `conversations` — userId (FK), mindId, mindName, squadId, title, messageCount, totalTokens, deletedAt
- `messages` — conversationId (FK), role (enum), content, tokensUsed, metadata (jsonb)

[Source: packages/api/src/db/schema.ts]

### Convencao de Datas

API retorna timestamps como ISO8601 strings (JSON serialization). Tipos shared devem usar `string` para campos de data (nao `Date`), pois sao consumidos como JSON no frontend.

### AC 5 — Integracao Futura

O AC 5 (importar de @loyola-x/shared) sera atendido progressivamente nas stories futuras (1.4.1, 1.5.1, 1.6.1) conforme os componentes frontend e backend sao criados. Nesta story, basta garantir que os tipos estao exportados e disponiveis.
