## 📖 Story

**Como** gestor de um projeto na Loyola Digital X,
**Quero** abrir a aba "Assinaturas", escolher um produto da Hotmart num dropdown e ver um dashboard de recorrência (total, vigentes, canceladas, reembolsadas, MRR, LTV, LT, retenção e renovações do próximo mês),
**Para** acompanhar a saúde da minha base de assinantes sem precisar entrar no painel da Hotmart.

---

## ✅ Acceptance Criteria

1. **Rota e gating:**
   - Página renderiza em `app/(app)/projects/[id]/subscriptions/page.tsx`, lendo `id` via `use(params)`.
   - `isAdmin = role !== null && role !== "guest"`. Guest enxerga o dashboard (leitura) mas NÃO acessa o painel de configuração (Story 34.4) — o dashboard em si é leitura para todos com acesso ao projeto.
   - Acessível pela subaba "Assinaturas" do project folder (subaba criada em Story 34.4).

2. **Estado não-configurado:**
   - Quando não há conexão Hotmart (`useHotmartConnection().connected === false`), o dashboard NÃO chama products/dashboard e exibe estado vazio com CTA para configurar (apenas para `isAdmin`; guest vê mensagem informativa "Integração não configurada").
   - Reaproveita o padrão visual de empty state de `sales/page.tsx` (ícone em círculo + título + subtítulo + botão).

3. **Dropdown de produto:**
   - Usa `useHotmartProducts(projectId, { months })` para popular um `Select` (shadcn) com `{ id, name }`.
   - Default = primeiro produto retornado (ou nenhum selecionado se a lista vier vazia).
   - Trocar o produto re-busca o dashboard daquele produto.
   - Enquanto products carrega, o Select aparece em estado desabilitado/skeleton.
   - Se `products` vier vazio (conexão ok mas sem assinaturas no período): exibir empty state "Nenhum produto com assinaturas no período".

4. **Filtro de período:**
   - Seletor de período (em meses) com default **12 meses** (reaproveitar `DayRangePicker` adaptado para meses OU um `Select` simples com opções 3/6/12/24 meses — decisão de UI no Dev, mas default = 12).
   - O valor de `months` alimenta tanto `useHotmartProducts` quanto `useHotmartDashboard`.

5. **KPI Cards (grid responsivo, padrão `KpiCard` + `MetricTooltip`):**
   - **Total de assinaturas** (período)
   - **Vigentes/Ativas** (status ACTIVE)
   - **Canceladas** (CANCELLED_BY_CUSTOMER + CANCELLED_BY_SELLER + CANCELLED_BY_ADMIN)
   - **Reembolsadas** (qtd + valor, de sales/summary REFUNDED)
   - **MRR** (moeda primária BRL; mostrar sub com outras moedas se houver)
   - **LTV** (média por assinante, moeda primária)
   - **LT (meses)** (lifetime médio)
   - **Retenção %** (ativas/total)
   - Cada card com `MetricTooltip` explicando a fórmula (ex.: "MRR = Σ price.value × 30 / recurrency_period das assinaturas ACTIVE").

6. **Card Renovações do próximo mês:**
   - Card destacado com **quantidade** de assinaturas ACTIVE cujo `date_next_charge` cai no próximo mês calendário + **soma de price.value** (receita prevista, moeda primária).
   - Vem pronto do backend (`dashboard.renewalsNextMonth`).

7. **Distribuição de status (gráfico/tabela):**
   - Gráfico de barras (recharts) OU tabela mostrando contagem por cada status de assinatura (ACTIVE, INACTIVE, DELAYED, OVERDUE, STARTED, CANCELLED_BY_*, etc.), espelhando a tabela "Distribuição" do dashboard da Hotmart.
   - Dados vêm de `dashboard.statusDistribution` (array `{ status, count }`).

8. **Estados de UI:**
   - **Loading:** skeletons no padrão das outras páginas (grid de KpiCards + bloco do gráfico) enquanto `useHotmartDashboard` carrega.
   - **Empty:** período sem assinaturas → mensagem clara, sem quebrar.
   - **Erro:** falha de fetch (token inválido, 5xx) → mensagem de erro amigável + botão "Tentar novamente" (refetch); NUNCA vazar `client_secret`/Basic.
   - **Não-configurado:** ver AC 2.

9. **Formatação:**
   - Moeda formatada por `currency_code` (BRL primária via `Intl.NumberFormat("pt-BR", { style: "currency", currency })`).
   - Percentuais com 1 casa decimal; LT em meses com 1 casa; contagens inteiras.
   - Datas (se exibidas) em pt-BR.

10. **Multi-moeda:**
    - Quando o dashboard retornar métricas em mais de uma moeda, a moeda primária (BRL) é exibida no valor principal e as demais aparecem no `sub`/tooltip — não somar moedas diferentes.

## 🔧 Dev Notes

### Contrato de Hooks (`packages/web/lib/hooks/use-hotmart.ts`)

Seguir o padrão de `use-mautic.ts` (React Query + `useApiClient`, `staleTime` ~5min). Esta story ADICIONA `useHotmartProducts` e `useHotmartDashboard` (os hooks de conexão — `useHotmartConnection`, `useSetHotmartConnection`, `useDeleteHotmartConnection` — são criados na Story 34.4; se ainda não existirem, criar `useHotmartConnection` aqui pois o dashboard precisa dele para o gating de não-configurado).

```ts
const STALE = 5 * 60 * 1000;
function projBase(projectId: string) {
  return `/api/projects/${projectId}/hotmart`;
}

export interface HotmartConnectionResponse { connected: boolean; }

export interface HotmartProduct { id: string; name: string; }

export interface HotmartMoneyMetric { value: number; currency: string; }

export interface HotmartStatusCount { status: string; count: number; }

export interface HotmartDashboard {
  productId: string;
  months: number;
  totalSubscriptions: number;
  active: number;              // vigentes (ACTIVE)
  cancelled: number;           // soma dos 3 CANCELLED_BY_*
  overdue: number;             // OVERDUE + DELAYED (inadimplentes)
  refunded: { count: number; value: HotmartMoneyMetric[] };
  mrr: HotmartMoneyMetric[];   // por moeda (BRL primeiro)
  ltv: HotmartMoneyMetric[];   // por moeda
  ltMonths: number;            // lifetime médio em meses
  retentionRate: number;       // 0..100 (ativas/total)
  churnRate: number;           // 0..100 (canceladas/total)
  renewalsNextMonth: { count: number; value: HotmartMoneyMetric[] };
  statusDistribution: HotmartStatusCount[];
}

export function useHotmartConnection(projectId: string) {
  const apiClient = useApiClient();
  return useQuery({
    queryKey: ["hotmart-connection", projectId],
    queryFn: () => apiClient<HotmartConnectionResponse>(`${projBase(projectId)}/connection`),
    staleTime: STALE,
  });
}

export function useHotmartProducts(projectId: string, opts: { months: number; enabled: boolean }) {
  const apiClient = useApiClient();
  return useQuery({
    queryKey: ["hotmart-products", projectId, opts.months],
    queryFn: () =>
      apiClient<{ products: HotmartProduct[] }>(`${projBase(projectId)}/products?months=${opts.months}`),
    enabled: opts.enabled,
    staleTime: STALE,
  });
}

export function useHotmartDashboard(
  projectId: string,
  opts: { productId: string | null; months: number; enabled: boolean },
) {
  const apiClient = useApiClient();
  return useQuery({
    queryKey: ["hotmart-dashboard", projectId, opts.productId, opts.months],
    queryFn: () =>
      apiClient<HotmartDashboard>(
        `${projBase(projectId)}/dashboard?productId=${encodeURIComponent(opts.productId!)}&months=${opts.months}`,
      ),
    enabled: opts.enabled && !!opts.productId,
    staleTime: STALE,
  });
}
```

> O shape EXATO de `HotmartDashboard` é fechado pela Story 34.3 (rotas). Se houver divergência ao integrar, o tipo do hook segue a rota — registrar no Change Log.

### Orquestração na página

```
connection = useHotmartConnection(projectId)
isConnected = connection.data?.connected === true

products = useHotmartProducts(projectId, { months, enabled: isConnected })
selectedProductId = state (default = products.data?.products[0]?.id)

dashboard = useHotmartDashboard(projectId, {
  productId: selectedProductId,
  months,
  enabled: isConnected,
})
```

Ordem de checagem de render:
1. `connection.isLoading` → skeleton geral.
2. `!isConnected` → empty "não configurado" (CTA configurar só se `isAdmin`).
3. `products.isLoading` → skeleton (dropdown desabilitado).
4. `products` vazio → empty "Nenhum produto com assinaturas no período".
5. `dashboard.isLoading` → skeletons de KPIs + gráfico.
6. `dashboard.isError` → erro + "Tentar novamente" (`dashboard.refetch()`).
7. `dashboard.data` → render completo.

function fmtMoney(m?: { value: number; currency: string }): string {
  if (!m) return "—";
  return new Intl.NumberFormat("pt-BR", { style: "currency", currency: m.currency }).format(m.value);
}
function primary(metrics: HotmartMoneyMetric[]): HotmartMoneyMetric | undefined {
  return metrics.find((m) => m.currency === "BRL") ?? metrics[0];
}
function fmtPct(v: number): string { return `${v.toFixed(1)}%`; }
function fmtInt(v: number): string { return v.toLocaleString("pt-BR"); }

### Notas de métricas (documentar como aproximação nossa nos tooltips)

A Hotmart NÃO expõe LTV/MRR/LT prontos — todos os números vêm calculados no backend (34.3). Os `MetricTooltip` devem deixar claro que são aproximações Loyola:
- MRR = Σ (price.value × 30 / recurrency_period) das assinaturas ACTIVE, por moeda.
- LTV = média de (price.value × lifetime) por assinante (do summary), por moeda.
- LT = média de (lifetime × recurrency_period / 30) em meses.
- Retenção% = ativas / total ; Churn% = canceladas / total.
- Renovações próximo mês = ACTIVE com `date_next_charge` no próximo mês calendário (qtd + Σ price.value).
