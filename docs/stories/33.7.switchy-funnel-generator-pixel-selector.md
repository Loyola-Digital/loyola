## 📖 Story

**Como** gestor de tráfego do projeto,
**Quero** gerar os shortlinks do Switchy **dentro da página do funil** (com o histórico filtrado por esse funil) e **selecionar** os pixels já cadastrados na conta Switchy ao configurar o projeto (em vez de digitar IDs de pixel/GTM manualmente),
**Para** que cada funil tenha seu próprio conjunto de links rastreados e que os pixels anexados sejam exatamente os do expert correto, sem erro de digitação.

> Incremento de frontend que consome o backend da Story 33.6 (funnel_id em `switchy_shortened_links`, `GET /switchy/pixels`, `funnelId` opcional no generate/history). Epic 33 já está em produção (PR #209).

---

## ✅ Acceptance Criteria

1. **Hook `useSwitchyAccountPixels(projectId)`** (`lib/hooks/use-switchy.ts`)
   - `GET /api/projects/:id/switchy/pixels` → `{ pixels }`, `select` retorna o array.
   - Tipo exportado `SwitchyAccountPixel = { id, platform, value, title }`.
   - `enabled: !!projectId`, `staleTime` de 5 min (padrão dos demais hooks), queryKey `["switchy-account-pixels", projectId]`.

2. **`useGenerateSwitchyLinks` inclui `funnelId` no payload**
   - `SwitchyGeneratePayload` ganha `funnelId?: string`.
   - O payload é repassado direto no body (sem mudança de assinatura do hook).
   - `onSuccess` invalida tanto `["switchy-history", projectId]` quanto `["switchy-history", projectId, funnelId]` quando `funnelId` presente.

3. **`useSwitchyHistory` aceita `funnelId`**
   - Assinatura `useSwitchyHistory(projectId, funnelId?)`.
   - Quando `funnelId` presente: query param `?funnelId=...` (encodado) e queryKey por funil `["switchy-history", projectId, funnelId]`.
   - Quando ausente: comportamento atual preservado (chave/URL sem funil).
   - `SwitchyHistoryItem` ganha `funnelId?: string | null`.

4. **Config panel = seletor de pixels da conta** (`switchy-config-panel.tsx`)
   - Os inputs manuais de **Meta Pixel ID** e **GTM Container** são **removidos**.
   - No lugar, um multi-select (lista com checkbox + filtro de busca) dos pixels retornados por `useSwitchyAccountPixels`, mostrando `title` + `platform` (+ `value`).
   - Seleção persiste em `project_switchy_settings.pixels` via `useSetSwitchySettings` no shape já existente `{ platform, value, title }` (id usado só na UI).
   - Plataforma da conta é normalizada para o enum aceito pelos settings (`facebook` | `gtm`); plataformas não suportadas são marcadas e ignoradas no save (com aviso no toast).
   - Pixels salvos que não voltam mais da conta seguem aplicados (aviso de "órfão"), sem sumir silenciosamente.
   - Presets manager mantido. utm_term/utm_content padrão e toggle GDPR mantidos.

5. **Tab Switch do projeto = CONFIG-ONLY** (`switchy-tab.tsx`)
   - Remove `SwitchyGenerator`, `SwitchyResultsTable` e `SwitchyHistoryList` do tab do projeto.
   - Mantém só `SwitchyConfigPanel` + `SwitchyPresetsManager`.

6. **Gerador + histórico vivem dentro do funil** (`switchy-funnel-section.tsx`)
   - Novo host `components/funnels/switchy-funnel-section.tsx` (card colapsável no padrão do `youtube-funnel-section.tsx`) que renderiza `SwitchyGenerator` + `SwitchyHistoryList`.
   - Plugado na página do funil: `app/(app)/projects/[id]/funnels/[funnelId]/page.tsx`, após a `SortableStageGrid`.
   - Funil **implícito**: `funnelId` da rota é passado no payload do generate e no history (sem picker de funil).
   - `utm_campaign` do gerador é pré-preenchido com o **nome do funil** (editável).

7. **Gating guest mantido nas escritas**
   - `canEdit = role !== null && role !== "guest"` em todas as ações de escrita (config save, generate). Leitura liberada.

8. **Não-regressão**
   - `pnpm --filter @loyola-x/web typecheck` e `lint` verdes (web não tem runner de teste → n/a).
   - Nenhuma assinatura/rota existente quebrada — props novas são opcionais; `funnelId` ausente preserva o comportamento anterior.

## ⚠️ Risks & Mitigations

| Risk | Severity | Mitigation |
|------|----------|-----------|
| Plataforma do pixel (GraphQL) fora do enum `facebook`/`gtm` quebrar o PUT settings | HIGH | `normalizePlatform` mapeia/normaliza; plataformas não suportadas são ignoradas no save com aviso no toast. |
| Pixel salvo sumir da conta e a seleção desaparecer silenciosamente | MEDIUM | Pixels "órfãos" (salvos mas ausentes da conta) seguem aplicados, com aviso visível. |
| `funnelId` novo quebrar chamadas do generate/history existentes | HIGH | Campo/props opcionais; ausência preserva URL e queryKey antigas. |
| Histórico do funil vazar links de outro funil/projeto | MEDIUM | Filtro `?funnelId=` sempre combinado ao `projectId` no backend (33.6); queryKey por funil isola o cache. |
| Gerador no project tab e no funil divergirem | LOW | Mesmo componente `SwitchyGenerator` reusado nos dois (funil só passa props extras); project tab agora é config-only. |

### Decisões do Lucas
1. Links atrelados ao **funil** (`funnel_id`).
2. Gerador + lista de links vivem **dentro da página do funil** (funil implícito, sem picker).
3. Pixels: **não digitar manual** — selecionar da lista de pixels já cadastrados no Switchy. O Switchy **não** herda pixel de folder/workspace (`pixels[]` é por-link), então o backend (33.6) busca via GraphQL e o frontend anexa via settings.

### Onde a seção Switch foi plugada
- `packages/web/app/(app)/projects/[id]/funnels/[funnelId]/page.tsx`, logo **após** o bloco da `SortableStageGrid` (e antes do Dialog "Nova Etapa"), via `<SwitchyFunnelSection projectId funnelId funnelName={funnel.name} />`.
- Funis de etapa única redirecionam para a página da etapa (early return existente), então a seção aparece em funis multi-etapa e perpetuais — o nível correto pro gerador por-funil.

### Normalização de plataforma
```ts
function normalizePlatform(raw: string): "facebook" | "gtm" | null {
  const p = raw.toLowerCase();
  if (p === "facebook" || p === "meta") return "facebook";
  if (p === "gtm" || ...) return "gtm";
  return null; // ignorado no save, com aviso
}
```

### UI do seletor
- Reusa o padrão de "checkbox-button" do gerador (não há componente `checkbox` no design system) + `ScrollArea` + `Input` de filtro + `Badge` de plataforma — tudo shadcn/ui já presente.
