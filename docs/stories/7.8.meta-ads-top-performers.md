## Story

**As a** gestor de trafego da Loyola Digital,
**I want** to see which ads/creatives performed best by key metrics (ROAS, CPL, conversions) and filter the dashboard directly by ad set,
**so that** I can quickly identify winning creatives and audiences without manually drilling down through every campaign.

---

## Acceptance Criteria

1. [ ] Nova secao "Top Performers" no dashboard `/traffic` — ranking automatico dos melhores ads/criativos
2. [ ] Ranking por KPI selecionavel: melhor ROAS, menor CPL, menor CPL qualificado, mais conversoes, mais leads, maior CTR
3. [ ] Top performers mostra top 5 (configuravel) com: nome do ad, nome da campanha, nome do adset, e metricas relevantes
4. [ ] Visual card para cada top performer com destaque da metrica principal e metricas secundarias
5. [ ] Filtro direto por ad set no nivel do dashboard — dropdown que lista todos adsets (sem precisar expandir campanha)
6. [ ] Ao filtrar por adset, tabela mostra apenas campanhas que contem aquele adset + os ads daquele adset
7. [ ] Filtro por adset combinavel com filtro de periodo existente (7d, 14d, 30d, 90d)
8. [ ] Endpoint `GET /api/traffic/analytics/:projectId/top-performers?metric=roas&limit=5&days=30` — retorna ranking ordenado
9. [ ] Endpoint considera dados cruzados (Meta Ads + leads + qualificados + vendas quando disponiveis)
10. [ ] Endpoint `GET /api/traffic/analytics/:projectId/adsets?days=30` — lista todos adsets do projeto (sem precisar de campaignId)
11. [ ] Hooks: `useTopPerformers(projectId, metric, limit, days)`, `useAllAdSets(projectId, days)`
12. [ ] Empty state quando nao ha dados suficientes para ranking
13. [ ] `pnpm typecheck` e `pnpm lint` passam sem erros

### Performance Note — getAllAdSets

```
IMPORTANTE: Listar adsets de todas as campanhas implica N chamadas à Meta API (1 por campanha).
Estratégia obrigatória:
1. Cache agressivo (TTL 15min, mesmo padrão do traffic-analytics)
2. Buscar campanhas primeiro, depois adsets em paralelo (Promise.allSettled)
3. Limitar por período — não buscar adsets de campanhas inativas/antigas
4. Se projeto tiver >50 campanhas, paginar ou limitar às 20 com maior spend
```
