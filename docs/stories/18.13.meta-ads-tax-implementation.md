## 🔧 Technical Details

### Implementação

**Arquivo Principal:** `packages/web/lib/utils/funnel-metrics.ts`

#### Nova Função
```typescript
export function applyMetaAdsTax(spendValue: number, dateStr: string): number {
  if (!dateStr || spendValue <= 0) return spendValue;
  const year = parseInt(dateStr.substring(0, 4), 10);
  if (year >= 2026) {
    return spendValue * META_ADS_TAX_RATE; // 1.1215
  }
  return spendValue;
}
```

#### Funções Modificadas
1. **`sumMetaInsights()`** — Aplica imposto ao somar insights por período
2. **`aggregateMetaDailyByDate()`** — Aplica imposto ao agregar por data

### Taxa
- **Percentual:** 12,15%
- **Multiplicador:** 1.1215
- **Aplicação:** Uma única vez durante agregação (evita duplicação)
- **Período:** Baseado em `date_start` do Meta Ads API

### Exemplo
| Data | Spend Original | Spend Ajustado | Cálculo |
|------|---|---|---|
| 2025-12-31 | R$ 100,00 | R$ 100,00 | sem imposto |
| 2026-01-01 | R$ 100,00 | R$ 112,15 | × 1.1215 |
| 2026-04-22 | R$ 500,00 | R$ 560,75 | × 1.1215 |
| 2027-01-01 | R$ 250,00 | R$ 280,38 | × 1.1215 |

## 💡 Notas Importantes

⚠️ **Taxa aplicada uma única vez:** A taxa é aplicada durante a agregação de dados (`sumMetaInsights()` e `aggregateMetaDailyByDate()`), não durante o cálculo de métricas derivadas. Isso evita duplicação e garante consistência.

📊 **Downstream Impact:** Todos os cálculos derivados (CPL, ROAS, CPM, CPC) recebem automaticamente o spend ajustado, sem necessidade de modificações adicionais.

🔍 **Validação por Data:** A validação é feita por `date_start` do Meta Ads API insight, garantindo que cada linha é processada individualmente.

---

## 🎓 Lições Aprendidas

1. **Ponto de aplicação importante:** Aplicar imposto na agregação (não na exibição) garante que todos os cálculos derivados usam o valor correto
2. **Testes unitários:** Validar edge cases (spend = 0, data inválida, etc) evita surpresas em produção
3. **Documentação QA:** Ter um plano de teste detalhado facilita validação manual e acelera deploy
