## 📖 Story

**Como** gestor de tráfego,  
**Quero** projetar leads baseado em custo (pacing + CPL histórico) e não em extrapolação direta,  
**Para** que minha previsão seja mais realista e considere variações de CPL, gasto diário e leads orgânicos separadamente.

---

## ✅ Acceptance Criteria

1. **Gráfico renderiza com 3 inputs:**
   - Data Final (input existente, mantido)
   - Meta Total de Leads (input existente, mantido)
   - Gasto Total Projetado em R$ (novo input, pré-preenchido por regressão do gasto/dia, editável)

2. **Dados diários exibidos:**
   - Leads pagos acumulados (reais + projetados)
   - Leads orgânicos acumulados (reais + projetados)
   - Gasto diário (reais até hoje, projetados para frente)
   - CPL projetado em eixo secundário (R$)

3. **Projeção diária (do "hoje" até Data Final):**
   - Gasto diário projetado = (Gasto Total Projetado − gasto acumulado real) ÷ dias restantes (pacing fixo)
   - CPL projetado(dia) = regressão linear do CPL acumulado histórico
   - Leads pagos projetados(dia) = gasto diário projetado ÷ CPL projetado(dia)
   - Leads orgânicos projetados(dia) = regressão linear sobre orgânicos/dia histórico
   - Acumulado projetado = acumulado real até hoje + Σ (pagos + orgânicos projetados)

4. **Guardas e comportamentos:**
   - Regressão só ativa após ≥ 5 dias de dados; antes disso, usa média simples
   - CPL projetado nunca ≤ 0 (piso = último CPL acumulado real)
   - Leads orgânicos projetados nunca < 0 (piso = 0)
   - Se gasto acumulado ≥ Gasto Total Projetado → gasto diário restante = 0

5. **Visuais:**
   - Barras diárias empilhadas (leads pagos em cor A, leads orgânicos em cor B)
   - Linha "Projeção" = acumulado projetado (derivada de custo)
   - Linha CPL projetado em eixo secundário
   - Linha Meta Acumulada (como hoje)
   - Banda min/máx de confiança do CPL (intervalo de confiança da regressão)
   - Rótulo "% da meta" no último ponto (como hoje)

6. **Arredondamento:**
   - Cálculos intermediários com frações (CPL, gasto, leads/dia)
   - Exibição: acumulado projetado = Math.round
   - CPL exibido: 2 casas decimais
   - Gasto exibido: 2 casas decimais (R$)

## 🔧 Dev Notes

### Definições Técnicas Rigorosas

#### 1. Regressão Linear (convenção única)
Toda regressão usa **mínimos quadrados ordinários**:
- X = índice do dia (inteiro, começando em 1)
- Y = métrica daquele dia
- Reta: Y = a + b·X
- Coeficientes: b = Sxy / Sxx, a = ȳ − b·x̄
- **Ativação:** ≥ 5 pontos válidos
- **Fallback:** Com < 5 pontos, use média simples (reta horizontal)

#### 2. CPL Histórico (entrada da regressão de CPL)
Para evitar ruído e divisão por zero:
- Regrida **CPL acumulado**, não CPL diário
- CPLacum(dia) = gastoAcumulado(dia) ÷ leadsPagosAcumulado(dia)
- Regrida CPLacum vs índice do dia
- Resultado: CPLproj(d) = a + b·d

#### 3. Gasto Diário Projetado (pacing — valor FIXO)
```
gastoDiarioProjetado = max(0, (GastoTotalProjetado − gastoAcumuladoReal) ÷ diasRestantes)
```
- Mesmo valor para todos os dias restantes
- Se gastoAcumuladoReal ≥ GastoTotalProjetado → gastoDiarioProjetado = 0

#### 4. Pré-preenchimento do Input "Gasto Total Projetado"
Ao carregar, sugira (campo editável):
```
sugestao = gastoAcumuladoReal + Σ (a_gasto + b_gasto·d) para cada d projetado
```
Regrida gasto diário (convenção 1), projete cada dia restante, some ao gasto real acumulado.

#### 5. CPL Projetado por Dia (com piso)
```
CPLproj(d) = a + b·d
```
**Piso:** Se CPLproj(d) ≤ 0, use CPLacum(hoje). CPL nunca pode ser ≤ 0.

#### 6. Leads Pagos Projetados por Dia
```
leadsPagosProj(d) = gastoDiarioProjetado ÷ CPLproj(d)
```

#### 7. Leads Orgânicos Projetados por Dia
Regrida leadsOrganicos diário (convenção 1):
```
orgProj(d) = max(0, a_org + b_org·d)
```
Piso = 0.

#### 8. Acumulado Projetado
```
acumuladoProjetado(d) = acumuladoRealAteHoje + Σ desde o primeiro dia projetado até d de (leadsPagosProj + orgProj)
acumuladoRealAteHoje = Σ (leadsPagos + leadsOrganicos) de todos os dias reais
```

#### 9. Banda Min/Máx (Intervalo de Confiança do CPL)
Calcule o erro-padrão da regressão do CPL:
```
SE = sqrt( Σ(yi − ŷi)² / (n − 2) )
CPLproj_baixo(d) = max(piso, CPLproj(d) − SE)
CPLproj_alto(d) = CPLproj(d) + SE
```
- Banda otimista (mais leads) usa CPLproj_baixo
- Banda pessimista (menos leads) usa CPLproj_alto
- Plote como área entre as duas curvas acumuladas
