## Story

Como gestor de tráfego do Loyola X, quero 3 meta-features no dashboard da aba Meta Ads:

1. **Banner de freshness** no topo, indicando quando os dados foram atualizados
2. **Botão de auditoria** permitindo marcar o dashboard como conferido (apenas Danilo/Lucas)
3. **Detector de campanhas novas** via prefixo configurável, avisando quando há campanhas não selecionadas

E finalizar o epic **removendo a aba Meta Ads 2** (que serviu só como referência durante a migração).

Pra que eu tenha visibilidade operacional (dados estão frescos?), rastreabilidade (alguém validou?) e segurança (não estou perdendo campanhas novas?).

## Escopo

### IN — Feature 1: Freshness banner
- Banner no topo do `LaunchDashboard` (acima do `DayRangePicker`)
- Fonte única monitorada: **Meta Ads API** — último insight gravado no DB
- Hook novo `useDataFreshness(projectId, funnelId)` retornando `{ lastUpdatedAt: Date, status: "fresh" | "stale" | "critical" }`
- Status por timestamp da última atualização:
  - **Verde ("fresh"):** `diff <= 24h` — texto "Dados atualizados há X horas"
  - **Amarelo ("stale"):** `24h < diff <= 72h` — texto "⚠️ Dados com N dias"
  - **Vermelho ("critical"):** `diff > 72h` — texto "❌ Dados desatualizados — X dias sem sync"

### IN — Feature 2: Botão de auditoria (com histórico)
- Tabela nova no DB: `funnel_audits` (`id`, `funnelId`, `userId`, `userEmail`, `userName`, `auditedAt`, `scope`)
- Endpoints novos:
  - `POST /api/projects/:projectId/funnels/:funnelId/audit` — cria nova auditoria (whitelist enforce)
  - `GET /api/projects/:projectId/funnels/:funnelId/audits` — lista histórico (última primeiro)
- **Whitelist de usuários autorizados** (hardcoded inicialmente em constante backend):
  ```
  AUDIT_AUTHORIZED_EMAILS = [
    "lucasvitalsilva17@gmail.com",            // Lucas

    "marketing@bonsaitrafegopago.com.br",     // Danilo (email 2 — Clerk prod)
    "marketing+localdev@bonsaitrafegopago.com.br", // Danilo (email 3 — dev local)
  ]
  ```
- Botão no header do `LaunchDashboard` (ao lado do "Gerenciar campanhas"):
  - **Se usuário autorizado:** botão "Marcar como auditado" (se nunca auditado ou quer re-auditar) ou selo "✓ Auditado por {nome} em {DD/MM/YYYY HH:mm}"
  - **Se usuário não autorizado:** mostra apenas o selo (se existir), sem botão
- **Alerta vermelho** quando `diff(now, last_audit) > 3 dias` → selo com fundo vermelho + texto "⚠️ Auditoria com mais de 3 dias — revisar"
- Auditoria **não invalida** por atualização de dados — só expira por tempo (3 dias)
- Ao clicar "Marcar como auditado": chama POST endpoint, refetch, mostra toast "Auditoria registrada"

### IN — Feature 3: Detector de campanhas novas (com prefixo configurável)

  - **Prefixo preenchido + newCampaigns > 0:** badge vermelho "🆕 N campanhas novas detectadas" clicável

- Ação ao clicar no badge de novas campanhas: abre o painel "Gerenciar campanhas" com as novas **pré-marcadas** + scroll até elas + toast explicativo

## Especificação

### Freshness banner — fonte dos dados

Usar timestamp da última inserção/atualização em uma das tabelas de insights persistidos. Tarefa investigativa inicial pro @dev: mapear qual coluna/tabela representa "última atualização do Meta Ads" (provavelmente `meta_ads_insights.createdAt` ou similar).

Endpoint novo (sugestão): `GET /api/projects/:projectId/funnels/:funnelId/freshness` → `{ lastUpdatedAt: ISO8601 }`

### Auditoria — schema proposto

```sql
CREATE TABLE funnel_audits (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  funnel_id   UUID NOT NULL REFERENCES funnels(id) ON DELETE CASCADE,
  user_id     UUID NOT NULL REFERENCES users(id),
  user_email  TEXT NOT NULL,
  user_name   TEXT NOT NULL,
  audited_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  scope       TEXT NOT NULL DEFAULT 'full-dashboard'
);
CREATE INDEX idx_funnel_audits_funnel_id ON funnel_audits(funnel_id);
CREATE INDEX idx_funnel_audits_audited_at ON funnel_audits(audited_at DESC);
```

**Nota:** sem constraint `UNIQUE(funnel_id, audited_at)` — evita bloqueio se 2 clicks chegarem no mesmo ms. Duplicatas de auditoria são benignas (apenas histórico extra).

### Detector de campanhas novas — migration

```sql
ALTER TABLE funnels ADD COLUMN campaign_prefix TEXT NULL;
```

### Visual layout final do header do LaunchDashboard

```
┌───────────────────────────────────────────────────────────────┐
│ [FreshnessBanner]  (verde/amarelo/vermelho, full-width)       │
├───────────────────────────────────────────────────────────────┤
│ [DayRangePicker]                                              │
│ [Prefixo: ____________] [🆕 N novas detectadas]              │
│ [✓ Auditado em ...] [Marcar como auditado] [N campanhas]     │
└───────────────────────────────────────────────────────────────┘
```

Layout responsivo — em mobile quebra em múltiplas linhas.
