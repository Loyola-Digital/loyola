# Story 18.5: Adicionar Detalhamento de Faturamento — Pago/Orgânico/Sem Track

## Descrição

O card **Faturamento** no dashboard de Captação Paga atualmente mostra **um número total**, mas gestor precisa entender a composição:

1. **Faturamento Pago** = vendas com `utm_source='meta'` (origem Meta Ads)
2. **Faturamento Orgânico** = vendas com `utm_source <> 'meta'` (outras fontes)
3. **Faturamento Sem Track** = vendas com `utm_source=''` ou NULL (não rastreado)

**Necessário:** Adicionar detalhamento **abaixo do valor principal** mostrando estes 3 componentes com valores e % do total.

**Impacto:** Gestor vê imediatamente quanto do faturamento vem de Meta vs. orgânico vs. não rastreado — ajuda a priorizar investimento em tráfego pago.

## Contexto

- Aba: Meta Ads 2 (dashboard Captação Paga)
- Componente: Card de Faturamento (`revenue-card.tsx` ou similar)
- Fonte de dados: Planilha de vendas (coluna `utm_source`)
- Layout: Valor principal em grande + 3 sub-linhas com detalhamento

## Acceptance Criteria

### AC1 — Layout do card atualizado
- [ ] Card mostra número principal de faturamento total (ex: R$ 50.000)
- [ ] Abaixo, exibe **3 linhas com detalhamento:**
  - `Pago (Meta Ads): R$ 35.000 (70%)`
  - `Orgânico: R$ 12.000 (24%)`
  - `Sem Track: R$ 3.000 (6%)`

### AC2 — Cálculos corretos
- [ ] Faturamento Pago = SUM(valor_vendas WHERE utm_source='meta')
- [ ] Faturamento Orgânico = SUM(valor_vendas WHERE utm_source <> 'meta' AND utm_source IS NOT NULL)
- [ ] Faturamento Sem Track = SUM(valor_vendas WHERE utm_source='' OR utm_source IS NULL)
- [ ] Total = Pago + Orgânico + Sem Track
- [ ] % calculado: (componente / total) * 100

### AC3 — Filtro de data respeitado
- [ ] Ao mudar filtro de data, valores atualizam corretamente
- [ ] Query na API respeita data_start e data_end

### AC4 — Visual limpo
- [ ] Detalhamento legível e alinhado (não poluído)
- [ ] Cores diferenciadas para Pago (azul/verde), Orgânico (cinza), Sem Track (laranja/alerta)
- [ ] Responde a hover com tooltip opcional (utm_source=value)
