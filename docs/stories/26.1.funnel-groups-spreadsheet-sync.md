## User Story

Como gestor de tráfego do Loyola X, quero conectar uma planilha do Google Sheets que registra snapshots diários dos grupos (WhatsApp/Telegram) ao funil, para acompanhar no dashboard quantas pessoas entraram, saíram e estão atualmente nos grupos por campanha — usando os dados já coletados pela operação.

---

## Contexto Técnico

### Planilha de origem

Aba (default "Planilha2") com cabeçalho:
```
Campaign ID | Campaign Name | Clicks Total Count | Criação |
Group Full Amount | Group Open Amount | Group Total Amount |
Input Amount | Output Amount | Participants Amount
```

Várias linhas/dia (ex: 15h e 20h). Cada linha = snapshot.

### Reuso de infra existente

- `services/google-sheets.ts` (service account) — usar `readSheetData(spreadsheetId, sheetName)`
- Padrão de link: `funnel_surveys` / `funnel_spreadsheets` (já existem na schema)
- Padrão de tab no funil: `survey-funnel-tab.tsx`

### Schema novo

```sql
-- Configuração do link da planilha (1 por funil)
CREATE TABLE funnel_groups_spreadsheets (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  funnel_id UUID NOT NULL UNIQUE REFERENCES funnels(id) ON DELETE CASCADE,
  spreadsheet_id VARCHAR(255) NOT NULL,
  spreadsheet_name VARCHAR(255) NOT NULL,
  sheet_name VARCHAR(255) NOT NULL,
  last_synced_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW() NOT NULL
);

-- Snapshots ingeridos (uma linha por linha da planilha)
CREATE TABLE funnel_group_snapshots (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  funnel_id UUID NOT NULL REFERENCES funnels(id) ON DELETE CASCADE,
  campaign_id VARCHAR(255) NOT NULL,
  campaign_name VARCHAR(500) NOT NULL,
  snapshot_at TIMESTAMPTZ NOT NULL,
  clicks_total INT NOT NULL DEFAULT 0,
  group_full INT NOT NULL DEFAULT 0,
  group_open INT NOT NULL DEFAULT 0,
  group_total INT NOT NULL DEFAULT 0,
  input_amount INT NOT NULL DEFAULT 0,
  output_amount INT NOT NULL DEFAULT 0,
  participants_amount INT NOT NULL DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW() NOT NULL,
  UNIQUE (funnel_id, campaign_id, snapshot_at)
);

CREATE INDEX idx_group_snapshots_funnel_date ON funnel_group_snapshots (funnel_id, snapshot_at DESC);
CREATE INDEX idx_group_snapshots_campaign ON funnel_group_snapshots (funnel_id, campaign_id);
```

### Sync logic

1. Ler dados da aba via `readSheetData(spreadsheetId, sheetName)`
2. Mapear cabeçalho → índice de coluna (case-insensitive, tolerante a variações de acento)
3. Para cada linha: parse + INSERT ON CONFLICT DO UPDATE no `funnel_group_snapshots`
4. Atualizar `last_synced_at` no link
5. Retornar `{ rowsProcessed, rowsInserted, rowsUpdated, errors[] }`

### Agregação para o dashboard

`GET /api/projects/:projectId/funnels/:funnelId/group-snapshots/daily`:
- Para cada `(campaign_id, day)`: pegar o snapshot **mais recente** desse dia
- Calcular `delta_input = input_amount - input_amount_dia_anterior`
- Calcular `delta_output = output_amount - output_amount_dia_anterior`
- Calcular `delta_participants = participants_amount - participants_amount_dia_anterior`
- Retornar série temporal por campanha + agregado total do funil

## Dev Notes

- **Reuso:** `readSheetData()`, `getSpreadsheetSheets()`, `listSpreadsheets()` já existem no `services/google-sheets.ts` (service account). NÃO criar nova auth.
- **Padrão FK auto-ref Drizzle:** `(): AnyPgColumn => table.id` (ver memory `project_loyola_digital_x.md`)
- **Padrão rotas Fastify:** dentro de `fp()` closure
- **Migration:** aplicar manual via `node + pg` conforme padrão do projeto
- **Mapeamento de coluna:** normalizar header (lowercase, sem acento) para tolerar "Criação" / "Criacao" / "criacao" — usar `.normalize("NFD").replace(/\p{Diacritic}/gu, "")`
- **Cabeçalho da planilha real (visto no exemplo):**
  ```
  Camapaign ID  Campaign Name  Clicks Total Count  Criação  Group Full Amount
  Group Open Amount  Group Total Amount  Input Amount  Outpu Amount  Participants Amount
  ```
  Atenção: "**Camapaign**" e "**Outpu**" estão **com typo** na planilha de origem. O mapeamento deve ser tolerante a esses typos OU normalizar via match aproximado (Levenshtein distance ≤ 2).
