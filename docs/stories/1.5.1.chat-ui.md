## Story

**As a** authenticated Loyola Digital user,
**I want** to chat with AI minds in real-time with streaming responses,
**so that** I can have natural conversations and see responses as they are generated.

## Dev Notes

### Backend API — POST /api/chat (SSE)

Endpoint ja implementado (Story 1.2.2). Aceita:
```json
POST /api/chat
{ "mindId": "string", "conversationId": "string|null", "message": "string" }
```

Retorna `text/event-stream` com eventos:
- `conversation` — `{ conversationId, isNew }`
- `text_delta` — `{ text }` (chunks de texto streaming)
- `task_detected` — `{ title, description, priority }`
- `usage` — `{ inputTokens, outputTokens }`
- `done` — `{ messageId }`
- `error` — `{ message }`

Rate limit: 20 req/min. Requires JWT Bearer token.

[Source: packages/api/src/routes/chat.ts]

### SSE Parsing Pattern (Client-Side)

O backend usa `reply.hijack()` com `text/event-stream`. O cliente deve parsear com `fetch` + `ReadableStream` (nao `EventSource`, pois precisa de POST + headers):

```typescript
const response = await fetch(`${API_URL}/api/chat`, {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    Authorization: `Bearer ${token}`,
  },
  body: JSON.stringify({ mindId, conversationId, message }),
});

const reader = response.body!.getReader();
const decoder = new TextDecoder();
let buffer = "";

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  buffer += decoder.decode(value, { stream: true });
  // Parse SSE lines: "event: type\ndata: json\n\n"
  const lines = buffer.split("\n\n");
  buffer = lines.pop()!; // keep incomplete
  for (const block of lines) {
    const eventMatch = block.match(/event: (.+)\ndata: (.+)/);
    if (eventMatch) {
      const [, event, data] = eventMatch;
      const parsed = JSON.parse(data);
      // handle event...
    }
  }
}
```

### Backend API — GET /api/conversations/:id/messages

Endpoint ja implementado (Story 1.2.3). Para carregar historico:

```typescript
GET /api/conversations/{id}/messages?limit=50
```

### API Client — JWT Injection

Para o chat stream, NAO usar `useApiClient()` (que retorna JSON parsed). Usar `fetch` direto com `useAuth().getToken()` para JWT. O `API_URL` esta em `process.env.NEXT_PUBLIC_API_URL`.

Para `loadHistory`, pode usar `useApiClient()` normalmente.

[Source: packages/web/lib/hooks/use-api-client.ts]
