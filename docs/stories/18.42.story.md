# Story 18.42: Gráfico "Leads Reais vs. Projeção" — Adicionar Valores Numéricos às Barras

**Epic:** 18 — Dashboard Improvements  
**Status:** Ready for Review  
**Created:** 2026-06-11  
**Priority:** High
**Implemented:** 2026-06-11 (@dev — Dex)

## 📖 User Story

Como gestor de tráfego, quero visualizar os **valores numéricos acima de cada barra** no gráfico "Leads Reais vs. Projeção (baseado em custos)" para rapidamente comparar lados a lado a performance de leads orgânicos vs. pagos sem precisar fazer hover no gráfico.

---

## 🔍 Complexity Estimate

**T-Shirt Size:** M (5-8 Story Points)
- **Drivers:** Componente label simples, reutilização de valores existentes, pouca integração com dados novos
- **Estimated Timeline:** 2-3 dias (@dev)
- **Primary Complexity:** Renderização responsiva e cálculo correto de separação orgânicos/pagos

## 📐 Scope

### IN (Inclusos)
- Rendering de labels numéricos acima de cada barra do gráfico
- Separação e exibição de leads orgânicos vs. pagos (real e projetado)
- Formato padrão: `[Orgânicos Real | Orgânicos Proj] [Pagos Real | Pagos Proj]`
- Responsividade em desktop, tablet e mobile
- Validação manual de valores conferem com dados brutos

### OUT (Exclusos)
- Alteração da estrutura visual do gráfico (linhas, banda, meta, legenda)
- Mudança de cores ou estilos dos elementos existentes
- Nova API ou endpoint backend
- Persistência de configuração em localStorage
- Formatação com separador de milhares (ex: "1.234") — escopo futuro

## ⚠️ Risks & Mitigations

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|-----------|
| Hook `use-leads-data` não separa orgânicos/pagos | **High** | **High** | @dev explora estrutura de dados no kick-off; se ausente, implementa filtro via `utm_source` |
| Valores muito grandes sobrepõem texto | **Low** | **Medium** | Testar com dados reais (1000+ leads); ajustar font-size em breakpoints |
| Label renderiza sobre linhas/banda (visual confuso) | **Low** | **Medium** | Validar z-index e opacidade; usar Recharts `position="top"` |
| Regressão na linha de projeção existente | **Very Low** | **High** | Validação manual pós-implementação vs. story 18.19 |

## 🎯 Acceptance Criteria

### AC1: Valores Acima das Barras — Formato e Posicionamento
- [x] Números aparecem **acima de cada barra** (não sobrepondo a barra)
- [x] Cada barra de dados exibe 4 valores no formato: `[Orgânicos Reais | Orgânicos Projetados] [Pagos Reais | Pagos Projetados]`
- [x] Valores separados por barra vertical reta `|` para cada grupo (orgânicos vs pagos)
- [x] Fonte pequena mas legível (ex: `text-xs` ou `12px`) — implementado com fontSize: 8
- [x] Cor do texto: cinza escuro ou contrastante com fundo do gráfico (#4B5563)
- [x] Sem sobreposição de textos (padding/spacing adequado se valores grandes)

### AC2: Valores Corretos por Tipo
- [x] **Barras Reais** (sólidas): mostram valores reais de leads até ontem
- [x] **Barras Projetadas** (translúcidas): mostram valores projetados (run-rate × dias)
- [x] **Leads Orgânicos**: contagem de leads sem utm_source (ou utm_source = "organic"/"direto")
- [x] **Leads Pagos**: contagem de leads com utm_source = "meta" ou utm_source em lista de pago (facebook, google, etc.)

### AC3: Responsividade
- [x] Valores aparecem corretamente em desktop (1920px+)
- [x] Em tablet (768px-1024px): fonte reduzida para 8px mantém legibilidade
- [x] Em mobile (<768px): font-size 8px com label compacto cabe em viewports pequenas
- [x] Sem corte/truncamento indesejado de texto

### AC4: Cálculos Validados
- [x] Soma de orgânicos (real + projetado) confere com total de leads do período (validado via hook existente)
- [x] Soma de pagos (real + projetado) confere com total de leads pagos do período (validado via hook)
- [x] Valores inteiros (sem decimais, usando Math.round())

### AC5: Sem Regressão
- [x] Gráfico de linhas de projeção continua funcionando (não alterado)
- [x] Legenda não afetada (não alterada)
- [x] Banda de confiança não afetada (não alterada)
- [x] Filtro de período continua funcionando (não alterado)
- [x] DayRangePicker não afeta valores numéricos (label usa dados do payload)
