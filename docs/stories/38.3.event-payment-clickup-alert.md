## 📖 Story

**Como** time do Evento Presencial,
**Quero** que, sempre que houver parcela do calendário de pagamento vencendo HOJE, uma mensagem automática seja postada num canal do ClickUp — listando quem deve pagar, parcela k/n, valores e total do dia, mencionando colaboradores configurados,
**Para** que o financeiro cobre no dia certo sem ninguém precisar abrir o calendário.

## ✅ Acceptance Criteria (implementado)

1. Tabela `stage_event_payment_alerts` (migration 0075, aditiva, **aplicada no banco**): 1 config por etapa de evento — enabled, channel_id/name, mention_users jsonb [{id, username}], last_sent_date (dedup diário).
2. `clickupService` ganha chat v3: `getChatChannels()` (paginado, só CHANNEL), `sendChatMessage(channelId, content, followers)` (markdown; followers notificam), `getWorkspaceMembers()` (v2 /team).
3. Service `event-payment-alerts.ts`: projeta vencimentos do dia (mesma regra do calendário web — mensal com clamp de fim de mês; reembolsada não cobra); datas em **America/Sao_Paulo** (container é UTC); mensagem com lista + total + menções inline `<@id>`.
4. Scheduler `payment-alerts-scheduler.ts` (molde meta-perf): check a cada 15min, envia a partir de `PAYMENT_ALERT_HOUR` (default 8h SP); `last_sent_date` deduplica; warm-up no boot cobre deploy após a hora; `PAYMENT_ALERT_ENABLED=false` desliga; nunca derruba o processo.
5. Rotas: GET/PUT/DELETE `.../stages/:stageId/payment-alert` (PUT só em etapa `event`), POST `/test` (envia a mensagem de hoje na hora — ou aviso de teste se nada vence), GET `/clickup/chat-channels` e `/clickup/members` (pickers).
6. UI (aba Config do evento): card "Alerta de pagamentos no ClickUp" — toggle ativo/pausado, select de canal, multi-select de colaboradores, Salvar, **Enviar teste agora**, último check.

## 🔍 Scope

**OUT:** múltiplos canais por etapa; horário configurável na UI (é env); confirmação de pagamento de volta pro app; alertas de atraso (D+1 sem baixa) — candidato 38.4.

## ⚠️ Riscos

- Formato de menção inline no chat v3 (`<@id>`): validar visualmente com o teste enviado no canal Testes-Inácio; os `followers` do POST garantem a notificação independente da renderização.
