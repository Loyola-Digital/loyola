## Story

**As a** authenticated Loyola Digital user,
**I want** to browse a visual catalog of AI minds organized by squad,
**so that** I can discover available minds and select one to start a conversation.

---

## Acceptance Criteria

1. [ ] Pagina `/minds` com grid de squads (cards expandiveis ou secoes)
2. [ ] Cada squad section mostra: nome, descricao, contagem de minds
3. [ ] Grid de mind cards dentro de cada squad
4. [ ] Mind card mostra: avatar (iniciais coloridas), nome, especialidade, tags como badges
5. [ ] Barra de busca com filtro por nome/especialidade (debounced 300ms)
6. [ ] Loading skeleton enquanto carrega dados
7. [ ] Empty state se nenhuma mind encontrada na busca
8. [ ] Responsivo: 1 coluna mobile, 2 tablet, 3-4 desktop
9. [ ] Dados carregados via TanStack Query (staleTime 5min — ja configurado no provider)
10. [ ] Clicar em mind card navega para `/minds/[mindId]`
11. [ ] `pnpm typecheck` passa sem erros
12. [ ] `pnpm lint` passa sem erros

### API Client Pattern

```typescript
// Client-side (em hooks):
import { useApiClient } from "@/lib/hooks/use-api-client";
import { useQuery } from "@tanstack/react-query";

function useMinds(query?: string) {
  const apiClient = useApiClient();
  return useQuery({
    queryKey: ["minds", query],
    queryFn: () => apiClient<{ squads: Squad[] }>(
      `/api/minds${query ? `?q=${encodeURIComponent(query)}` : ""}`
    ),
  });
}
```

[Source: packages/web/lib/hooks/use-api-client.ts, packages/web/lib/api-client.ts]

### Backend API — GET /api/minds

Endpoint ja implementado (Story 1.1.2). Retorna:
```json
{
  "squads": [{
    "id": "string",
    "name": "string",
    "displayName": "string",
    "description": "string",
    "mindCount": number,
    "minds": [{ "id", "name", "squad", "specialty", "tags", "avatarUrl", "totalTokenEstimate" }]
  }]
}
```
Query param `?q=term` filtra por nome/especialidade/tags.

[Source: packages/api/src/routes/minds.ts]
