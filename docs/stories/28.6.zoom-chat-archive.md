## User Story

Como gestor de tráfego que rodou um CPL via Zoom, quero ver as **mensagens do chat da reunião** (quem mandou, em que horário e o quê) dentro do dashboard da etapa CPL, pra (1) achar leads quentes que perguntaram muito, (2) ler perguntas frequentes pra alimentar copy do próximo lançamento, e (3) ter esse histórico **arquivado no banco** já que vamos cancelar a API do Zoom em 22/05.

## Comportamento desejado

### Backend

1. Novo helper em `zoom.ts`:

```ts
export async function fetchMeetingChat(
  meetingId: string,
  accessToken: string,
): Promise<ZoomChatMessage[]>
```

2. Tenta endpoints em ordem (mesmo pattern do `fetchMeetingParticipants` linha 159-162):
   - `/metrics/meetings/{meetingId}/chat`
   - `/past_meetings/{meetingId}/chat`
   - Se ambos falham com 4xx específico → retorna `[]` + log info "chat não disponível nesta reunião" (não bloqueia sync de participants)
3. Tipo `ZoomChatMessage`:

```ts
interface ZoomChatMessage {
  sender: string;             // nome do remetente
  senderEmail?: string;       // email se Zoom expor
  dateTime: string;           // ISO 8601
  message: string;            // texto
}
```

4. No `syncMeetingParticipants` ([zoom-stage.ts:237-353](packages/api/src/routes/zoom-stage.ts#L237)):
   - Após coletar participants + rawSessions, chama `fetchMeetingChat` em **paralelo** (Promise.all)
   - Adiciona `chat: ZoomChatMessage[]` ao objeto `cachedData` antes do `.update()`
5. Retro-compat: response da rota `GET /zoom/meetings/{id}` agora pode incluir `chat` no payload. Se `cachedData` antigo não tem `chat`, retorna `chat: []` (não quebra UI antiga).

### Frontend

1. Tipo `ZoomChatMessage` em `use-zoom-stage.ts`, exportado
2. Hook retorna `chat?: ZoomChatMessage[]` no dado de meeting
3. UI: **novo card "Chat da Reunião"** dentro de `zoom-meeting-analytics.tsx` (5º card no grid 2-col), com:
   - Header: "💬 Chat — N mensagens"
   - Lista scrollável (max-height 400px) com cada msg: `HH:mm` · **Nome** · texto
   - Filtro por nome (input search) — filtra na lista, client-side
   - Empty state: "Sem chat capturado pra esta reunião" (quando `chat = []`)

## Acceptance Criteria

### AC1 — Backend captura
- **AC1.1:** `fetchMeetingChat(meetingId, token)` tenta os 2 endpoints e retorna `ZoomChatMessage[]` ou `[]`
- **AC1.2:** Erros 4xx (chat não habilitado / reunião não tem chat) retornam `[]` silencioso (log info), **não quebram** o sync de participants
- **AC1.3:** Erros 5xx ou rede propagam (sync inteiro falha como hoje)
- **AC1.4:** Quando há mensagens, são ordenadas por `dateTime` ascendente

### AC2 — Persistência
- **AC2.1:** `cached_data.chat` é populado no `funnel_stage_zoom_meetings` quando o sync rodar
- **AC2.2:** Sem migration de schema (JSONB aceita campo novo direto)
- **AC2.3:** Force re-sync ([zoom-stage.ts:557](packages/api/src/routes/zoom-stage.ts#L557)) atualiza chat também

### AC3 — UI
- **AC3.1:** Card "💬 Chat — N mensagens" aparece quando `chat.length > 0`
- **AC3.2:** Lista mostra `20:15 · Maria Silva · Quanto custa o curso?`
- **AC3.3:** Filtro por nome funciona client-side (case-insensitive)
- **AC3.4:** Estilo dark-mode-safe — segue o pattern do hotfix #151 (`hsl(var(--card))`, `hsl(var(--foreground))`)
- **AC3.5:** Quando `chat = []` ou `chat === undefined`, card mostra estado vazio (não some — usuário precisa saber que o sync rodou mas não tinha chat)

### AC4 — Sem regressão

- **AC4.2:** Charts existentes (Retenção, Audiência, Distribuição, Histograma) não são afetados
- **AC4.3:** Falha em buscar chat não impede sync dos demais dados (graceful degradation)

## Implementation guidance

### `fetchMeetingChat` — pattern dos endpoints alternativos

Reusar a estratégia do `fetchMeetingParticipants` ([zoom.ts:159-162](packages/api/src/services/zoom.ts#L159)):

```ts
export async function fetchMeetingChat(
  meetingIdOrUuid: string,
  accessToken: string,
): Promise<ZoomChatMessage[]> {
  const endpoints = [
    `/metrics/meetings/${encodeURIComponent(meetingIdOrUuid)}/chat`,
    `/past_meetings/${encodeURIComponent(meetingIdOrUuid)}/chat`,
  ];
  for (const path of endpoints) {
    try {
      const data = await fetchZoom<{ chat_messages?: Array<{ sender: string; sender_email?: string; date_time: string; message: string }> }>(
        path,
        accessToken,
      );
      const msgs = data.chat_messages ?? [];
      return msgs
        .map((m) => ({
          sender: m.sender,
          senderEmail: m.sender_email,
          dateTime: m.date_time,
          message: m.message,
        }))
        .sort((a, b) => a.dateTime.localeCompare(b.dateTime));
    } catch (err) {
      // Tenta próximo endpoint se 404; propaga 5xx
      if (isNotFoundError(err)) continue;
      throw err;
    }
  }
  return [];
}
```
