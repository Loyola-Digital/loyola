# Story 28.5 — Zoom analytics: eixo X em horário real (HH:mm) em vez de minuto-desde-início

**Epic:** 28 — follow-up de qualidade dos dashboards
**Status:** InReview
**Priority:** Medium
**Estimate:** XS (~30min de código, 1 arquivo)
**Type:** Enhancement (UX) — pequena melhora num componente existente
**Surface:** `packages/web/components/funnels/zoom-meeting-analytics.tsx`

---

## User Story

Como gestor de tráfego do Loyola que rodou um CPL (aula ao vivo via Zoom), quando olho os gráficos de **Curva de Retenção** e **Audiência Simultânea**, quero ver no eixo X **a hora real do dia** (ex: 20:00, 20:15, 20:30) em vez de "minuto desde o início" (0, 5, 10...). Isso me permite cruzar o pico/quedas com pontos específicos do roteiro ("às 20h25 abri o pitch") e comparar entre lives que começaram em horários diferentes.

---

## Estado atual

`zoom-meeting-analytics.tsx:62-77` monta `temporalData` com `{ minute: 0..240, concurrent, retention }`. Os 2 charts que consomem (`AreaChart` linha 140 e `LineChart` linha ~189) usam `dataKey="minute"` no `XAxis`.

Internamente o cálculo já parte de timestamps absolutos (`minStart` = `Date.parse(joinTime)`), só converte pra `minute` na hora de plotar. Logo, **o timestamp absoluto já está disponível** — basta expor + formatar no eixo.

---

## Comportamento desejado

1. `temporalData` ganha campo extra `clockTime: string` (formato `HH:mm`) por ponto
2. Charts trocam `dataKey="minute"` por `dataKey="clockTime"` no `XAxis`
3. Mantém `minute` no objeto (pode aparecer no tooltip pra contexto: "Min 15 · 20h15 · 87 pessoas")
4. Timezone: usa o do navegador (Brasília na prática) — `toLocaleTimeString("pt-BR", { hour: "2-digit", minute: "2-digit", hour12: false })`
5. Tick interval: `Math.max(1, Math.floor(totalMinutes / 12))` — ~12 ticks no eixo, evita overlap

---

## Acceptance Criteria

- **AC1:** Eixo X dos charts "Curva de Retenção" e "Audiência Simultânea (PICO)" mostra horário no formato `HH:mm` (ex: `20:15`)
- **AC2:** Hover/tooltip mantém info útil — mostra horário real + contagem; se possível inclui também minuto-desde-início entre parênteses
- **AC3:** Quando a reunião dura >120min, eixo X não fica poluído (ticks bem espaçados)
- **AC4:** Sem regressão nos outros 2 charts (Distribuição Hot/Warm/Cold/Bounce + Histograma de Duração — não usam timestamp)
- **AC5:** Sem mudança no formato dos dados puxados da API Zoom (rawSessions intactos)

---

## Out of Scope

- ❌ Adicionar toggle "minuto vs hora real" — Lucas escolheu hora real direto, sem dual mode
- ❌ Comparar múltiplas lives no mesmo gráfico (Tier 2 visualization, fora de escopo)
- ❌ Anotações de roteiro/manual markers no gráfico — feature futura
- ❌ Timezone configurável — fica do navegador

## Implementation guidance

No `useMemo` do `temporalData` (linha 40-80):

```ts
const clockTime = new Date(t).toLocaleTimeString("pt-BR", {
  hour: "2-digit",
  minute: "2-digit",
  hour12: false,
});
points.push({ minute: m, clockTime, concurrent, retention });
```

Nos XAxis (Recharts):

```tsx
<XAxis
  dataKey="clockTime"
  interval={Math.max(0, Math.floor(temporalData.length / 12))}
  fontSize={10}
/>
```
