## 📖 Story

**Como** desenvolvedor da plataforma Loyola Digital X,
**Quero** criar o schema de persistência do Switchy Link Generator (settings por projeto, presets de canal editáveis e histórico de shortlinks gerados),
**Para** que as stories seguintes (service REST, rotas e frontend) tenham as tabelas, índices, FKs e o seed dos 7 presets default prontos e migrados no banco.

---

## ✅ Acceptance Criteria

1. **Tabela `project_switchy_settings` criada** com as colunas e constraints EXATAS do contrato:
   - `id` uuid PK default `gen_random_uuid()`
   - `project_id` uuid NOT NULL **UNIQUE** FK → `projects(id)` ON DELETE cascade
   - `pixels` jsonb NOT NULL default `'[]'` (array de `{platform,value,title}`)
   - `show_gdpr` boolean NOT NULL default `false`
   - `default_utm_term` varchar(120) (nullable)
   - `default_utm_content` varchar(120) (nullable)
   - `created_at` timestamptz default `now()`
   - `updated_at` timestamptz default `now()`
   - índice `idx_project_switchy_settings_project` on (`project_id`)

2. **Tabela `switchy_channel_presets` criada** com as colunas e constraints EXATAS do contrato:
   - `id` uuid PK default `gen_random_uuid()`
   - `project_id` uuid NOT NULL FK → `projects(id)` ON DELETE cascade
   - `label` varchar(120) NOT NULL
   - `utm_medium` varchar(120) NOT NULL
   - `utm_source` varchar(120) NOT NULL
   - `sort_order` int NOT NULL default `0`
   - `enabled` boolean NOT NULL default `true`
   - `created_at` timestamptz default `now()`
   - `updated_at` timestamptz default `now()`
   - índice `idx_switchy_presets_project` on (`project_id`)

3. **Tabela `switchy_shortened_links` criada** com as colunas e constraints EXATAS do contrato:
   - `id` uuid PK default `gen_random_uuid()`
   - `project_id` uuid NOT NULL FK → `projects(id)` ON DELETE cascade
   - `folder_id` varchar(64) NOT NULL
   - `folder_name` varchar(500) (nullable)
   - `checkout_base_url` text NOT NULL
   - `channel_label` varchar(120) (nullable)
   - `utm_campaign` varchar(120) (nullable)
   - `utm_medium` varchar(120) (nullable)
   - `utm_source` varchar(120) (nullable)
   - `utm_term` varchar(120) (nullable)
   - `utm_content` varchar(120) (nullable)
   - `sck` text (nullable)
   - `vk_source` text (nullable)
   - `full_url` text NOT NULL
   - `short_url` text (nullable)
   - `switchy_link_id` varchar(255) (nullable)
   - `switchy_uniq` bigint (nullable)
   - `created_at` timestamptz default `now()`
   - índices em `project_id` (`idx_switchy_links_project`) e em `created_at` (`idx_switchy_links_created_at`)

4. **Definições Drizzle** das 3 tabelas adicionadas em `packages/api/src/db/schema.ts`, seguindo o padrão do bloco Mautic (`pgTable` + `.references(() => projects.id, { onDelete: "cascade" })` + `index(...)` no callback de tabela). Nomes de coluna em snake_case no banco, camelCase no objeto Drizzle.

5. **Arquivo de migration SQL** criado em `packages/api/src/db/migrations/0048_switchy_link_generator.sql`, seguindo o formato das migrations existentes (`CREATE TABLE IF NOT EXISTS`, `--> statement-breakpoint`, FKs via `ALTER TABLE ... ADD CONSTRAINT`, índices via `CREATE INDEX IF NOT EXISTS`). Comentário de cabeçalho descrevendo a story.

6. **Script de aplicação** criado em `packages/api/scripts/apply-migration-0048.mjs`, replicando o padrão node+pg dos scripts existentes (dotenv `../.env`, `DATABASE_URL`, BEGIN/COMMIT, ROLLBACK on error, verificação pós-migration via `information_schema`).

7. **Seed dos 7 presets default** definido como SQL idempotente (não duplica se o projeto já tem presets). Os 7 presets default são EXATAMENTE:

   | label | utm_medium | utm_source | sort_order |
   |-------|-----------|-----------|-----------|
   | bio | bio | ig | 0 |
   | direct | direct | ig | 1 |
   | stories | stories | ig | 2 |
   | manychat | automacao | manychat | 3 |
   | chatwoot | disparo | chatwoot | 4 |
   | email | email | mautic | 5 |
   | grupo | grupo | whatsapp | 6 |

   > **Decisão de design (Dev Notes):** o seed dos 7 presets NÃO é executado dentro da migration `0048` (que só roda 1x e a tabela `projects` pode ter N projetos criados depois). O seed é **lazy / por-projeto**, disparado pela rota `GET /switchy/presets` quando o projeto ainda não tem presets (story 33.3). Esta story 33.1 entrega a **constante/helper de seed** (`DEFAULT_SWITCHY_PRESETS`) e a função que insere os 7 quando ausentes, deixando o ponto de uso pronto para a story de rotas. A tabela de cima é a fonte de verdade dos valores.

8. **Migration aplicada com sucesso** no banco de desenvolvimento (`apply-migration-0048.mjs` roda sem erro, as 3 tabelas existem e os índices/FKs são confirmados via query de verificação).

9. **`npm run typecheck` e `npm run lint` passam** no pacote `packages/api` após a adição das definições Drizzle.

## ⚠️ Risks & Mitigations

| Risk | Severity | Mitigation |
|------|----------|-----------|
| **Colisão de número de migration** (outra branch usar 0048) | MEDIUM | 0048 confirmado como próximo livre (último é `0047_mautic_integration.sql`). Se houver conflito no merge, renumerar para o próximo livre e renomear o script `.mjs` correspondente. |
| **`bigint` do `switchy_uniq` estourar precisão no JS** | LOW | Drizzle mapeia `bigint` com `mode: "number"` (seguro até 2^53) ou `mode: "bigint"`. Como `uniq` do Switchy é um id numérico, usar `bigint("switchy_uniq", { mode: "number" })` — alinhado ao restante do schema. Se valores excederem 2^53 no futuro, migrar para `mode: "bigint"`. |
| **Seed duplicado de presets** | MEDIUM | Seed é lazy e idempotente: só insere quando `count(*) = 0` para o `project_id`. NÃO roda dentro da migration. Documentado em AC7. |
| **FK para `projects` falhar se projeto for deletado** | LOW | Todas as FKs usam `ON DELETE cascade` (mesmo padrão Mautic), garantindo limpeza automática. |
| **`default '[]'` em jsonb não aplicado** | LOW | Usar `default sql\`'[]'::jsonb\`` no Drizzle e `DEFAULT '[]'::jsonb` no SQL, padrão já usado em colunas jsonb do schema. Verificação pós-migration confere o default. |
