## Story

Como gestor de tráfego do Loyola X, quero visualizar a **Taxa de Conversão do Checkout** (Tx. Checkout) ao lado do card CPM na aba Meta Ads, calculada como **número de vendas / número de visitas no checkout**, para acompanhar a eficiência de conversão do funil de vendas.

## Contexto

### Problema atual

Hoje o `LaunchDashboard` não exibe a taxa de conversão do checkout. Para calcular esse métrica, o gestor precisa:
1. Contar vendas manualmente na planilha de captação paga
2. Contar visitas de checkout via Meta Ads API
3. Dividir manualmente

Essa taxa é crítica para avaliar a **qualidade do tráfego** que chega ao checkout.

### Solução proposta

Adicionar um card **Taxa Checkout** (ou "Tx. Checkout") ao lado direito do card CPM, mostrando:
- **Numerador:** número de vendas (planilha de captação paga)
- **Denominador:** número de visitas no checkout (Meta Ads API, `actions.action_type="landing_page_view"` para o pixel de checkout)
- **Fórmula:** `(vendas ÷ visitas_checkout) × 100` em percentual

### Decisão arquitetural

Frontend-only, utilizando dados existentes:
- Vendas da planilha de captação paga (já carregada em Story 18.14)
- Visitas de checkout via Meta Ads API (action_type='landing_page_view', evento rastreado no pixel de checkout)

Extrair esses valores do hook `useCrossedFunnelMetrics` ou criar cálculo auxiliar inline.

## Escopo

### IN
- Criar card **Taxa Checkout** (Tx. Checkout) exibindo percentual de conversão
- Fórmula: `(vendas ÷ visitas_checkout) × 100`
- Posicionar imediatamente à direita do card CPM
- Respeitar filtro de período `DayRangePicker`
- Exibir com 1 ou 2 casas decimais (ex: "15.5%" ou "15.45%")
- Se visitas_checkout = 0, exibir "—" (divisão por zero)
- Se planilha captação não vinculada, exibir "—" com tooltip explicativo

### OUT
- Alterações em outros cards ou métricas
- Alterações em tabelas ou gráficos
- Mudanças em backend ou eventos de tracking
- Configuração nova de pixels de checkout

## Fontes de dados

| Dado | Fonte | API/Hook |
|------|-------|----------|
| Número de Vendas | Planilha de Captação Paga | `useFunnelSpreadsheetData` (tipo `captures_paid`) |
| Visitas Checkout | Meta Ads API | `useQueries` com `/api/traffic/analytics/:projectId/campaign-daily` (action_type='landing_page_view') |
| Período | DayRangePicker | Estado `days` |

**Nota:** "Visitas do Checkout" é interpretado como o total de `landing_page_view` events capturados do pixel de Meta no checkout. Se o projeto usa um evento customizado diferente, ajustar conforme mapeamento do projeto.

## Fórmulas

```
visitas_checkout = Σ actions[action_type="landing_page_view"] no período
vendas           = count(linhas planilha de captação paga onde "Venda ingressos" = 1)

taxa_checkout    = (vendas ÷ visitas_checkout) × 100

Resultado: X% (com 1-2 casas decimais)
Divisão por zero: "—"
Planilha ausente: "—" + tooltip
```

## Riscos Técnicos

| Risco | Severidade | Impacto | Mitigação |
|-------|-----------|--------|-----------|
| **Evento landing_page_view mapeado incorretamente** | High | Taxa calculada com denominador errado, números inconsistentes | Validar mapeamento do pixel de checkout na Meta API; adicionar teste com dados conhecidos; documentar qual evento = checkout |
| **Planilha de captação não vinculada** | Medium | Card exibe "—", métrica indisponível | Documentar requirement de vinculação; tooltip explicativo |
| **Meta Ads API indisponível ou timeout** | Medium | Card não renderiza ou fica em loading | Implementar timeout e fallback ("—" com tooltip); adicionar retry logic |
| **Divisão por zero (zero visitas checkout)** | Low | Card exibe "—", é comportamento esperado | Validado em AC3; documentar no tooltip que zero visitas = "—" |
| **Filtro de data inconsistência** | High | Numerador e denominador em períodos diferentes | Garantir mesmo `days` para Meta API e planilha (feedback obrigatório: `feedback_dashboard_date_filter.md`) |
