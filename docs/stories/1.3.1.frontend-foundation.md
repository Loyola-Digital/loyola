### Tailwind CSS v4 — Breaking Changes

Tailwind CSS 4 usa uma abordagem diferente:
- **Sem `tailwind.config.ts`** — configuracao via CSS
- Import via `@import "tailwindcss"` no CSS
- PostCSS plugin: `@tailwindcss/postcss`
- Temas via CSS variables (nao mais `extend.colors` no config)
- Dark mode via `@media (prefers-color-scheme)` ou classe `dark` no html

```css
/* globals.css — Tailwind v4 */
@import "tailwindcss";

@theme {
  --color-background: hsl(0 0% 100%);
  --color-foreground: hsl(240 10% 3.9%);
  /* ... mais variables conforme shadcn/ui */
}

.dark {
  --color-background: hsl(240 10% 3.9%);
  --color-foreground: hsl(0 0% 98%);
}
```

### API Client — Pattern

Dois patterns de uso:

**Server Components (RSC):**
```typescript
import { auth } from "@clerk/nextjs/server";

async function getMinds() {
  const { getToken } = await auth();
  const token = await getToken();
  const res = await fetch(`${API_URL}/api/minds`, {
    headers: { Authorization: `Bearer ${token}` },
  });
  return res.json();
}
```

**Client Components:**
```typescript
"use client";
import { useAuth } from "@clerk/nextjs";

function useApiClient() {
  const { getToken } = useAuth();
  return async (path: string, options?: RequestInit) => {
    const token = await getToken();
    const res = await fetch(`${API_URL}${path}`, {
      ...options,
      headers: {
        "Content-Type": "application/json",
        Authorization: `Bearer ${token}`,
        ...options?.headers,
      },
    });
    if (!res.ok) throw new Error(`API error: ${res.status}`);
    return res.json();
  };
}
```
