## Story

Como usuario do Loyola X, quero conectar minha conta de Google Ads nas configuracoes, para que o sistema possa puxar dados de campanhas de YouTube Ads.

## Contexto Tecnico

O Google Ads API usa autenticacao via:
- **Developer Token** — token fixo da conta MCC (My Client Center)
- **Customer ID** — ID da conta de anuncio (formato: 123-456-7890)
- **Refresh Token** — OAuth2 refresh token para acesso continuo

A UI segue o **mesmo padrao do Meta Ads** em `/settings/traffic` — formulario inline + lista de contas + vinculacao a projetos.

## Acceptance Criteria

### AC1: Schema DB
- [ ] Tabela `google_ads_accounts` com: id, account_name, customer_id (varchar 20), developer_token_encrypted, developer_token_iv, refresh_token_encrypted, refresh_token_iv, is_active, created_by, created_at, updated_at
- [ ] Tabela `google_ads_account_projects` (many-to-many): id, account_id, project_id, created_at
- [ ] Unique constraint em customer_id
- [ ] Cascade delete em account_projects
- [ ] Migration gerada e aplicavel

### AC2: API Routes
- [ ] `POST /api/google-ads/accounts` — criar conta (validar token contra Google Ads API)
- [ ] `GET /api/google-ads/accounts` — listar contas com projetos vinculados
- [ ] `DELETE /api/google-ads/accounts/:id` — remover conta
- [ ] `POST /api/google-ads/accounts/:id/projects/:projectId` — vincular projeto
- [ ] `DELETE /api/google-ads/accounts/:id/projects/:projectId` — desvincular projeto
- [ ] Guests bloqueados (403)
- [ ] Tokens encriptados com mesma funcao de encryption existente

### AC3: Service de Validacao
- [ ] `validateGoogleAdsAccount(customerId, developerToken, refreshToken)` — valida credenciais chamando Google Ads API
- [ ] Retorna nome da conta se valido
- [ ] Throw error com mensagem clara se invalido

### AC4: Settings UI
- [ ] Nova pagina `/settings/google-ads`
- [ ] Formulario com 3 campos: Nome da conta, Customer ID, Developer Token, Refresh Token
- [ ] Lista de contas conectadas (cards)
- [ ] Vincular/desvincular projetos (Select dropdown)
- [ ] Delete com confirmacao (AlertDialog)
- [ ] Toast de sucesso/erro
- [ ] Loading states (Skeleton)
- [ ] Empty state quando nao tem contas

### AC5: Navegacao
- [ ] Adicionar grupo "Google" no sidebar de settings (ao lado do grupo "Meta")
- [ ] Sub-item "YouTube Ads" apontando para `/settings/google-ads`

## Dev Notes

- Usar Google Ads API v18 REST endpoint: `https://googleads.googleapis.com/v18/customers/{customerId}/googleAds:searchStream`
- Customer ID deve ser formatado sem hifens internamente (1234567890)
- Developer Token e Refresh Token devem ser encriptados separadamente
- O Google Ads API requer header `developer-token` em todas as requests
- Validacao: fazer query simples `SELECT customer.descriptive_name FROM customer WHERE customer.id = '{customerId}'`
