## Story

**As a** Loyola Digital employee managing client Instagram accounts,
**I want** a visual dashboard with charts and metrics for each account,
**so that** I can quickly assess account performance and identify trends without leaving the platform.

---

## Acceptance Criteria

1. [ ] Nova rota `/instagram` acessivel pela sidebar (icone Instagram de Lucide)
2. [ ] Account selector (dropdown) no topo — lista todas contas cadastradas
3. [ ] Period selector: 7d, 14d, 30d, 90d, custom range (date picker)
4. [ ] Secao **Overview**: 4 KPI cards — Followers (com variacao %), Following, Posts, Engagement Rate
5. [ ] Secao **Reach & Impressions**: line chart com reach e impressions ao longo do tempo
6. [ ] Secao **Posts Performance**: tabela/grid dos ultimos posts com thumbnail, caption truncada, likes, comments, reach — ordenavel por qualquer metrica
7. [ ] Secao **Stories**: cards dos stories ativos com impressions, reach, replies
8. [ ] Secao **Reels**: cards dos reels recentes com plays, likes, shares
9. [ ] Secao **Audience**: bar chart distribuicao idade/genero + top cidades/paises
10. [ ] Loading skeletons para cada secao durante fetch
11. [ ] Empty state quando nenhuma conta cadastrada — link para /settings/instagram
12. [ ] Responsivo: cards empilham no mobile, charts scrollam horizontal
13. [ ] Refresh button global e por secao
14. [ ] TanStack Query com staleTime adequado por tipo de dado
15. [ ] `recharts` adicionado como dependencia no packages/web
16. [ ] Link "Instagram" adicionado na sidebar

### UI Design Guidelines

- KPI cards: usar `Card` do shadcn/ui com numero grande, label, e badge de variacao (+/- %)
- Charts: cores consistentes com design system (gold primary, muted para secundario)
- Tabela: usar `Table` do shadcn/ui com sort headers
- Skeletons: usar `Skeleton` do shadcn/ui para cada secao
- Period selector: usar `Select` + `Popover` com `Calendar` para custom range
