## 📖 Contexto

A tabela **Desempenho de Testes de LPs** (`lp-performance-table.tsx`, renderizada pela `LpPerformanceSection` do `launch-dashboard.tsx`) lista as LPs do stage — mas as LPs são **derivadas dinamicamente** do Campaign Name (Story 18.46: "lpX" no nome da campanha → "LPA", "LPB"...). Não existe entidade LP no banco, portanto não há onde guardar a URL real de cada página.

O usuário quer **cadastrar manualmente o link de cada LP direto na tabela** e, uma vez cadastrado, o **nome da LP vira hiperlink** — um clique abre a página real para conferir o criativo/copy sem caçar a URL no Notion/planilha.

## 🗳️ Decisões de escopo (elicitação com o usuário, 2026-07-16)

| Dúvida | Decisão |
|--------|---------|
| UX de edição | **Lápis na linha:** ícone de editar ao lado do nome da LP; clica → popover com input de URL → salvar |
| Escopo do link | **Por etapa:** cada stage tem seus próprios links (LPA da Paga ≠ LPA da Gratuita). Persistência em `funnel_stages` (JSONB), seguindo o padrão de `campaigns`/`switchyLinkedLinks` |

---

## ✅ Acceptance Criteria

**AC1 — Backend: persistência `lpLinks` por stage**
- Nova coluna `lp_links` (JSONB, default `{}`) em `funnel_stages`, tipada `Record<string, string>` — chave = `lpName` normalizado (`trim().toLowerCase()`, ex.: `"lpa"`), valor = URL.
- Migration nova (seguir numeração `packages/api/src/db/migrations/`, próxima após `0079`).
- **Given** o PUT `/api/projects/:projectId/funnels/:funnelId/stages/:stageId` existente, **when** o body inclui `lpLinks`, **then** o campo é validado (Zod: record de strings; cada valor deve ser URL `http(s)://` válida OU string vazia — vazia remove o link) e persistido; `stageShape` passa a retornar `lpLinks`. Demais campos do PUT inalterados.
- Guest continua bloqueado pelo endpoint (comportamento atual, linha 361).

**AC2 — Edição inline na tabela (lápis por linha)**
- **Given** a tabela Desempenho de Testes de LPs, **when** renderiza cada linha, **then** um ícone de lápis aparece ao lado do nome da LP (visível em hover da linha ou sempre — seguir padrão visual das tabelas existentes).
- **When** clico no lápis, **then** abre um popover com input de URL (pré-preenchido se já houver link), botões Salvar/Cancelar.
- **When** salvo, **then** o PUT do stage é chamado com o `lpLinks` atualizado (merge: só a chave da LP editada muda) e a UI reflete sem reload (invalidation/optimistic do React Query).
- Salvar com o campo **vazio remove** o link da LP (nome volta a texto puro).

**AC3 — Nome hiperlinkado**
- **Given** uma LP com link cadastrado, **when** a tabela renderiza, **then** o nome (`row.lpName`) vira `<a href={url} target="_blank" rel="noopener noreferrer">` com affordance visual de link (cor/underline discreto e/ou ícone de link externo).
- LP **sem** link cadastrado: nome continua texto puro, idêntico a hoje.
- O match usa a mesma normalização do agregado (`lpName.trim().toLowerCase()`, como `lpTotals` no `useLpPerformanceData.ts:94`).

**AC4 — Validação de URL**
- **Given** o popover, **when** digito algo que não é URL `http(s)://` válida (e não-vazio), **then** o Salvar é bloqueado com mensagem de erro inline ("URL inválida — use http:// ou https://"). Trim aplicado antes de validar.

**AC5 — Escopo por etapa**
- **Given** o mesmo funil, **when** cadastro o link da LPA na Captação Paga, **then** a LPA da Captação Gratuita continua sem link (e vice-versa) — cada stage lê/escreve seu próprio `lpLinks`.

**AC6 — Regressão zero**
- Stage sem nenhum link cadastrado: tabela byte-a-byte como hoje (exceto o lápis). Métricas, ordenação, filtro de público (Hot/Cold/Todos) e colunas paid/free intactos.
- O PUT de stage sem `lpLinks` no body não toca o campo (partial update, padrão das linhas 385-393 de `funnel-stages.ts`).

## 📊 Complexidade & Valor

- **Estimativa:** S (small) — 1 coluna JSONB + extensão de endpoint existente + popover de edição + render condicional do nome. Sem endpoint novo, sem cruzamento de dados.
- **Valor de negócio:** o gestor confere a página real de cada LP em 1 clique durante a análise de testes — hoje a URL vive fora do dashboard (Notion/planilha/memória), quebrando o fluxo de análise.

## ⚠️ Riscos

- **R1 (baixo)** — LPs somem da tabela quando o período/campanhas mudam (são derivadas dos Campaign Names do range): o link cadastrado fica órfão no JSONB, invisível mas persistido. **Mitigação:** aceitável — se a LP reaparecer, o link volta; órfãos não têm efeito colateral.
- **R2 (baixo)** — URL maliciosa/typo (`javascript:` etc.): **Mitigação:** validação AC4 restringe a `http(s)://` no frontend E no Zod do backend; `rel="noopener noreferrer"` no anchor.
- **R3 (baixo)** — Corrida entre dois usuários editando links do mesmo stage (PUT com objeto inteiro): último ganha. **Mitigação:** aceitável no tamanho do time; merge por chave no client reduz janela.
