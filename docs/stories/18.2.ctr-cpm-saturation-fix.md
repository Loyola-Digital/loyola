# Story 18.2: Corrigir Cálculo CTR × CPM — Indicador de Saturação

## Descrição

O gráfico **CTR × CPM** (Indicador de Saturação) no dashboard de Captação Paga está exibindo dados quebrados:
- Mostrando apenas dias **20-04 até 25-04**
- Dia 25 aparecendo com valor zerado
- Expectativa: mostrar série completa de dados dentro do filtro de data selecionado

**Impacto:** Gestor de tráfego não consegue visualizar tendência correta de saturação.

## Contexto

- Componente: `meta-ads-spreadsheet-tab.tsx` (aba Meta Ads 2)
- Gráfico: CTR (%) no eixo X × CPM (R$) no eixo Y
- Métrica de Saturação: indica quando campanha está saturada (alto CPM + baixo CTR)
- Problema: filtro de data ou cálculo de agregação diária pode estar ignorando alguns dias ou incluindo valores zerados incorretamente

## Dependências

- [ ] **Story 18.1** (Meta Ads 2 — Tabela diária cruzada) **DEVE estar Done**
  - Necessário: Aba "Meta Ads 2" existente no funil + componente base `meta-ads-spreadsheet-tab.tsx` funcional
  - Sem 18.1 completa, esta story não tem base para corrigir o gráfico

## Riscos & Blockers

- **Risco Alto:** Problema pode estar em 3 camadas diferentes (componente React, hook de dados, API, ou query no banco)
  - **Mitigação:** Pistas de Debugging guiam investigação por camada — começar de cima (componente) e descer até API
  - **Impacto:** Se diagnosticar errado, pode gastar tempo no lugar errado

- **Blocker Potencial:** Se problema estiver em schema DB (coluna faltante ou mapeamento errado)
  - **Mitigação:** Validar schema DB (`packages/api/src/db/schema.ts`) PRIMEIRO antes de implementar
  - **Impacto:** Pode descobrir que precisa de migration de DB (escopo ampliado)

- **Risco Médio:** Filtro de data hardcoded pode estar em múltiplos lugares (hook, API, componente)
  - **Mitigação:** Usar grep para procurar padrão "20-04" ou "25-04" nos arquivos sugeridos
  - **Impacto:** Se em múltiplos lugares, correção leva mais tempo que estimado

## Acceptance Criteria

### AC1 — Série de dados completa
- [ ] Gráfico exibe todos os dias dentro do **intervalo de data selecionado no filtro** (não fixo 20-04:25-04)
- [ ] Se filtro = últimos 30 dias, mostra 30 pontos (um por dia)

### AC2 — Sem valores zerados incorretos
- [ ] Dia 25-04 exibe valor correto de CPM/CTR (não zero)
- [ ] Apenas dias SEM dados no período ficam vazios (gap no gráfico)

### AC3 — Cálculo de média diária correto
- [ ] CTR diário = (soma de cliques no dia) / (soma de impressões no dia)
- [ ] CPM diário = (soma de gasto no dia) / (soma de impressões no dia) * 1000
- [ ] Validar query/hook que agrega dados por dia

### AC4 — Filtro de data respeitado
- [ ] Quando usuário muda filtro de data, gráfico atualiza automaticamente
- [ ] Query na API respeita data_start e data_end passados

## Scope

- Revisar fórmula de cálculo CTR e CPM
- Testar com intervalo de 30, 60 dias para validar série completa
- Corrigir zero-values ou dados inválidos na série

### ❌ OUT

## Pistas de Debugging

1. **Arquivo principal:** `packages/web/components/funnels/meta-ads-spreadsheet-tab.tsx`
2. **Hook de dados:** Procure por `useMeta` ou `useMetaAdsData` que alimenta este gráfico
3. **Query na API:** Verifique `packages/api/src/routes/funnels.ts` — procure por endpoint que retorna dados diários de CTR/CPM
4. **Possível culpado:**
   - Filtro de data hardcoded em algum lugar (20-04:25-04)?
   - Agregação diária exluindo último dia?
   - Valores zerados não sendo filtrados na query?
