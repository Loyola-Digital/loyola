## Story

**As a** authenticated Loyola Digital user chatting with a mind,
**I want** to see task suggestions detected inline in chat and manage my delegated tasks,
**so that** I can create ClickUp tasks directly from AI conversations and track them in the app.

---

## Dev Notes

### Chat SSE Events

O backend (Story 1.2.2) emite os seguintes SSE events:
- `conversation` — `{ conversationId }` (inicio do stream)
- `text_delta` — `{ text }` (tokens incrementais)
- `task_detected` — `{ title, description, priority, tags }` (task sugerida pela mind)
- `usage` — `{ inputTokens, outputTokens }` (metricas)
- `done` — `{}` (fim do stream)
- `error` — `{ message }` (erro)

O hook `useChatStream` atualmente NAO trata o evento `task_detected`. Precisa ser adicionado.

[Source: packages/api/src/routes/chat.ts, packages/web/lib/hooks/use-chat-stream.ts]

### Backend API — POST /api/tasks

Endpoint ja implementado (Story 1.6.1). Request body:
```json
{
  "conversationId": "uuid",
  "messageId": "uuid",
  "mindId": "string",
  "title": "string",
  "description": "string (optional)",
  "priority": "normal (optional: urgent|high|normal|low)",
  "tags": ["string"]
}
```
Response 201: `DelegatedTask` object
Response 502: ClickUp API failure (nao salva localmente)
Idempotencia: se `clickupTaskId` ja existe, retorna 200 com task existente.

[Source: packages/api/src/routes/tasks.ts]

### Backend API — GET /api/tasks

Endpoint ja implementado (Story 1.6.1). Response:
```json
{
  "tasks": [{
    "id": "uuid", "conversationId": "uuid", "messageId": "uuid | null",
    "userId": "uuid", "mindId": "string", "clickupTaskId": "string",
    "clickupUrl": "https://app.clickup.com/t/xxx", "title": "string",
    "description": "string | null", "status": "open",
    "priority": "normal", "tags": ["string"], "createdAt": "ISO", "updatedAt": "ISO"
  }],
  "total": 10
}
```
Query params: `?status=open&limit=20&offset=0`

[Source: packages/api/src/routes/tasks.ts]
