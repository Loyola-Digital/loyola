## 📖 Story

**Como** desenvolvedor da plataforma Loyola Digital X,
**Quero** um plugin Fastify `kiwify.ts` que exponha o CRUD de conexão (credenciais criptografadas por projeto), a lista de produtos recorrentes e o dashboard agregado (com cache SWR L1+L2),
**Para** que o frontend (35.4/35.5) consuma a integração Kiwify por endpoints REST seguros, sem conhecer o contrato da Kiwify.

---

## ✅ Acceptance Criteria

1. **Plugin fp** `packages/api/src/routes/kiwify.ts` (espelha `routes/hotmart.ts`): `getProjectAccess` (guest sem vínculo → 404), `getConnectionRow`, `getCreds` (decifra client_id/secret/account_id, ou null).

2. **`GET /api/projects/:projectId/kiwify/connection`** → `{ connected: boolean }` (nunca retorna credenciais).

3. **`PUT /api/projects/:projectId/kiwify/connection`** (guest → 403): body `{ clientId, clientSecret, accountId }` (zod, `min(1)`). **Valida batendo no token endpoint + 1 chamada autenticada** (`GET /v1/products?page_size=1` com o accountId, p/ confirmar que o account-id casa) **antes** de criptografar e fazer upsert. Invalida cache do projeto. Retorna `{ connected: true }`. Em falha → 502 com mensagem clara.

4. **`DELETE /api/projects/:projectId/kiwify/connection`** (guest → 403): remove a linha + invalida cache → `{ connected: false }`.

5. **`GET /api/projects/:projectId/kiwify/products?months=12`** → `{ products: [{id,name}] }` (produtos recorrentes derivados, cacheado SWR). 409 se não conectado.

6. **`GET /api/projects/:projectId/kiwify/dashboard?productId=&months=12`** → `KiwifyDashboard` (cacheado SWR). zod valida `productId` (min 1) e `months` (1..36, default 12). 409 se não conectado.

7. **Cache SWR L1+L2** idêntico ao Hotmart: L1 LRU em memória (TTL 30min) + L2 tabela `kiwify_cache` (stale-while-revalidate, guard de stampede via `refreshing` set, `backgroundRefresh`). `invalidateKiwifyCache(projectId)` limpa L1 (prefixo) + L2 (linhas do projeto), chamado no PUT/DELETE. `cache_key` = `products:<months>` | `dashboard:<productId>:<months>`.

8. **Plugin registrado** onde os demais plugins de rota são (espelha registro do `hotmartRoutes`).

9. **Segurança:** GET connection nunca serializa segredo; nenhum log de secret/token/PII; guest bloqueado em PUT/DELETE.

10. **Qualidade:** `npm run typecheck` + `npm run lint` passam.

## ⚠️ Risks & Mitigations

| Risk | Severity | Mitigation |
|------|----------|-----------|
| Credencial inválida salva sem validar | MEDIUM | PUT valida token + 1 chamada autenticada antes de persistir (AC3). |
| account_id errado passa na auth mas falha depois | MEDIUM | Validação do PUT inclui `GET /products?page_size=1` com o accountId (não só o token). |
| Cache servir dados de outro produto | LOW | `cache_key` inclui `productId` + `months`; invalidação por prefixo de projeto. |
| Guest acessar dados | MEDIUM | `getProjectAccess` + guard guest em PUT/DELETE (espelha Hotmart). |

### Diferenças do contrato

- `connectionBodySchema = z.object({ clientId: z.string().min(1), clientSecret: z.string().min(1), accountId: z.string().min(1) })`.
- `getCreds` retorna `{ clientId, clientSecret, accountId }`.
- Compute chama `computeKiwifyDashboard(token, accountId, { productId, months })` e `listKiwifyProducts(token, accountId)` (precisa do accountId no header — passar adiante).
- Validação do PUT: `const token = await getKiwifyToken(clientId, clientSecret); await kiwifyGet(token, accountId, "/products", { page_size: 1 });` dentro de try/catch → 502 em falha.
