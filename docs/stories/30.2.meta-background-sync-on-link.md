# Story 30.2 — Background Sync ao Vincular Campanha

**Epic:** 30
**Status:** Ready for Review
**Points:** 3
**Depende de:** 30.1

## User Story

Quando vinculo uma campanha nova a um funnel, quero que o sistema pré-popule em background o cache histórico (365 dias retroativos) dessa campanha, para que quando eu abrir o dashboard, os dados já estejam instantaneamente disponíveis.

## Acceptance Criteria

- [x] **AC1:** Helper `syncCampaignHistoryInBackground(db, projectId, metaAccountId, campaignId, accessToken, days = 365)` em `meta-insights-cache.ts`:
  - Determina quais (campaign, date) já estão no cache
  - Pros dias faltantes, chama `fetchCampaignDailyInsightsForIds` (que faz chunks de 90 internamente — Story 30.1)
  - Upserta resultado
  - Captura erros internamente (não propaga) e loga no `fastify.log`
- [x] **AC2:** Trigger na rota `POST /api/projects/:projectId/funnels`: depois de inserir, identificar IDs de campanhas novas em `funnel.campaigns` e disparar `syncCampaignHistoryInBackground` para cada, **sem await** (fire-and-forget).
- [x] **AC3:** Trigger na rota `PUT /api/projects/:projectId/funnels/:funnelId`: comparar `campaigns` antigas vs novas, identificar IDs adicionados, disparar sync pra cada novo.
- [x] **AC4:** Trigger na rota `PUT /api/projects/:projectId/funnels/:funnelId/stages/:stageId`: idem pra `stage.campaigns`.
- [x] **AC5:** Sync NÃO bloqueia a resposta da rota — usuário vê resposta imediata (≤500ms), background continua.
- [x] **AC6:** Erros do sync não derrubam o processo Node (try/catch global).
- [x] **AC7:** Quando user re-vincula uma campanha já cached (ex: removeu e adicionou de novo no mesmo dia), o sync NÃO duplica trabalho — `loadCachedCampaignInsights` filtra dias já fresh.
- [x] **AC8:** Typecheck zero erros.

## Dev Notes

### Fire-and-forget seguro

```ts
function fireAndForget(promise: Promise<unknown>, label: string) {
  promise.catch((err) => {
    fastify.log.error({ err, label }, "background sync failed");
  });
}
```

Sem await na rota; sem Promise.reject vazado.

### Por que só novas campanhas?

Re-sync de todas a cada update sobrecarrega Meta API. Só sync incremental do que entrou no funnel.

### Idempotência

Sync chamado N vezes pra mesma campanha → 1ª popula, demais não fazem nada (cache check no início).

### Acesso ao token

`metaAdsAccounts` tem `accessTokenEncrypted` + `accessTokenIv`. Mesma pattern de `stage-sales-data.ts` (já vi anteriormente).
