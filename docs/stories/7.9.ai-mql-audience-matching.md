## Story

**As a** gestor de trafego da Loyola Digital,
**I want** to describe my ideal audience in natural language (e.g., "mulheres catolicas com renda acima de 5k e no maximo 2 filhos") and have the AI analyze my survey spreadsheet to automatically generate qualification rules,
**so that** I don't need to manually map field names and values — the AI reads my sheet headers and actual responses to create exact matching rules.

---

## Acceptance Criteria

1. [ ] Novo campo de texto na secao "Perfil de Lead Qualificado" em `/settings/traffic` — "Descreva seu publico ideal"
2. [ ] Botao "Gerar regras com IA" que envia a descricao + dados da planilha para o backend
3. [ ] `POST /api/traffic/qualification/:projectId/ai-generate` — recebe descricao em texto livre + projectId
4. [ ] Backend le headers e amostra de respostas (primeiras 50 rows) da aba survey via Google Sheets API (reutiliza `getTabData()`)
5. [ ] Backend envia para Claude API: descricao do publico + headers + amostra de valores unicos por coluna → recebe `QualificationRule[]`
6. [ ] Prompt do Claude inclui: headers da planilha, valores unicos por campo (max 20 por coluna — para colunas numericas enviar min/max/5 exemplos em vez de lista de unicos), descricao do usuario, e formato de saida esperado (JSON)
7. [ ] Claude retorna regras no formato existente: `[{ field, operator, value }]` — mesmo formato que o sistema ja usa
8. [ ] Regras geradas sao exibidas na UI para revisao antes de salvar (mesmo layout de regras existente)
9. [ ] Usuario pode editar/remover regras individuais geradas pela IA antes de confirmar
10. [ ] Preview automatico apos geracao: "X de Y leads qualificados com este perfil" (reutiliza endpoint de preview existente)
11. [ ] Botao "Confirmar e Salvar" persiste as regras (reutiliza `POST /api/traffic/qualification/:projectId` existente)
12. [ ] Se aba survey nao mapeada, exibir mensagem clara: "Configure a aba de pesquisa no Google Sheets primeiro"
13. [ ] Loading state durante geracao (Claude pode levar alguns segundos)
14. [ ] Tratamento de erro: se Claude nao conseguir gerar regras validas, exibir mensagem e sugerir ajustar descricao
15. [ ] Fluxo manual de regras continua funcionando (IA e complementar, nao substitui)
16. [ ] `pnpm typecheck` e `pnpm lint` passam sem erros

### AI Generation Flow

```
1. Usuario digita: "Mulheres catolicas com renda acima de 5k e no maximo 2 filhos"
2. Frontend chama: POST /api/traffic/qualification/:projectId/ai-generate { description }
3. Backend:
   a. Busca connection + tab mapping do projeto (aba survey)
   b. Le dados da aba survey via getTabData()
   c. Extrai headers + valores unicos por coluna (max 20 por campo)
   d. Monta prompt para Claude com contexto da planilha + descricao
   e. Claude retorna QualificationRule[]
4. Frontend recebe regras → exibe para revisao → usuario confirma → salva via endpoint existente
```

### Claude Prompt Design

```typescript
const prompt = `Voce e um assistente que gera regras de qualificacao de leads.

O usuario descreveu o publico ideal como:
"${description}"

A planilha de pesquisa tem os seguintes campos (headers):
${headers.join(", ")}

Valores encontrados por campo:
${Object.entries(fieldSummaries).map(([field, summary]) =>
  summary.isNumeric
    ? `- ${field} (numerico): min=${summary.min}, max=${summary.max}, exemplos: ${summary.examples.join(", ")}`
    : `- ${field}: ${summary.uniqueValues.slice(0, 20).join(", ")}`
).join("\n")}

Gere regras de qualificacao no formato JSON:
[
  { "field": "nome_exato_do_header", "operator": "equals|not_equals|gte|lte|contains|in", "value": "valor_exato_da_planilha" }
]

REGRAS:
- Use EXATAMENTE os nomes dos headers da planilha no campo "field"
- Use EXATAMENTE os valores encontrados na planilha no campo "value"
- Operadores disponiveis: equals, not_equals, gte (>=), lte (<=), contains, in
- Para numeros use gte/lte. Para texto exato use equals. Para lista use in.
- Retorne APENAS o array JSON, sem explicacao.`;
```
