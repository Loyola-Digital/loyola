## 📖 User Story

Como gestor de tráfego, quero ver um gráfico unificado de leads que mostre:
1. Uma **linha única** azul: sólida (dados reais até ontem) + tracejada (projeção até fim do lançamento)
2. **Barras diárias** por trás: reais opacas até ontem + projetadas translúcidas de hoje em diante
3. Uma **linha de referência** da meta total (reta horizontal)
4. Uma **banda de confiança** ao redor da projeção (±1*sigma*sqrt(k))

para que eu possa acompanhar visualmente se o leads está no caminho certo e com que margem de incerteza.

## 🎯 Acceptance Criteria

### AC1: Inputs Obrigatórios
- [ ] Campo de entrada para **Data Final do Lançamento** (datepicker, persistido)
- [ ] Campo de entrada para **Meta Total de Leads** (número, persistido)
- [ ] Campo configurável para **N** (janela de dias para run-rate, default=5, persistido)
- [ ] Todos os valores restauráveis via localStorage entre sessões

### AC2: Série Diária e Run-Rate Ponderada
- [ ] **Derivar série diária** a partir do acumulado: `d[i] = C[i] - C[i-1]`
- [ ] Calcular run-rate **NUNCA sobre acumulado, só sobre série diária**
- [ ] **Run-rate = média móvel PONDERADA** dos últimos N dias
  - Pesos lineares: dia 1 = weight 1, dia 2 = weight 2, ..., dia N = weight N
  - Fórmula: `r = Σ(d[i] * weight[i]) / Σ(weights)`
- [ ] Se menos de N dias reais disponíveis: usar todos os dias disponíveis
- [ ] Recalcular run-rate sempre que data_final ou N mudam

### AC3: Projeção de Barras Diárias Futuras
- [ ] **Barras diárias futuras** = run-rate `r` (constante para cada dia projetado)
- [ ] Barras projetadas: `bar[t+k] = r` para k=1,2,...,dias_até_final
- [ ] **Barras reais:** `bar[t] = C[t] - C[t-1]` (dados históricos)
- [ ] Opacidade visual: reais ~0.85, projetadas ~0.35

### AC4: Linha Acumulada Unificada
- [ ] **Linha única**, mesmo tom azul em toda a série
- [ ] **Sólida (opaca):** até ontem (dados reais)
- [ ] **Tracejada (clara):** de ontem até data_final (projeção)
- [ ] **Continuidade:** primeiro ponto projetado = último ponto real (sem degrau)
- [ ] Construir acumulado projetado como: `C[t+k] = C[t] + r*k`
- [ ] Validação: `acumulado_projetado[final] == acumulado[hoje] + Σ(barras_projetadas)`

### AC5: Banda de Confiança
- [ ] **Banda translúcida** ao redor da projeção (sólida, não tracejada)
- [ ] Limite superior: `C[t+k] + 1*sigma*sqrt(k)`
- [ ] Limite inferior: `C[t+k] - 1*sigma*sqrt(k)`
- [ ] Onde: `sigma = desvio_padrão(série_diária_da_janela)`
- [ ] Banda aparece **APENAS no trecho projetado** (de hoje em diante)
- [ ] Cor: mesmo azul da linha, opacidade ~0.15

### AC6: Linha de Meta (Referência Horizontal)

### AC7: Marcador "Hoje" e Legenda

## 🎬 Implementation Notes

### MÉTODO OBRIGATÓRIO (Não Inovar)

Seguir exatamente este método, passo a passo:

#### Passo 1: Derivar Série Diária
```
A partir do acumulado C[i], calcular a série diária:
d[i] = C[i] - C[i-1]  para i > 0
d[0] = C[0]

Exemplo:
C = [40, 85, 120, 160, ...]  (acumulado)
d = [40, 45,  35,  40, ...]  (diária)
```

#### Passo 2: Calcular Run-Rate Ponderada (NUNCA no acumulado)
```
Última N dias da série diária: d[t-N+1], d[t-N+2], ..., d[t]

Pesos lineares: w[1]=1, w[2]=2, ..., w[N]=N

Run-rate = Σ(d[i] * w[i]) / Σ(w)
         = Σ(d[i] * w[i]) / (N * (N+1) / 2)

Exemplo (N=5, últimos 5 dias reais [40, 45, 35, 40, 50]):
weights = [1, 2, 3, 4, 5]
numerador = 40*1 + 45*2 + 35*3 + 40*4 + 50*5 = 40 + 90 + 105 + 160 + 250 = 645
denominador = 1+2+3+4+5 = 15
r = 645 / 15 = 43 leads/dia
```

#### Passo 3: Projetar Barras Diárias Futuras
```
Para cada dia futuro t+k (k=1,2,...,dias_até_final):
  bar[t+k] = r  (run-rate constante)

Verificação: Σ(bar[t+1..t+dias_restantes]) = r * dias_restantes
```

#### Passo 4: Construir Linha Acumulada Projetada
```
C[t+k] = C[t] + r * k

Validação crítica:
C[final] == C[hoje] + Σ(barras_projetadas)

Esta igualdade é lei — sem ela, o método está errado.
```

#### Passo 5: Calcular Banda de Confiança
```
sigma = desvio_padrão(série_diária da janela de N dias)

Para cada dia projetado t+k:
  limite_superior[t+k] = C[t+k] + 1 * sigma * sqrt(k)
  limite_inferior[t+k] = C[t+k] - 1 * sigma * sqrt(k)

Exemplo:
sigma = 8 (desvio dos 5 dias reais)
Dia +1: banda = [43 + 8*sqrt(1), 43 - 8*sqrt(1)] = [51, 35]
Dia +5: banda = [215 + 8*sqrt(5), 215 - 8*sqrt(5)] = [233, 197]
```

#### Passo 6: Verificação de Continuidade
```
O ponto da projeção em t (ontem, último dado real) deve ser igual ao primeiro ponto projetado:
C[t] == C[t] + r * 0

Sim, óbvio. O que importa é: garantir que a emenda não tem degrau visual.
No gráfico: último ponto real = primeiro ponto projetado, mesma altura.
```

## 🚀 Implementation Roadmap

### Steps (Ordem Rigorosa)

1. **Inputs HTML** (3 campos)
   - `dataFinal` (datepicker, default +20 dias)
   - `metaTotal` (número, default 0)
   - `N` (número, default 5, range [2, 30])
   - Persistência em localStorage

2. **Função `calculateDailyFromCumulative()`**
   - Input: array de acumulados
   - Output: array de séries diárias
   - `d[i] = C[i] - C[i-1]`

3. **Função `calculateRunRateWeighted(dailySeries, N)`**
   - Input: série diária, N
   - Output: run-rate ponderada `r`
   - Pesos lineares: 1, 2, ..., N
   - Fórmula: `r = Σ(d[i] * w[i]) / Σ(w)`

4. **Função `calculateSigma(dailySeries, N)`**
   - Input: série diária, N
   - Output: desvio padrão dos últimos N dias
   - Fórmula: `sigma = sqrt(Σ((d[i] - média)²) / N)`

5. **Função `projectAccumulated(cumulativeToday, runRate, daysAhead, sigma)`**
   - Input: último acumulado, r, dias até final, sigma
   - Output: array com {cumulative, bandUpper, bandLower} para cada dia
   - Fórmulas:
     - `C[t+k] = C[t] + r * k`
     - `band[t+k] = C[t+k] ± sigma * sqrt(k)`

6. **Função `expandChartData()`**
   - Combina dados reais + projetados
   - Adiciona meta (reta horizontal)
   - Estrutura final: `{date, dailyReal, dailyProjected, cumReal, cumProjected, bandUpper, bandLower, meta}`

7. **Renderização (ComposedChart)**
   - 2 Barras (real + projetada)
   - 2 Linhas (real sólida + projetada tracejada, ambas azul)
   - 1 Banda (ao redor da projeção)
   - 1 ReferenceLine (meta horizontal cinza)
   - 1 ReferenceLine (marcador "Hoje")
   - Legenda diferenciando: "Real", "Projeção", "Banda de Confiança", "Meta"

8. **Teste Manual**
   - Validar com exemplo numérico (usar o exemplo da Implementation Notes)
   - Verificar: `C[final] == C[hoje] + Σ(barras_projetadas)`
   - Verificar: continuidade na emenda real→projeção
   - Verificar: opacidades (real 0.85, projetado 0.35, banda 0.15)

9. **Visual & Regressão**
   - Sem degrau na emenda
   - Cores não conflitam
   - DayRangePicker não afeta tendência/meta
   - Outros elementos dashboard não afetados
