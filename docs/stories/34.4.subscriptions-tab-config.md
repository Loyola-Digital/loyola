## 📖 Story

**Como** gestor de um projeto na Loyola Digital X,
**Quero** uma aba unificada "Assinaturas" no projeto onde eu conecto a credencial da Hotmart (client_id + client_secret),
**Para** que eu consiga, num passo seguinte, ver o dashboard de assinaturas/recorrência do produto escolhido — com a porta já aberta para o Kiwify no futuro.

> Esta story entrega a **camada de entrada** do Epic 34: o item de navegação, a rota da página e o painel de configuração da conexão (criar / ver status / desconectar). O **dashboard de métricas** em si é entregue em stories irmãs (service + rotas + UI de dashboard). Aqui o foco é: plugar a aba, criar a página, e construir o fluxo de conexão Hotmart no frontend reusando o padrão Mautic.

---

## ✅ Acceptance Criteria

1. **Item de navegação "Assinaturas":**
   - Adicionado a `PROJECT_SUBITEMS` em `components/layout/project-folder.tsx` com `{ label: "Assinaturas", href: "subscriptions", icon: CreditCard }`.
   - `CreditCard` importado de `lucide-react`.
   - Aparece na lista de módulos do projeto (junto a Minds, Ads, Vendas, etc.), navega para `/projects/{id}/subscriptions` e fica `secondary` (ativo) quando a rota casa.

2. **Rota da página:**
   - Criado `app/(app)/projects/[id]/subscriptions/page.tsx` (Client Component, `"use client"`).
   - Resolve `projectId` via `use(params)` (params é `Promise<{ id: string }>`, padrão das outras sub-rotas de projeto).
   - Renderiza título "Assinaturas" com ícone `CreditCard`.

3. **Painel de configuração (quando NÃO conectado):**
   - Formulário com dois campos: **Client ID** (`type="text"`) e **Client Secret** (`type="password"`, placeholder mascarado).
   - Botão "Conectar" que chama `useSetHotmartConnection`; estado de loading com spinner (`Loader2`).
   - Validação no front: bloqueia submit se `clientId` ou `clientSecret` vazios; `toast.error` explicando.
   - Sucesso → `toast.success("Hotmart conectado!")`; erro → `toast.error(mensagem do backend)`.
   - Texto auxiliar curto explicando que a credencial é por projeto e onde gerá-la na Hotmart.

4. **Estado conectado:**
   - Quando `useHotmartConnection().data.connected === true`, esconde o formulário e mostra um cabeçalho "conectado" (estilo `ConnectedHeader` do Mautic) com botão "Desconectar" que chama `useDeleteHotmartConnection`.
   - Desconectar → `toast.success("Hotmart desconectado")`; invalida as queries da conexão.
   - **Placeholder de dashboard:** abaixo do cabeçalho conectado, renderizar um placeholder ("Dashboard de assinaturas em breve" ou slot reservado) — o dashboard real entra na story irmã. Esta story NÃO implementa o dropdown de produto nem os KPIs.

5. **Seção Kiwify (stub):**
   - Bloco visualmente desabilitado (opacity reduzida, `disabled`/`pointer-events-none`) com label "Kiwify" e badge/texto "em breve".
   - Não dispara nenhuma chamada de rede; é puramente decorativo.

6. **Gating de permissão (guest read-only):**
   - `isAdmin = role !== null && role !== "guest"` via `useUserRole()`.
   - Guest: vê o status da conexão (conectado / não conectado) mas **não** vê o formulário de conexão nem os botões "Conectar" / "Desconectar" (ou eles vêm desabilitados com aviso). Alinha com o padrão do app (binário: guest=leitura, resto=edição).

7. **Hooks de conexão (`lib/hooks/use-hotmart.ts`):**
   - `useHotmartConnection(projectId)` → GET `/api/projects/:projectId/hotmart/connection` → `{ connected }`.
   - `useSetHotmartConnection(projectId)` → PUT `/api/projects/:projectId/hotmart/connection`, body `{ clientId, clientSecret }`; invalida a query de conexão no sucesso.
   - `useDeleteHotmartConnection(projectId)` → DELETE `/api/projects/:projectId/hotmart/connection`; invalida a query de conexão.
   - Padrão idêntico ao `use-mautic.ts`: `useApiClient`, React Query, `staleTime` ~5min, `useQueryClient().invalidateQueries`.

8. **Loading / erro:**
   - Enquanto a conexão carrega: `Skeleton` (padrão Mautic/sales).
   - Erro de carregamento da conexão: mensagem de erro discreta com botão "Tentar de novo" (`refetch`).

9. **Segurança:**
   - O `clientSecret` nunca é renderizado de volta nem logado; o GET de conexão devolve só `{ connected }` (sem segredo).
   - Campo de secret sempre `type="password"`.

### Contrato de rotas consumidas (backend — fora do escopo desta story)

Apenas para referência (implementadas na story de backend de conexão):
- `GET    /api/projects/:projectId/hotmart/connection` → `{ connected }`
- `PUT    /api/projects/:projectId/hotmart/connection` → body `{ clientId, clientSecret }` (valida batendo no token endpoint antes de salvar; cripto + upsert; bloqueia guest)
- `DELETE /api/projects/:projectId/hotmart/connection` → remove (bloqueia guest)

### UI / Design system

- shadcn/ui (`Button`, `Input`, `Label`, `Skeleton`, `Select` se necessário), `lucide-react`, `sonner` (toast).

- Ícone principal: `CreditCard`.

### Dependências

Cadeia de backend que sustenta esta story (front): **34.1 → 34.2 → 34.3 → 34.4**.

- **Story 34.1 (schema + migration)** — cria a tabela `hotmart_connections`. NÃO entrega rotas.
- **Story 34.2 (service / cripto)** — service `services/hotmart.ts` (validação de credencial + criptografia/upsert).
- **Story 34.3 (rotas)** — **dependência direta de backend** desta story. Entrega as 3 rotas `connection` (GET/PUT/DELETE `/api/projects/:projectId/hotmart/connection`) que esta story (34.4) consome. O teste E2E de conexão real depende da 34.3 pronta.

Front (34.4) pode ser desenvolvido em paralelo (hooks degradam graciosamente), mas o consumo real das rotas `connection` depende da **34.3**, não da 34.1.
