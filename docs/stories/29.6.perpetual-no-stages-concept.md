# Story 29.6 — Perpetual: sem conceito de stages (1 dashboard só)

**Epic:** 29
**Status:** Ready for Review
**Points:** 3
**Depende de:** —

## User Story

Como usuário criando um funil de tipo **perpétuo**, quero ir direto pro dashboard sem precisar criar/escolher "etapas" (Captação/Paga/Vendas/CPL) — em produto perpétuo essa noção não faz sentido, é uma operação contínua única.

## Acceptance Criteria

- [x] **AC1:** Ao criar um funil com `type='perpetual'` via `POST /api/projects/:id/funnels`, o backend automaticamente cria 1 stage default:
  - `name`: igual ao nome do funnel (ex: "Vendas Perpétuas")
  - `stageType`: `"paid"`
  - `metaAccountId`: igual ao do funnel
  - `campaigns`: igual ao do funnel
  - `googleAdsAccountId`: igual ao do funnel
  - `googleAdsCampaigns`: igual ao do funnel
  - `switchyFolderIds`, `switchyLinkedLinks`: iguais ao do funnel
  - `sortOrder`: 0
- [x] **AC2:** Para funnels com `type='launch'`, comportamento atual preservado (nenhuma stage auto-criada).
- [x] **AC3:** Na funnel page (`[funnelId]/page.tsx`), botão **"Nova Etapa"** é escondido quando `funnelType === "perpetual"`.
- [x] **AC4:** Na funnel page, o dialog de criação de etapa também não é renderizado quando perpetual (defesa em profundidade).
- [x] **AC5:** Auto-redirect para a única stage segue funcionando (já existe) — usuário cai direto no dashboard.
- [x] **AC6:** Na stage page (`stages/[stageId]/page.tsx`), quando `funnelType === "perpetual"`, o `<h1>` exibe `funnel.name` (não `stage.name`), e o subtítulo mostra apenas "Funil Perpétuo" sem mencionar etapa.
- [x] **AC7:** Perpetuals com stages já existentes (legados) continuam mostrando a lista de stages — comportamento atual preservado.
- [x] **AC8:** Typecheck passa em api/web.

## Dev Notes

### Por que stage default em vez de remover stages?

Reusa toda a infra existente (tabs Meta Ads / YouTube / Survey / Spreadsheets / Switchy / Lead Scoring / Orgânicos) sem duplicar 470 linhas na funnel page. Stage default é invisível pro usuário — ele acha que está no "funil" direto.

### Por que "paid" como stageType default?

Pra perpetual, o tipo da stage é meramente cosmético (filtra algumas seções condicionais). "paid" desbloqueia o `StageSalesSection` e o seletor de planilha de captação que algumas integrações usam.

### Por que mostrar funnel.name no h1 (AC6)?

Eliminar qualquer pista visual de que existe uma "stage" debaixo do capô. Usuário não precisa saber.

### Risco: perpetuals criados ANTES desta story

Sem a auto-criação, perpetuals legados que não têm nenhuma stage vão continuar caindo na funnel page (sem auto-redirect porque stages.length === 0). Aceitável — usuário pode criar manualmente uma stage, ou abrimos story de migração futura.
