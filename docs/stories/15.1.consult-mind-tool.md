**Epic:** EPIC-15 Cross-Mind Consultation
**Status:** Draft
**Priority:** High
**Estimate:** L

---

## Story

Como Mind do Loyola X, quero poder consultar outros Minds durante uma conversa, para trazer perspectivas e conhecimentos de diferentes especialistas na minha resposta.

## Contexto Tecnico

### Arquitetura Existente

- `chat-tools.ts` define tools disponiveis (clickup, instagram_metrics, get_past_conversations)
- `claude.ts` executa `agentLoop()` que chama tools e reenvia resultados
- `mind-engine.ts` / `prompt-builder.ts` monta system prompts por Mind
- `mind-registry.ts` tem catalogo de todos os Minds com metadados

### Nova Tool: consult_mind

```typescript
{
  name: "consult_mind",
  description: "Consulta outro Mind/especialista para obter perspectiva sobre um topico. Use quando o usuario mencionar outro Mind com /nome ou quando precisar de expertise complementar.",
  input_schema: {
    type: "object",
    properties: {
      mind_id: { type: "string", description: "ID do Mind a consultar" },
      question: { type: "string", description: "Pergunta ou topico para consultar" },
      context: { type: "string", description: "Contexto da conversa atual para o Mind consultado entender o cenario" }
    },
    required: ["mind_id", "question"]
  }
}
```

### Fluxo de Execucao

1. Claude decide usar tool `consult_mind` (automaticamente ou por `/mindName`)
2. Backend recebe `executeToolCall("consult_mind", { mind_id, question, context })`
3. Busca Mind no registry → monta system prompt (tier 2)
4. Chama `claude.stream()` com:
   - System prompt do Mind consultado
   - Mensagem: question + context
   - Sem tools (consulta simples, nao recursiva)
5. Coleta resposta completa
6. Retorna como tool result pro Claude principal
7. Mind principal integra a resposta na sua resposta final

### Protecao Anti-Loop

- Consulta nao pode usar tools (sem recursao)
- Max 3 consultas por mensagem
- Timeout de 30 segundos por consulta

### AC5: Resposta formatada
- [ ] Tool result inclui: mindName, mindSquad, response
- [ ] Formato: "Resposta de {mindName} ({squad}): {response}"

## Dev Notes

- Usar `claude.stream()` (nao `agentLoop`) pra consulta — sem tools, sem recursao
- O Mind consultado recebe tier 2 (identity + frameworks) — suficiente pra resposta curta
- Max tokens da consulta: 1000 (resposta curta e direta)
- A consulta nao persiste mensagens — é efemera
- Buscar Mind por ID ou por slug/nome normalizado (lowercase, sem acentos)
- Se Mind nao encontrado, retornar erro amigavel na tool result
