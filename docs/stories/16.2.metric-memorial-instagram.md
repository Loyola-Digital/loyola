## Pré-requisitos (extensões aditivas à 16.1 — não-breaking)

### Task 0.a — Criar `<MetricTooltip>` (wrapper-only)

**Por quê:** O `<MetricWithTooltip>` da 16.1 renderiza layout próprio (label + value + children). Não serve para envolver o `KpiCard` interno do Instagram (que tem design autoral: gradientes coloridos, ícones top-right, sub-texto). Precisamos de um wrapper que **apenas** adiciona o tooltip ao redor de qualquer children, sem layout próprio.

**Novo arquivo:** `packages/web/components/metrics/metric-tooltip.tsx`

```tsx
"use client";
import type { ReactNode } from "react";
import { Tooltip, TooltipContent, TooltipTrigger } from "@/components/ui/tooltip";
import { FormulaTooltipContent } from "@/components/metrics/formula-tooltip-content";
import type { MetricFormula } from "@/lib/types/metric-formula";

interface MetricTooltipProps {
  label?: string;
  value?: string | number;
  formula?: MetricFormula;
  children: ReactNode;
  side?: "top" | "right" | "bottom" | "left";
}

/** Wraps any content with a formula tooltip. Does NOT render its own layout. */
export function MetricTooltip({ label, value, formula, children, side = "top" }: MetricTooltipProps) {
  if (!formula) return <>{children}</>;
  return (
    <Tooltip>
      <TooltipTrigger asChild>{children}</TooltipTrigger>
      <TooltipContent side={side} className="p-0">
        <FormulaTooltipContent label={label} value={value} formula={formula} />
      </TooltipContent>
    </Tooltip>
  );
}
```

### Task 0.b — Estender `<FormulaChartTooltip>` para per-series formulas

**Por quê:** Hoje o componente só lê `payload[i].payload.formula` (singular, comum a todas as séries do mesmo ponto). Para reach-chart com múltiplas séries (reach/impressions/engaged), cada série precisa de fórmula própria.

**Mudança retrocompatível:** Adicionar lookup `formulasByKey[entry.dataKey]` **antes** do fallback `formula`:

```tsx
// Em formula-chart-tooltip.tsx, dentro do for-of loop:
const byKey = (entry.payload?.formulasByKey as Record<string, MetricFormula> | undefined);
const candidate =
  (entry.dataKey && byKey?.[String(entry.dataKey)]) ??
  entry.payload?.formula;
```

**RechartsPayloadEntry interface** ganha opcional:
```tsx
payload?: Record<string, unknown> & {
  formula?: MetricFormula;
  formulasByKey?: Record<string, MetricFormula>;
};
```

Comportamento antigo (`formula` singular) continua funcionando.

## Acceptance Criteria

### AC1: 9 KPIs do Instagram com tooltip de memorial — **sem regressão visual**
- [ ] Cada card em `overview-cards.tsx` permanece com seu design autoral (gradientes, borders coloridos, ícone, sub-texto)
- [ ] Cada card é envolvido por `<MetricTooltip formula={...}>` (não substituído por `<MetricWithTooltip>`)
- [ ] Fórmula, valores e source batem com a tabela acima
- [ ] Source string segue padrão `Instagram Graph API · {campo}` (exceto "Engajamento" → `Derivado (total_interactions ÷ reach)`)
- [ ] Período exibido no tooltip usa `period.since` / `period.until` formatados em pt-BR (`dd/MM`)

### AC2: Reach chart com memorial por série (LineChart)
- [ ] `reach-chart.tsx` usa `<FormulaChartTooltip>` no prop `content` do `<Tooltip>`
- [ ] Data items enriquecidos com `formulasByKey: { reach, impressions?, engaged? }` — uma fórmula por série presente
- [ ] Hover em um ponto mostra N memoriais empilhados (um por série ativa)
- [ ] Cada memorial mostra o valor daquela série naquele dia + fonte
- [ ] Tooltip atual (estilo inline em `contentStyle`) é removido — substituído pelo custom

### AC3: Helper centralizado de fórmulas
- [ ] Novo arquivo `packages/web/lib/formulas/instagram.ts`
- [ ] Exporta factory functions puras (sem side-effects): `buildFollowersFormula(...)`, `buildReachFormula(...)`, `buildEngagementFormula(...)`, etc.
- [ ] Cada factory recebe `{ rawValue(s), period: { since, until } }` e retorna `MetricFormula`
- [ ] Usa `Intl.NumberFormat("pt-BR")` para formatação de números dentro do memorial
- [ ] Usa `date-fns` + locale `ptBR` para formatação do período

### AC4: Graceful fallback
- [ ] Se `isLoading` ou dado ausente: card não renderiza tooltip (omite `formula` prop) — MetricTooltip já cai no passthrough
- [ ] Engajamento com `reach=0`: não mostra tooltip (divisão por zero evitada)
- [ ] Reach chart sem data: continua mostrando "Sem dados disponíveis" (sem tooltip)

### AC5: Sem regressão funcional
- [ ] Filtro de período continua funcionando — refetch atualiza valores E fórmulas
- [ ] Dark mode: tooltip herda tema automaticamente (já testado na 16.1)
- [ ] Valores numéricos formatados exatamente como antes nos cards (fmtNumber existente preservado)

### AC6: Demo page atualizada
- [ ] Adicionar seção em `/dev/metrics-demo/page.tsx` demonstrando `<MetricTooltip>` envolvendo um card customizado (simulando KpiCard do Instagram)
- [ ] Demonstrar LineChart com `formulasByKey` e múltiplas séries
