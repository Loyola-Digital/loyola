## Story

**As a** Loyola Digital employee,
**I want** a conversations view scoped to a specific project,
**so that** I can manage client conversations without mixing them with other clients.

---

## Acceptance Criteria

1. [ ] Rota `/projects/[id]/conversations` criada — lista conversas do projeto
2. [ ] Ao criar nova conversa dentro de `/projects/[id]/conversations`: `conversation.project_id` = ID do projeto
3. [ ] Conversa criada no contexto do projeto aparece no sidebar do projeto, não na lista global
4. [ ] Página global `/conversations` continua mostrando todas as conversas (project_id = null ou qualquer valor)
5. [ ] Breadcrumb: "Projetos > [Nome do Cliente] > Conversas"
6. [ ] Loading state e empty state por projeto
7. [ ] `useConversations(projectId?)` hook aceita `projectId` opcional
8. [ ] Backend: `GET /api/projects/:id/conversations` retorna conversas filtradas por `project_id`
9. [ ] Backend: `POST /api/chat` quando chamado com `project_id` no header ou body: salva `conversation.project_id`

Ao iniciar chat dentro do contexto do projeto:
