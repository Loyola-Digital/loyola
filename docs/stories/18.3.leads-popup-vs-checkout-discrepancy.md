# Story 18.3: Investigar Discrepância Funil — Leads Popup > Visita Checkout

## Descrição

No **Funil de Conversão** (dashboard Captação Paga), o card **Visita Checkout** está exibindo um número MENOR que o card **Leads Popup**, o que logicamente não faz sentido:

- Leads Popup = pessoas que clicaram no popup de leads
- Visita Checkout = pessoas que chegaram na página de checkout
- **Expectativa:** Visita Checkout ≥ Leads Popup (pois vem depois no funil)

**Impacto:** Gestor vê discrepância confusa, não consegue interpretar a qualidade do funil.

## Contexto

- Aba: Meta Ads 2 (dashboard Captação Paga no funil)
- Componente: `funnel-card.tsx` ou similar que exibe os cards do funil
- Fonte de dados: Planilha mapeada de leads (Leads Popup) + eventos de checkout
- Possível causa: Filtro de data, mapeamento de coluna, ou diferença em data de registro (Leads vs. Checkout)

## Acceptance Criteria

### AC1 — Ordem lógica do funil
- [ ] Visita Checkout ≥ Leads Popup (sempre)
- [ ] Se não atender, bug está encontrado e documentado

### AC2 — Fonte de dados validada
- [ ] Leads Popup conta pessoas que **clicaram no popup** (evento rastreado corretamente)
- [ ] Visita Checkout conta pessoas que **chegaram no checkout** (evento rastreado corretamente)
- [ ] Ambas usam **mesmo filtro de data** e **mesmo UTM_source**

### AC3 — Mapeamento de coluna verificado
- [ ] Coluna da planilha de "Leads" mapeia corretamente para campo do schema
- [ ] Coluna de "Checkout" mapeia corretamente
- [ ] Sem duplicação ou exclusão acidental de registros

### AC4 — Relatório de achado
- [ ] Se discrepância confirmada: documentar causa raiz (em comentário da story)
- [ ] Apontar qual é o filtro errado ou qual coluna está mal mapeada

## Scope

### ✅ IN
- Revisar lógica de contagem de Leads Popup
- Revisar lógica de contagem de Visita Checkout
- Validar mapeamentos de coluna na planilha
- Validar filtro de data em ambas as contagens
- Documentar discrepância encontrada

### ❌ OUT
- Implementar novos eventos de rastreamento (out of scope de esta story)
- Mudar fonte de dados (já mapeada)
- Corrigir dados históricos (future story, se needed)

## Pistas de Debugging

1. **Componente:** Procure pelo componente que exibe os cards do funil (`leads-popup-card`, `checkout-card`, etc.)
2. **Hook de dados:** Qual hook busca dados de Leads Popup? Qual busca Checkout?
3. **Query na API:** Verifique query que conta registros — está filtrada corretamente?
4. **Possíveis culpados:**
   - Leads Popup inclui duplicatas?
   - Checkout exclui registros de uma certa data por acidente?
   - Diferentes filtros de UTM/data entre os dois cards?
   - Timestamp vs. date — inconsistência de fuso horário?
