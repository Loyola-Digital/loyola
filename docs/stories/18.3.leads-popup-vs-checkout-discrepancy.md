# Story 18.3: Investigar Discrepância Funil — Leads Popup > Visita Checkout

## Descrição

No **Funil de Conversão** (dashboard Captação Paga), o card **Visita Checkout** está exibindo um número MENOR que o card **Leads Popup**, o que logicamente não faz sentido:

- Leads Popup = pessoas que clicaram no popup de leads
- Visita Checkout = pessoas que chegaram na página de checkout
- **Expectativa:** Visita Checkout ≥ Leads Popup (pois vem depois no funil)

**Impacto:** Gestor vê discrepância confusa, não consegue interpretar a qualidade do funil.

## Contexto

- Aba: Meta Ads 2 (dashboard Captação Paga no funil)
- Componente: `funnel-card.tsx` ou similar que exibe os cards do funil
- Fonte de dados: Planilha mapeada de leads (Leads Popup) + eventos de checkout
- Possível causa: Filtro de data, mapeamento de coluna, ou diferença em data de registro (Leads vs. Checkout)

## Riscos & Blockers

- **Risco Alto:** Problema pode estar em 3 camadas diferentes (componente React, hook de dados, API, ou query no banco)
  - **Mitigação:** Começar de cima (componente) e descer até API/DB — validar schema DB PRIMEIRO
  - **Impacto:** Se diagnosticar errado, pode gastar tempo no lugar errado

- **Blocker Potencial:** Se problema estiver em schema DB (coluna faltante ou mapeamento errado)
  - **Mitigação:** Validar schema DB (`packages/api/src/db/schema.ts`) ANTES de investigar lógica
  - **Impacto:** Pode descobrir que precisa de migration de DB ou ajuste de query

- **Risco Médio:** Diferentes filtros de data/UTM entre os dois cards (Leads Popup vs. Checkout)
  - **Mitigação:** Validar que ambas as contagens usam MESMO período de data e MESMO filtro de UTM
  - **Impacto:** Se descasados, discrepância é esperada (não é bug, é filtro)

### Investigation Log — Phase 1: Code Tracing (2026-04-30)

**Objetivo:** Rastrear onde Leads Popup e Visita Checkout são contados, identificando a origem da discrepância.

**Camada 1: Componente React**
- ✅ Encontrado: `packages/web/components/funnels/conversion-funnel.tsx` exibe os cards do funil
- ✅ ConversionFunnel recebe `leads={metrics.totalLeads}` e `checkoutVisits={metrics.checkoutVisits}`
- Renderização: ambos são passados corretamente do LaunchDashboard

**Camada 2: Hook de Dados**
- ✅ Encontrado: `packages/web/lib/hooks/use-crossed-funnel-metrics.ts` é a fonte única de verdade
- ✅ `totalLeads` = soma de (leadsPagos + leadsOrg + leadsSemTrack) — contados da **planilha de leads**
- ✅ `checkoutVisits` = `totals.checkoutInitiations` — contados do **Meta Ads API** (events)
- **ACHADO CRÍTICO:** Duas fontes diferentes com metodologias diferentes!

**Camada 3: Cálculo de Métricas**
- `useCrossedFunnelMetrics` combina:
  1. **metaData:** Dados diários agregados do Meta Ads (múltiplas campanhas, chamada em `useQueries`)
  2. **sheetData:** Dados da planilha de leads (planilha vinculada)
  3. Ambos são agregados por data com `buildDailyRows()`

**Fontes de Dados Identificadas:**
- **Leads:** `aggregateSpreadsheetByDate()` — e-mails **deduplicated** por categoria (pagos/org/sem-track)
- **Checkout:** `getCheckoutValue()` extraído de `row.actions[]` da Meta API — eventos **brutos** (não dedup)

**ACHADO CRÍTICO — Filtro de Campanha:**
- ✅ Meta Ads API: Filtra por `funnel.campaigns` (apenas campanhas linked)
  - Route: `/api/traffic/analytics/:projectId/campaign-daily?campaignId=X`
  - Cada query é por campaign ID específico
- ❌ Spreadsheet: **NÃO filtra por campaign ID**
  - Route: `/api/projects/:projectId/funnels/:funnelId/spreadsheets/:id/data`
  - Retorna TODOS os dados da planilha (sem filtro de campanha)
  - Apenas filtra por `funnelId` + `stageId` (se fornecido)

**Conclusão Diagnóstica:**
A discrepância ocorre porque:
1. Leads são contados de **toda a planilha** (potencialmente incluindo fontes fora do funil)
2. Checkout events são contados apenas de **campanhas linked ao funil**
3. Se planilha tem leads de campanha X, mas funnel.campaigns contém campanha Y, ocorre mismatch

**Hipótese Principal:** Planilha vinculada ao funil contém leads de origem diferente das campanhas linked (ex: leads de Google Ads enquanto checkout vem de Meta Ads).

**Próximo Passo (Phase 2):** 
Validar via dashboard qual é a fonte de utm_source dos leads na planilha vs. quais campanhas estão linked ao funil.

### Recomendação de Fix

**Opção A (Recomendada):** Filtrar spreadsheet por campaign ID
- Modificar `useCrossedFunnelMetrics` para passar `funnel.campaigns` ao buscar dados da planilha
- Garantir que apenas leads das campanhas linked sejam contados
- Alinhar metodologia de ambas as fontes

**Opção B (Alternativa):** Manter ambas unfiltered
- Remover filtro de campaign ID da Meta API query
- Mostrar série completa de leads + checkout de todas as fontes
- Trade-off: Perder isolamento por campanha

**Impacto:** Fix vai direto na interface do LaunchDashboard → precisa coordenar com Story 18.1 (Meta Ads 2 base) e Story 18.4 (Hot/Cold).

### Phase 2: Implementation Decision (2026-04-30)

**Opção A escolhida:** Filtrar spreadsheet por campaign ID

**Plano de implementação:**
1. Hook `useCrossedFunnelMetrics` deve passar `funnel.campaigns` ao buscar dados da planilha
2. Endpoint de spreadsheet (`/api/projects/.../funnels/.../spreadsheets/:id/data`) deve aceitar query param `campaignIds` e filtrar registros
3. Após mudança: Leads Popup (spreadsheet filtrado) ≤ Checkout Visits (Meta API — já filtrado por campaign)
4. Alinhamento de metodologia garantido
