# Story 18.4: Ativar Distribuição Hot/Cold Buyers com utm_term

## Descrição

O card **Distribuição de Compradores (Hot/Cold)** está mapeado para usar a coluna **utm_term** da planilha de vendas (Captação), mas **nenhum dado aparece** no dashboard.

A coluna está mapeada no schema/banco, mas não está sendo exibida no gráfico.

**Impacto:** Gestor não consegue ver qual termo de pesquisa (utm_term) converteu em buyer hot vs. cold, impossível otimizar bids por termo.

## Contexto

- Aba: Meta Ads 2 (dashboard Captação Paga)
- Componente: Gráfico de distribuição Hot/Cold (donut/bar chart)
- Fonte: Coluna `utm_term` mapeada na planilha de vendas
- Problema: Coluna está mapeada, mas dado não chega ao gráfico ou é filtrado por engano

## Acceptance Criteria

### AC1 — Dados aparecem no gráfico
- [ ] Gráfico Hot/Cold Buyers exibe **distribuição por utm_term**
- [ ] Cada termo de pesquisa mostra: count buyers, % hot, % cold
- [ ] Sem valores vazios ou "(undefined)"

### AC2 — Mapeamento de coluna confirmado
- [ ] Coluna `utm_term` da planilha está corretamente mapeada para o schema do DB
- [ ] Query na API busca esta coluna e a agrupa
- [ ] Sem conflito de nome ou alias errado

### AC3 — Filtro de data aplicado
- [ ] Gráfico respeita filtro de data do dashboard
- [ ] Apenas dados dentro do intervalo aparecem

### AC4 — Dados validados
- [ ] Amostra de 5-10 termos com counts conferem vs. planilha original
- [ ] Sem duplicação ou registros perdidos

## Scope

### ✅ IN
- Revisar hook/query que alimenta gráfico Hot/Cold
- Validar se utm_term está sendo buscado na query
- Revisar se há filtro silencioso (null, empty string) excluindo dados
- Validar mapeamento de coluna no schema
- Testar com dados da planilha para conferir counts

### ❌ OUT
- Mudar visual do gráfico (já implementado)
- Adicionar novas dimensões além de utm_term
- Retroativar dados históricos não mapeados

## Pistas de Debugging

1. **Componente gráfico:** `packages/web/components/funnels/hot-cold-count-donut.tsx` (já existe!)
2. **Hook de dados:** Procure por `use-stage-hot-cold-buyers.ts` ou similar
3. **Query na API:** Verifique endpoint de vendas que retorna buyers por utm_term
4. **Possíveis culpados:**
   - utm_term não está sendo buscado na SELECT da query?
   - Há um WHERE que filtra registros com utm_term null/empty?
   - Hook mapeia dado errado (utm_source em vez de utm_term)?
   - Coluna na planilha tem nome diferente e mapeamento está desatualizado?
