## Story

**As a** developer on the Loyola Digital X project,
**I want** a MindRegistry service that scans the `squads/` directory on startup, indexes all minds and agents with metadata, and exposes REST endpoints for the catalog,
**so that** the frontend can display the mind catalog and the Mind Engine (Story 1.2.1) can locate artifacts for prompt building.

### API Contract (Architecture Doc Section 8)

**GET /api/minds Response:**

```typescript
{
  squads: [
    {
      id: string,
      name: string,
      displayName: string,
      description: string,
      mindCount: number,
      minds: MindSummary[]
    }
  ]
}
```

**GET /api/minds/:mindId Response (MindDetail):**

```typescript
{
  id, name, squad, specialty, tags, avatarUrl, totalTokenEstimate,
  bio: string,                     // First ~500 chars of COGNITIVE_OS
  frameworks: string[],            // Extracted framework names
  communicationStyle: {
    tone: string,
    vocabulary: string[],
    forbiddenWords: string[]
  },
  stats: {
    artifactCount: number,
    heuristicCount: number,
    conversationCount: number      // From DB
  }
}
```

```typescript
interface MindMetadata {

  heuristicPaths: string[];
  totalTokenEstimate: number;
}
```

### Performance Targets

- Registry initialization: < 500ms for 27+ minds at startup
- `GET /api/minds`: < 200ms (in-memory, no DB)
- `GET /api/minds/:mindId`: < 300ms (registry + 1 DB query for stats)

### Key Implementation Notes

1. **Minds NAO estao no banco** — filesystem e a source of truth (version-controlled .md files)
2. **conversations.mind_id** e TEXT, nao FK — referencia o MindRegistry, nao uma tabela
3. **Nao carregar conteudo dos artifacts** durante scan — apenas indexar paths e sizes
4. **MMOS vs Content-Engine** tem estruturas completamente diferentes — scanner deve detectar automaticamente
5. **Dopamine Learning** tem path nested: `dopamine-learning/dopamine-learning/agents/`
6. **config.json** pode nao existir em todas as minds — tratar como opcional
