## Contexto

A tabela **Desempenho de Criativos** atualmente divide cada criativo em duas linhas: uma para dados **Hot** (período selecionado) e outra para **Cold** (dados históricos compilados). Quando o usuário seleciona "Todos" nos três filtros disponíveis, essa divisão gera ruído e redundância visual.

**Requisito:** Quando "Todos" estiver selecionado nos 3 filtros, compilar valores (somar) por `ad_name` e exibir uma única linha por criativo, ao invés de dividi-lo em Hot/Cold.

## Notas Técnicas

### Compilação de Métricas

**Soma simples:**
- `spend`, `impressions`, `clicks`, `conversions`

**Média aritmética:**
- `cpc`, `cpm`, `roi` (são médias diretas dos sub-valores)

**Média ponderada (por impressões):**
- `ctr` = sum(clicks) / sum(impressions)
- `cvr` = sum(conversions) / sum(impressions)

**ROAS:**
- ROAS_compilado = sum(revenue) / sum(spend)

### Estrutura de Dados

```typescript
// Raw (da API)
interface CreativeMetric {
  ad_name: string;
  variant: 'hot' | 'cold'; // Será ignorado no modo "Todos"
  spend: number;
  impressions: number;
  // ... outras
}

// Compilado (modo "Todos")
interface CompiledCreativeMetric {
  ad_name: string;
  spend: number; // sum
  impressions: number; // sum
  ctr: number; // weighted avg
  // ... outras
}
```
