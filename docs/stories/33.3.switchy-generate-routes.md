## 📖 Story

**Como** gestor de tráfego do projeto,
**Quero** rotas de API que persistam as configurações Switchy do projeto (pixels + defaults), gerenciem os presets de canal e gerem em lote os shortlinks rastreados via Switchy,
**Para** que a aba "Switch" (nível projeto) consiga, a partir de 1 link de checkout cru + folder + utm_campaign, produzir N shortlinks (um por canal) com os pixels do projeto e guardar histórico.

---

## ✅ Acceptance Criteria

1. **Settings — GET** `GET /api/projects/:projectId/switchy/settings`
   - Retorna `{ pixels, showGdpr, defaultUtmTerm, defaultUtmContent }`.
   - Se o projeto ainda não tem linha em `project_switchy_settings`, retorna defaults vazios: `{ pixels: [], showGdpr: false, defaultUtmTerm: null, defaultUtmContent: null }` (NÃO cria a linha no GET).
   - Acessível por guest (leitura).

2. **Settings — PUT** `PUT /api/projects/:projectId/switchy/settings`
   - Faz **upsert** (insert ou update) da linha por `project_id` (UNIQUE).
   - Body validado por zod: `pixels` array de `{ platform: "facebook" | "gtm", value: string, title?: string }`, `showGdpr` boolean, `defaultUtmTerm` string|null, `defaultUtmContent` string|null.
   - **Bloqueia guest** → `403 { error: "Acesso negado" }`.
   - Retorna o objeto persistido no mesmo shape do GET.

3. **Presets — GET** `GET /api/projects/:projectId/switchy/presets`
   - Lista os presets do projeto ordenados por `sort_order` asc, depois `created_at` asc.
   - **Seed lazy:** se o projeto ainda não tem nenhum preset, semeia os 7 defaults (bio, direct, stories, manychat, chatwoot, email, grupo) e retorna a lista recém-criada.
   - Acessível por guest (leitura).

4. **Presets — POST** `POST /api/projects/:projectId/switchy/presets`
   - Cria 1 preset: `{ label, utmMedium, utmSource, sortOrder?, enabled? }`.
   - **Bloqueia guest** → 403.
   - Retorna o preset criado.

5. **Presets — PUT** `PUT /api/projects/:projectId/switchy/presets/:presetId`
   - Atualiza campos do preset (label, utmMedium, utmSource, sortOrder, enabled) — todos opcionais (partial update).
   - 404 se o preset não existe **ou** não pertence ao `projectId` da rota.
   - **Bloqueia guest** → 403.

6. **Presets — DELETE** `DELETE /api/projects/:projectId/switchy/presets/:presetId`
   - Remove o preset. 404 se não existe ou não pertence ao projeto.
   - **Bloqueia guest** → 403.

7. **Generate (lote) — POST** `POST /api/projects/:projectId/switchy/generate`
   - Body: `{ checkoutUrl, folderId, folderName?, campaign, term?, content?, channels: [{ label, medium, source }] }`.
   - `campaign` é **obrigatório** (zod min(1)). `channels` deve ter ≥ 1 item.
   - Carrega os `pixels` e `showGdpr` do projeto (de `project_switchy_settings`; se não houver, pixels=[]/showGdpr=false).
   - Para **cada canal**, na ordem recebida:
     - `buildTrackedCheckoutUrl({ baseUrl: checkoutUrl, campaign, medium, source, term, content })` → `fullUrl`.
     - `createSwitchyLink(token, { url: fullUrl, folderId, pixels, showGDPR })` → short url + id/uniq.
     - Persiste 1 linha em `switchy_shortened_links` (independente de sucesso/erro? — ver Dev Notes: persistir somente em sucesso; erro entra no `results` mas NÃO grava linha sem `full_url`/`short_url` válido… `full_url` SEMPRE existe pois é montado localmente, então grava sempre; `short_url`/`switchy_link_id` ficam null no erro).
   - **Falha por-canal não derruba o batch:** captura erro por canal, registra `error` no item de resultado e segue para o próximo (parcial OK).
   - **Bloqueia guest** → 403.
   - Retorna `{ results: [{ label, medium, source, fullUrl, shortUrl, switchyLinkId, error? }] }`.

8. **History — GET** `GET /api/projects/:projectId/switchy/links/history`
   - Retorna os últimos links gerados do **nosso banco** (`switchy_shortened_links` do projeto), ordenados por `created_at` desc, com limite (default 50, query `?limit=` opcional, máx 200).
   - Acessível por guest (leitura).

9. **Padrões transversais**
   - Todas as rotas reusam `getProjectAccess` e respondem `404 { error: "Projeto não encontrado" }` se sem acesso.
   - `projectId`/`presetId` inválidos → `400 { error: "ID inválido" }`.
   - Body inválido → `400 { error: "Dados inválidos", details: ... }`.
   - Erros do Switchy (createSwitchyLink) por-canal viram `error` no `results` (200 no batch); falha de infra geral → `500`. Erros logados via `fastify.log.error(...)` nos catches.
   - Reusa o `fp` plugin existente em `packages/api/src/routes/switchy.ts` (NÃO criar novo plugin/arquivo de rotas).

## ⚠️ Risks & Mitigations

| Risk | Severity | Mitigation |
|------|----------|-----------|
| **Shape da response do `createSwitchyLink` indefinido** (Switchy REST não 100% documentado) | HIGH | Service (33.2) trata defensivamente e normaliza (aceita `{link:{...}}` ou flat). Esta rota apenas consome o retorno normalizado `{ shortUrl, switchyLinkId, uniq }`; se o service lançar, vira `error` no item do `results`. |
| **Falha de 1 canal derrubar o batch inteiro** | HIGH | `try/catch` por canal dentro do loop. Erro vira string no campo `error` do resultado; batch continua. Resposta 200 com resultados parciais. |
| **Seed de presets duplicado** (race em 2 requests simultâneos) | MEDIUM | Checar `count` antes de semear; aceitar que, em corrida rara, possa haver dupla seed — mitigar fazendo o insert dos 7 numa única operação e relendo. Não há UNIQUE em label, então duplicata é tolerável e editável pelo usuário. Documentar como aceitável. |
| **Upsert de settings sem UNIQUE funcional** | MEDIUM | `project_id` é UNIQUE (definido em 33.1). Usar `onConflictDoUpdate` no `project_id`. Se a constraint não existir, fallback select+update. |
| **Token global ausente em runtime** | MEDIUM | Reusar `getSwitchyToken()` existente, que já lança erro claro; catch no handler retorna 502/500 com mensagem. |
| **Guest conseguir escrever** | HIGH | Checagem `request.userRole === "guest"` → 403 no início de cada handler de escrita (mesmo padrão de `funnels.ts`). |
