**Epic:** 32
**Status:** Ready for Review

## Story

Pivot da aba Mautic: em vez de vincular 1 campanha por etapa, a aba abre já mostrando um **dashboard geral de emails**:
- Por padrão mostra os emails do funil (match pelo código no nome, ex.: `dg-pg02`/`fz-l2` — não existe campo "tag" no Mautic; o código vive no nome `[dg-pg02-abr-26]`).
- Toggle "Todos os emails" pra ver/incluir os sem o código.
- **Ranking por performance** (taxa de clique/abertura, enviados, etc.).
- **Filtro por email** (busca por nome).
- Métricas por email: enviados, abertura%, clique%, bounces, descadastros.
- Resumo agregado no topo (emails, enviados, abertura média, clique médio).

## Acceptance Criteria

- [x] Backend lista TODOS os emails (`/api/emails` paginado) com sent/opens.
- [x] Enriquece cada email com cliques (channel_url_trackables.unique_hits), bounces/descadastros (lead_donotcontact) via `/api/stats`, com concorrência (6) e fallback gracioso.
- [x] Endpoint `GET .../mautic-emails` retorna `{ matchToken, emails[], statsAvailable }`.
- [x] UI: dashboard com filtro funil/todos, busca, sort, ranking (troféu no #1 por taxa), resumo agregado.
- [x] Typecheck ✅ + Lint ✅ (api + web). Validado contra Mautic real (26 emails, cliques OK).

## Dev Notes

- A "tag" do funil = código no nome do email. matchToken = 2 primeiros segmentos do nome do funil.
- Mantidos (sem uso na UI) os endpoints de campanha por etapa da 32.1 — podem ser removidos depois.
- `/api/stats` exige admin; sem isso, cliques/bounces ficam null e a UI avisa.
