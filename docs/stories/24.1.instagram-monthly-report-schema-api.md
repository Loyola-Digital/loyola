# Story 24.1 — Schema & Endpoint de Geração: Instagram Monthly Report

## Status

Done

## Story

**As an** analista do Loyola Digital X,
**I want** um endpoint que agrega 1 mês completo de métricas de Instagram em um snapshot persistido em DB,
**so that** as stories 24.2 (página HTML) e 24.3 (trigger + histórico) tenham uma fonte consistente de dados que não dependa de re-fetchar a Meta a cada visualização.

## Acceptance Criteria

### Database Schema

1. Tabela `instagram_monthly_reports` em `packages/api/src/db/schema.ts`:
   - `id` (uuid, PK, defaultRandom)
   - `project_id` (uuid, FK → `projects.id`, ON DELETE CASCADE, NOT NULL)
   - `month` (varchar(7), NOT NULL) — formato `YYYY-MM` (ex: `"2026-04"`)
   - `data` (jsonb, NOT NULL) — snapshot completo do relatório
   - `generated_by` (uuid, FK → `users.id`, NOT NULL)
   - `generated_at` (timestamptz, default NOW(), NOT NULL)

2. Constraints e índices:
   - `UNIQUE (project_id, month)` — 1 relatório por projeto por mês (regenerar substitui via upsert)
   - `INDEX (project_id, generated_at DESC)` — pra listagem ordenada por recência

3. Migration aplicada em PROD via `node -e` com `pg` Pool. Migration file commitada em `packages/api/src/db/migrations/`.

### Snapshot data shape (JSONB)

4. Estrutura do `data`:
   ```typescript
   interface MonthlyReportData {
     month: string;                    // "2026-04"
     monthLabel: string;               // "Abril 2026"
     generatedAt: string;              // ISO
     periodStart: string;              // "2026-04-01"
     periodEnd: string;                // "2026-04-30"
     accounts: AccountReport[];        // 1 por conta IG vinculada ao projeto
     hasComparison: boolean;           // true se relatório do mês anterior existe
   }

   interface AccountReport {
     accountId: string;                // UUID interno
     instagramUsername: string;
     accountName: string;
     profilePictureUrl: string | null;

     totals: {
       postsPublished: number;
       reach: number;
       views: number;
       interactions: number;
       likes: number;
       comments: number;
       saves: number;
       shares: number;
     };

     followers: {
       startOfMonth: number;           // followers_count no início (best-effort)
       endOfMonth: number;             // followers_count no final
       gained: number;
       lost: number;
       net: number;
     };

     mediaDistribution: {
       reels: { count: number; reach: number };
       feed:  { count: number; reach: number };
     };

     dailyTrend: Array<{
       date: string;                   // "2026-04-15"
       followersDelta: number;         // gained - lost no dia
       reach: number;
     }>;

     topByEngagement: PostSummary[];   // Top 5
     bottomByEngagement: PostSummary[];// Bottom 5
     topByReach: PostSummary[];        // Top 5
     bottomByReach: PostSummary[];     // Bottom 5

     demographics: {                   // null se conta tem <100 seguidores ou Meta indisponível
       ageGender: Array<{ key: string; value: number }> | null;
       cities: Array<{ name: string; count: number }> | null;     // Top 5
       countries: Array<{ name: string; count: number }> | null;  // Top 5
     } | null;

     comparison: AccountReportDelta | null;  // null no 1º relatório, populado a partir do 2º
   }

   interface PostSummary {
     mediaId: string;
     mediaType: string;                // VIDEO/IMAGE/CAROUSEL_ALBUM
     mediaProductType: string | null;  // FEED/REELS
     timestamp: string;
     thumbnailUrl: string | null;
     permalink: string | null;
     caption: string | null;           // truncada a 200 chars
     reach: number | null;
     likes: number;
     comments: number;
     saves: number | null;
     engagementRate: number | null;
     follows: number | null;           // só vem em FEED
   }

   interface AccountReportDelta {
     followersNet: { current: number; previous: number; deltaPct: number | null };
     reach:        { current: number; previous: number; deltaPct: number | null };
     views:        { current: number; previous: number; deltaPct: number | null };
     interactions: { current: number; previous: number; deltaPct: number | null };
     postsPublished:{current: number; previous: number; deltaPct: number | null };
   }
   ```

### API Endpoints

5. `POST /api/projects/:projectId/reports/instagram/generate`
   - Body: `{ month: string }` (formato `YYYY-MM`)
   - Valida com Zod
   - Verifica `getProjectAccess(userId, projectId)`
   - Lista contas IG do projeto via `instagramAccountProjects`
   - Pra cada conta, agrega:
     - Lista de mídias publicadas no mês (filtra por `timestamp` ∈ [periodStart, periodEnd])
     - Pra cada mídia, lê insights (reusa cache do `getMediaInsights`)
     - Conta media types e soma reach por tipo
     - Account insights (reach, views, follows_and_unfollows com breakdown daily)
     - Demographics via `getAudienceDemographics`
   - Calcula top/bottom 5 por engagement e alcance
   - Procura relatório do mês anterior pra calcular `comparison`. Se não existir, `comparison: null`
   - Upsert em `instagram_monthly_reports` (substituí se já existe pra `(project_id, month)`)
   - Retorna o registro completo
   - Timeout estendido: 60s (Fastify default 30s pode não cobrir múltiplas contas)

6. `GET /api/projects/:projectId/reports/instagram/:reportId`
   - Retorna o registro completo (`data` + metadata)
   - Verifica `getProjectAccess`
   - 404 se reportId não existe ou não pertence ao projeto

7. `GET /api/projects/:projectId/reports/instagram`
   - Lista todos os relatórios do projeto, ordenados por `generated_at DESC`
   - Retorna `[{ id, month, generatedAt, generatedBy }]` (sem o `data` pesado)
   - Verifica `getProjectAccess`

8. Endpoints registrados em `app.ts` dentro de `fp()` closure.

## Dev Notes

### Reuso de helpers existentes

- `fastify.instagramService.getMediaList(accountId, 100)` — lista mídia (já enriquece com reach/saved/engagement_rate)
- `fastify.instagramService.getMediaInsights(mediaId, accountId, mediaType)` — insights por post (cache 15min)
- `fastify.instagramService.getAccountInsights(accountId, "day", since, until)` — daily breakdown
- `fastify.instagramService.getAudienceDemographics(accountId)` — demografia
- `fastify.instagramService.getProfile(accountId)` — followers_count atual

### Cálculo de start/end-of-month followers

- `endOfMonth` = `profile.followers_count` no momento da geração (snapshot)
- `startOfMonth` = `endOfMonth - net (do mês)`. Se relatório anterior existe, usa `previous.followers.endOfMonth` como sanity check.

### Comparison logic

- Para cada métrica em `AccountReportDelta`, `deltaPct = ((current - previous) / |previous|) × 100`. Se `previous === 0`, retorna `null` (não dá pra calcular % de zero).

### Migration

Padrão do projeto (per memory): `node -e` com `pg` Pool, NÃO drizzle-kit push.
