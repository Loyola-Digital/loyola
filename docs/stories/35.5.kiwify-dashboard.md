## 📖 Story

**Como** usuário com acesso a um projeto da Loyola Digital X,
**Quero** um dashboard de recorrência da Kiwify por produto recorrente escolhido num dropdown, com filtro de período,
**Para** acompanhar receita recorrente, cobranças (pagas/reembolsadas/chargeback/pendentes), MRR aproximado e novos vs renovações — com clareza sobre o que a API da Kiwify **não** entrega.

---

## ✅ Acceptance Criteria

1. **`KiwifySubscriptionsDashboard`** (`components/subscriptions/kiwify-subscriptions-dashboard.tsx`) montado quando a conexão Kiwify está ativa (ponto de montagem da 35.4).

2. **Dropdown de produto recorrente** (reusa `product-picker.tsx`) populado por `useKiwifyProducts`. **Filtro de período** (reusa `period-picker.tsx`, default 12 meses). Dashboard só busca quando há `productId` selecionado.

3. **KPIs honestos** (reusa `KpiCard`/`subscription-kpis.tsx` + `format.ts` para moeda em centavos→BRL):
   - **Receita recorrente** (período, por moeda) · **MRR aproximado** (últimos 30d, com tooltip "aproximação: receita do mês, não MRR contratual") ·
   - **Cobranças pagas** (count + valor) · **Reembolsadas** (count + valor) · **Chargeback** (count + valor) · **Pendentes** (count) ·
   - **Taxa de reembolso** e **Taxa de chargeback** (de `/stats`) ·
   - **Novos vs renovações** (via `parent_order_id`).

4. **Cards de gap honestos:** "Assinaturas vigentes", "Churn" e afins aparecem **desabilitados/com tooltip** "Requer integração de webhooks (fase 2) — a API pública da Kiwify não expõe estado de assinatura". **Nunca** mostrar número inventado.

5. **Gráfico de distribuição por status de venda** (reusa `status-distribution-chart.tsx`, recharts) com os buckets/contagens de `statusDistribution`.

6. **Estados:** loading (skeleton), erro (com retry), vazio (produto sem vendas no período) tratados.

7. **Qualidade:** `npm run typecheck` + `npm run lint` (web) passam; sem warning de recharts/hidratação.

## 🔧 Dev Notes

### Espelhar `subscriptions-dashboard.tsx` (Story 34.5)

Mesma casca (dropdown + período + grid de KpiCards + gráfico). Trocar hooks Hotmart→Kiwify e o set de cards. **Diferença conceitual importante:** o dashboard Hotmart mostra estado de assinatura (vigentes/canceladas); o Kiwify mostra **fluxo de vendas recorrentes** + cards de gap honestos. Não copiar os cards de "vigentes/churn" como se tivessem dado — usar o tratamento de gap (AC4).

### Componentes reutilizáveis (`components/subscriptions/`)

`product-picker.tsx`, `period-picker.tsx`, `subscription-kpis.tsx` (KpiCard), `status-distribution-chart.tsx`, `format.ts`. Verificar assinaturas/props de cada um antes de reusar.

### Formatação

`net_amount`/valores vêm em **centavos** do backend → `format.ts` divide por 100 e formata BRL. Multi-moeda: exibir por moeda (primária BRL).

### Reuso

shadcn/ui, recharts, lucide, `MetricTooltip` (se existir no padrão sales/traffic) para os tooltips de aproximação/gap.
