## Story

Dentro das etapas de cada campanha, uma seção pra **conectar o Mautic** (URL, email, senha). Conectado, puxa as campanhas do Mautic via API e faz **match automático pelo nome do funil** (ex.: `fz-l2`), com opção de **seleção manual**. Exibe métricas de email (enviados / aberturas / taxa) da campanha vinculada.

## Decisões (confirmadas com o usuário)

1. **Conexão por projeto** (uma instância Mautic por cliente; reusa entre etapas). Vínculo de campanha é por etapa.
2. **Exibir métricas de email**: enviados + aberturas + taxa de abertura.
3. **Auto-match pelo nome do funil** (token = 2 primeiros segmentos, ex.: `fz-l2-jun-26` → `fz-l2`) + seleção manual sempre disponível.

## Dev Notes

**Pesquisa Mautic API:** Basic Auth `Authorization: Basic base64(user:pass)`, base `{url}/api`. `GET /api/campaigns` (objeto keyed por id). `GET /api/campaigns/{id}` traz `events[]` (email.send → `properties.email`). `GET /api/emails/{id}` traz `sentCount`/`readCount`. **Cliques não existem** no objeto email padrão.

**⚠️ Pendente de validação:** não foi possível testar contra uma instância Mautic real (sem credenciais no ambiente de dev). Código espelha o padrão do Zoom; precisa de smoke test do Lucas conectando o Mautic real (e confirmar que Basic Auth está habilitado).
