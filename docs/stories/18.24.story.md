## Overview

**Objetivo:** Adicionar tabela de desempenho de criativos (ads) no funil de lançamento pago (**Paid Stage**), permitindo análise detalhada do ROI e eficiência de cada anúncio publicado.

**Contexto:** Análogos já existem para Meta Ads (Story 18.2, 18.3). Esta tabela consolida métricas críticas de performance com filtro por temperatura do público (hot/cold).

**Valor de Negócio:** Permite identificar rapidamente quais criativos têm melhor custo-benefício (CPL, ROAS) e orientar decisões de pausar/escalar anúncios.

## Technical Notes

### Dados Source
- **Meta Ads:** Vêm do endpoint `/ads` → `AdInsights` com filtro `stage_id = 'Paid'`
- **Fields mínimos necessários:**
  - `meta_adid` — ID do anúncio
  - `meta_adname` — Nome do anúncio
  - `spend` — Gasto (Invest)
  - `impressions` — Impressões
  - `clicks` — Cliques
  - `leads` — Leads (se disponível)
  - `revenue` — Faturamento
  - `utm_term` — Para identificar temperatura (hot/cold)

### Arquitetura

**Frontend:**
- `stage-creative-performance-table.tsx` — Renderiza tabela e gerencia estado de filtro/sort
- `useStageCreativePerformance` — Chama API, mantém cache, refetch em mudanças de período
- `creative-metrics-calculator.ts` — Funções puras para ROAS, CPL, CTR (testáveis)

**Backend:**
- `stage-creative-performance.ts` — Query creativa, agrupa por `meta_adid`, calcula agregados
- Response structure:
```json
[
  {
    "adId": "123456",
    "adName": "Ad Jan 2026",
    "spend": 5000.00,
    "impressions": 50000,
    "clicks": 1500,
    "leads": 300,
    "revenue": 15000.00,
    "utm_term": "hot-audience-jan"
  }
]
```

### Temperatura Filter Logic
```typescript
const temperatureFilter = (ad) => {
  const term = ad.utm_term?.toLowerCase() || '';
  if (!filters.hot && !filters.cold) return true; // padrão: mostrar todos
  if (filters.hot && term.includes('hot')) return true;
  if (filters.cold && term.includes('cold')) return true;
  return false;
};
```
