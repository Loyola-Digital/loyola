## Story

**As a** authenticated Loyola Digital user,
**I want** to create tasks in ClickUp directly from my conversations with minds,
**so that** actionable items identified by minds are tracked in the team's project management tool.

---

## Acceptance Criteria

1. [ ] `ClickUpService` como Fastify plugin (`packages/api/src/services/clickup.ts`)
2. [ ] Metodo `createTask({ name, description, listId, priority, tags })` chama ClickUp API v2
3. [ ] Mapeia priority enum para ClickUp priority (urgent=1, high=2, normal=3, low=4)
4. [ ] `POST /api/tasks` endpoint:
   - Recebe `{ conversationId, messageId, mindId, title, description?, priority?, tags? }`
   - Cria task no ClickUp
   - Salva em `delegated_tasks` com clickup_task_id e clickup_url
   - Retorna DelegatedTask
5. [ ] `GET /api/tasks` — lista tasks do user autenticado
   - Query params: `?status=open&limit=20&offset=0`
   - Ordenado por `created_at DESC`
6. [ ] Error handling: se ClickUp API falha, retorna erro claro (NAO salva task local)
7. [ ] Idempotencia: se clickup_task_id ja existe, retorna task existente (nao duplica)
8. [ ] Env vars `CLICKUP_API_TOKEN` e `CLICKUP_LIST_ID` adicionadas ao schema Zod
9. [ ] Resolucao de user: `request.userId` (Clerk ID) resolvido para `users.id` (UUID) via `conversationService.resolveUserId`
10. [ ] `pnpm typecheck` passa sem erros
11. [ ] `pnpm lint` passa sem erros

### Schema da Tabela `delegated_tasks`

```typescript
export const delegatedTasks = pgTable("delegated_tasks", {
  id: uuid("id").defaultRandom().primaryKey(),
  conversationId: uuid("conversation_id").notNull().references(() => conversations.id, { onDelete: "cascade" }),
  messageId: uuid("message_id").references(() => messages.id, { onDelete: "set null" }),
  userId: uuid("user_id").notNull().references(() => users.id, { onDelete: "cascade" }),
  mindId: text("mind_id").notNull(),
  clickupTaskId: text("clickup_task_id").notNull(),  // UNIQUE
  clickupUrl: text("clickup_url").notNull(),          // CHECK: LIKE 'https://%'
  title: text("title").notNull(),                     // CHECK: length > 0
  description: text("description"),
  status: taskStatusEnum("status").notNull().default("open"),
  priority: taskPriorityEnum("priority").notNull().default("normal"),
  tags: text("tags").array(),
  createdAt, updatedAt
}, (table) => [
  index("idx_tasks_user_created").on(table.userId, table.createdAt),
  index("idx_tasks_user_status").on(table.userId, table.status),
  index("idx_tasks_conversation").on(table.conversationId),
  uniqueIndex("uq_tasks_clickup_task_id").on(table.clickupTaskId),
]);
```

[Source: packages/api/src/db/schema.ts]

### ClickUp API v2 — Create Task

```
POST https://api.clickup.com/api/v2/list/{list_id}/task
Authorization: {api_token}
Content-Type: application/json

Body:
{
  "name": "Task title",
  "description": "Task description",
  "priority": 3,           // 1=urgent, 2=high, 3=normal, 4=low
  "tags": ["tag-name"]
}

Response 200:
{
  "id": "abc123",
  "url": "https://app.clickup.com/t/abc123",
  ...
}
```

### Env Vars — Optional Pattern

`CLICKUP_API_TOKEN` e `CLICKUP_LIST_ID` devem ser `.optional()` no schema Zod (dev pode nao ter ClickUp configurado localmente). O service deve verificar se estao definidos antes de tentar criar task.

[Source: packages/api/src/config/env.ts — pattern usado com CLERK_WEBHOOK_SECRET]

### Dependency: conversationService.resolveUserId

O `resolveUserId` ja esta disponivel via `fastify.conversationService.resolveUserId(clerkId)`. Reusar em vez de reimplementar.

[Source: packages/api/src/services/conversation.ts]
