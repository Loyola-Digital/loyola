# Story 23.2 — UI Dash Orgânico: Botão "Vincular a Etapa"

## Status

InReview

## Story

**As a** membro do projeto,
**I want** um botão em cada card de vídeo (Dash YouTube Orgânico) e em cada post (Dash Instagram) que abre um modal para escolher Funil e Etapa,
**so that** eu possa marcar quais conteúdos orgânicos sustentam quais etapas do funil — e ver no card se ele já está vinculado e em quais etapas.

## Acceptance Criteria

### Componente compartilhado

1. Criado componente `<LinkPostToStageModal>` em `packages/web/components/funnels/link-post-to-stage-modal.tsx`:
   - Props: `{ projectId: string, source: 'youtube' | 'instagram', externalId: string, postTitle: string, alreadyLinkedStageIds: string[], open: boolean, onOpenChange: (v: boolean) => void }`
   - 2 selects encadeados: **Funil** (`useFunnels(projectId)`) → **Etapa** (`useFunnel(projectId, funnelId)` filtrando `stages`)
   - Botão "Vincular" chama `useLinkOrganicPost.mutate({ stageId, source, externalId })`
   - Lista de etapas já vinculadas exibida acima dos selects (chips removíveis com botão de desvincular cada uma)
   - Toast de sucesso/erro (sonner)
   - Fecha automaticamente após sucesso da primeira vinculação (mas usuário pode reabrir e vincular mais)

### Hooks Web

2. Criado `packages/web/lib/hooks/use-organic-posts.ts` com:
   - `useOrganicPostLinks(projectId, source)` → GET `/api/projects/:id/organic-posts/links?source=...` — retorna `{ externalId: string, stageIds: string[] }[]`
   - `useLinkOrganicPost(projectId, funnelId)` → POST `/api/projects/:id/funnels/:fid/stages/:sid/organic-posts`
   - `useUnlinkOrganicPost(projectId, funnelId)` → DELETE
   - Invalidação de queries: `['organic-post-links', projectId, source]` e `['stage-organic-posts', stageId]` em todas as mutations

### Dash YouTube Orgânico

3. Em `packages/web/app/(app)/projects/[id]/youtube-organic/page.tsx`, cada card de vídeo (no grid "Vídeos Recentes") ganha:
   - Novo botão `<Button size="icon" variant="ghost">` com ícone `Link2` (lucide-react) no canto superior direito do thumbnail (z-index acima da imagem, hover: `bg-background/80`)
   - Click no botão → abre `<LinkPostToStageModal>` com `source='youtube'` e `externalId={v.videoId}`
   - **Indicador visual quando vinculado:** se `videoId` aparece em `useOrganicPostLinks(projectId, 'youtube')`, exibir badge no canto inferior do card mostrando contagem (`📍 {n}` etapas) com tooltip listando nomes das etapas

### Dash Instagram

4. Em `packages/web/components/instagram/...` (identificar o componente que renderiza grid de posts no `social-tab-instagram` ou na página dedicada do Instagram do projeto):
   - Aplicar o mesmo padrão do AC 3: botão `Link2`, modal, badge de "vinculado a N etapas"
   - `source='instagram'`, `externalId={post.mediaId}` (verificar campo correto do shape atual do Instagram media)

### UX

5. Modal usa o mesmo design system do projeto (shadcn Dialog/Sheet — verificar qual padrão é usado em outros modais da etapa, como `CampaignSelector`).

6. Click fora do modal ou ESC fecha sem confirmar.

7. Estado vazio "nenhum funil ainda criado neste projeto" exibe link/dica para criar funil em `/projects/:id/funnels`.

8. Estado de loading: skeleton nos selects enquanto carrega `useFunnels` / `useFunnel`.

### Permissão

9. Não há gate de permissão na UI — qualquer membro do projeto pode usar (per AC do Epic). API valida `getProjectAccess` (já coberto na 23.1).

### Validação

10. `pnpm --filter @loyola-x/web typecheck` → 0 erros
11. `pnpm --filter @loyola-x/web lint` → 0 erros
12. Smoke test manual:
    - Abrir Dash Orgânico de um projeto que tem funil + canal YouTube vinculado
    - Click no `Link2` de um vídeo → modal abre → seleciona funil → seleciona etapa → "Vincular" → toast de sucesso
    - Card mostra badge `📍 1`
    - Reabre modal → chip da etapa aparece → click no X → desvincula → badge some
