## Story

**As a** developer building funnel dashboards,
**I want** all traffic analytics hooks to accept an optional `campaignId` filter,
**so that** funnel dashboards load only the data from the linked campaign instead of the entire account.

---

## Acceptance Criteria

### Backend — Campaign-Scoped Endpoints

1. [x] `GET /api/traffic/analytics/:projectId/overview` — accept `?campaignId=X` to filter aggregation to one campaign
2. [x] `GET /api/traffic/analytics/:projectId/campaigns` — when `campaignId` is set, return only that campaign
3. [x] `GET /api/traffic/analytics/:projectId/top-performers` — already supports `campaignId` param (verified)
4. [x] `GET /api/traffic/analytics/:projectId/placements` — add `?campaignId=X` to filter placement breakdown by campaign
5. [x] All existing behavior (without campaignId) remains unchanged — backward compatible

### Backend — Placement by Campaign

6. [x] `fetchPlacementBreakdown()` in `meta-ads.ts` — add optional `campaignId` param
7. [x] When campaignId provided, add filtering: `&filtering=[{"field":"campaign.id","operator":"EQUAL","value":"X"}]`

### Frontend Hooks

8. [x] `useTrafficOverview(projectId, days, campaignId?)` — pass campaignId as query param
9. [x] `useTopPerformers(projectId, metric, limit, days, campaignId?)` — already supports (verified)
10. [x] `usePlacementBreakdown(projectId, days, campaignId?)` — add campaignId param
11. [x] All hooks: when campaignId is provided, include in queryKey for proper caching

### Funnel Context Hook

12. [x] Create `useFunnel(projectId, funnelId)` hook — fetches single funnel and returns `{ funnel, campaignId, funnelType }`
13. [x] Funnel dashboard pages use this hook to get the campaignId for all traffic hooks
