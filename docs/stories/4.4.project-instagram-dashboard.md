## Story

**As a** Loyola Digital employee,
**I want** an Instagram dashboard scoped to a specific project,
**so that** I can view metrics for only the accounts belonging to that client.

---

## Acceptance Criteria

1. [ ] Rota `/projects/[id]/instagram` criada e funcional
2. [ ] Dashboard idêntico ao global (`/instagram`) — mesmos componentes reutilizados
3. [ ] Account selector exibe apenas contas vinculadas ao projeto (`?project_id=`)
4. [ ] Se projeto tem apenas 1 conta: auto-seleciona sem exibir dropdown
5. [ ] Breadcrumb: "Projetos > [Nome do Cliente] > Instagram"
6. [ ] Botão "Gerenciar contas" leva para `/settings/instagram` (fase 2 terá `?project_id=`)
7. [ ] `useInstagramAccounts(projectId?)` hook aceita `projectId` opcional para filtrar
8. [ ] Dashboard global (`/instagram`) mantido sem alteração — mostra todas as contas
9. [ ] Empty state quando projeto não tem contas: "Nenhuma conta vinculada a este projeto. Adicione em Settings."

## Dev Notes

### Architecture Context

- **Reutilizar componentes:** Todos os componentes do dashboard global (`overview-cards`, `reach-chart`, `posts-table`, `stories-section`, `reels-section`, `audience-charts`) devem ser reutilizados sem modificação
- **Hook a atualizar:** `use-instagram-accounts.ts` — `useInstagramAccounts()` aceita `projectId?: string` opcional
- **API:** `GET /api/instagram/accounts?project_id=UUID` já filtrado pela Story 4.2
- **Layout do projeto:** herda de `app/(app)/projects/[id]/layout.tsx` da Story 4.3
