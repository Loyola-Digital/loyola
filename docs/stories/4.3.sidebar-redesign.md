## Story

**As a** Loyola Digital employee,
**I want** a sidebar with collapsible project folders,
**so that** I can navigate per-client resources quickly without losing access to global views.

---

## Acceptance Criteria

1. [ ] Seção "Global" mantida no topo: Dashboard (`/`), Conversas (`/conversations`), Tasks (`/tasks`), Instagram (`/instagram`), Settings (`/settings`)
2. [ ] Separador visual entre seção Global e seção Projetos
3. [ ] Seção "Projetos": cada projeto é um folder colapsável com chevron toggle
4. [ ] Folder exibe cor do projeto como dot indicator + nome do projeto
5. [ ] Estado de colapso de cada projeto persistido em `localStorage` (key: `project-folder-{id}`)
6. [ ] Subitens visíveis quando expandido: 📊 Instagram → `/projects/[id]/instagram`, 💬 Conversas → `/projects/[id]/conversations`
7. [ ] Item ativo destacado (variant `secondary`)
8. [ ] Botão "+ Novo Projeto" ao fim da lista → abre `CreateProjectDialog`
9. [ ] `CreateProjectDialog`: campos `name`, `clientName`, `color` (color picker simples — 6 cores predefinidas) + botão Criar
10. [ ] Loading skeleton enquanto projetos carregam
11. [ ] Empty state inline quando sem projetos: "Nenhum projeto. Crie o primeiro."
12. [ ] Responsivo: comportamento de sheet no mobile mantido
13. [ ] Rota `/projects/[id]` criada — overview do projeto (nome, clientName, cor, lista de contas vinculadas)
14. [ ] `useProjects()` hook TanStack Query para listar projetos

#### Concerns (advisory — não bloqueiam merge)

**1. CreateProjectDialog — sem error handling no mutateAsync** (Medium):
`handleSubmit` chama `createProject.mutateAsync(...)` sem try/catch. Se a API retornar erro (ex: 400, 500), o dialog fecha silenciosamente sem feedback ao usuário. Padrão correto seria envolver em try/catch e exibir toast de erro. Adicionar em story futura ou Story 4.4 como melhoria incremental.

**2. /projects/[id] fetches lista completa para encontrar projeto por ID** (Low):
`page.tsx:19-20` usa `useProjects()` + `find(p => p.id === id)` em vez de `GET /api/projects/:id` diretamente. Ineficiente com muitos projetos. Endpoint individual já existe na Story 4.2. Refatorar quando volume justificar.

**3. Active state condition no ProjectFolder** (Low):
`project-folder.tsx:62` usa `!pathname.includes("/instagram") && !pathname.includes("/conversations")` — frágil se futuras rotas tiverem "instagram" no path em contexto diferente. Preferível comparação exata `pathname === /projects/${id}`. Risco atual: mínimo dado escopo das rotas.

**4. Rotas /projects/[id]/instagram e /projects/[id]/conversations ainda não existem** (Low / design intencional):
Links criados corretamente como placeholders para Stories 4.4 e 4.5. Comportamento esperado: 404 até implementação. Não é bug — é sequência planejada do EPIC-4.

**5. AC1 — "Dashboard" não aparece na seção Global** (Low / wording):
AC menciona "Dashboard (`/`)" mas o sidebar existente sempre teve "Minds" como primeiro item. O @dev manteve o comportamento pré-existente (correto). AC foi redigido de forma imprecisa — não é falha de implementação.
