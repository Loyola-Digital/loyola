## 📖 Story

**Como** membro do time discutindo um debriefing,
**Quero** deixar o comentário fixado no ponto exato do documento a que ele se refere (como no Figma ou Google Docs), com um pin visível para todos,
**Para** que quem lê saiba exatamente sobre qual trecho/número/gráfico do doc a pessoa está falando.

_Pedido do Lucas: "o comentário devia ficar igual é no Figma ou Google Doc, no lugar que a pessoa quer que veja onde tá o comentário"._

---

## ✅ Acceptance Criteria

1. **Âncora no banco:** colunas nullable `anchor_x`/`anchor_y` (double precision, % da largura/altura do doc) em `debriefing_comments` — migration 0069 aditiva/idempotente. NULL = comentário geral (comportamento 37.1 preservado).
2. **API:** POST comments aceita `anchorX`/`anchorY` opcionais (0–100, ambos juntos, validado por zod); GET retorna os campos.
3. **Modo "Comentar no doc":** botão no header ativa overlay com cursor de mira sobre o doc; clique captura a posição em % e abre um **composer flutuante** naquele ponto (textarea + Comentar/Cancelar). Banner explicativo enquanto ativo.
4. **Pins:** comentários ancorados aparecem como **pins numerados** sobre o doc (overlay React no parent — o iframe sandbox NÃO é tocado; funciona porque a altura do iframe acompanha o doc desde a 37.2). Clicar no pin abre popover com avatar, nome, data, texto, e excluir (se `mine`).
5. **Sidebar ↔ pin:** comentário ancorado na sidebar mostra badge "Pin N — ver no doc"; clicar rola até o pin e destaca ambos. Numeração estável (ordem de criação).
6. **Interações:** modo edição (37.2) desativa pins/placing; comentários gerais continuam existindo (composer da sidebar).
7. **Qualidade:** typecheck/lint/testes API ok; `pnpm --filter @loyola-x/web build` compila com types ok.
