## Story

**As a** Loyola Digital employee chatting with an AI Mind,
**I want** the mind to access Instagram metrics of my client accounts,
**so that** I can ask questions like "como esta a performance do @cliente" and get real data in the response.

---

## Acceptance Criteria

1. [ ] Novo tool `instagram_metrics` registrado em `chat-tools.ts`
2. [ ] Tool input schema: `{ account_name_or_username: string, metric_type?: "overview" | "posts" | "demographics" | "full", period?: string }`
3. [ ] Tool description clara para o LLM saber quando usar (ex: "Busca metricas de Instagram de uma conta cadastrada")
4. [ ] Busca conta por nome amigavel ou @username (case-insensitive, partial match)
5. [ ] Tipo "overview": followers, engagement rate, reach summary do periodo
6. [ ] Tipo "posts": ultimos 10 posts com likes, comments, reach
7. [ ] Tipo "demographics": audiencia resumida (top 3 cidades, distribuicao idade/genero)
8. [ ] Tipo "full": overview + posts + demographics combinados
9. [ ] Default metric_type: "overview" se nao especificado
10. [ ] Dados formatados como markdown estruturado para o LLM processar
11. [ ] Se conta nao encontrada: retorna lista de contas disponiveis como sugestao
12. [ ] Se token expirado: retorna aviso pedindo renovacao em Settings
13. [ ] Usa cache do Instagram service (nao faz chamadas extras a API)
14. [ ] Testes: tool retorna dados corretos, handles erros gracefully

## Dev Notes

### Architecture Context

- **Chat Tools:** `packages/api/src/services/chat-tools.ts` — array de tools disponíveis para o Claude [Source: packages/api/src/services/chat-tools.ts]
- **Tool Pattern:** Cada tool tem `name`, `description`, `input_schema` (JSON Schema), e `handler(input, context)`
- **Claude Tool Use:** Anthropic SDK tool_use format — tool registrado no system prompt e chamado pelo LLM
- **Instagram Service:** `packages/api/src/services/instagram.ts` (Story 3.2.1) — usa cache

### Tool Definition

```typescript
{
  name: "instagram_metrics",
  description: "Busca metricas e dados de uma conta de Instagram cadastrada na plataforma. Use quando o usuario perguntar sobre performance, engagement, followers, ou dados de Instagram de um cliente.",
  input_schema: {
    type: "object",
    properties: {
      account_name_or_username: {
        type: "string",
        description: "Nome da conta ou @username do Instagram (busca parcial)"
      },
      metric_type: {
        type: "string",
        enum: ["overview", "posts", "demographics", "full"],
        description: "Tipo de metricas: overview (resumo), posts (ultimos posts), demographics (audiencia), full (tudo)"
      },
      period: {
        type: "string",
        description: "Periodo para metricas: 7d, 14d, 30d, 90d (default: 30d)"
      }
    },
    required: ["account_name_or_username"]
  }
}
```

### Output Format (Markdown for LLM)

```markdown

### Overview (ultimos 30 dias)

- **Engagement Rate:** 4.8%
- **Reach Total:** 45,200
- **Impressions Total:** 62,100

### Ultimos Posts
| Post | Likes | Comments | Reach |
|------|-------|----------|-------|
| "Lancamento campanha..." | 245 | 32 | 3,200 |
| ...
```
