## Story

Como usuário dos dashboards de funis (Launch e Perpetual), quero que **todos** os elementos visuais que mostram valores calculados — KPI cards, gráficos (pie/line/bar), o próprio funil de conversão e as métricas dos creatives — **convidem visualmente ao hover** e exibam o memorial de cálculo ao passar o mouse, para que em reuniões eu saiba imediatamente que qualquer número é auditável sem precisar adivinhar quais têm memorial e quais não têm.

## Contexto

As Stories 16.7 e 16.8 fecharam os KPI cards e as tabelas drill-down dos funis. Em 2026-04-15 o usuário reportou que os cards "não aparecem hover" — o diagnóstico mostrou que os KPI cards **já têm `<MetricTooltip>` wrapper** desde 16.7, mas **não há qualquer pista visual** de que são hover-able (sem cursor-help, sem underline dotted).

Além disso, o mapeamento identificou que 5 tipos de elementos visuais ainda estão sem memorial:
1. KPI cards (wrapped mas sem visual hint) — problema de UX, não de cobertura
2. Gráficos usando `<Tooltip contentStyle={...}>` default do Recharts (3 gráficos)
3. `ConversionFunnel` — barras do funil e taxas de conversão sem tooltip
4. `TopCreativesGallery` — cards de criativo (Spend/CTR/CPC) e lightbox (5 métricas)

Esta story fecha o último gap do Epic 16.

## Contexto Técnico

### Grupo 1 — Visual hints nos KPI cards já wrapped

**Problema:** `<MetricTooltip>` envolve o `<KpiCard>` inteiro, mas nada no card comunica ao usuário que é hover-able. O usuário passou por cima dos cards e achou que não tinha memorial.

**Cards afetados (já wrapped):**
- `launch-dashboard.tsx` — 8 KPI cards (linhas 187-219)
- `perpetual-dashboard.tsx` — 7 KPI cards (linhas 247-272) + 3 rate cards (linhas 280-288)

**Estratégia recomendada:** modificar `<KpiCard>` local dos dois dashboards para aceitar prop `hintTooltip?: boolean` e quando true:
- Adicionar `cursor-help` no container do card
- Adicionar `underline decoration-dotted decoration-muted-foreground/40 underline-offset-4` no `<p>` do valor

Alternativa: aplicar esse visual hint automaticamente no `<MetricTooltip>` (envolver `children` num `<span>` com essas classes quando `formula` está presente). Riscos: KpiCard do launch e perpetual são `<div>` não `<span>`, e inserir um span extra no DOM pode quebrar layout Flexbox/Grid. → **Usar a prop explicit é mais seguro.**

## Dev Notes

### Padrão de visual hint (recomendado)

```tsx
// Dentro de KpiCard local
function KpiCard({ icon: Icon, label, value, hintTooltip }: { ..., hintTooltip?: boolean }) {
  return (
    <div className={cn(
      "rounded-xl border ... transition-colors",
      hintTooltip && "cursor-help"
    )}>
      <div>...label...</div>
      <p className={cn(
        "text-xl font-bold tracking-tight",
        hintTooltip && "underline decoration-dotted decoration-muted-foreground/40 underline-offset-4"
      )}>
        {value}
      </p>
    </div>
  );
}
```

### ConversionFunnel — factories

Para não explodir `funnels.ts`, sugerido criar 2 factories novas:

```ts
export function buildFunnelStageFormula(stageLabel: string, value: number, apiSource: string): MetricFormula {
  return { expression: `Σ ${stageLabel.toLowerCase()}`, values: [{ label: stageLabel, value: nf.format(value), source: apiSource }], result: nf.format(value) };
}

export function buildFunnelStageConversionFormula(fromLabel: string, fromValue: number, toLabel: string, toValue: number): MetricFormula | undefined {
  if (fromValue <= 0) return undefined;
  const rate = (toValue / fromValue) * 100;
  return {
    expression: `${toLabel} ÷ ${fromLabel} × 100`,
    values: [
      { label: fromLabel, value: nf.format(fromValue), source: "Meta Ads / CRM" },
      { label: toLabel, value: nf.format(toValue), source: "Meta Ads / CRM" },
    ],
    result: `${nf.format(toValue)} ÷ ${nf.format(fromValue)} × 100 = ${rate.toFixed(1)}%`,
  };
}
```

## Risks

- **R1 (Baixo):** underline dotted em KPI cards pode conflitar com design já aprovado. Mitigação: usar `decoration-muted-foreground/40` (cor sutil, praticamente invisible em estado não-hover, visível em hover)
- **R2 (Baixo):** TopCreativesGallery lightbox tem click handlers no container — tooltip no span interno não deveria conflitar (mesmo pattern do drill-down tables)
- **R3 (Médio):** ConversionFunnel stages são dinâmicos. Factories precisam receber valores opcionais sem quebrar (atendem AC3 quando value > 0). Stages com value=0 nem são renderizados, então não é problema.
