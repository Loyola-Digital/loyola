## Story

**As a** Loyola Digital employee managing Instagram accounts,
**I want** my access tokens stored securely with encryption,
**so that** client credentials are protected even if the database is compromised.

## Acceptance Criteria

1. [ ] Tabela `instagram_accounts` criada via Drizzle migration com campos: id (UUID PK), user_id (FK users), account_name, instagram_user_id, instagram_username, access_token_encrypted, access_token_iv, token_expires_at, profile_picture_url, is_active, last_synced_at, created_at, updated_at
2. [ ] Tabela `instagram_metrics_cache` criada com campos: id (UUID PK), account_id (FK instagram_accounts ON DELETE CASCADE), metric_type, metric_data (JSONB), period_start, period_end, fetched_at, expires_at
3. [ ] Unique constraint em `instagram_accounts.instagram_user_id`
4. [ ] Unique constraint em `instagram_metrics_cache(account_id, metric_type, period_start, period_end)`
5. [ ] Indices: `idx_ig_accounts_user(user_id)`, `idx_ig_metrics_account(account_id)`, `idx_ig_metrics_expires(expires_at)`
6. [ ] Service `encryption.ts` com funcoes `encrypt(plaintext): { encrypted, iv }` e `decrypt(encrypted, iv): plaintext`
7. [ ] Usa AES-256-GCM via `node:crypto` (built-in, sem dependencia extra)
8. [ ] `ENCRYPTION_KEY` lido de `process.env` (32 bytes, hex-encoded)
9. [ ] IV aleatorio gerado por encriptacao (16 bytes via `crypto.randomBytes`)
10. [ ] Testes unitarios para encryption: round-trip (encrypt → decrypt), IV unico por chamada, key invalida rejeita, tampered ciphertext rejeita
11. [ ] Drizzle migration gerada e aplicavel
12. [ ] `pnpm typecheck` e `pnpm lint` passam

### Schema SQL Reference

```sql
CREATE TABLE instagram_accounts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(id),
  account_name VARCHAR(100) NOT NULL,
  instagram_user_id VARCHAR(50) NOT NULL UNIQUE,
  instagram_username VARCHAR(50),
  access_token_encrypted TEXT NOT NULL,
  access_token_iv TEXT NOT NULL,
  token_expires_at TIMESTAMPTZ,
  profile_picture_url TEXT,
  is_active BOOLEAN DEFAULT true,
  last_synced_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE instagram_metrics_cache (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  account_id UUID NOT NULL REFERENCES instagram_accounts(id) ON DELETE CASCADE,
  metric_type VARCHAR(50) NOT NULL,
  metric_data JSONB NOT NULL,
  period_start DATE,
  period_end DATE,
  fetched_at TIMESTAMPTZ DEFAULT now(),
  expires_at TIMESTAMPTZ NOT NULL,
  UNIQUE(account_id, metric_type, period_start, period_end)
);
```

4. decrypt com ciphertext adulterado lanca erro
