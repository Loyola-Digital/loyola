## Story

Ajustes no Sprint Dashboard a partir do feedback do Lucas:

1. **Visão Geral — faltam tasks.** Causa-raiz: `getTasks`/`getCustomTypeTasks` do ClickUp **não paginam** (a API retorna no máx. 100 tasks/página e exige `&page=N`). Listas com +100 tasks perdiam o resto. (Filtro de sem-responsável **mantido** por decisão do usuário.)
2. **Pendentes movem pra Visão Geral.** A `PendingTasksList` (atraso + hoje) sai do card do Calendário Macro e passa pro card da Visão Geral (`SprintBlockCard`).
3. **Calendário Macro** — remover a lista de pendentes do card.
4. **Saúde — "Em progresso" junta não-iniciadas + em-andamento.** Separar usando o `type` do status do ClickUp (`open` = não iniciado, `custom`/demais = em progresso, `done`/`closed` = concluído), alinhando com a nomenclatura do ClickUp.

## Acceptance Criteria

### AC1: Paginação ClickUp
- [x] `getTasks` pagina via `?page=N` até `last_page === true`.
- [x] `getCustomTypeTasks` idem.
- [x] Sem regressão de performance (páginas sequenciais; listas pequenas = 1 chamada).

### AC2: Pendentes na Visão Geral
- [x] `SprintBlockCard` renderiza `PendingTasksList`.
- [x] `MacroCalendarView`/`CampaignCard` não renderiza mais `PendingTasksList`.

### AC3: Split Não iniciado × Em progresso (Saúde)
- [x] Backend expõe `statusType` por task (de `status.type`).
- [x] `/metrics` retorna `notStarted` + `inProgress` (não-done split por type); `overdue`/`done` mantidos; health denom = done + notStarted + inProgress (inalterado).
- [x] Aba Saúde (FolderMetricCard) + linha de métricas globais mostram "Não iniciado" e "Em progresso" separados.
- [x] Fallback por nome quando `type` não vier (to do/backlog/aberto = não iniciado).

### AC4: Validação
- [x] Typecheck ✅ + Lint ✅ (api + web)
- [x] Sem regressão nos testes (falhas pré-existentes documentadas)
