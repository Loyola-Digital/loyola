# Story 29.16 — Funil Perpétuo: Vendas e ROAS da tabela de Campanha vindos da planilha

**Epic:** 29
**Status:** InReview
**Points:** 5
**Tipo:** Feature / Data Correctness
**Prioridade:** P1

## Contexto

Na tabela "Desempenho por Canal (Campanha)" do Funil Perpétuo, as colunas **Vendas** e **ROAS** vinham do Meta (pixel purchases). Danilo quer que venham da **planilha vinculada**, casando `utm_campaign` (que carrega o **Campaign ID** no padrão Loyola) com a campanha, e exibindo o Campaign Name.

**Descoberta:** o `PerpetualSalesData` **não tinha** breakdown por `utm_campaign` (só source/medium/content) — precisou ser criado no backend. O Campaign Name **já está na linha** da tabela (`c.campaignName`), então não precisa resolver via Meta Names — basta cruzar por `campaignId`.

## Acceptance Criteria

- [x] **AC1:** Backend: `PerpetualSalesData` ganha `porUtmCampaign: { campaign, vendas, bruto, liquido }[]`, agregado por `utm_campaign` (espelhando `porUtmSource/Medium/Content`).
- [x] **AC2:** Backend: endpoint lê `mapping.utm_campaign`, captura no dedup e agrega em `utmCampaignMap`; `EMPTY_SALES_DATA` inclui `porUtmCampaign: []`.
- [x] **AC3:** Frontend: com planilha conectada (`usingSpreadsheet`), a tabela de campanha usa **Vendas** e **Receita** da planilha (match `utm_campaign === campaignId`).
- [x] **AC4:** Frontend: **ROAS = receita_planilha / spend_Meta** (spend continua Meta). **CAC = spend_Meta / vendas_planilha**.
- [x] **AC5:** Campaign sem match na planilha → Vendas/Receita = 0, ROAS/CAC = "—" (não herda Meta). Nome continua vindo da linha.
- [x] **AC6:** Fallback: sem planilha conectada → comportamento atual (dados Meta) preservado.
- [x] **AC7:** Corrige as **duas** tabelas de uma vez (a "Desempenho por Canal" e o "Detalhamento" no filtro Campanha), pois ambas leem `funnelCampaigns`. Filtros Público/Criativo (adset/ad) seguem Meta — fora do escopo.
- [x] **AC8:** Typecheck + build (API + Web) OK.
