Como **desenvolvedor da plataforma**, preciso de uma tabela e endpoint que armazenem a configuração do Sprint Dashboard global (1 row única) com a lista de blocos configurados (cada bloco aponta pra listas ClickUp + filtros + visual), pra que stories subsequentes (31.2–31.6) possam consumir essa config como source-of-truth do que renderizar.

## Dev Notes

### Por que singleton via UNIQUE em coluna boolean default true?

Truque clássico: `singleton` é sempre `true`, e UNIQUE garante que só 1 row exista. Tentar inserir uma segunda row dá conflict. Mais simples que CHECK constraint ou trigger.

### Pattern de referência

Migrations 0037 (perpetual_sales_enum), 0038 (funnel_spreadsheets.platform), 0039 (funnels.leads_goal). Mesmo padrão node+pg+dotenv.

### Rota não precisa de projectId

É uma config GLOBAL (toda Loyola compartilha). Endpoint não tem `:projectId` na URL.
