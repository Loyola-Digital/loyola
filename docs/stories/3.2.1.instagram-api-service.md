## Story

**As a** backend service,
**I want** a centralized Instagram Graph API client with intelligent caching,
**so that** the app can fetch Instagram metrics efficiently while respecting Meta's rate limits.

---

## Acceptance Criteria

1. [ ] Service `instagram.ts` com metodo `validateToken(accessToken)` — GET /me valida token, retorna profile data ou erro
2. [ ] Metodo `getProfile(accountId)` — retorna followers_count, follows_count, media_count, biography, profile_picture_url, username
3. [ ] Metodo `getMediaList(accountId, limit?, after?)` — lista posts com paginacao cursor (id, caption, media_type, media_url, thumbnail_url, timestamp, like_count, comments_count)
4. [ ] Metodo `getMediaInsights(mediaId)` — metricas de post: impressions, reach, likes, comments, saves, shares, plays (para reels)
5. [ ] Metodo `getAccountInsights(accountId, period, since, until)` — metricas da conta: impressions, reach, follower_count, profile_views por dia
6. [ ] Metodo `getAudienceDemographics(accountId)` — audience_city, audience_country, audience_gender_age (period=lifetime)
7. [ ] Metodo `getStories(accountId)` — stories ativos com metricas (impressions, reach, replies, exits)
8. [ ] Metodo `getReels(accountId)` — reels com metricas (plays, likes, comments, reach, saves, shares)
9. [ ] Cache layer: verifica `instagram_metrics_cache` antes de chamar API
   - Profile: TTL 5 min
   - Post insights: TTL 15 min
   - Account insights: TTL 30 min
   - Demographics: TTL 1 hora
10. [ ] Cache hit retorna dados do banco; cache miss chama API, salva no banco, retorna
11. [ ] Rate limit tracking: contador interno, max 200 calls/hora por token
12. [ ] Se rate limit atingido: retorna erro claro "Rate limit atingido, tente novamente em X minutos"
13. [ ] Error handling: token expirado retorna erro especifico com mensagem de renovacao
14. [ ] Error handling: Graph API errors mapeados para mensagens uteis
15. [ ] Decripta access_token do banco via encryption service antes de usar
16. [ ] Graph API version: v21.0 (hardcoded como constante)
17. [ ] Testes unitarios com mocks da Graph API (fetch mockado)

## Dev Notes

### Architecture Context

- **HTTP Client:** Usar `fetch` nativo do Node.js 18+ (sem axios/got) [Source: architecture/fullstack-architecture-loyola-digital-x.md#5]
- **Fastify Service Pattern:** Decorar Fastify instance com service, similar ao `claude.ts` e `clickup.ts` [Source: packages/api/src/app.ts]
- **Encryption:** Usar `decrypt()` de `services/encryption.ts` (Story 3.1.2)
- **DB:** Consultar `instagram_metrics_cache` via Drizzle antes de fetch externo

### Meta Graph API v21.0 Endpoints

```
Base URL: https://graph.instagram.com/v21.0

GET /me?fields=id,name,username&access_token={token}  — Token validation
GET /{ig-user-id}?fields=id,username,name,biography,followers_count,follows_count,media_count,profile_picture_url  — Profile
GET /{ig-user-id}/media?fields=id,caption,media_type,media_url,thumbnail_url,timestamp,like_count,comments_count&limit={n}&after={cursor}  — Media list
GET /{media-id}/insights?metric=impressions,reach,likes,comments,saved,shares,plays  — Media insights
GET /{ig-user-id}/insights?metric=impressions,reach,follower_count,profile_views&period=day&since={unix}&until={unix}  — Account insights
GET /{ig-user-id}/insights?metric=audience_city,audience_country,audience_gender_age&period=lifetime  — Demographics
GET /{ig-user-id}/stories?fields=id,media_type,media_url,timestamp  — Stories
GET /{story-id}/insights?metric=impressions,reach,replies,exits  — Story insights
```

### Rate Limits

- 200 calls/user/hour (standard Graph API)
- Implementar contador in-memory com reset a cada hora
- Se proximo do limite (>180): logar warning

### Cache TTL Strategy

| Metric Type | TTL | Justificativa |
|------------|-----|---------------|
| profile | 5 min | Muda raramente, mas followers atualiza |
| post_insights | 15 min | Metricas estabilizam apos 24h |
| account_insights | 30 min | Agregados diarios, nao mudam rapido |
| demographics | 1 hora | Lifetime data, atualiza ~1x/dia |
| stories | 5 min | Stories expiram em 24h, dados mudam rapido |
| reels | 15 min | Similar a posts |
