## Story

**As a** Loyola Digital employee chatting with an AI Mind,
**I want** the backend to stream Claude API responses in real-time via SSE,
**so that** I see text appearing progressively (like ChatGPT) instead of waiting for the full response.

---

## Acceptance Criteria

1. [ ] `ClaudeService` as Fastify plugin wrapping `@anthropic-ai/sdk` (`src/services/claude.ts`)
2. [ ] Method `stream(systemPrompt, messages)` returns AsyncIterable of stream events
3. [ ] Uses `claude-sonnet-4-6` as default model
4. [ ] `POST /api/chat` endpoint conforme API contract (architecture doc section 8.2)
5. [ ] Receives `{ mindId, conversationId?, message }` — validated with Zod
6. [ ] Returns `text/event-stream` with events: `conversation`, `text_delta`, `task_detected`, `usage`, `done`
7. [ ] If `conversationId` is null, creates new conversation in DB
8. [ ] Loads conversation message history for context (last 20 messages)
9. [ ] Detects `json:task` block in assistant response and emits `task_detected` event
10. [ ] Persists user and assistant messages in DB after stream completes
11. [ ] Updates `conversations.updated_at` and counters (via DB trigger)
12. [ ] Limits message history in context to last 20 messages
13. [ ] Error handling: returns SSE event `error` on failure
14. [ ] Rate limit specific for chat: 20 req/min/user

### Anthropic SDK — Streaming Pattern

```typescript
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic({ apiKey: "..." });

// Usar messages.stream() que retorna MessageStream com eventos
const stream = client.messages.stream({
  model: "claude-sonnet-4-6",
  max_tokens: 4096,
  system: systemPrompt,
  messages: [{ role: "user", content: "Hello" }],
});

// Eventos do MessageStream:
stream.on('text', (text) => { /* delta text */ });
// Ao final:
const finalMessage = await stream.finalMessage();
// finalMessage.usage = { input_tokens, output_tokens }
// finalMessage.stop_reason = "end_turn" | "max_tokens" | ...
```

### API Contract — POST /api/chat (Section 8.2)

```
POST /api/chat
  Content-Type: application/json
  Authorization: Bearer {clerk_jwt}

  Body: { mindId: string, conversationId: string | null, message: string }

  → 200 (text/event-stream):
  event: conversation   → data: {"conversationId":"uuid","isNew":true}
  event: text_delta     → data: {"text":"Let me"}
  event: text_delta     → data: {"text":" tell you"}
  event: task_detected  → data: {"title":"...","description":"...","priority":2}
  event: usage          → data: {"inputTokens":5200,"outputTokens":340}
  event: done           → data: {"messageId":"uuid"}
```

### SSE Format

Cada evento SSE segue o formato:
```
event: {event_name}\ndata: {json_string}\n\n
```

Helper function para escrever eventos:
```typescript
function sendSSE(reply: FastifyReply, event: string, data: unknown) {
  reply.raw.write(`event: ${event}\ndata: ${JSON.stringify(data)}\n\n`);
}
```

### Database Schema Relevante

**conversations** table:
- `id` (uuid PK), `user_id` (FK→users), `mind_id`, `mind_name`, `squad_id`
- `title` (auto-gerado do 1o msg), `message_count`, `total_tokens`
- `deleted_at` (soft delete), `created_at`, `updated_at`

**messages** table:
- `id` (uuid PK), `conversation_id` (FK→conversations), `role` (enum: user|assistant)
- `content` (text), `tokens_used` (int), `metadata` (jsonb)
- `metadata` shape: `{ model?, inputTokens?, outputTokens?, taskDetected?, finishReason? }`

**Triggers no DB** (serao criados em story de migrations):
- `trg_messages_update_counters` — incrementa `conversations.message_count` e `total_tokens`
- `trg_messages_auto_title` — gera `conversations.title` do 1o user message (first 80 chars)

> **NOTA:** Se as migrations/triggers ainda nao existirem, a story deve funcionar sem eles — o counter update pode ser feito manualmente no codigo da route por enquanto. Os triggers serao adicionados na story de DB migrations (1.0.1 ou similar).

### Task Detection Regex

O TASK_DELEGATION_PROMPT (incluido em todo system prompt pelo MindEngine) instrui o mind a emitir:
````
```json:task
{
  "action": "create_task",
  "title": "...",
  "description": "...",
  "priority": 2,
  "tags": ["tag1"]
}
```
````

Regex para detectar: `` /```json:task\s*\n([\s\S]*?)```/ ``

### Rate Limiting


### Auth Pattern

O middleware de auth (`middleware/auth.ts`) ja extrai o userId do JWT e coloca em `request.userId`.
Na rota de chat, usar `request.userId` para:
1. Criar novas conversas vinculadas ao user
2. Verificar ownership de conversas existentes
3. Persistir mensagens
