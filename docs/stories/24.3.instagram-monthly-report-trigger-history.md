# Story 24.3 — Trigger no Dashboard + Listagem de Histórico

## Status

Done

## Story

**As a** membro do projeto,
**I want** um botão "Gerar relatório do mês" no dashboard de Instagram e uma página listando todos os relatórios passados,
**so that** eu possa gerar relatórios sob demanda e reabrir relatórios antigos para comparar performance ao longo do tempo, sem precisar re-fetchar a Meta.

## Acceptance Criteria

### Trigger no Dashboard

1. Botão "Gerar relatório do mês" adicionado ao header das duas páginas Instagram:
   - `app/(app)/instagram/page.tsx` (global)
   - `app/(app)/projects/[id]/instagram/page.tsx` (project)
   - Posicionado ao lado de "Atualizar"
   - Ícone `FileText` (lucide-react)
   - Label muda em mobile: ícone só

2. Click abre Dialog "Gerar relatório":
   - Dropdown "Mês": lista últimos 12 meses + opcionalmente "mês corrente" (mês atual ainda em curso)
     - Default: mês passado (ex: hoje é Maio/26 → default Abril/26)
     - Format dropdown: "Abril 2026", "Março 2026"...
   - Texto explicativo: "O relatório agrega todas as contas IG vinculadas a este projeto."
   - Botão "Gerar" + Botão "Cancelar"
   - Dialog reusa shadcn `<Dialog>` existente

3. Ao clicar "Gerar":
   - Mostra spinner com texto "Gerando relatório... (pode levar até 1 minuto)"
   - Dialog não fecha durante geração
   - POST para `/reports/instagram/generate` com `{ month: "YYYY-MM" }`
   - Em sucesso: redireciona pra `/projects/:id/reports/instagram/:reportId` (página da 24.2)
   - Em erro: toast com mensagem específica + dialog continua aberto pra retry
   - Timeout client-side de 90s (UI) — backend tem 60s

### Listagem de Histórico

4. Nova route `/projects/:id/reports/instagram` (sem `:reportId`):
   - Server component lista todos os relatórios do projeto via `GET /reports/instagram?projectId=...`
   - Header: breadcrumb "Projetos / {projectName} / Relatórios Instagram"
   - Tabela ou lista de cards com:
     - Mês (formato "Abril 2026")
     - Gerado em (data)
     - Gerado por (nome do usuário)
     - Botão/link "Ver" → `/reports/instagram/:reportId`
   - Ordenação: mais recente primeiro
   - Empty state: "Nenhum relatório gerado ainda. Vá ao Dashboard de Instagram e clique em 'Gerar relatório do mês'."

5. Botão "Gerar novo relatório" no topo da listagem → abre o mesmo Dialog do AC 2

### Sidebar / Navegação

6. Item "Relatórios Instagram" no menu lateral do projeto (sidebar) → leva pra `/projects/:id/reports/instagram`
   - Localizar a sidebar atual (`packages/web/components/layout/project-sidebar.tsx` ou similar) e adicionar entry
   - Ícone `FileBarChart` (lucide-react)

### Comparativo automático vs mês anterior

7. AC 24.1 já calcula `comparison` no snapshot. Esta story apenas garante que ao gerar relatório do mês N+1, ele encontra o do mês N e popula o campo. Verificação: gerar 2 meses consecutivos e abrir o 2º — deltas aparecem.

### Erros & Edge cases

8. Tentar gerar relatório de mês onde a conta IG não existia ainda → retorna snapshot com `accounts: []` e a página HTML mostra empty state ("Nenhuma conta IG vinculada ou nenhum dado no período")

9. Tentar regenerar relatório do mesmo mês → upsert substitui (já coberto pelo schema 24.1, garantir UI deixa claro: "Já existe relatório de Abril/2026 — gerar substitui o anterior")

10. Conta com 0 posts no mês → ainda gera o relatório, conta aparece com seção "Sem publicações no período"

### Validação

11. typecheck `@loyola-x/web` — 0 erros
12. lint dos arquivos novos/modificados — 0 erros
13. Smoke E2E: clicar "Gerar relatório" → escolher Abril/2026 → aguardar geração → ver redirect pra página HTML com dados reais. Voltar à listagem, ver entry. Gerar Maio/2026 (vazio talvez) — abrir, ver `comparison` populada vs Abril.

### Geração de lista de meses (dropdown)

```ts
function getLast12Months(): { value: string; label: string }[] {
  const out = [];
  const now = new Date();
  for (let i = 1; i <= 12; i++) {
    const d = new Date(now.getFullYear(), now.getMonth() - i, 1);
    const value = `${d.getFullYear()}-${String(d.getMonth() + 1).padStart(2, "0")}`;
    const label = d.toLocaleDateString("pt-BR", { month: "long", year: "numeric" });
    out.push({ value, label });
  }
  return out;
}
```
