# Story 18.20 — Dashboard: Corrigir Top Criativos (Busca e Filtro)

## 📖 User Story

Como gestor de tráfego, quero que o painel "Top Criativos" busque corretamente informações de cada criativo (incluindo dados de pesquisa) e aplique filtros de investimento com precisão, para que eu possa identificar quais anúncios estão gerando melhor ROI sem ver dados inconsistentes ou incorretos.

---

## 🎯 Acceptance Criteria

### AC1: Busca de Campos Completos do Criativo
- [ ] Retornar para **cada criativo único** (agrupado por `ad_id` ou `ad_name`):
  - `ad_id` (identificador único)
  - `ad_name` (nome do anúncio)
  - `spend` (gasto total agregado)
  - `impressions` (total de impressões)
  - `clicks` (total de cliques)
  - `ctr` (click-through rate)
  - `cpm` (custo por mil impressões)
  - `reach` (alcance total)
  - `frequency` (frequência média)
  - `status` (ativo/pausado)
- [ ] Dados devem vir direto da Meta Ads API (sem cache desatualizado)
- [ ] Não trazer campos incompletos ou null (usar padrão 0 ou omitir)

### AC2: Agregação de Spend e Filtro de Investimento
- [ ] **Agregar spend por criativo único** (mesmo `ad_id` ou `ad_name` de múltiplas fontes):
  - Se um criativo aparece em múltiplas campanhas/períodos, **somar spend total**
  - Exemplo: Ad "v07-ia" com R$ 15k em Campaign A + R$ 20k em Campaign B = **R$ 35k total**
- [ ] **Validar spend agregado** antes de exibir:
  - Spend deve bater com total gasto em Meta Ads (conforme período do DayRangePicker)
  - Se spend de um criativo parece baixo (< R$ 1k), investigar se está sendo agregado corretamente
- [ ] **Ordenação por "Maior Investimento"**: ordenar `spend` descrescente
  - 1º lugar deve ser o criativo com spend máximo (~R$ 100k+ se total é R$ 100k+)
  - **Validação:** Soma dos top 3 criativos nunca deve exceder total gasto em Meta (sanity check)
- [ ] **Bug Fix:** Criativo com R$ 28.76 NÃO deve estar no topo (raiz: falta agregação ou filtro wrong)

### AC3: Busca de Dados de Pesquisa (Survey/Research)
- [ ] **Se criativo contém "research" ou "survey"** na `ad_name`:
  - Buscar respostas correspondentes na planilha de pesquisa (Google Sheets)
  - Agregar por `ad_name` (mesma lógica de spend)
- [ ] **Campos de pesquisa a retornar:**
  - `response_count` (total de respostas)
  - `response_rate` (% de respondentes vs impressões)
  - `avg_sentiment` (se disponível, ex: positivo/neutro/negativo)
- [ ] **Se nenhuma pesquisa** associada ao criativo: mostrar "—" (sem erro)
- [ ] **Não bloquear carregamento** se pesquisa não está disponível

### AC4: Validação de Dados Antes de Exibir
- [ ] Antes de renderizar, validar que `spend > 0` e dados são coerentes
- [ ] Se dados faltam ou estão inconsistentes, mostrar placeholder ou loading state
- [ ] Log de erro se API retorna dados incoerentes (importante para debug)

### AC5: Sincronização com DayRangePicker
- [ ] Filtro de data aplicado corretamente aos dados de criativo
- [ ] Ao mudar intervalo de datas, dados refetch automaticamente
- [ ] Spend e métricas são recalculadas para o novo período

### AC6: Sem Regressão
- [ ] Outros filtros (por Status, por Campanha, etc.) continuam funcionando
- [ ] Performance não degrada
- [ ] Padrão visual mantém consistência
