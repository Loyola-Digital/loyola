**Epic:** EPIC-18 Cross-Reference Meta Ads + Planilhas no Funil

## Story

Como gestor de tráfego do Loyola X, quero que a seção **Top Criativos** agregue criativos que tenham o mesmo nome (somando suas métricas), mostre **Investimento + CTR + CPL Pago** em cada card, ordene por CPL (menor primeiro) e fique posicionada no **final do dashboard**, para que eu consiga identificar rapidamente quais criativos estão performando melhor sem duplicação.

## Contexto

### Problema atual

O `TopCreativesGallery` hoje:
- Busca top N ads via `useTopPerformers` — retorna rows **por `ad_id`** (cada variação de criativo vira uma linha)
- Quando um criativo é duplicado (mesmo nome, lançado em campanhas diferentes, ou re-uploadado), aparece várias vezes na lista — poluindo o ranking
- Usa `ad.cpl` vindo do backend que é calculado via Meta Ads API (`actions[action_type=lead]`) — divergente da metodologia da Story 18.2 (leads da planilha)
- O default de ordenação é **CTR** (título "Maior CTR")
- Posicionada **antes** do Funil+Placements (não é o último bloco)

### Decisão arquitetural

- **Agregação frontend:** buscar N=100 do `useTopPerformers` e agregar por `campaignName` no cliente (zero backend)
- **CPL Pago recalculado no frontend:** cruzar `ad_id` (Meta) com `utm_content` da planilha, filtrar apenas leads onde `utm_source ∈ PAID_SOURCES`, dividir spend pela contagem
- **Props opcionais de pesquisa:** deixar o componente preparado pra receber `surveyDataByAdId?` na Story 18.6 (F) — zero refactor nessa Story

## Especificação

### Algoritmo de agregação (frontend)

```
1. useTopPerformers(projectId, metric, 100, days, campaignId)
   → data.topPerformers (até 100 ads por ad_id)

2. Filtrar:
   → withCreatives = topPerformers.filter(ad => ad.creative?.imageUrl || ad.creative?.thumbnailUrl)
   → withName = withCreatives.filter(ad => ad.campaignName && ad.campaignName.trim() !== "")

3. Agrupar por campaignName (exato, case-sensitive — nomes são strings literais do Meta):
   grouped: Map<string, TopPerformerAd[]>

4. Para cada grupo (mesmo nome):
   aggregated = {
     name: group[0].campaignName,
     ids: group.map(a => a.campaignId),          // lista de ad_ids agregados
     spend: sum(group.spend),
     impressions: sum(group.impressions),
     clicks: sum(group.clicks),
     reach: sum(group.reach),
     ctr: (sumClicks / sumImpressions) × 100,
     cpc: sumSpend / sumClicks,
     creative: group.sortBy(-spend)[0].creative,  // criativo com maior spend
     parentInfo: group[0].parentCampaignName + " › " + group[0].adsetName,

     // CPL Pago cruzando com planilha
     leadsPagos: count(planilha.rows where
                       utm_content ∈ aggregated.ids AND
                       utm_source.toLowerCase() ∈ PAID_SOURCES),
     cplPago: spend / leadsPagos  // null se leadsPagos == 0
   }

5. Ordenar pela métrica selecionada:
   - "cpl" (default): ASC por cplPago (menor primeiro, nulls ao final)
   - "ctr": DESC por ctr
   - "spend": DESC por spend (se adicionado)
   - demais: comportamento atual do Select

6. Mostrar top 20 (com botão "Ver todos (N)" pra expandir)
```

### Fonte dos dados

| Dado | Fonte |
|------|-------|
| Ads (id, name, creative, spend, impressions, clicks, ctr, reach) | `useTopPerformers(100)` |
| Leads pagos por ad_id | Cruzamento com `useFunnelSpreadsheetData` → filtrar `utm_content ∈ ids && utm_source ∈ PAID_SOURCES` |
| PAID_SOURCES | `["meta", "meta-ads", "google-ads"]` (mesma constante da 18.1/18.2) |

### Campo CPL no card

- **Principal (grande):** `cplPago` formatado como moeda (`R$ X,XX`) ou `—` se `leadsPagos == 0`
- Exibido ao lado de Investimento e CTR no overlay (que hoje mostra Spend + CTR + CPC — **substituir CPC por CPL**)

### Título dinâmico

- Default (CPL): `"Top Criativos — Menor CPL"`
- Outras métricas: `"Top Criativos — {sortLabel}"` (já existe essa lógica hoje)

## Acceptance Criteria

### AC1: Agregação por nome
- [ ] `useTopPerformers` chamado com `N=100` (em vez de 20)
- [ ] Função de agregação agrupa ads por `campaignName` exato (case-sensitive)
- [ ] Ads com nome vazio/null são **ignorados** (não renderizados)
- [ ] Métricas somadas: `spend`, `impressions`, `clicks`, `reach`
- [ ] Métricas recalculadas: `ctr` = (sumClicks / sumImpressions) × 100; `cpc` = sumSpend / sumClicks
- [ ] `creative` do grupo = do ad com maior `spend` (desempate: primeiro encontrado)

### AC2: CPL Pago e Leads cruzados com planilha
- [ ] Pra cada grupo agregado, contar linhas da planilha onde `utm_content ∈ ids_do_grupo` **e** `utm_source.toLowerCase() ∈ {meta, meta-ads, google-ads}` = **`leadsPagos` do grupo**
- [ ] `cplPago = spend / leadsPagos` (null se `leadsPagos == 0`)
- [ ] Exibição CPL: `fmtCurrency(cplPago)` ou `"—"` se null
- [ ] Quando Select em `leads`: ordenação usa `leadsPagos` (metodologia nova), **não** o `ad.leads` do backend
- [ ] Se não houver planilha vinculada: CPL e Leads exibem `"—"` + aviso no empty state

### AC3: Card do criativo — métricas exibidas

## Dev Notes

- **Naming confuso no tipo `TopPerformerAd`:** o campo `campaignId` parece ser na verdade o `ad_id` (pois `parentCampaignName` existe como campo separado). **Verificar antes de implementar** — se for mesmo `ad_id`, manter uso atual; se for `campaignId`, identificar o campo correto (provavelmente `ad.id` ou similar).
- **Case-sensitivity do nome:** nomes são strings literais do Meta Ads (uppercase/lowercase importam). Agrupar exato por padrão. Se surgir caso de "BBE Hot" vs "BBE hot" na prática, revisitar com normalização.
- **PAID_SOURCES:** reusar a constante já existente na 18.1 (`["meta", "meta-ads", "google-ads"]`). Se ainda não estiver em local compartilhado, extrair pra `lib/constants/paid-sources.ts`.
- **Ordenação com nulls:** quando CPL Pago é `null` (zero leads pagos), colocar no **final** da lista (não confundir com CPL = 0).
- **Não confundir ordenação e título:** se o Select estiver em "Leads", ordena por Leads (decrescente), mas o CPL Pago continua sendo o valor exibido no overlay (spec do AC3).
- **Lightbox:** ao abrir um grupo agregado, mostrar a imagem do ad com maior spend (primeiro do grupo ordenado). Navegação Lightbox navega entre **grupos**, não entre ads individuais do grupo.
- **Performance:** com N=100 ads + cruzamento com planilha pode ser computado em `useMemo` dependente de `[topPerformersData, spreadsheetData]`. Sem necessidade de otimização mais agressiva.

## Risks

- **Médio:** N=100 ads pode resultar em muitos grupos após agregação, fazendo o "Ver todos" virar impraticável. Mitigar com top 20 visível + expandir progressivo (já existe). Em funis pequenos isso não é problema.
- **Médio:** Lightbox perde contexto quando grupo agrega múltiplos ads. UX pode ficar confusa ("estou vendo qual?"). Mitigar com sub-texto tipo "1 de N criativos com este nome" no lightbox — anotado como possível iteração em Story futura.
- **Baixo:** Caso `utm_content` da planilha não tenha valor igual ao `ad_id` (drift de convenção), `leadsPagos` fica 0 pra vários criativos. Mitigar com log/warning em dev.
- **Baixo:** Performance — cruzamento é O(grupos × planilha.rows). Em volume típico (200 grupos × 5000 rows) = 1M operações, ainda aceitável. Se virar problema, otimizar com Map invertido por `utm_content`.
