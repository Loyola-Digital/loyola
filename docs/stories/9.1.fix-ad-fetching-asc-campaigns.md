## Story

**As a** gestor de tráfego da Loyola Digital,
**I want** to see ad creatives (thumbnails, titles, video indicators) in Top Performers and drill-down views,
**so that** I can visually identify which creatives are performing best without leaving the dashboard.

---

## Root Cause Analysis

The current implementation fetches ads using a hierarchical chain:
`campaigns → adsets (per campaign) → ads (per adset) → creatives (per ad)`

This breaks for **Advantage+ Shopping Campaigns (ASC)** and other campaign types where Meta flattens the hierarchy. These campaigns return 0 adsets via the insights API with campaign filtering, so the chain never reaches the ad level.

**Evidence (tested against production account):**
- Method `insights?level=adset&filtering=[campaign_id=X]` → **0 results** for ASC campaigns
- Method `/{campaign_id}/adsets` → **0 results** for ASC campaigns
- Method `insights?level=ad` (account-level) → **returns all ads correctly**, including from ASC campaigns

All 19 active campaigns in the Loyola account are affected.

## Acceptance Criteria

### Backend (traffic-analytics.ts)

1. [x] `getTopPerformers()` refactored: fetch ALL ads via single `insights?level=ad` call with `campaign_id,campaign_name,adset_id,adset_name` fields, instead of hierarchical campaign→adset→ad chain
2. [x] `getProjectAdAnalytics()` updated: if adset-level fetch returns 0 ads, fallback to account-level `insights?level=ad&filtering=[adset_id=X]`
3. [x] Creative fetching still works via `/{adId}?fields=creative{...}` for top N ads (rate limit friendly)
4. [x] Cache keys updated to reflect new query pattern

### Backend (meta-ads.ts)

5. [x] New function `fetchAllAdInsights(metaAccountId, accessToken, days)` → returns all ads with `campaign_id, campaign_name, adset_id, adset_name, ad_id, ad_name` + video metrics, using `level=ad` at account level
6. [x] Optional `campaignId` filter parameter for campaign-scoped queries
7. [x] Pagination support (Meta API pages at 25 results; use `after` cursor for accounts with many ads)

### Frontend (page.tsx)

8. [x] Top Performers section displays creative thumbnails correctly
9. [x] Drill-down ad rows show thumbnails and video retention sparklines
10. [x] No UI changes required — existing components already handle creative data when present

### Non-functional

11. [x] Total API requests for Top Performers reduced from O(campaigns × adsets) to O(1) + O(top_N creatives)
12. [x] Rate limit consumption decreased significantly
13. [x] Graceful fallback: if creative fetch fails, ad still shown without thumbnail

## Dev Notes

- The Meta Graph API `insights` endpoint with `level=ad` at account level returns ALL ads across all campaigns and adsets in a single paginated response
- Fields available at ad level: `ad_id, ad_name, adset_id, adset_name, campaign_id, campaign_name, spend, impressions, reach, clicks, ctr, cpc, cpm` + video metrics
- This eliminates the N+1 query problem entirely
- Creative fetch remains per-ad via `/{adId}?fields=creative{...}` but only needed for top 5-10 ads
- The `fetchAdSetInsights` function is still valid for the drill-down table when user expands a specific campaign — it works for non-ASC campaigns. For ASC campaigns, the adset level simply won't have data (which is correct behavior)
